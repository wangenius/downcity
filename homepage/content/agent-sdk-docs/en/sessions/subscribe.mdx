# `session.subscribe()`

`session.subscribe()` is the only live subscription entry point for a Session. It emits Message snapshots, Assistant Parts, text deltas, Turn lifecycle, and Session state changes.

```ts
const unsubscribe = session.subscribe(async (mutation, reply) => {
  if (mutation.variant === "delta" && mutation.type === "text") {
    process.stdout.write(mutation.delta);
  }

  if (
    mutation.variant === "part" &&
    mutation.type === "tool" &&
    mutation.part.state === "approval-required"
  ) {
    await reply.approval({ decision: "approved" });
  }
});
```

Mutations have five variants: `message`, `part`, `delta`, `turn`, and `session`. Message-backed Mutations are published after the corresponding Message or Assistant draft is persisted. Turn and Session Mutations describe live runtime state.

Mutations are a live protocol, not the history format. They do not replay history or recover a disconnected stream; reload `messages()` and `get_info()` after reconnecting.

An approval-required Tool Part keeps the same `part_id` while moving through `ready -> approval-required -> running | failed`. Reply in the callback with `reply.approval({ decision: "approved" })`, or resolve it later with `session.resolve_approval(...)`.
