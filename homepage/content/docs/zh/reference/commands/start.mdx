# `sma start`

启动 SMA Manager 后台进程。

## 用法

```bash
sma start
```

## 行为

- 首次执行会创建 manager 运行目录并启动后台进程。
- 若 manager 已在运行，会直接返回“already running”。
- manager 进程用于全局管理 agent 运行态，不负责具体业务调度。

## 运行文件

默认写入用户目录下：

- `~/.ship/manager/manager.pid`
- `~/.ship/manager/manager.log`

## 常见组合

```bash
# 1) 先启动 manager
sma start

# 2) 再进入某个 agent 目录启动 agent（默认后台）
cd /path/to/agent
sma agent create .
sma agent on
```
