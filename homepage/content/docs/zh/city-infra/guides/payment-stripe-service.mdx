title: Stripe 支付服务
description: 用 @downcity/services 把一次性 Stripe 支付同步成全局余额充值。
---

Stripe 支付服务当前只处理一件事：

> 用户完成一次性 Stripe 付款后，把付款结果同步成一笔 `balance topup`。

它不处理 subscription，也不处理 entitlement。

## 启用

```ts
import { balanceService } from "@downcity/services";
import {
  paymentService,
  stripePaymentMethod,
  stripePaymentService,
  type StripePaymentServiceBalanceBridge,
} from "@downcity/services";

const balance = balanceService({
  unit: "credits",
});
const stripeBalanceBridge: StripePaymentServiceBalanceBridge = {
  readTopup: async (topup_id) => await balance.readTopup(topup_id),
  finishTopup: async (topup_id, extra) => await balance.finishTopup(topup_id, extra),
};

base.use(balance);
base.use(paymentService({
  methods: [
    stripePaymentMethod(),
  ],
}));
base.use(stripePaymentService({
  balance: stripeBalanceBridge,
  secret_key: process.env.STRIPE_SECRET_KEY,
  webhook_secret: process.env.STRIPE_WEBHOOK_SECRET,
}));
```

## 最常见链路

### 1. 先读支付方式

```ts
const methods = await guest.service("payment").get("methods");
```

前端可以先根据 `methods.items` 判断当前 InfraRuntime 是否真的开放了 Stripe 支付入口，再决定是否展示充值按钮。

### 2. 先创建 topup

```ts
const topup = await user.service("balance").action("topups/create").invoke({
  amount: 500,
  note: "recharge",
});
```

### 3. 再创建 Stripe Checkout

```ts
const checkout = await user.service("payment.stripe").action("checkout/create").invoke({
  topup_id: topup.topup_id,
});
```

返回值里会包含：

- `payment_id`
- `stripe_checkout_session_id`
- `checkout_url`

前端跳转到 `checkout_url` 即可。

### 3. Stripe 回调 webhook

Stripe 直接请求服务路由：

```txt
POST /v1/payment.stripe/webhook
```

所以你在 Stripe Dashboard 里注册 endpoint 时，默认就填：

```txt
{base_url}/v1/payment.stripe/webhook
```

配置 `webhook_secret` 或 `STRIPE_WEBHOOK_SECRET` 后，服务会校验 `stripe-signature`。

当收到 `checkout.session.completed` 时，服务会自动调用 `balance.finishTopup()`，真正把余额充进去。

如果你部署在 Worker bindings 这类宿主环境里，`STRIPE_SECRET_KEY` 和 `STRIPE_WEBHOOK_SECRET` 都要显式提供给运行时。

如果你没有单独配置 `STRIPE_SUCCESS_URL` / `STRIPE_CANCEL_URL`，服务会按下面顺序自动生成默认跳转页：

1. `DOWNCITY_INFRA_BASE_URL`
2. 当前请求的 origin

生成路径都是：

- `/v1/payment.stripe/redirect/success`
- `/v1/payment.stripe/redirect/cancel`

也就是说，很多场景下你不必再单独配置 `STRIPE_SUCCESS_URL` / `STRIPE_CANCEL_URL`，甚至不必手动配置 `DOWNCITY_INFRA_BASE_URL`。

只有在你希望跳到自己的前端页面时，才建议显式覆写：

- `STRIPE_SUCCESS_URL`
- `STRIPE_CANCEL_URL`

## 它和 balance 的关系

这两个服务的职责要分清：

- `balance` 负责全局钱包、充值单和流水
- `payment.stripe` 负责 Stripe Checkout、webhook 和支付结果同步

也就是说，Stripe 服务不会自己直接改余额，而是统一走 `finishTopup()` 完成到账。

## 路由

- `GET /v1/payment/methods`
- `POST /v1/payment.stripe/checkout/create`
- `GET /v1/payment.stripe/payments/me`
- `GET /v1/payment.stripe/payments`
- `GET /v1/payment.stripe/events`
- `POST /v1/payment.stripe/webhook`
