# web Plugin

`web` 是联网任务的方法论层。它向 Agent 注入网页搜索、资料核实、网页读取和浏览器操作的通用策略，但不提供真实的联网执行能力。

它提供一个 `install` action。这个 action 只返回安装指引，不执行命令、不访问网络，也不会安装依赖或修改文件。Agent 需要读取返回的 `data.prompt`，再使用当前可用的 Plugin 与 Shell 能力完成安装。

## 注册

```ts
import { Agent } from "@downcity/agent";
import { WebPlugin } from "@downcity/plugins";

const agent = new Agent({
  id: "research-agent",
  path: "/path/to/project",
  plugins: [new WebPlugin()],
});
```

注册后，`WebPlugin.system()` 返回的方法论会自动进入 Agent 的 system prompt。

## install action

```ts
const result = await agent.plugins.runAction({
  plugin: "web",
  action: "install",
  payload: {
    target: "agent-browser",
    scope: "project",
  },
});
```

返回结果的核心结构为：

```ts
{
  kind: "instructions",
  target: "agent-browser",
  scope: "project",
  prompt: "...给 Agent 的安装步骤...",
}
```

可选 `target`：

- `web-access`：返回 `web-access` Skill 安装工作流，默认值
- `agent-browser`：返回 `agent-browser` Skill 与 CLI 安装工作流
- `all`：返回以上全部能力的安装工作流

可选 `scope` 只控制 `agent-browser` CLI 的安装提示：

- `user`：提示使用 npm 全局安装，默认值
- `project`：提示检查项目锁文件，并使用现有包管理器安装 devDependency

Skill 的安装位置不由 `scope` 决定。它由当前 `SkillPlugin` 的 `use / paths` 扫描配置决定。

## 安装工作流

`web.install` 会告诉 Agent 按以下流程操作：

1. 对每个需要的 Skill 调用 `skill.install`。
2. 按 `skill.install` 返回的扫描感知 Shell 指引完成安装。
3. `agent-browser` 还需要 CLI 时，按 `scope` 对应的提示安装并验证命令。
4. 调用 `skill.list`，确认新 Skill 已进入当前扫描范围。
5. 调用 `skill.lookup` 读取 `SKILL.md`，然后再使用该能力。

例如，`agent-browser` 的 Skill 安装仍由 Skill Plugin 提供指引：

```ts
plugin_call({
  plugin: "skill",
  action: "install",
  payload: {
    spec: "agent-browser",
  },
});
```

这样可以保证 SkillPlugin 的实际扫描根始终是安装位置的唯一事实源，WebPlugin 不需要复制项目目录、用户目录或自定义目录的规则。

## 能力边界

`web` 不负责：

- 选择联网 provider
- 决定运行时默认使用哪个联网实现
- 执行真实搜索、抓取、点击或浏览器操作
- 执行 Skill 或 CLI 安装命令
- 读取或写入项目级 web 配置

真实联网能力来自当前 Agent 环境中的 tools、skills 或其他 Plugin。例如，搜索或 fetch 工具适合公开网页检索，browser、Chrome、agent-browser 或 computer-use 能力适合动态页面与登录态交互。

如果必要能力不存在，Agent 应直接说明，不应假装已经联网。

## System Prompt

WebPlugin 注入的方法论会引导 Agent：

- 先检查当前实际可用的 tools 与 skills
- 根据搜索、读取、结构化抓取或浏览器交互选择路径
- 优先回到官方页面、原始文档和源码等一手来源核实结论
- 路径不再产生进展时及时切换方法
- 达成目标后停止，不为形式完整继续增加无效步骤
