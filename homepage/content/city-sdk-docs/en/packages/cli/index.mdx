The `downcity` package installs two command families:

- `fed` / `downfed`: manages Federation infrastructure, including project scaffolding, Worker deploys, saved admin connections, and direct Federation HTTP queries.
- `city` / `downcity`: manages local Agents, local plugins, chat/gateway, and the local user's membership and login state for a Federation.

Use `fed` when you are deploying or operating the shared backend. Use `city` when you are connecting, signing in, running Agents, or working from a local user workspace.

## How To Read This Section

The recommended order is:

1. Start with [Quickstart](/en/city-sdk-docs/packages/cli/quickstart) to separate the `fed` and `city` entry points.
2. For Federation deploys, read [Deploy to Cloudflare Workers](/en/city-sdk-docs/operations/deploy-cloudflare-workers).
3. For local Agent or Federation-login debugging, read [Operations and Debugging](/en/city-sdk-docs/packages/cli/operations-and-debug).

## fed / downfed

`fed` is for Federation operators. Common commands:

```bash
fed
fed create .
fed deploy
fed deploy --dry-run
fed deploy --verify
fed server add
fed server manage
fed query GET /health
```

It owns the shared backend surface: Federation projects, Cloudflare Workers deploys, D1 / Queue / R2 resources, the admin workspace, and HTTP debugging.

## city / downcity

`city` is for the local workspace and Agent runtime. Common commands:

```bash
city
city federation status
city federation join <url>
city federation login
city agent create .
city agent list
city agent start
city agent chat
```

It owns local state: selecting a Federation, storing user login state, running local Agents, and managing plugins plus chat/gateway.

## Boundaries

- Do not use `city deploy` to deploy a Federation; the deploy entry is `fed deploy`.
- Do not use `city create` to scaffold a Federation project; the scaffold entry is `fed create`.
- Do not use `fed` as the local Agent manager; Agent commands live under `city agent ...`.
- Programmatic management should still use `Admin City`; CLI is best for human terminal workflows.
