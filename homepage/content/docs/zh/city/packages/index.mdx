如果你已经知道自己要看哪个 package，这一组文档会比场景指南更直接。

Downcity 的 package 设计很克制：先用 `@downcity/city` 搭出 City，再选 `@downcity/gate` 作为产品调用入口；数据库由你的运行环境创建 Drizzle db 后直接传给 City，最后按产品需要接官方服务。

这套顺序背后的意思是，Downcity 其实有 4 个清晰角色：

- `city` 是 Kernel，负责 City 本体
- `services` 是 Capabilities，负责官方能力
- `gate` / `cli` 是 Interfaces，负责 studio 接入与运维操作
- `cities/*` 是 Composition，负责把这些东西组装成可运行实例

如果你只从 npm package 角度理解，最值得先记住的是前 3 个：`city`、`services`、`gate`。

## 推荐理解顺序

1. 先读 [@downcity/city](/docs/city/packages/city)，理解 City 本体负责什么
2. 再读 [@downcity/gate](/docs/city/packages/gate)，理解产品怎么调用它
3. 需要运维工具时读 [City CLI](/docs/city/packages/cli)
4. 需要账号、余额、usage 或 Stripe 支付能力时，再看 `@downcity/services`

## 如果你只想先搞清接入关系

建议把 package 层先理解成这条链：

```txt
@downcity/city
  -> 提供 City Kernel
@downcity/services
  -> 把官方 capabilities 装进 City
@downcity/gate
  -> 用 UserClient / AdminClient 调这些能力
```

也就是说，`@downcity/gate` 不只是调 AI。

它统一负责：

- AI service 调用
- custom service 调用
- 官方 service 调用
- 管理侧 studio / token / env 调用

## 一张图理解职责

```txt
Kernel        -> @downcity/city
Capabilities  -> @downcity/services
Interfaces    -> @downcity/gate + downcity
Composition   -> cities/node + cities/edge
```

## 典型组合

### 最小 AI 产品

- `@downcity/city`
- `@downcity/gate`

### 要最小注册登录

- `@downcity/city`
- `@downcity/gate`
- `@downcity/services`

### 要 usage 计量

- `@downcity/city`
- `@downcity/gate`
- `@downcity/services`

### 要全局余额与充值单

- `@downcity/city`
- `@downcity/gate`
- `@downcity/services`

### 要 Stripe 一次性充值

- `@downcity/city`
- `@downcity/gate`
- `@downcity/services`
