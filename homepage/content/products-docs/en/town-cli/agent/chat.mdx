# `town agent chat`

Chat with a target agent from the terminal.

> This page remains as a command entry point. For the fuller explanation, see [Chat with an Agent](/en/docs/agent/getting-started/chat-with-an-agent) and [CLI / chat](/en/docs/agent/cli/interaction/chat).

This command uses a dedicated local CLI main session: `local-cli-chat-main`.

## Usage

```bash
town agent chat [--to <id>]
town agent chat [--to <id>] --message <text> [--json]
```

## Key options

- `-t, --to <id>`: target agent id; if omitted, the CLI opens an interactive picker from running agents
- `-m, --message <text>`: send one message and exit instead of entering the REPL
- `--json`: print structured output for one-shot mode
- `--host <host>`: override the auto-resolved RPC host

## Behavior

- Defaults to interactive terminal chat when `--message` is omitted.
- In interactive mode, once you pick an agent and enter the chat loop, the terminal shows spinner-based progress states.
- When the agent runs tools, interactive mode shows the tool name, a compact input summary, and the completion state.
- One-shot `--message` mode keeps the simpler output flow and does not render the interactive spinner / tool status blocks.
- One-shot mode works without entering the REPL.
- Requires the target agent to be running.
- Keeps sending every line into the same local CLI session instead of sharing Console UI context.
- Type `/exit` or `/quit` to leave the chat loop.
