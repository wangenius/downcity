---
title: "Balance Service"
description: "Maintain one global user balance, ledger, topup orders, and redeem_code flows while keeping charging policy in business hooks."
---

`@downcity/services` gives you a global user wallet, not a hard-coded pricing engine.

It is responsible for:

- balance accounts
- balance ledger
- topup orders
- redeem_code flows

It does not decide:

- which actions should charge
- how much each call should cost
- which products are free or paid

Those rules should stay in your own business hooks.

## Reading order

1. Start with [Global wallet model](/en/docs/city-infra/packages/services/balance/global-wallet) to understand why balance is not bound to product.
2. Continue with [Charge directly from hooks](/en/docs/city-infra/packages/services/balance/hook-charging) for the recommended integration pattern.
3. Continue with [Topups and ledger](/en/docs/city-infra/packages/services/balance/topups-and-ledger) for create-first, confirm-later recharge flows.
4. Finish with [redeem_code flows](/en/docs/city-infra/packages/services/balance/redeem-codes) for one-time gift code credits.

## Minimal setup

```ts
import { balanceService } from "@downcity/services";

const balance = balanceService({
  init: 100,
  unit: "credits",
});

base.use(balance);
```

## The most important boundary

### Balance is one global user wallet

One user shares the same balance across multiple products.

### Charging policy stays in business code

You should write:

- a pre-call balance check
- a post-success debit

inside your own hooks instead of putting pricing rules into service config.

## Common combinations

- pair it with [@downcity/services](/en/docs/city-infra/packages/services) for one wallet after login
- pair it with [@downcity/services](/en/docs/city-infra/packages/services) to separate "what happened" from "what was charged"
- pair it with [@downcity/services](/en/docs/city-infra/packages/services) to translate payment results into wallet topups
- use `redeem_code` on its own for gift credits, compensation codes, and campaign codes
