# `town update`

Update the currently installed global `downcity` CLI bundle to the latest version.

By default, Downcity detects both the package manager and the installed package name. If you installed the CLI from `downcity`, `town update` upgrades `downcity`;

## Usage

```bash
town update
```

```bash
town update --manager npm
```

```bash
town update --manager pnpm
```

## Options

- `--manager <manager>`: choose `auto`, `npm`, or `pnpm` (default: `auto`)

## Important behavior

- `town update` updates only the currently installed global CLI package.
- It does not automatically restart already running Town runtime / Console processes.
- After update completes, run [`town restart`](/en/docs/commands/restart) if you want running processes to load the new version.

## Related commands

- [`town restart`](/en/docs/commands/restart)
- [`town status`](/en/docs/commands/status)

## Example output

```text
$ town update
town v1.0.0

Updating downcity
  manager   npm
  command   npm install -g downcity@latest

Downcity updated
  package   downcity@latest
  manager   npm
  note      If Town runtime / Console is already running, use `town restart` to load the new version.
```
