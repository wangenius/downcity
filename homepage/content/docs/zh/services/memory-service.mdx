# Memory Service

`memory` service 提供运行时记忆操作能力。

## 职责

- 状态查看与索引重建
- 记忆检索与按文件读取
- 显式写入 memory
- 把会话窗口刷写到 daily memory
- 在本地 session 执行链里自动召回相关记忆
- 在本地 session 执行结束后自动写入 session 级 working memory

## 自动接入

当你使用默认本地 session 执行器时，`memory` 会自动参与一轮执行：

- 执行前：按当前用户请求做一次 recall，并把少量相关记忆注入本轮上下文
- 执行后：把当前轮问答写入 `.downcity/session/<sessionId>/memory/working.md`
- 执行后：追加一条轻量 daily journal 到 `.downcity/memory/daily/<date>.md`
- 执行后：如果检测到明显的长期偏好 / 规则表达，再额外写入 `.downcity/memory/MEMORY.md`
- 写入 `longterm` 前会先做一次简单去重，避免同一条长期规则不断重复追加
- 当前 longterm 候选会被整理成更稳定的 Canon 表述，并区分 `preference / fact / decision / rule`

如果你需要手动排查或管理记忆，仍然使用 `city memory` 命令。

## 运行态操作

```bash
city service status memory
city memory status
city memory restart
city service command memory search --payload '{"query":"release decision"}'
```
