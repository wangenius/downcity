# RemoteAgent 类

`RemoteAgent` 的核心公开使用面包括：

- `new RemoteAgent({ url })`
- `agent.sessions`
- `agent.runPluginAction(input)`

`agent.sessions.create()` 和 `agent.sessions.get()` 返回 `RemoteAgentSession`。

## Sessions

`agent.sessions` 返回一个 `AgentSessions`，包含以下方法：

- `agent.sessions.create(input?)`
- `agent.sessions.get(sessionId)`
- `agent.sessions.list(input?)`
- `agent.sessions.archive({ id })`
- `agent.sessions.archived(input?)`
- `agent.sessions.clean_archive()`

## `runPluginAction`

当你的客户端 UI 需要调用远程 runtime 上的 plugin action 时，使用 `runPluginAction()`。

```ts
await agent.runPluginAction({
  plugin: "shell",
  action: "approve",
  payload: {
    approvalId: "ap_xxx",
  },
});
```

对于 unrestricted shell 审批流程，前端订阅 session event，遇到 `tool-approval-request` 后调用 approve 或 deny：

```ts
session.subscribe(async (event) => {
  if (event.type !== "tool-approval-request") return;

  const displayText = event.operation === "write"
    ? event.inputPreview || event.cmd
    : event.cmd;
  const approved = window.confirm(`${displayText}\n\n${event.reason}`);

  await agent.runPluginAction({
    plugin: "shell",
    action: approved ? "approve" : "deny",
    payload: {
      approvalId: event.approvalId,
    },
  });
});
```

## `url`

当前构造参数是：

```ts
new RemoteAgent({
  url: "http://127.0.0.1:5314/agents/repo-helper",
});
```

而不是 `baseUrl`。

`url` 当前支持：

- `http://...`
- `https://...`
- `rpc://...`

## 核心语义

它本身不执行任务，只负责连接远程 session 能力。

如果你把本地 `Agent` 看成服务端执行壳，那 `RemoteAgent` 就是与之配对的客户端。

## 一个重要限制

远程 session 当前不支持客户端直接传本地模型实例。

也就是说：

```ts
await session.set({ model });
```

当前不支持。

模型应该由服务端本地 `Agent` 或服务端本地 session 持有。
