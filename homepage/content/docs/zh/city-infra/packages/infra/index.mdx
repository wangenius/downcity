`@downcity/infra` 是 Downcity 的 InfraRuntime 本体。只要你真的要部署一个能服务多个产品的 InfraRuntime，最后都会落到这个包。

这不是“某个 AI SDK 的补充包”，而是整个 server-side InfraRuntime 的核心运行时。你注册的 AI service、自定义 service、官方服务，以及统一的 `/v1/*` 调用边界，都会在这里汇合。

## 这组文档怎么读

建议按下面顺序读：

1. 先读 [InfraRuntime 运行时与启动方式](/docs/city-infra/packages/infra/infra-runtime)，理解 `InfraRuntime` 实例到底承载什么。
2. 再读 [Service 与 Action 设计](/docs/city-infra/packages/infra/service-and-action)，理解为什么 AI service、custom service 最后都能走统一路由。
3. 需要扩展能力时读 [Hooks 与服务挂载机制](/docs/city-infra/packages/infra/hooks-and-service)，理解服务和钩子怎么接入。
4. 需要理解持久化与表结构时读 [Store 与 Table 数据模型](/docs/city-infra/packages/infra/store-and-table)。

如果你只想先建立总体印象，可以先看完本页，再按顺序读上面四页。

## 这个包真正负责什么

- 创建 `InfraRuntime` 实例
- 注册 `Service`、官方服务和 `AIService`
- 管理内置的 product、env 和 token 相关能力
- 暴露统一的 `/v1/*` 路由与调用边界

## 它不负责什么

- 不负责产品侧调用 SDK，这部分在 [@downcity/conduit](/docs/city-infra/packages/conduit)
- 不负责创建具体数据库连接；你用 Drizzle ORM 创建 db 后传给 `new InfraRuntime({ db })`
- 不负责现成账号、usage、支付业务，这些通过官方服务接入

## 什么时候应该先看它

- 你要搭一个真正可运行的 InfraRuntime
- 你要声明自己的 AI service 或自定义 service
- 你要把多产品共用的鉴权、hooks、模型目录放进同一个后端

## 最常见的组合关系

| 目标 | 组合方式 |
| --- | --- |
| 本地或常规 Node 部署 | `@downcity/infra` + Drizzle SQLite / Postgres |
| 产品前端或 App 调用 | `@downcity/infra` + `@downcity/conduit` |
| 账号、usage、支付 | `@downcity/infra` + 对应官方服务 |

## 安装

```bash
pnpm add @downcity/infra drizzle-orm better-sqlite3
```

## 最小示例

```ts
import { InfraRuntime, AIService } from "@downcity/infra";
import Database from "better-sqlite3";
import { drizzle } from "drizzle-orm/better-sqlite3";

const sqlite = new Database("./data.sqlite");
sqlite.pragma("journal_mode = WAL");

const db = Object.assign(drizzle(sqlite), {
  $client: { exec: (sql: string) => sqlite.exec(sql) },
});

const base = new InfraRuntime({ db, dialect: "sqlite", raw: sqlite });

const ai = new AIService();

ai.use({
  id: "local-echo",
  name: "Local Echo",
  default: ["text"],
  actions: {
    text: async (ctx) => ({
      id: crypto.randomUUID(),
      role: "assistant",
      parts: [
        {
          type: "text",
          text: String(ctx.input.prompt ?? ""),
          state: "done",
        },
      ],
    }),
  },
});

base.use(ai);
```

这个示例只说明一件事：`@downcity/infra` 负责把数据库、service 和服务挂到同一个 InfraRuntime 上；它本身不解决产品怎么登录、前端怎么调这些外围问题。
