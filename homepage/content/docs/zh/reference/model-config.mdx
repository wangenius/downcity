# Model 配置参考

Downcity 不把完整模型实现细节放进 Agent 项目。项目只保存模型 ID；provider key、provider routing 和模型目录由共享 AI service 负责。

## 心智模型

```txt
共享 AI service 模型目录
  -> town model list
  -> town model use <modelId>
  -> downcity.json execution.modelId
  -> agent session runtime
```

这样每个 Agent 项目都保持轻量，多个产品或 Agent 可以复用同一套模型基础设施。

## 项目绑定

Agent 项目通过 `execution.modelId` 绑定模型：

```json
{
  "execution": {
    "type": "api",
    "modelId": "quality"
  }
}
```

说明：

- `execution.type` 应写为 `api`
- `execution.modelId` 应对应 `town model list` 返回的某个模型
- provider secrets 和 provider 连接细节不写进项目配置

## 常用命令

列出、绑定和测试共享模型：

```bash
town model list
town model use <modelId>
town model test <modelId>
```

模型绑定变更后重启 agent：

```bash
town agent restart .
```

## 项目不应该保存什么

项目 `downcity.json` 不应该保存 provider secrets 或 provider 连接细节。这些属于共享服务 runtime。
