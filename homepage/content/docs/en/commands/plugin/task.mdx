# `city task`

`city task` is the managed task plugin command group.

Use it when you want a reusable automation definition instead of a one-shot delayed plugin action.

## Usage

```bash
city task <subcommand> [options]
```

## Main subcommands

- `list`
- `create`
- `run <title>`
- `update <title>`
- `delete <title>`
- `status <title> <enabled|paused|disabled>`
- `enable <title>`
- `disable <title>`

## Key boundaries

- Use `city task` for reusable or long-lived automation.
- Use `city chat --delay/--time` or `city memory --delay/--time` for one existing action that should happen later once.
- Task definitions live in project storage and can use `@manual`, cron, or `time:<ISO8601>`.

## Examples

```bash
city task list
city task create --title "Daily check" --description "Run tests" --session-id telegram-chat-123 --when "0 10 * * 1-5" --activate
city task run daily-check
city task update daily-check --when "@manual"
city task disable daily-check
```
