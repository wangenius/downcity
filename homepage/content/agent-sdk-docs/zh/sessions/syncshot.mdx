# `session.syncshot()`

本地 Session 的 system 第一次生成后会固定在内存中。需要显式采用 Agent 当前最新 instruction、managed system blocks 和 plugin system 时，调用：

```ts
await session.syncshot();
```

`syncshot()` 会替换当前 Session 的内存 system snapshot。已经发给 provider 的请求不受影响，后续 step 使用新 system。它只刷新 system，不改变 Session 的 plugin execution view。

持久化规则取决于当前 Session 是否已经存在 `instruction.md`：

- 文件已存在：原子覆盖为重新生成后的完整 system。
- 文件不存在：只更新内存，不自动创建文件。

如果需要首次创建 `instruction.md`，在刷新后显式调用：

```ts
await session.syncshot();
await session.snapshot();
```

`syncshot()` 和 `snapshot()` 会在同一 Session 内串行执行，避免并发写入覆盖顺序不确定。该方法只属于本地 `AgentSession`，远程 Session 不提供。
