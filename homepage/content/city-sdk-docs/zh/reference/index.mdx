如果你还不清楚 Downcity 是什么、为什么要这样拆、边界在哪里，先回到 [理解 Downcity City](/zh/city-sdk-docs/concepts) 或 [快速开始](/zh/city-sdk-docs/quickstart)。

## 推荐顺序

1. 如果你在写产品前端，先读 [User City](/zh/city-sdk-docs/reference/city-user)
2. 如果你在写服务端接入，先读 [City](/zh/city-sdk-docs/reference/city-runtime)
3. 如果你在写登录和签 token 逻辑，再读 [Admin City](/zh/city-sdk-docs/reference/city-admin)
4. 如果你需要自己手写请求，最后读 [HTTP API](/zh/city-sdk-docs/reference/http-api)
