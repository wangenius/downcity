# Action

**Action** 是 Agent 可以调用的离散操作。它是 Service/Action 层中的工作单元。Action 由 Service 暴露，可以组合成工作流。

## 示例

- `send_email` — 向收件人发送邮件
- `query_database` — 运行 SQL 查询
- `call_api` — 调用外部 HTTP API
- `upload_file` — 将文件存储到文件存储

## Schema

每个 Action 声明：

- 输入 schema（接受的参数）
- 输出 schema（返回的内容）
- 错误条件
- 所需权限

## 调用

Agent 通过 City 层调用 Action。City 将请求路由到拥有该 Action 的 Service，然后将结果返回给 Agent。

## 组合

Action 可以组合成更高级的工作流。例如，"用户入职"工作流可能按顺序调用 `create_account`、`send_welcome_email` 和 `add_to_crm`。
