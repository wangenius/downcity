 `Provider` 是抽象基类，封装第三方 AI 提供商的 action 实现和连接信息。通过 `.model()` 生成模型配置，传给 `AIService.use()` 注册。

 ## 快速开始：OpenAI-compatible 文本 Provider

 继承 `Provider`，覆盖 `createClient`，即可自动获得 `text` / `stream` 能力：

 ```ts
 import { createOpenAICompatible } from "@ai-sdk/openai-compatible";
 import { Provider, type OpenAICompatibleClientConfig } from "@downcity/city";

 class DeepSeekProvider extends Provider {
   constructor() {
     super({
       id: "deepseek",
       envKey: "DEEPSEEK_API_KEY",
       baseURL: "https://api.deepseek.com/v1",
       passthroughModel: "deepseek-chat",
     });
   }

   protected createClient({ apiKey, baseURL }: OpenAICompatibleClientConfig) {
     return createOpenAICompatible({ apiKey, baseURL, name: "deepseek" });
   }
 }

 const deepseek = new DeepSeekProvider();

 const ai = new AIService();
 ai.use(deepseek.model({ id: "deepseek-v4-flash", name: "DeepSeek V4 Flash" }));
 base.use(ai);
 ```

 注册后自动生成路由：

 ```
 POST /v1/ai/text              → SDK 通路
 POST /v1/ai/stream            → SDK 通路
 POST /v1/ai/chat/completions  → 自动透传到 DeepSeek
 GET  /v1/ai/models            → 模型目录
 ```

 `passthroughModel` 控制自动透传时发往上游的 model 值。

 ## 覆盖默认 text / stream

 如果默认的 OpenAI-compatible 实现不满足需求，可以直接覆盖 `text` / `stream` 方法：

 ```ts
 import { Provider, type Context, type AIProviderChargedOutput } from "@downcity/city";
 import type { UIMessage } from "ai";

 class CustomTextProvider extends Provider {
   constructor() {
     super({ id: "custom", envKey: "CUSTOM_API_KEY" });
   }

   async text(ctx: Context): Promise<AIProviderChargedOutput<UIMessage>> {
     // 自定义实现
   }

   async stream(ctx: Context) {
     // 自定义实现
   }
 }
 ```

 ## 多模态转文本

 覆盖 `extractPrompt` 可以把多模态 messages 规整为纯文本：

 ```ts
 class DeepSeekProvider extends Provider {
   // ...
   protected extractPrompt(input: { prompt?: string; messages?: Array<{ role: string; parts?: Array<{ type: string; text?: string }> }> }): string {
     if (typeof input.prompt === "string") return input.prompt;
     return input.messages
       ?.find((m) => m.role === "user")
       ?.parts?.map((part) => (part.type === "text" ? part.text : ""))
       .filter(Boolean)
       .join("\n") ?? "";
   }
 }
 ```

 ## 图片 Provider

 `@downcity/city` 不内置 OpenAI、DeepSeek 或其他具体图片 Provider 构造器。City 包只提供 `Provider` 抽象、`AIService` 注册机制，以及 `client.ai.image_create()` / `client.ai.image_result()` 的统一任务协议。

具体上游适配应放在具体 City 项目里实现。这些 adapter 可以把不同上游的响应归一成 AI SDK `UIMessage` file parts，然后通过 `Provider.model()` 注册给 `AIService`：

```ts
import {
  Provider,
  type AIImageProviderCreateResult,
  type AIImageProviderPersistResult,
  type AIImageProviderResult,
  type Context,
} from "@downcity/city";

interface ImageJobStore {
  create(record: {
    job_id: string;
    input: Record<string, unknown>;
    status: "running";
    user_id?: string;
  }): Promise<void>;
  get(job_id: string): Promise<{
    input: Record<string, unknown>;
    status: "running" | "succeeded" | "failed";
    user_id?: string;
    result?: AIImageProviderResult["result"];
    usage?: unknown;
    error?: string;
  } | undefined>;
  update(record: {
    job_id: string;
    status: "running" | "succeeded" | "failed";
    result?: AIImageProviderResult["result"];
    usage?: unknown;
    error?: string;
  }): Promise<void>;
}

 class MyImageProvider extends Provider {
   constructor(private readonly job_store: ImageJobStore) {
     super({ id: "my-image", envKey: "MY_IMAGE_API_KEY" });
  }

  async image_create(ctx: Context): Promise<AIImageProviderCreateResult> {
    const job_id = crypto.randomUUID();
    await this.job_store.create({
      job_id,
      input: ctx.input as Record<string, unknown>,
      status: "running",
      user_id: ctx.user?.user_id,
    });

    return {
      job_id,
      status: "running",
      poll_after_ms: 2000,
    };
  }

  async image_persist(ctx: Context): Promise<AIImageProviderPersistResult> {
    const job_id = String(ctx.input.job_id);
    const job = await this.job_store.get(job_id);
    if (!job) return { job_id, status: "failed", error: "image job is missing" };
    if (job.status === "succeeded") {
      return {
        job_id,
        status: "succeeded",
        result: job.result,
        billing_ref: `image:${job_id}`,
        metadata: { user_id: job.user_id, usage: job.usage },
      };
    }

    const apiKey = ctx.env("MY_IMAGE_API_KEY");
    const response = await fetch("https://image.example.com/generate", {
       method: "POST",
       headers: {
         authorization: `Bearer ${apiKey}`,
         "content-type": "application/json",
       },
       body: JSON.stringify(job.input),
     });
     const data = await response.json();
     const result: AIImageProviderResult = {
       job_id,
       status: "succeeded",
       result: {
         id: crypto.randomUUID(),
         role: "assistant",
         parts: [
           {
             type: "file",
             mediaType: "image/png",
             url: data.image_url,
           },
         ],
       },
     };

     await this.job_store.update({
       job_id,
       status: "succeeded",
       result: result.result,
       usage: data.usage,
     });
     return {
       ...result,
       billing_ref: `image:${job_id}`,
       metadata: { user_id: job.user_id, usage: data.usage },
     };
   }

  async image_result(ctx: Context): Promise<AIImageProviderResult> {
    const job_id = String(ctx.input.job_id);
    const job = await this.job_store.get(job_id);
    if (!job) {
      return { job_id, status: "failed", error: "image result is missing" };
    }
    if (job.status === "succeeded") return { job_id, status: "succeeded", result: job.result };
    if (job.status === "failed") return { job_id, status: "failed", error: job.error };
    return { job_id, status: "running", poll_after_ms: 2000 };
  }
 }

 const imageProvider = new MyImageProvider(jobStore);

 ai.use(imageProvider.model({
   id: "image-basic",
   name: "Image Basic",
   default: ["image"],
 }));
 ```

 调用端仍然保持统一：

 ```ts
 const job = await client.ai.image_create({
   model: "image-basic",
   prompt: "A clean product photo of a ceramic mug",
   ratio: "1:1",
   quality: "standard",
   count: 1,
 });

 const current = await client.ai.image_result({ job_id: job.job_id });
 const file = current.result?.parts.find((part) => part.type === "file");
 ```

## Provider 任务协议

调用方只使用 `client.ai.image_create()` 和 `client.ai.image_result()`。Provider 实现侧统一使用三个方法：

```ts
import {
  type AIImageProviderCreateResult,
  type AIImageProviderPersistResult,
  type AIImageProviderResult,
  type Context,
} from "@downcity/city";

class LuchiImageProvider extends Provider {
  // ...
  async image_create(ctx: Context): Promise<AIImageProviderCreateResult> {
    // 创建/启动上游 job，并用你的运行环境保存需要的状态
    return { job_id: "upstream_job_id", status: "running", poll_after_ms: 2000 };
  }

  async image_persist(ctx: Context): Promise<AIImageProviderPersistResult> {
    // 由后台 worker / admin action 写入最终结果到 Provider 自己的存储
    return { job_id: String(ctx.input.job_id), status: "running", poll_after_ms: 2000 };
  }

  async image_result(ctx: Context): Promise<AIImageProviderResult> {
    // 只通过 ctx.input.job_id 读取 Provider 自己已经保存的状态
    return { job_id: String(ctx.input.job_id), status: "running" };
  }
}
```

AIService 只校验固定返回格式并原样返回。任务如何存储、如何后台执行、如何重试、图片 URL 是 HTTPS、R2 URL 还是 data URL，都由具体 Provider 实现决定。生产环境里通常由 queue / cron / admin worker 在 `image_create` 后调用 `image_persist`，不要让前端直接调用它。

 ## 自动透传

 有 `baseURL` + `envKey` 但没有 `openai` 方法时，AIService 自动生成透传 action：

 - 第三方工具的 OpenAI body 原样转发到上游
 - 上游 Response 原样返回
 - `passthroughModel` 替换 `body.model`（有则替换）

 ```ts
 // 这行不用写 — AIService 自动做
 // async openai(ctx) { return fetch(`${baseURL}/chat/completions`, ...) }
 ```

 ## 自定义 OpenAI 兼容 Handler

 当 OpenAI 兼容上游需要自定义请求或响应处理时，可以写 `openai` 方法：

 ```ts
 import {
   Provider,
   type AIProviderChargedResponse,
   type Context,
   normalizeOpenAICompatibleBody,
   readRequiredEnv,
   trimTrailingSlash,
 } from "@downcity/city";

 class OpenAICustomProvider extends Provider {
   constructor() {
     super({ id: "openai-custom", envKey: "OPENAI_API_KEY", baseURL: "https://api.openai.com/v1" });
   }

   async openai(ctx: Context): Promise<AIProviderChargedResponse> {
     const apiKey = readRequiredEnv(ctx, this.envKey ?? "");
     const body = normalizeOpenAICompatibleBody(
       ctx.input as Record<string, unknown>,
       this.passthroughModel ?? "",
     );
     const response = await fetch(`${trimTrailingSlash(this.baseURL ?? "")}/chat/completions`, {
       method: "POST",
       headers: { authorization: `Bearer ${apiKey}`, "content-type": "application/json" },
       body: JSON.stringify(body),
     });
     return { response };
   }
 }
 ```

 ## Provider 出账

当 Provider 或某个具体 model 需要把一次完成的 AI 调用转换成账单草稿时，使用 `bill(ctx, output)`。`bill` 不直接扣余额；AIService 会读取它返回的 charge line，并通过配置的 `BalanceService` bridge 调用 `balance.charge()`。

图片任务的扣费只会在 `image_persist()` 返回 `status: "succeeded"` 后触发。`image_create()` 不扣费，`image_result()` 只读取已保存状态。如果后台 persist 没有当前用户上下文，`bill()` 应该从持久化任务归属里返回 charge line 的 `user_id`。

 通常优先级是：

 1. action 显式返回 `charge`
 2. model 提供 `bill(ctx, output)`
 3. provider 提供 `bill(ctx, output)`
 4. 不产生扣费

 ```ts
 import { normalizeAIUsage } from "@downcity/city";

 class DeepSeekProvider extends Provider {
   // ...
   protected bill(ctx: Context, output: unknown) {
     const usage = output?.metadata?.usage;
     const normalized = normalizeAIUsage(usage);
     // 根据 usage 计算金额
     return {
       amount_microcredits: 0,
       note: `DeepSeek ${ctx.variant?.id}`,
       metadata: { raw_usage: usage },
     };
   }
 }
 ```

 单个 model 也可以覆盖 provider 的出账逻辑：

 ```ts
 ai.use(deepseek.model({
   id: "deepseek-v4-pro",
   name: "DeepSeek V4 Pro",
   bill(ctx, output) {
     return {
       amount_microcredits: 20_000,
       note: "DeepSeek V4 Pro request",
       metadata: {
         model_id: ctx.variant?.id,
         usage: output?.metadata?.usage,
       },
     };
   },
 }));
 ```

 也可以在 action 方法里直接返回 charge：

 ```ts
 async text(ctx: Context) {
   const result = await callUpstream(ctx.input);
   return {
     output: toUIMessage(result),
     charge: {
       amount_microcredits: priceUpstreamUsage(result.usage),
       note: "custom text",
     },
   };
 }
 ```

 ## Provider 字段与方法

 | 字段 | 必填 | 说明 |
 |---|---|---|
 | `id` | 是 | Provider 唯一 ID |
 | `env` | 否 | 环境变量声明 |
 | `baseURL` | 透传时必填 | 上游 API 地址 |
 | `envKey` | 透传时必填 | API Key 环境变量名 |
 | `passthroughModel` | 否 | 透传时替换 body.model |

 | 方法 | 说明 |
 |---|---|
 | `createClient` | 需要 text/stream 时覆盖，返回 OpenAI-compatible client |
 | `text` | SDK 文本生成 action（基类默认提供） |
 | `stream` | SDK 流式生成 action（基类默认提供） |
 | `image_create` | 创建并启动图片 provider 任务 |
 | `image_persist` | 持久化最终图片结果；成功后触发 `bill` |
 | `image_result` | 读取图片 provider 任务结果 |
 | `video` | SDK 视频生成 action |
 | `openai` | /chat/completions action（未提供则自动透传） |
 | `bill` | 为本次完成的调用返回账单草稿 |
 | `extractPrompt` | 从输入提取 prompt（可覆盖） |
