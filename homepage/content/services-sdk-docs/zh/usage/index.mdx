`@downcity/services` 用来记录 City 里发生过哪些 service 调用。它只记录事实，不替你定义价格、套餐或扣费规则。

它的价值在于先把“发生了什么”沉淀下来，再由你决定这些事实后面要接报表、额度还是计费。

## 这组文档怎么读

建议按下面顺序读：

1. 先读 [事件模型](/zh/services-sdk-docs/usage/event-model)，理解一条 usage event 里会记录哪些字段。
2. 再读 [查询与汇总](/zh/services-sdk-docs/usage/querying-and-summary)，理解管理侧如何读取事件和聚合结果。
3. 最后读 [与计费集成](/zh/services-sdk-docs/usage/billing-integration)，理解 usage 和支付、套餐、额度之间的关系边界。

## 它记录什么

- downcity
- user
- service
- model
- 调用状态

## 它不替你做什么

- 不直接扣费
- 不定义价格和套餐
- 不自动决定哪些调用应该收费

## 什么时候用它

- 你要统计调用量
- 你要把 usage 送进计费系统
- 你要先把调用事实沉淀下来，再决定产品怎么收费

## 安装

```bash
pnpm add @downcity/services
```

## 启用方式

```ts
import { UsageService } from "@downcity/services";

base.use(new UsageService({
  record_errors: true,
}));
```

## 常见查询

```ts
const events = await admin.service("usage").get("events");
const summary = await admin.service("usage").get("summary");
```

## 最常见的组合方式

- 和 [@downcity/city](/zh/city-sdk-docs/packages/city/city-runtime) 配合，在可信环境查询 usage
- 和 [@downcity/services](/zh/services-sdk-docs) 配合，把 usage 与权益或账单逻辑衔接起来
- 和你自己的产品数据库或 BI 系统配合，做更细的报表
