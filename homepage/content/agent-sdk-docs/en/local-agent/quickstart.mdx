# Local Agent Quickstart

## Install

```bash
pnpm add @downcity/agent
```

## Minimal example

```ts
import { Agent } from "@downcity/agent";
import { createOpenAI } from "@ai-sdk/openai";

const openai = createOpenAI({
  apiKey: process.env.OPENAI_API_KEY!,
});

const agent = new Agent({
  id: "repo-helper",
  path: "/path/to/project",
  tools: {},
  model: openai.responses("gpt-5"),
});

const session = await agent.sessions.create();
const turn = await session.prompt({
  query: "Summarize the current repository structure",
});

const result = await turn.finished;
console.log(result.text);
```

## The three most important steps

1. `new Agent(...)`
2. `await agent.sessions.create()`
3. make sure the Agent or Session has a model

Step 3 is the easiest one to forget, and it is also the most common local SDK mistake.

You can satisfy step 3 in either way. Runtime resolution always prefers the Session model, then falls back to the Agent model:

- pass a model to `new Agent({ model })`
- call `await session.set({ model })` for one specific local session

If you skip step 3 entirely, the local session has no model and cannot execute.

When you are using the SDK for the first time, avoid stacking complex tools, managed plugins, extra plugins, or Downcity HTTP exposure immediately. Get the shortest execution path working first.

## Agent lifecycle

After `new Agent(...)`, plugin lifecycle and ActionSchedule start automatically. Most callers use sessions directly:

```ts
const session = await agent.sessions.create();
const turn = await session.prompt({ query: "hello" });
await turn.finished;
```

Session entry points already wait for the current Agent's background services internally; model execution starts after plugin lifecycle is ready. You can still call `await agent.ready()` as an explicit checkpoint, for example before inspecting plugin state right after construction or before exposing the Agent over RPC / HTTP.

When you are done, call `await agent.dispose()` to stop plugin lifecycle, ActionSchedule, and shell.

If you need to expose the agent to other processes over RPC or HTTP, use the separate package [`@downcity/server`](/en/agent-sdk-docs/local-agent/http-and-rpc). The Agent itself no longer starts any transport.
