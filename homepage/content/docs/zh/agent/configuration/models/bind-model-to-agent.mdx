# 把模型绑定到 Agent

项目和模型之间的绑定点是：

```json
{
  "execution": {
    "type": "api",
    "modelId": "quality"
  }
}
```

## 修改方式

你可以直接编辑 `downcity.json`，也可以通过相关模型管理命令来辅助完成。

不管通过哪种方式，最终落到项目上的结果都应该是同一件事：

- `execution.type = "api"`
- `execution.modelId = "<some-id>"`

## 修改后要做什么

通常建议：

```bash
city agent restart
```

这样最稳妥。

## 一个常见错误

把项目绑定到一个不存在的 `modelId`。

这会导致启动或执行失败。

如果你遇到这种问题，通常不会等到“多轮对话很奇怪”才暴露，而会更早地出现在启动或首次执行阶段。
