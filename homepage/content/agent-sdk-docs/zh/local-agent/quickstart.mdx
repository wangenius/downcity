# 本地 Agent 快速开始

## 安装

```bash
pnpm add @downcity/agent
```

## 最小示例

```ts
import { Agent } from "@downcity/agent";
import { createOpenAI } from "@ai-sdk/openai";

const openai = createOpenAI({
  apiKey: process.env.OPENAI_API_KEY!,
});

const agent = new Agent({
  id: "repo-helper",
  path: "/path/to/project",
  tools: {},
  model: openai.responses("gpt-5"),
});

const session = await agent.session();
const turn = await session.prompt({
  query: "总结一下当前仓库结构",
});

const result = await turn.finished;
console.log(result.text);
```

## 最重要的三步

1. `new Agent(...)`
2. `await agent.session()`
3. 确保 session 有 model

这三步里第 3 步最容易被遗漏，也是本地 SDK 最常见的踩坑点。

第 3 步有两种做法：

- 在 `new Agent({ model })` 里传默认模型
- 或者对某个 session 单独调用 `await session.set({ model })`

如果两种都没做，本地 session 没有模型，无法执行。

第一次上手时，建议先不要叠加复杂 tools、services、plugins 或 HTTP 暴露，先把最短执行链跑通。

## 什么时候需要 `agent.start()`

`new Agent(...)` 只是在当前进程里创建一个本地 agent 实例。

如果你只是想在宿主代码里直接跑 session：

```ts
const session = await agent.session();
const turn = await session.prompt({ query: "hello" });
await turn.finished;
```

通常不需要调用 `agent.start()`。

只有在你需要这些长期运行能力时，才需要调用 `agent.start(...)`：

- 启动 HTTP 端口，让别的进程通过 `RemoteAgent` 调用它
- 启动本地 RPC 端口，让宿主或其他本机进程通过 RPC 调用它
- 启动 plugin lifecycle，让 chat、schedule 或其他后台 loop 真正开始运行

可以把它简单理解成：

- 不 `start`：把 Agent 当本地库直接调用
- `start`：把 Agent 变成一个会对外服务、会跑后台能力的长期运行实例
