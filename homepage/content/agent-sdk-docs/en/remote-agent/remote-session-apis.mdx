# Remote Session API

Remote Sessions provide `get_info()`, `prompt()`, `stop()`, `subscribe()`, `messages()`, `system()`, approval methods, and `fork()`.

```ts
const session = await agent.sessions.create();

const unsubscribe = session.subscribe(apply_mutation);
const turn = await session.prompt({ query: "Continue" });
await turn.finished;

const page = await session.messages();
unsubscribe();
```

The subscription emits `message | part | delta | turn | session`. An approval request is an approval-required Tool Part: read a local tool's complete request from `mutation.part.approval.request` and resolve it with `session.resolve_approval(...)`.

Remote clients do not select models through the Agent SDK. The server host owns model IDs, defaults, recovery, and runtime instance injection.
