# Skills

Skills are reusable instructions packaged around `SKILL.md`.

They let an Agent learn a repeatable workflow without hard-coding that workflow into the core runtime.

## Discovery Roots

Downcity can discover skills from:

- project-level skill roots
- user-level skill roots
- extra roots declared by the skill plugin configuration

Project-level roots are useful when the workflow belongs to one repository. User-level roots are useful when the workflow should follow the operator across projects.

## Runtime Semantics

Skills are selected before execution and then injected into the Agent context.

That means the runtime can keep the base Agent general, while still adding precise procedural knowledge when a task calls for it.

## Important Boundary

Installed skills are available knowledge. They are not automatically trusted actions.

The Agent still has to respect each skill's declared tool boundaries and the active permission model.
