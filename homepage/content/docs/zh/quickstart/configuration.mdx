# 配置

ShipMyAgent 的核心配置由 4 部分组成：

- `PROFILE.md`：Agent 角色与行为边界（必需）
- `SOUL.md`：稳定执行原则（可选）
- `USER.md`：用户偏好（可选）
- `ship.json`：运行时配置（必需）

## `ship.json` 关键字段

### `start`

控制启动参数（可被 CLI 参数覆盖）：

```json
{
  "start": {
    "port": 3000,
    "host": "0.0.0.0"
  }
}
```

### `model`

agent 绑定模型 ID（映射到 console 全局模型池）：

```json
{
  "model": {
    "primary": "default"
  }
}
```

### `services.chat`

聊天通道与队列控制：

```json
{
  "services": {
    "chat": {
      "method": "direct",
      "queue": {
        "maxConcurrency": 2,
        "mergeDebounceMs": 600,
        "mergeMaxWaitMs": 2000
      },
      "channels": {
        "telegram": {
          "enabled": true,
          "botToken": "${TELEGRAM_BOT_TOKEN}",
          "auth_id": "123456789"
        }
      }
    }
  }
}
```

`services.chat.queue` 说明：
- `maxConcurrency`：不同 chatKey lane 的全局并发上限。
- `mergeDebounceMs`：连续入站消息的短防抖窗口，窗口内消息会尽量并入同一次 run。
- `mergeMaxWaitMs`：合并等待的硬上限，避免持续来消息导致无限延迟。

### `services.skills`

配置 skills 扫描目录：

```json
{
  "services": {
    "skills": {
      "paths": [".agents/skills"],
      "allowExternalPaths": false
    }
  }
}
```

### `extensions.*`（说明）

extension 配置不在项目 `ship.json` 里维护，而是统一在 console 全局 `~/.ship/ship.json` 中维护。

例如 voice：

```bash
sma voice on SenseVoiceSmall
sma console config get extensions.voice
```

### `permissions`

仓库写入与命令执行权限：

```json
{
  "permissions": {
    "read_repo": true,
    "write_repo": { "requiresApproval": false },
    "exec_command": {
      "deny": ["rm"],
      "requiresApproval": false,
      "maxOutputChars": 12000,
      "maxOutputLines": 200
    }
  }
}
```

## 常用命令

```bash
sma console config get
sma console config get model.primary
sma console model list
sma console config set start.port 3100
sma console config unset services.chat.channels.telegram
```
