# `sma task`

任务系统命令组。

> 层级说明：`sma task` 是 service 动作命令，不是一级命令。一级入口是 `sma service`。

## 用法

```bash
sma task <subcommand> [options]
```

## 子命令

- `list`
- `create`
- `run <taskId>`
- `update <taskId>`
- `delete <taskId>`
- `status <taskId> <enabled|paused|disabled>`
- `enable <taskId>`
- `disable <taskId>`

## `create` 关键参数

- `--title <title>`（必填）
- `--description <description>`（必填）
- `--context-id <contextId>`（必填，任务执行上下文标识）
- `--cron <cron>`（默认 `@manual`）
- `--activate`（等同 `--status enabled`）
- `--kind <agent|script>`
- `--time <ISO8601>`（带时区；建议配合 `--cron @manual`）

## `update` 常用参数

- `--cron` / `--timezone` / `--status`
- `--required-artifact <path>`（可重复）
- `--clear-required-artifacts`
- `--min-output-chars` / `--clear-min-output-chars`
- `--max-dialogue-rounds` / `--clear-max-dialogue-rounds`

## 示例

```bash
sma task list
sma task create --title "Daily check" --description "Run tests" --context-id telegram-chat-123 --activate
sma task run daily-check
sma task update daily-check --cron "0 10 * * 1-5"
sma task disable daily-check
```

## 说明

任务调度依赖运行中的 runtime：

```bash
sma agent on
```

任务执行完成后不会自动回发到 chat channel，结果请在任务 run 目录查看。
