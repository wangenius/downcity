# `memory` Plugin

`memory` 是 agent 的记忆 plugin。

它提供：

- memory 存储
- memory 检索
- memory 文件读取
- session 到 memory 的刷写
- memory system text

## 主要形状

- `lifecycle`
- `actions`
- `system`

## 它主要解决什么问题

当你需要这些能力时，应该关注 `memory`：

- 可检索的 long-term 或 daily memory
- 显式写入记忆
- 读取记忆文件片段
- 把某个 session 的最近消息刷写进 memory

## 用户怎么使用

普通本地使用直接走 `town memory` 快捷命令：

```bash
town memory status
town memory search "project decision" --max-results 5
town memory store --content "Prefer concise release notes." --target longterm
town memory get .downcity/memory/daily/2026-06-06.md --from 1 --lines 20
town memory flush --session-id <sessionId> --max-messages 30
```

`store` 支持 `longterm`、`daily`、`working`。写入 `working` 时还要传 `--session-id`，这样 memory 才能绑定到正确会话。

## SDK action 用法

```ts
await agent.plugins.runAction({
  plugin: "memory",
  action: "store",
  payload: {
    content: "Prefer concise release notes.",
    target: "longterm",
  },
});
```

action 名称是 `status`、`search`、`get`、`store`、`flush`。

## 关键语义

- runtime state 归属于 plugin 实例
- memory 是否启用，和代码里是否存在这个 plugin 不是一回事
- 部分 action 具备 session-aware 语义，尤其是 `flush`

## 公开性说明

`MemoryPlugin` 在代码里是 built-in，但更偏平台 runtime plugin，不属于包根主公开的 SDK plugin export。
