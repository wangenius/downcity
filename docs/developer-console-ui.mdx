# 1. 项目基本情况

当前仓库包含两层与 Console UI 相关的实现：

- packages/downcity/：CLI、console daemon、agent runtime、API 网关。
- console-ui/：独立前端包（Vite + React），构建输出到 packages/downcity/public。

设计目标：

- 前端独立开发、独立构建。
- city console ui start 只负责托管构建产物并代理 API。
- 避免把页面逻辑继续写在 packages/downcity/public 的手工静态脚本中。

---

开发环境

1. 在 console-ui/ 开发 React（vite dev 热更新）。
2. API 仍走 console ui 网关（或 Vite proxy 到网关），请求 /api/*。
3. 目标是快速迭代 UI，不依赖每次打包 package。

生产环境

1. 发布前先构建 console-ui：vite build。
2. 构建产物输出到 packages/downcity/public（已在 vite.config.ts 配好）。
3. 用户执行 city console ui start 时，ConsoleUIGateway 直接托管这些静态文件并代理 /api/*。
4. 不需要用户机器再跑 Vite 或前端构建。

一句话：  

开发用 vite dev，生产用“预构建静态产物 + gateway 托管”。

# 2. 目录与职责

# 2.1 前端包

- console-ui/index.html：前端入口模板。
- console-ui/src/main.tsx：React 启动入口。
- console-ui/src/App.tsx：页面主组件（当前为最小可运行版本）。
- console-ui/vite.config.ts：构建配置，outDir 指向 ../packages/downcity/public。

# 2.2 后端网关

- packages/downcity/src/console/ui/ConsoleUIGateway.ts托管 packages/downcity/public 静态资源（含 Vite assets/）。提供 /api/ui/agents。代理 /api/ 到当前选中 agent runtime。提供 SPA fallback（未知前端路由回退 index.html）。

---

# 3. 构建与发布逻辑

仓库级构建顺序（scripts/build.sh）：

1. 先构建 console-ui（生成 packages/downcity/public）。
2. 再构建 package（tsc + tsc-alias + copy prompt assets）。
3. 最后全局安装 package（当前脚本行为）。

核心点：

- console-ui 的 dist 不单独发布目录，而是直接写入 packages/downcity/public，保证 npm 包里天然包含 UI 产物。
- 生产环境不依赖用户本地再执行 Vite 构建。

---

# 4. 运行时调用链路

以浏览器打开 http://127.0.0.1:3001 为例：

1. 用户执行 city console ui start。
2. CLI 启动 Console UI Gateway 进程。
3. 浏览器访问 /：Gateway 读取 packages/downcity/public/index.html 并返回。
4. 前端加载 /assets/*：Gateway 直接返回对应静态文件。
5. 前端请求 /api/ui/agents：Gateway 返回 console registry 中可用 agent 列表。
6. 前端请求 /api/tui/ 或 /api/services/：Gateway 选择当前 agent。转发请求到该 agent runtime（默认 http://127.0.0.1:3000 或按 registry/daemon meta 解析）。

---

# 5. 前端 API 调用约定

前端统一通过同域 /api/* 调用，不直接跨域访问 agent 端口。

常用接口：

- /api/ui/agents：获取 agent 列表与默认选中项。
- /api/tui/overview：概览数据。
- /api/tui/services：service 状态。
- /api/services/command：执行 service action（例如 chat status/test/reconnect）。

参数约定：

- 多 agent 场景可带 ?agent=<projectRoot> 指定目标。
- Gateway 会剔除该 query 并转发到上游 runtime。

# 6. 常见问题与排查

# 6.1 city console ui start 后页面 503

原因通常是 packages/downcity/public/index.html 不存在。  

处理：

1. 先执行仓库级 npm run build。
2. 确认 console-ui 已安装依赖并可构建。

# 6.2 前端资源 404

检查：

- console-ui 是否构建成功。
- packages/downcity/public/assets/* 是否生成。
- Gateway 是否为新版（支持 assets/* + SPA fallback）。

# 6.3 API 显示连接失败

检查：

- city console status / city console ui status。
- 目标 agent 是否在线（city agent status）。
- UI log 与 agent log 中具体 404/500 路径。
