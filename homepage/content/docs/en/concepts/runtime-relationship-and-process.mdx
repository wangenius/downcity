# Runtime Relationship & Process

## 1. Role Summary

- `console`: manages registry, model pool, shared storage, and UI
- `agent`: owns one project runtime
- `service`: owns the main workflow and lifecycle
- `plugin`: provides optional enhancement through service-defined points
- `plugin`: handles its own dependency resolution and config internally

In one sentence:

`console` governs, `agent` executes, `service` orchestrates, and `plugin` augments while owning its dependencies.

## 2. Startup Order

1. `city console start`
2. `city agent start`
3. agent loads project config from `downcity.json`
4. runtime assembles the shared execution context and registers service/plugin infrastructure
5. plugins register `pipeline / guard / effect / resolve` handlers on service-defined points
6. messages or tasks enter a service workflow
7. services trigger those plugin points at the right stages
8. runtime traces are written into `.downcity/*`
