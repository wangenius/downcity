# 调用路由与隔离

## 简短答案

- 一个 agent runtime 只绑定一个项目根目录。
- `service`、`plugin`、`capability` 默认都在这个 runtime 内执行。
- 只有显式改了 host/port 或 target project，才会跨 agent 路由。

## 主调用链

### 1）CLI -> service / plugin

1. 解析目标项目
2. 解析 daemon endpoint
3. 调 `/api/services/*` 或 `/api/plugins/*`
4. 在当前 runtime 内分发

### 2）service -> capability

service 应该调 capability，而不是直接耦合 plugin 私有实现。

- capability 调度发生在进程内
- 复用同一份 runtime state
- 默认不经过 shell

## endpoint 解析优先级

1. CLI 显式 `--host/--port`
2. `DC_SERVER_HOST/DC_SERVER_PORT`
3. `DC_CTX_SERVER_HOST/DC_CTX_SERVER_PORT`
4. 当前项目 runtime metadata
5. 默认 `127.0.0.1:5314`
