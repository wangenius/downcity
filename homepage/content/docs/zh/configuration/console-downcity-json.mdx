# Console 配置存储（downcity.db）

从当前版本开始，Console 不再使用：

1. `~/.downcity/downcity.json`
2. `~/.downcity/.env`

Console 全局配置统一存储在：

- `~/.downcity/downcity.db`

## 存储内容

`downcity.db` 中主要包含以下数据域：

1. 模型池（providers/models）
2. Console 侧管理的 env 记录（`env_entries`）
4. Bot 账号凭据池（`channel_accounts`）
5. Chat 用户授权（`chat_auth_roles`、`chat_auth_role_permissions`、`chat_auth_channel_defaults`、`chat_auth_user_roles`）

其中敏感字段（如 API Key）采用加密存储。

## 与 Agent 项目配置的边界

以下文件仍然是 Agent 项目级配置（保留不变）：

1. `<agent>/downcity.json`
2. `<agent>/.env`

也就是：

- Console：`~/.downcity/downcity.db`
- Agent：项目目录下的 `downcity.json/.env`

## 环境变量作用域（重要）

当前版本按明确分层工作：

1. 宿主显式注入 env：`new Agent({ env })`
2. Agent 本地运行时覆盖：`<agent>/.env`（用户维护）
3. Chat 渠道凭据：`channel_accounts`（由 `downcity.json` 的 `channelAccountId` 绑定）
4. Chat 用户授权：`chat_auth_*` 结构化表（city 全局生效）

关键行为：

1. Agent 运行时 env 最终是 `new Agent({ env }) + <agent>/.env`（`.env` 优先）。
2. 当宿主是 `city` 时，会先从 `downcity.db` 读取 Console 全局 env，再显式传给 `Agent`。
3. `downcity.json` 只保存渠道绑定（`channelAccountId`），不保存明文密钥。
4. 渠道真实凭据来自 `channel_accounts`。
5. 用户聊天权限来自 `chat_auth_*` 表，不保存在 agent 项目里。
6. plugin 参数来自项目 `downcity.json`，plugin 启停属于 Console 全局配置。

## 常用操作

初始化 Console（写入 `downcity.db`）：

```bash
city init
```

查看项目 plugin 参数：

```bash
city config get plugins.asr
city config set plugins.asr.modelId '\"SenseVoiceSmall\"'
```

切换 city 级 plugin 启停：

```bash
city asr on
city asr off
```

管理模型池：

```bash
city model list
city model create
```
