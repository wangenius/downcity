# 项目结构

Agent 的管理配置统一保存在全局数据库。项目目录只保留项目环境、Skills 和运行数据，不保存配置副本。

## 目录布局

```text
my-project/
├── .env                 # 项目环境变量
├── .agents/
│   └── skills/          # 用户管理、可进入版本控制的 Skills
├── .downcity/           # Agent 与 Plugin 运行数据
│   ├── agents/          # 会话与消息数据
│   ├── chat/            # Chat Access、路由和历史数据
│   ├── logs/            # 运行日志
│   ├── memory/          # 记忆数据
│   └── task/            # 任务运行数据
└── src/                 # 业务代码（可选）
```

## 全局配置

默认保存在 `~/.downcity/downcity.db`，设置 `DC_PLATFORM_ROOT` 后改为 `$DC_PLATFORM_ROOT/downcity.db`。其中包含：

- Agent ID、项目路径和启动参数
- 模型绑定与 Plugin 配置
- Chat channel account 和 Agent 渠道绑定
- City 登录态、全局环境变量和其他平台管理状态

CLI、交互管理器和 Agent daemon 都读取同一份数据库，不维护项目配置副本。

## 项目资产

`.env` 的优先级高于平台全局环境和当前 CLI 进程环境，并默认加入 `.gitignore`。`.agents/skills` 是项目中唯一由用户管理并进入版本控制的 Agent 能力资产。

## 运行数据

`.downcity/` 只保存会话、Chat Access、任务、记忆、日志和缓存等运行数据。不要把管理配置写入该目录。
