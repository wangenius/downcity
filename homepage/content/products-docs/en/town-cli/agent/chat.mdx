# `town agent chat`

Chat with a target agent from the terminal.

Use this command after the target agent is running.

This command uses a dedicated local CLI main session: `local-cli-chat-main`.
Interactive mode lets you choose an existing session or create a fresh one before the TUI opens.
Use `--new-session` when you want to skip the picker and start with a fresh conversation context.

## Usage

```bash
town agent chat [--to <id>] [--session-id <sessionId>]
town agent chat [--to <id>] --new-session
town agent chat [--to <id>] --message <text> [--json] [--session-id <sessionId> | --new-session]
```

## Key options

- `-t, --to <id>`: target agent id; if omitted, the CLI opens an interactive picker from running agents
- `-m, --message <text>`: send one message and exit instead of entering the REPL
- `--session-id <sessionId>`: enter or reuse a specific CLI chat session
- `--new-session`: create a fresh isolated CLI chat session before sending messages
- `--json`: print structured output for one-shot mode
- `--host <host>`: override the auto-resolved RPC host

## Behavior

- Defaults to interactive terminal chat when `--message` is omitted.
- In interactive mode, once you pick an agent, the CLI asks you to choose an existing session or create a new one.
- Pass `--session-id` or `--new-session` to skip the interactive session picker.
- Inside the TUI, type `/session` to switch sessions or `/new` to create and switch to a new session.
- Once you enter the chat loop, the terminal shows spinner-based progress states.
- When the agent runs tools, interactive mode shows the tool name, a compact input summary, and the completion state.
- One-shot `--message` mode keeps the simpler output flow and does not render the interactive spinner / tool status blocks.
- One-shot mode works without entering the REPL.
- Requires the target agent to be running.
- Keeps sending every line into the selected local CLI session instead of sharing Console UI context.
- `--new-session` generates a `local-cli-chat-*` session ID and the TUI title shows the active session.
- Type `/exit` or `/quit` to leave the chat loop.

## If you get no reply

Check the runtime before debugging prompts or sessions:

1. [`town agent status`](/en/products-docs/town-cli/agent/status)
2. model binding in `downcity.json.execution.modelId`
3. whether the selected target is actually running

If the problem is a polluted terminal session, use [`town agent history clean`](/en/products-docs/town-cli/agent/history-clean) only when you know the exact session to reset.
