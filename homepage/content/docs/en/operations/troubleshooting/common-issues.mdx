# Common Issues

## Command not found (`town: command not found` / `town: command not found`)

```bash
npm config get prefix
echo $PATH
export PATH=$(npm config get prefix)/bin:$PATH
rehash
town --version
```

Or run directly with:

```bash
town --version
```

Notes:

- `town` and `city` come from the same `downcity` package but expose different command surfaces.
- New installs should not require manual `chmod +x`.
- If `sudo town ...` fails but `town ...` works, it is usually a `sudo` PATH issue rather than an install failure.

## Port conflict (5314 already in use)

```bash
lsof -nP -iTCP:5314 -sTCP:LISTEN
town agent doctor . --fix
town agent restart .
```

## `town task/chat/...` cannot connect

Make sure daemon is running:

```bash
town agent start
town plugin list
```

## LLM configuration mismatch

Check:

```bash
town model list
town config get execution.modelId
```

Ensure:

1. `execution.modelId` points to an existing model id in `~/.downcity/downcity.db`
2. console-global SQLite providers/models entries are valid
3. For Kimi (Moonshot), use `provider.type` as `moonshot-cn` or `moonshot-ai`
4. `moonshot-cn` defaults to `https://api.moonshot.cn/v1`
5. `moonshot-ai` defaults to `https://api.moonshot.ai/v1`
6. Use a model name that matches the target endpoint, such as `kimi-k2-0905-preview` or `kimi-k2.5`
