# 工作区架构总览

这个仓库现在围绕两个 runtime package，再向外展开多个产品面。

## 主要工作区

- `packages/agent/`：本地 agent 内核、session 执行、plugin runtime、HTTP/RPC 宿主、SDK
- `cli/city/`：City 管理命令，负责连接和管理已部署的 City 服务资源
- `cli/studio/`：本机 Agent 宿主、Console 网关、模型池、项目编排与 daemon 控制
- `packages/city/`：可复用 City runtime、service/action、auth、env 与 studio 访问边界
- `packages/ui/`：可复用 UI 基础组件与应用侧组件
- `products/console/`：浏览器里的控制台
- `products/chrome-extension/`：把网页上下文送入 agent 的扩展
- `homepage/`：官网、用户文档、开发文档、SDK 文档与 plugin 文档

## 边界规则

- `/docs` 只写用户文档
- `/devdocs` 只写协作者文档
- `packages/agent` 负责单 agent 执行
- `cli/city` 负责 City 服务管理
- `cli/studio` 负责本机控制面与多项目协调
- 产品界面只消费 runtime API，不复制 runtime 逻辑

## 当前统一架构哲学

- `console` 负责全局治理
- `agent host` 负责单项目 runtime
- `session` 负责执行
- `plugin` 负责能力暴露与增强
- 可追踪状态尽量贴近真实工作区

## 构建关系

```text
packages/agent build -> SDK 与 runtime package
cli/city build -> City 管理命令
cli/studio build -> 本机 Studio 宿主命令
products/console build -> 浏览器控制台
products/chrome-extension build -> 扩展 bundle
homepage build -> 官网与文档站
```
