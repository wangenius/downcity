# `sma voice`

`voice` extension 命令组，用于统一管理本地语音识别模型与 `extensions.voice` 配置。

## 用法

```bash
sma voice <subcommand>
```

## 子命令

- `models`：列出内置模型目录
- `status`：查看当前 `extensions.voice` 配置
- `init [models...]`：一键初始化（启用 + 安装模型 + 自动安装转写依赖）
- `on [models...]`：启用 voice extension；默认安装所选模型
- `off`：关闭 voice extension（保留已安装记录）
- `install [models...]`：仅安装模型，不改变启用状态
- `use <modelId>`：切换激活模型
- `transcribe <audioPath>`：转写本地音频文件（联调用）

## 示例

```bash
# 1）从零初始化（推荐）
sma voice init SenseVoiceSmall

# 2）显式安装到指定目录
sma voice install SenseVoiceSmall --models-dir ~/.ship/models/voice

# 3）只写配置，不下载
sma voice on SenseVoiceSmall --no-install

# 4）切换激活模型
sma voice use whisper-large-v3-turbo

# 5）本地音频转写测试
sma voice transcribe .ship/.cache/telegram/demo.ogg --language zh
```

## 说明

- 若 daemon 正在运行，`sma voice ...` 会通过 `/api/extensions/command` 远程执行。
- 若 daemon 不可达，`sma voice ...` 会自动降级为本地执行（同样会写入 `ship.json` 并执行模型安装）。
- `voice init` 默认会自动安装 Python 转写依赖（`funasr` 或 `transformers/torch`，按模型自动选择）。
- 若系统 Python 命中 PEP 668（不允许全局 pip install），`voice init` 会自动回退到 `~/.ship/venvs/voice` 虚拟环境安装依赖。
- 模型默认下载到 `~/.ship/models/voice`（可通过 `--models-dir` 覆盖）。
- 内置转写 runner 依赖本地 Python 包（`funasr` / `transformers` 至少其一）。
- Telegram 收到 voice/audio 附件时，会通过该 extension 的 `transcribe` action 做 best-effort 转写。
