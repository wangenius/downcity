# 渠道账号

聊天平台的机器人凭证（Telegram、飞书、QQ）由 City 存储和管理，而不是放在每个 Agent 项目中。这保持密钥集中管理，并便于在多个项目之间复用同一个机器人。

## 工作原理

1. 用户通过控制台或 CLI 添加机器人凭证（`downcity chat add`）
2. City 将凭证安全存储在 `~/.downcity/downcity.db` 中
3. 凭证获得一个 `channelAccountId`
4. 项目通过 ID 在 `downcity.json` 中引用该凭证

## 支持的平台

- Telegram
- 飞书 (Lark)
- QQ

## 安全

- 凭证在静态时加密
- 项目永远不会看到原始 token
- 撤销会立即影响所有使用该账号的项目

## 配置

```json
{
  "plugins": {
    "chat": {
      "channels": {
        "telegram": {
          "enabled": true,
          "channelAccountId": "telegram-main"
        }
      }
    }
  }
}
```
