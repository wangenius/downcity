# `sma console ui`

Manage the UI gateway hosted by `console`.

This UI is not bound to one `agent on` process. One page can switch across multiple running agents.
It is a state-first dashboard; chat is only a secondary capability.

## Usage

```bash
sma console ui [action] [options]
```

## Key options

- `-p, --port <port>`: UI port (default `3001`)
- `-h, --host <host>`: UI host (default `127.0.0.1`)

## action

- `start`: start UI in background (default)
- `stop`: stop background UI
- `restart`: restart background UI
- `status`: show UI status

## Behavior

- Requires console first: `sma console start`.
- On startup, UI loads active agents from console registry.
- Global Agents panel shows each agent's runtime endpoint, current model binding, and chat identity snapshots (e.g. Telegram bot username), and supports direct model switching from the table.
- Global / Overview shows console + selected agent config file status (exists/readable/updated time).
- Global / Model now supports full provider/model operations (create, update, test, discover, pause, remove).
- Agent switcher in top bar controls all `Contexts/Services/Tasks/Logs` views.
- Agent / Tasks now includes task definition details and per-run execution history (run list + run detail).
- Dashboard shows `Chat Channels` link status (Telegram/Feishu/QQ) with `test` and `reconnect` actions.
- Chat input is fixed to `local_ui` context only; UI cannot publish instructions to arbitrary contexts.
- UI shows grouped `system prompt` composition (static/service/runtime).
- `sma console status` also shows UI status.

## Typical flow

```bash
# 1) start console
sma console start

# 2) start multiple agents
sma agent on /path/to/agent-a
sma agent on /path/to/agent-b

# 3) start UI in background
sma console ui start --port 3001

# 4) check UI status
sma console ui status

# 5) restart UI (reload latest gateway code/routes)
sma console ui restart

# 6) stop UI
sma console ui stop
```
