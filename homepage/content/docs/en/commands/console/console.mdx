# `city console`

`console` is the global control plane.

Use it to manage console runtime and inspect managed agent runtimes.

## Command hierarchy

Top-level command groups are:

- `city console`
- `city agent`
- `city service`
- `city plugin`

Under `console`, use `init/model/config/start/stop/status/agents/ui`.

## Usage

```bash
city console init
city console model <subcommand>
city console config <subcommand>
city console start
city console stop
city console restart
city console status
city console agents [--json]
city console ui [start|stop|restart|status] [--port 5315]
```

## Behavior

- `init`: initialize global console config (`~/.ship/*`).
- `model`: dedicated management for console-global language model pool (provider/model/test/pause).
- `config`: manage project `ship.json` and alias.
- `start`: start console in background.
- `stop`: stop console and managed child agent runtimes.
- `restart`: restart console and automatically restore managed agents that were running before restart.
- `status`: print console process state and active managed agents.
- `agents`: print active managed agents list.
- `ui`: manage single UI instance (start/stop/restart/status) and switch across agents in one page.

## Typical Flow

```bash
# 1) start global console
city console start

# 2) start project runtime
cd /path/to/agent-project
city agent create .
city agent start

# 3) check status
city console status
city agent status .
```
