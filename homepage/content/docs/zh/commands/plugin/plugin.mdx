# `bay plugin`

`bay plugin` 是当前 plugin 系统的通用入口。

> 推荐先看：plugin 概念层文档在 [Agent SDK / Plugins](/zh/agent-sdk-docs/plugins/overview)，具体手册在 [Plugins Docs](/zh/plugins-docs)。

适合这些场景：

- 查看当前注册了哪些 plugin
- 检查某个 plugin 的静态启用态与能力面
- 面向指定 agent 启动、停止、重启托管 plugin
- 不走短别名，直接转发托管 plugin command
- 在当前本地项目内直接执行本地 plugin action
- 查看或取消一次性 plugin 调度任务

## 常用示例

```bash
bay plugin
bay plugin list
bay plugin status asr
bay plugin start chat --agent my-agent
bay plugin command task run --payload '{"title":"daily-check"}' --agent my-agent
bay plugin action asr status
bay plugin schedule list --path /path/to/agent
```

## 说明

- 在交互式终端里直接运行 `bay plugin`，会进入 plugin 管理器。
- `bay plugin list` 和 `bay plugin status` 是静态 console-level catalog 视图。
- `chat`、`task`、`memory`、`contact`、`shell` 这类托管 plugin 支持 `start/stop/restart/command`。
- `bay plugin action ...` 是本地执行形式，默认直接在当前项目里运行。
- 日常使用更推荐具体命令入口，比如 `bay chat ...`、`bay task ...`、`bay memory ...`、`bay shell ...`、`bay web ...`、`bay skill ...`、`bay asr ...`、`bay tts ...`。
