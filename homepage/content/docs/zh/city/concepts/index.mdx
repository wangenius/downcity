概念页不是额外的理论层，而是帮你先把 Downcity 的边界看清楚。

如果一开始就把它当成“另一个后端框架”或“对外统一 provider API 的网关”，后面的接入方式、产品拆分和权限边界都会走偏。

## 什么时候应该先读概念页

- 你准备让多个 AI 产品共用一套底层能力
- 你想知道 `bay_id`、`user_token` 和模型目录各自扮演什么角色
- 你不确定 City、业务后端、AI Gateway 的边界应该怎么划

如果你现在只想先跑通一条调用链，直接去 [快速开始](/docs/city/quickstart) 更合适。
