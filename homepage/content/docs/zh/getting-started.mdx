# 快速开始

在几分钟内启动并运行 ShipMyAgent。本指南涵盖从安装到第一个自主 agent 的所有内容。

## 前提条件

在开始之前，请确保您拥有：

- **Node.js** >= 18.0.0 已安装
- **代码仓库**（已初始化 Git）
- **LLM API key**（Anthropic、OpenAI 或兼容的）
- **基本终端知识**

:::info
还没有 LLM API key？从 [Anthropic](https://console.anthropic.com/) 或 [OpenAI](https://platform.openai.com/) 获取免费额度。
:::

---

## 安装

### 选项 1：全局安装（推荐）

全局安装 ShipMyAgent 以在任何项目中使用：

```bash
npm install -g shipmyagent
```

验证安装：

```bash
shipmyagent --version
```

### 选项 2：使用 npx（无需安装）

无需安装即可使用 ShipMyAgent：

```bash
npx shipmyagent <command>
```

### 选项 3：项目本地

在项目中作为开发依赖安装：

```bash
npm install --save-dev shipmyagent
```

然后使用 `npx` 或添加到 package.json 脚本：

```json
{
  "scripts": {
    "agent": "shipmyagent",
    "agent:start": "shipmyagent agent on --daemon"
  }
}
```

### 安装故障排查

**安装后"命令未找到"？**

将 npm 全局 bin 添加到 PATH：

```bash
# 查找 npm bin 路径
npm config get prefix

# 添加到 PATH（macOS/Linux）
export PATH=$(npm config get prefix)/bin:$PATH

# 添加到 ~/.bashrc 或 ~/.zshrc 以持久化
echo 'export PATH=$(npm config get prefix)/bin:$PATH' >> ~/.bashrc
```

**权限错误？**

不要对 npm 使用 sudo。而是：

```bash
# 修复 npm 权限
mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
```

## 快速开始（5 分钟）



### 步骤 1：初始化项目

导航到项目目录并运行：

```bash
cd /path/to/your/project
shipmyagent init
```

交互式设置将询问：

1. **Agent 名称**：给您的 agent 起个名字（默认为当前目录名，例如"my-assistant"）
2. **LLM 模型**：选择模型，支持 Claude Sonnet 4（推荐）、Claude Haiku、Claude 3.5 Sonnet、GPT-4、GPT-4o、DeepSeek Chat 等
3. **聊天适配器（可多选）**：可选启用 Telegram / 飞书 / QQ；**仅选中的 adapters 会写入 `ship.json.services.chat.adapters`**
4. **QQ 沙箱环境**（仅选择 QQ 时）：是否使用沙箱环境进行测试
5. **Skills 安装**（可选）：可选安装推荐的技能插件，如 Vercel 最佳实践、Web 设计指南、浏览器自动化等

这将创建：

```
your-project/
├── PROFILE.md           # Agent 系统提示词
├── SOUL.md            # 可选：核心原则提示词
├── USER.md            # 可选：用户偏好提示词
├── ship.json          # 配置文件
├── .claude/           # Claude 兼容目录
│   └── skills/        # Skills 存放位置
└── .ship/             # Agent 数据目录
    ├── task/          # 定时任务
    ├── logs/          # 操作日志
    ├── .cache/        # 缓存数据
    ├── profile/       # 用户画像记忆
    │   ├── Primary.md
    │   └── other.md
    ├── data/          # 数据存储
    ├── chat/          # 聊天记录
    ├── public/        # 公共资源
    ├── config/        # 配置文件
    ├── skills/        # 项目级技能
    ├── schema/        # JSON Schema (IDE 智能提示)
    │   └── ship.schema.json
    └── .debug/        # 调试信息
```

> **提示**：Profile 记忆系统会保存用户偏好，Schema 文件提供 IDE 自动补全。

### 步骤 2：配置 API Key

打开 `ship.json` 并添加您的 API key：

```json
{
  "llm": {
    "activeModel": "default",
    "providers": {
      "default": {
        "type": "anthropic",
        "baseUrl": "https://api.anthropic.com/v1",
        "apiKey": "${ANTHROPIC_API_KEY}"
      }
    },
    "models": {
      "default": {
        "provider": "default",
        "name": "claude-sonnet-4-5"
      }
    }
  }
}
```

**设置环境变量**：

```bash
# 临时（当前会话）
export ANTHROPIC_API_KEY="sk-ant-..."

# 永久（添加到 ~/.bashrc 或 ~/.zshrc）
echo 'export ANTHROPIC_API_KEY="sk-ant-..."' >> ~/.bashrc
source ~/.bashrc
```

:::warning
永远不要提交带有硬编码 API key 的 `ship.json`！始终使用环境变量。
:::

### 步骤 3：测试连接

验证您的 API key 有效：

```bash
shipmyagent test-llm
```

您应该看到：

```
✅ LLM 连接成功
提供商：Anthropic
模型：claude-3-5-sonnet-20241022
响应时间：245ms
```

### 步骤 4：自定义您的 Agent

编辑 `PROFILE.md` 以定义 agent 的角色（可选再编辑 `SOUL.md` 与 `USER.md` 定义稳定原则和用户偏好）：

```markdown
# Agent 角色

你是一位专注于 React 和 Node.js 的高级全栈开发人员。

## 目标

- 编写干净、可维护的代码
- 遵循最佳实践和设计模式
- 确保新功能的测试覆盖率
- 审查代码的安全问题

## 上下文

本项目使用：
- React 18 with TypeScript
- Node.js 后端使用 Express
- PostgreSQL 数据库
- Tailwind CSS 用于样式

## 约束

- 未经批准不得修改 package.json
- 所有新代码必须包含测试
- 遵循现有代码风格
- 未经明确请求不要重构
```

### 步骤 5：启动 Agent

启动您的 agent：

```bash
shipmyagent .
# `shipmyagent .`：前台启动（当前终端运行）
# `shipmyagent agent on --daemon [path]`：后台启动（终端关闭也保持运行）
```

输出：

```
🚀 正在启动 ShipMyAgent
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ 100%

✅ 配置已加载
✅ LLM 连接已建立
✅ Agent 已初始化：my-assistant
✅ 任务调度器已启动
✅ 服务器运行在 http://localhost:3000

🤖 您的 agent 现在在线！
💬 通过 Telegram 聊天（如果已配置）
📋 查看任务：shipmyagent tasks list
📊 查看日志：shipmyagent logs
```

## 第一次交互

### 通过 CLI

向您的 agent 提问：

```bash
shipmyagent chat "这个项目的目的是什么？"
```

### 通过 Telegram（如果已配置）

只需向您的 bot 发送消息：

```
嘿，你能审查最近的更改吗？
```

Agent 将分析您的仓库并响应。

- [ ] `shipmyagent --version` 显示正确版本

## 高级设置

### 自定义配置

对于非标准设置：

**Open-compatible 网关（Chat Completions）**：

```json
{
  "llm": {
    "activeModel": "default",
    "providers": {
      "gateway": {
        "type": "open-compatible",
        "baseUrl": "https://your-api.com/v1",
        "apiKey": "${LLM_API_KEY}"
      }
    },
    "models": {
      "default": {
        "provider": "gateway",
        "name": "gpt-4o-mini"
      }
    }
  }
}
```

**Open-responses 网关**：

```json
{
  "llm": {
    "activeModel": "default",
    "providers": {
      "gateway": {
        "type": "open-responses",
        "baseUrl": "https://your-api.com/v1",
        "apiKey": "${LLM_API_KEY}"
      }
    },
    "models": {
      "default": {
        "provider": "gateway",
        "name": "gpt-5.2"
      }
    }
  }
}
```

**同一项目配置多个模型**：

```json
{
  "llm": {
    "activeModel": "quality",
    "providers": {
      "anthropic_main": {
        "type": "anthropic",
        "apiKey": "${ANTHROPIC_API_KEY}"
      },
      "google": {
        "type": "gemini",
        "apiKey": "${GEMINI_API_KEY}"
      }
    },
    "models": {
      "quality": {
        "provider": "anthropic_main",
        "name": "claude-sonnet-4-5"
      },
      "fast": {
        "provider": "google",
        "name": "gemini-2.5-flash"
      }
    }
  }
}
```

## 故障排查

**问题**：Agent 无法启动

```bash
# 检查配置
shipmyagent validate

# 查看日志
shipmyagent logs --tail 50

# 运行诊断
shipmyagent doctor
```

**问题**：API 连接失败

```bash
# 测试 LLM 连接
shipmyagent test-llm --verbose

# 检查 API key
echo $ANTHROPIC_API_KEY

# 测试网络
curl https://api.anthropic.com/v1/messages
```

更多帮助，请参阅[故障排查指南](/docs/troubleshooting)。

## 词汇表

- **Agent**：管理您仓库的 AI 助手
- **Task**：自动运行的定时作业（通过 cron）
- **Prompt**：给 LLM 的指令（在 PROFILE.md / SOUL.md / USER.md / 任务中）
- **Service**：外部服务连接（Telegram、GitHub 等）
- **Permission**：控制 agent 可以做什么的规则
- **Approval**：敏感操作需要的人工确认
