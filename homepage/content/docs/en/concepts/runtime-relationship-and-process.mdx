# Runtime Relationship & Process

This page explains two things:

- how Service and Extension relate
- how processes are started and collaborate

## Service vs Extension relationship

- Service handles core runtime business domains (`chat`, `task`, `skill`, `memory`).
- Extension handles optional capability plugins (for example `voice`).
- Both share a unified action model: CLI and HTTP routes call into runtime control/command handlers.
- Agent execution may involve both:
  - Service actions for domain workflows
  - Extension actions for specialized capabilities

## Process model

1. Start manager process: `sma start`
2. Start agent runtime process in project: `sma agent on`
3. Runtime loads configured `services.*` and `extensions.*`
4. Inbound messages/tasks are scheduled
5. Agent execution triggers service/extension actions
6. Outputs and traces are persisted to `.ship/*`

## Practical checklist

```bash
sma start
sma agent on
sma service list
sma extension list
```
