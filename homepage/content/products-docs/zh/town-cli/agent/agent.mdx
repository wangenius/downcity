# `town agent`

管理 Agent 生命周期和 registry 相关操作。

这是 Town CLI 中管理项目 Agent 的命令组。Agent 的设计边界和概念模型见 [Agent 是什么](/zh/docs/agent/overview/what-is-agent)。

在交互式终端中直接运行 `town agent` 会进入 agent manager；直接运行裸 `town` 会先进入全屏 Town TUI dashboard。需要脚本化调用时，继续使用明确子命令。

## 命令层级

核心命令组为：`town start/stop/restart/status`、`town city`、`town agent`、`town plugin`，以及各个 plugin 快捷入口。

## 用法

```bash
town agent
town agent <subcommand>
```

## 版本查看

- `town -v` 只显示 town CLI 自身版本。
- `town agent -v` 会同时显示当前 town CLI 版本，以及这份 town 实际绑定的 `@downcity/agent` runtime 版本。

## 交互式管理器

交互式管理器可以完成：

- 查看已登记 agents
- 创建 agent 项目
- 启动 agent
- 选择 agent 时直接看到当前状态、名称、执行绑定和 chat account 关联
- 配置 agent id，或连接 Town 级 chat account

它不会替代下面的子命令，只是把常用本地操作收进一个引导入口。

聊天凭据属于 `town chat` 中的 Town 级 chat account。agent manager 只负责把当前 agent 连接到已有账号，即写入 `plugins.chat.channels.<platform>.channelAccountId`。如果发现 agent 绑定了不存在的 account，会自动清理这个悬空关联。

## 子命令

- `create`：创建/初始化 agent 项目
- `list`：列出已登记的 agent 项目
- `start`：启动 agent
- `stop`：停止 agent daemon
- `chat`：在终端中与 agent 对话，支持交互式和一次性模式
- `history clean`：硬清理指定 session history
- `status`：查看 agent 状态
- `doctor`：诊断状态文件，并可修复僵尸 pid/meta
- `restart`：重启 agent

## 环境变量注入

每次执行 `town` 时，CLI 会自动推导当前 agent 上下文，并在 `town` 进程及其子进程中注入：

- `DC_AGENT_ID`：当前 agent runtime id
- `DC_AGENT_PATH`：当前 agent 绝对路径

> 注意：这是进程级注入，不会直接回写你当前 shell 会话的环境变量。
