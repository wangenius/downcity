---
title: AIService
description: AIService unifies model registration, routing, LanguageModelV3, and OpenAI compatibility.
---

`AIService` is the Federation AI service. It registers `AIModelDefinition` values and owns authentication, availability, fallback, reasoning, metering, and billing.

| Concept | Purpose |
|---|---|
| `AIChannel` | Federation execution boundary for one upstream AI channel |
| `AIModelSpec` | Model declaration passed to `channel.model()` |
| `AIModelDefinition` | Complete AIService runtime model |
| `CityModelDescriptor` | Public client model catalog shape |
| `CityModel` | Client model implementing AI SDK `LanguageModelV3` |

Every language entry point converges on `AIChannel.stream(ctx, call)`:

```text
city.ai.text()          -> LanguageModelV3 -> AIChannel.stream
city.ai.stream()        -> CityModel       -> AIChannel.stream
CityModel.doGenerate()  -> CityModel       -> AIChannel.stream
chat/completions        -> OpenAI adapter   -> AIChannel.stream
```

The OpenAI-compatible endpoint only adapts protocols. It supports system/user/assistant/tool messages, image URLs, function tools, tool choice, reasoning, and JSON/SSE output. It never forwards the raw request upstream and does not require `AIChannel.openai()`.
