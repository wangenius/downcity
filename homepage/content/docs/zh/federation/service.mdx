# Service

**Service** 是 Agent 可以调用的长期运行后端能力。与工具（在 Agent 进程内运行）不同，Service 作为独立后端运行，通过 City 网络访问。

## 示例

- 知识库（文档的语义搜索）
- 搜索引擎（网页搜索、内部搜索）
- 数据库（结构化数据存储和查询）
- 文件存储（文档上传和检索）

## 注册

Service 向 City 注册，通过：

1. 声明其 API schema
2. 暴露健康和能力的端点
3. 接收一个 service ID

## 发现

Agent 通过 City 发现 Service。当 Agent 需要某种能力时，它向 City 询问可用的 Service，然后将请求路由到合适的那个。
