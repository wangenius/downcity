# Memory Service

`memory` service provides runtime memory operations.

## Responsibilities

- Status and index rebuild
- Search and read memory entries
- Store explicit memory records
- Flush conversation window into daily memory
- Inject memory service usage rules for the agent

## Integration model

`memory` is an independent service and is not embedded into the local session execution core.

- The system prompt injects a small stable longterm memory pack from `MEMORY.md` plus rules for using the memory service
- Injected memory only uses stable Canon text, with dedupe, item limits, and truncation for prompt stability
- When the agent needs more memory, it should call `memory.search`, `memory.get`, and `memory.store` itself
- There is no automatic per-turn recall / capture inside the session executor

Use `city memory` when you want to inspect or manage memory directly.

## Runtime operations

```bash
city service status memory
city memory status
city memory restart
city service command memory search --payload '{"query":"release decision"}'
```
