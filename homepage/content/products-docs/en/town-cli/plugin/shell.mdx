# shell plugin

`shell` is a built-in runtime plugin for shell sessions that belong to a specific agent runtime.

Use it when you want shell execution that belongs to the agent runtime instead of an ad-hoc local terminal step.

## What this plugin shape means

- shell session ownership belongs to the managed shell plugin runtime
- start/stop cleanup belongs to plugin lifecycle
- command execution and session state stay tied to the target agent
- shell runtime parameters come from the SDK constructor and are optional; the CLI only calls the shell plugin already registered on the target agent

## Recommended path

- inspect lifecycle and boundaries in [Agent Plugins Docs / shell](/en/plugins-docs/builtins/shell)
- There is currently no `town shell` shortcut root; for scripted calls use `town plugin command shell <action> --payload <json>`
- Common actions include `exec`, `start`, `status`, `read`, `write`, `wait`, and `close`
