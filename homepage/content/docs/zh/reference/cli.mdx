## 运行模型（简版）

- `city start`：仅启动 city runtime
- `city start --console` / `city start -a`：同时启动 city runtime 和 Console
- `city console start`：只启动 Console，前提是 city runtime 已经运行
- `city stop`：先停止 Console，再停止 city runtime 与托管 agent
- `city restart`：重启 city runtime，恢复之前运行中的 agent，再重启 Console
- `city update`：更新当前全局安装来源对应的 `downcity` CLI 聚合包
- `city public`：管理 Console 的持久化公网暴露，支持交互式 manager 和 `on/off/status`
- `studio agent`：在交互式终端中打开 agent manager
- `studio agent create`：初始化项目运行文件
- `studio agent list`：查看已登记 agent，也可加 `--running` 只看正在运行的项目
- `studio agent start`：启动项目 agent daemon
- `studio agent stop`：停止项目 agent daemon
- `studio agent chat`：在终端中与 agent 对话，也可通过 `--message` 发送一次性消息
- `city console`：管理 Console 模块本身
- `studio plugin ...`：查看 plugin catalog、管理 plugin lifecycle，或使用高级 plugin command 形式
- `studio chat/task/memory/contact/shell ...`：日常托管 plugin runtime 命令
- `studio web/skill/asr/tts ...`：日常本地 plugin 命令
- `city model`：在 TTY 中打开交互式全局模型管理器
- `city model ...`：管理运行时模型配置与自动化脚本入口
