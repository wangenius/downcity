# `sma console ui`

管理由 `console` 托管的 UI 网关。

该 UI 不绑定某一个 `agent on` 进程，而是可以在同一页面切换查看多个已运行 agent。
它是状态优先的 Dashboard，对话只在 `local_ui` context 中开放。

## 用法

```bash
sma console ui [action] [options]
```

## 常用参数

- `-p, --port <port>`：UI 端口（默认 `3001`）
- `-h, --host <host>`：UI 主机（默认 `127.0.0.1`）

## action

- `start`：后台启动 UI（默认）
- `stop`：停止后台 UI
- `restart`：重启后台 UI
- `status`：查看 UI 状态

## 行为说明

- 需要先启动 console：`sma console start`。
- UI 启动后会读取 console registry 中的活跃 agent 列表。
- `Global Agents` 面板会展示每个 agent 的 runtime 地址、当前绑定模型与 chat 身份快照（例如 Telegram bot 用户名），并支持直接下拉切换模型。
- 侧边栏按三层分区：`Global` / `Agent` / `Context`。
- `Global`：全局概览、console runtime、model 管理、agents 管理、extensions。
- `Global / Model` 现已支持 provider/model 全量操作（创建、更新、测试、发现、暂停、删除）。
- `Global / Overview` 会展示 console 与当前 agent 的关键配置文件状态（存在性、可读性、更新时间）。
- `Agent`：`Overview`、`Services`、`Tasks`、`Logs`。
- `Agent / Tasks` 会展示任务定义详情与每次执行记录（run history + run detail）。
- `Context`：`Context Overview` + 每个 context 条目（包含 `local_ui`）。
- `Context Overview` 内会展示 channel 链路状态与 context 对应关系，并提供 `test` / `reconnect`。
- 只有 `local_ui` 支持发送消息；其他 context 在 UI 中为只读。
- `Context Workspace` 会展示当前 context 的消息历史、channel 历史、system prompt 解析结果。
- `sma console status` 会同时显示 console 与 UI 状态。

## 典型流程

```bash
# 1) 启动 console
sma console start

# 2) 启动多个 agent
sma agent on /path/to/agent-a
sma agent on /path/to/agent-b

# 3) 后台启动 UI
sma console ui start --port 3001

# 4) 查看 UI 状态
sma console ui status

# 5) 重启 UI（加载最新网关代码/路由）
sma console ui restart

# 6) 停止 UI
sma console ui stop
```
