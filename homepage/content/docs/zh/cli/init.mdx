# 项目初始化

`city agent create` 命令创建一个新 Agent 项目，包含运行所需的最小文件，并把 Agent 配置写入 CLI 全局 DB。

## 基本用法

```bash
city agent create my-project
```

这会创建：

```text
my-project/
├── PROFILE.md
├── SOUL.md
├── .downcity/
└── .env
```

## 生成的文件

- `PROFILE.md` — Agent 的可见角色和输出风格。见 [Profile](/zh/docs/cli/profile)。
- `SOUL.md` — 更深层的长期运行原则。见 [Profile](/zh/docs/cli/profile)。
- `.downcity/` — 运行时数据目录（会话、日志、缓存）。
- `.env` — 项目环境变量。
- CLI 全局 DB 中的 Agent 配置 — 包含 Agent ID、模型绑定、启动参数和 plugin 参数。

## 选项

```bash
# 允许覆盖已有 Agent 配置
city agent create my-project --force
```

## 初始化后

1. 编辑 `PROFILE.md` 和 `SOUL.md` 塑造行为。
2. 使用 `city agent config` 或交互式 `city agent` 管理模型、端口和 chat account 绑定。
3. 运行 `city agent start` 启动 Agent。
