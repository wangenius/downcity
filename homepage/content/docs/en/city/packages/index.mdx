If you already know which package you want to inspect, this section is the fastest route.

Downcity keeps the package graph intentionally small: start with `@downcity/city` for the City itself, add `@downcity/gate` for studio calls, create a Drizzle db in your host runtime, and add official services only when the studio needs them.

That order maps to four clear roles:

- `city` is the Kernel for the City itself
- `services` are Capabilities that add official behavior
- `gate` and `cli` are Interfaces for studio and operator entry
- `cities/*` is the Composition layer that assembles runnable examples

If you only care about npm packages, the three most important pieces to remember first are `city`, `services`, and `gate`.

## Recommended order

1. Start with [@downcity/city](/en/docs/city/packages/city) to understand the City itself
2. Then read [@downcity/gate](/en/docs/city/packages/gate) to see how studios call it
3. If you want terminal tooling, read [@downcity/cli](/en/docs/city/packages/cli)
4. Add `@downcity/services` when your studio needs auth, wallet, usage, or Stripe payment flows

## If you only want the integration picture first

Read the package layer as this chain:

```txt
@downcity/city
  -> provides the City Kernel
@downcity/services
  -> adds official capabilities into City
@downcity/gate
  -> uses UserClient and AdminClient to call those capabilities
```

So `@downcity/gate` is not only for AI calls.

It also unifies:

- AI service calls
- custom service calls
- official service calls
- trusted-side studio, token, and env management calls

## One responsibility map

```txt
Kernel        -> @downcity/city
Capabilities  -> @downcity/services
Interfaces    -> @downcity/gate + @downcity/cli
Composition   -> cities/node + cities/edge
```

## Typical combinations

### Minimal AI studio

- `@downcity/city`
- `@downcity/gate`

### Minimal registration and login

- `@downcity/city`
- `@downcity/gate`
- `@downcity/services`

### Usage metering

- `@downcity/city`
- `@downcity/gate`
- `@downcity/services`

### Global wallet and topup orders

- `@downcity/city`
- `@downcity/gate`
- `@downcity/services`

### One-time Stripe topups

- `@downcity/city`
- `@downcity/gate`
- `@downcity/services`
