# `downcity.json` 参考

当前版本采用两层模型配置：

1. `~/.downcity/downcity.db`（console 全局 SQLite）管理完整模型池  
2. 项目 `downcity.json`（agent 本地）只声明 `execution`

## 1) Agent 项目最小结构

```json
{
  "$schema": "./.downcity/schema/downcity.schema.json",
  "name": "my-agent",
  "version": "1.0.0",
  "execution": {
    "type": "acp",
    "agent": {
      "type": "kimi"
    }
  },
  "services": {}
}
```

如果你使用 console 模型池，`execution.type` 应为 `api`，并且 `execution.modelId` 必须能在 `~/.downcity/downcity.db` 的模型池中找到同名 model id。

如果你使用本地 llama / GGUF，`execution.type` 应为 `local`，并在 `plugins.lmp.model` 中声明模型文件名或绝对路径。留空 `plugins.lmp.modelsDir` 时默认从 `~/.models` 查找。

如果你使用 ACP coding agent，`execution.type` 应为 `acp`，并在 `execution.agent.type` 中声明 `kimi`、`claude` 或 `codex`。

## 2) Console 全局模型池（`~/.downcity/downcity.db`）

模型细节统一在 SQLite：

- providers
- models
- provider.apiKey 为加密存储（`api_key_encrypted`）

通过命令管理：

```bash
city model list
city model get provider openai_main
```

## 3) 运行时解析规则

agent 启动时：

1. 先检查项目是否声明了 `execution`
2. 如果 `execution.type = "api"`，到 `~/.downcity/downcity.db` 的 models 表取模型
3. 再由 `providerId` 关联 providers 表
4. 如果 `execution.type = "local"`，读取 `plugins.lmp`，解析本地模型文件并启动/复用 llama.cpp server
5. 如果 `execution.type = "acp"`，直接启动对应 ACP agent

`execution` 必须合法，否则启动失败。

## 4) Plugin 配置

插件行为配置写在项目 `downcity.json` 的 `plugins.*`。

插件如果需要额外模型、命令模板或本地路径，也直接写在对应的 `plugins.<name>.*` 下。

例如 asr：

```json
{
  "plugins": {
    "asr": {
      "enabled": true,
      "provider": "local",
      "modelId": "SenseVoiceSmall"
    }
  }
}
```


## 5) Chat 渠道绑定

`downcity.json` 中的 chat 渠道应只绑定全局 bot 账户 ID：

```json
{
  "services": {
    "chat": {
      "channels": {
        "telegram": {
          "enabled": true,
          "channelAccountId": "telegram-main"
        }
      }
    }
  }
}
```

`channelAccountId` 会解析到 `~/.downcity/downcity.db` 的 `channel_accounts.id`。

## 6) ACP 执行模式

当前支持以下 ACP coding agent：

- `kimi`：默认命令 `kimi acp`
- `claude`：默认命令 `npx -y @zed-industries/claude-code-acp`
- `codex`：默认命令 `npx -y @zed-industries/codex-acp`

示例：

```json
{
  "execution": {
    "type": "acp",
    "agent": {
      "type": "kimi"
    }
  }
}
```

也可以覆盖命令：

```json
{
  "execution": {
    "type": "acp",
    "agent": {
      "type": "claude",
      "command": "claude-acp",
      "args": []
    }
  }
}
```

## 7) Local 执行模式

本地 llama 示例：

```json
{
  "execution": {
    "type": "local"
  },
  "plugins": {
    "lmp": {
      "model": "gemma-4-E4B-it-UD-Q4_K_XL.gguf"
    }
  }
}
```

可选字段：

- `plugins.lmp.modelsDir`：本地模型目录，默认 `~/.models`
- `plugins.lmp.command`：启动命令，默认 `llama-server`
- `plugins.lmp.contextSize`：上下文窗口，默认 `8192`
- `plugins.lmp.autoStart`：是否自动拉起 `llama-server`，默认 `true`

当前限制：

- `execution.type="acp"` 只作用于 session/chat/task 执行链。
- `execution.type="api"` 时，必须提供 `execution.modelId`。
- `execution.type="local"` 时，必须提供 `plugins.lmp.model`，并确保本机已安装 `llama-server`。

## 8) 推荐操作方式

1. 如果你走远程 API 模式，在 console 层维护 provider/model，再在 agent 层配置 `execution.type="api"` 与 `execution.modelId`
2. 如果你走本地 GGUF 模式，在 agent 层配置 `execution.type="local"` 与 `plugins.lmp.*`
3. 如果你走 coding agent 模式，在 agent 层配置 `execution.type="acp"`
4. 修改后重启 agent：

```bash
city agent restart .
```

管理本地模型时，优先使用：

```bash
city lmp status
city lmp models
city lmp use gemma-4-E4B-it-UD-Q4_K_XL.gguf
```
