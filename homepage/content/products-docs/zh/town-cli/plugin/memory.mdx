# `town memory`

`town memory` 是托管 memory plugin 的命令组。

当 agent 或操作者需要显式查看、写入、整理 memory，而不是只依赖 prompt 注入时，使用它。

## 用法

```bash
town memory <subcommand> [options]
```

## 主要子命令

- `status`
- `search <query> [--include-sources]`
- `read <memoryPath> [--from <line>] [--lines <count>]`
- `remember --content <text> [--topic <topic>] [--wiki-path <path>] [--source <source>]`
- `digest --session-id <sessionId> [--max-messages <count>]`
- `revise <memoryPath> --instruction <text> [--evidence <text>]`

## 说明

- MemoryPlugin 使用 LLM Wiki 结构：`.downcity/memory/wiki/` 是知识层，`.downcity/memory/sources/` 是证据层。
- system prompt 只会注入少量稳定 wiki 摘要；更深检索走 `memory.search` 与 `memory.read`。
- `memory.search` 默认只查 wiki 和 working memory；需要证据层时加 `--include-sources`。
- 所有 memory action 都支持一次性 `--delay` / `--time`。

## 示例

```bash
town memory status
town memory search "release decision"
town memory read .downcity/memory/wiki/project-decisions.md --from 10 --lines 20
town memory remember --topic "plugin system" --content "Plugin config should be passed through constructors."
town memory digest --session-id telegram-chat-123 --max-messages 40
town memory revise .downcity/memory/wiki/plugin-system.md --instruction "Merge latest constructor config decision"
```
