# asr Plugin

`asr` 是语音转文字 plugin。它只定义 agent 侧协议，不绑定本地模型、Python 依赖或 provider。

它的真实转写能力从 constructor 传入。推荐把 City 的 AI 能力接进来：

```ts
import { Agent } from "@downcity/agent";
import { AsrPlugin } from "@downcity/plugins";

const agent = new Agent({
  id: "voice-agent",
  path: "/path/to/project",
  plugins: [
    new AsrPlugin({
      asr: (input) => city.ai.asr(input),
      auto: true,
      language: "auto",
    }),
  ],
});
```

## 设计边界

`asr` 负责：

- 暴露 `transcribe` action
- 在 `auto: true` 时监听 chat 入站语音附件
- 把转写结果追加到用户文本里
- 给 agent 注入如何使用 `asr` 的 system 提示

`asr` 不负责：

- 安装本地模型
- 选择 ASR provider
- 管理 Python 依赖
- 持久化项目配置
- 提供 CLI 管理命令

## Constructor 参数

| 参数 | 作用 | 默认值 |
| --- | --- | --- |
| `asr` | 必填，真实语音转写函数 | 无 |
| `auto` | 是否自动转写 chat 入站语音附件 | `false` |
| `language` | 默认语言提示，例如 `auto`、`zh`、`en` | 无 |
| `name` | plugin 名称 | `asr` |
| `title` | plugin 展示标题 | `ASR` |
| `description` | plugin 说明 | 内建说明 |

## 显式调用

如果你只想单独转写一个音频，可以直接调用 action：

```ts
const result = await agent.plugins.runAction({
  plugin: "asr",
  action: "transcribe",
  payload: {
    audio_path: "/path/to/voice.ogg",
    language: "zh",
  },
});
```

payload 至少需要提供一个输入来源：

- `audio_path`
- `url`
- `data_url`

## 自动转写

当 `auto: true` 时，chat 入站消息如果带有 `voice` 或 `audio` 附件，`asr` 会在 `CHAT_PLUGIN_POINTS.augmentInbound` 阶段尝试转写。

转写成功后，它不会替换原消息，而是把结果追加进正文：

```xml
<voice src="voice.ogg">明天下午三点提醒我开会</voice>
```

如果自动转写失败，主 chat 链路会继续执行。这个设计避免单个语音附件或外部 ASR 服务失败时阻塞 agent。

## Agent 如何调用

agent 仍然通过标准 plugin action 调用 `asr`：

```ts
plugin_call({
  plugin: "asr",
  action: "transcribe",
  payload: {
    audio_path: "...",
  },
});
```

如果开启了 `auto: true`，很多语音消息不需要模型显式调用 action，因为转写文本已经在消息进入 agent 前写进正文。
