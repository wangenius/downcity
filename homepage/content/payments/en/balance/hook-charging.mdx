---
title: Charge directly from hooks
description: "Use service-defined before hooks for balance prechecks and post-success hooks or provider charge lines for real wallet debits."
---

Balance is a wallet, not a pricing policy engine.

The recommended pattern is:

1. register `BalanceService()`
2. pass it into services that need wallet access
3. let each service define its own `before` hook
4. call `precheck()` before execution
5. call `charge()` after a successful call

## Why this is more flexible

Pricing usually belongs to business logic:

- which actions should charge
- whether one model costs more
- whether one downcity should be free
- whether one user segment gets discounts

That logic fits your own hooks better than service config.

## Precheck before execution

`precheck(user_id)` uses `needed_credits = 0` by default. It blocks users whose balance is already negative and lets zero or positive balances continue.

```ts
await balance.precheck(ctx.user!.user_id);
```

For fixed-price or estimated-price actions, pass a minimum balance:

```ts
await balance.precheck(ctx.user!.user_id, 10_000);
```

## Define it inside the service

Do not attach balance rules next to `fed.use(service)`. The service should own which actions need balance access.

```ts
const balance = new BalanceService({
  init_credits: 100_000_000,
});

base.use(balance);

class ExportService extends Service {
  constructor(options: { balance: BalanceService }) {
    super({ id: "export", name: "Export" });

    this.action("pdf", async (ctx) => {
      return await export_pdf(ctx.input);
    }, {
      auth: ["user"],
    }).before(options.balance.precheck_hook({
      needed_credits: 10_000,
    }));
  }
}

base.use(new ExportService({ balance }));
```

Complex rules can be plain code:

```ts
this.action("render", render_action, { auth: ["user"] })
  .before(async (ctx) => {
    const needed_credits = await estimate_render_credits(ctx.input);
    await balance.precheck(ctx.user!.user_id, needed_credits);
  });
```

## Debit after success

`charge()` records the real debit and can move the wallet below zero. Use it after success, or let AIService call it from provider charge lines.

```ts
this.action("pdf", async (ctx) => {
  const output = await export_pdf(ctx.input);

  await balance.charge({
    user_id: ctx.user!.user_id,
    credits: 10_000,
    note: "pdf export",
    metadata: {
      city_id: ctx.city?.city_id,
    },
  });

  return output;
});
```

## AIService

When you construct AIService with a balance bridge, AIService defines the precheck internally for model-consuming actions:

```ts
const ai = new AIService({ balance });
base.use(ai);
```

AIService prechecks only whether the user is already negative, then charges the actual provider usage after success.
