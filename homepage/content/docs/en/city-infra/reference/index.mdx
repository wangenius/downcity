This section answers one question only: **how do I use it concretely?**

If you are still unsure what Downcity is, why it is split this way, or where the boundaries are, go back to [Concepts](/en/docs/city-infra/concepts) or [Quickstart](/en/docs/city-infra/quickstart) first.

## Read by role

<div className="dc-doc-grid">
  <a className="dc-doc-card" href="/en/docs/city-infra/reference/infra-runtime">
    <strong>InfraRuntime</strong>
    <p>The server runtime entry. It initializes state, registers services, runs hooks, and exposes `handleRequest()`, `serve()`, and `router()`.</p>
  </a>
  <a className="dc-doc-card" href="/en/docs/city-infra/reference/user-client">
    <strong>UserClient</strong>

  </a>
  <a className="dc-doc-card" href="/en/docs/city-infra/reference/admin-client">
    <strong>AdminClient</strong>
    <p>The trusted-side entry. It manages products, `user_token`, and runtime env.</p>
  </a>
  <a className="dc-doc-card" href="/en/docs/city-infra/reference/http-api">
    <strong>HTTP API</strong>

  </a>
</div>

## Suggested order

1. Writing product-side code: start with [UserClient](/en/docs/city-infra/reference/user-client)
2. Wiring server-side runtime: continue with [InfraRuntime](/en/docs/city-infra/reference/infra-runtime)
3. Writing login and token issuance: read [AdminClient](/en/docs/city-infra/reference/admin-client)
4. Hand-writing requests: finish with [HTTP API](/en/docs/city-infra/reference/http-api)
