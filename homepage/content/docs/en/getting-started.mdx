# Getting Started

Get ShipMyAgent up and running in minutes. This guide covers everything from installation to your first autonomous agent.

## Prerequisites

Before you begin, ensure you have:

- **Node.js** >= 18.0.0 installed
- **A code repository** (Git initialized)
- **An LLM API key** (Anthropic, OpenAI, or compatible)
- **Basic terminal knowledge**

:::info
Don't have an LLM API key? Get free credits from [Anthropic](https://console.anthropic.com/) or [OpenAI](https://platform.openai.com/).
:::

---

## Installation

### Option 1: Global Install (Recommended)

Install ShipMyAgent globally for use in any project:

```bash
npm install -g shipmyagent
```

Verify installation:

```bash
shipmyagent --version
```

### Option 2: Using npx (No Install)

Use ShipMyAgent without installing:

```bash
npx shipmyagent <command>
```

### Option 3: Project Local

Install as a dev dependency in your project:

```bash
npm install --save-dev shipmyagent
```

Then use with `npx` or add to package.json scripts:

```json
{
  "scripts": {
    "agent": "shipmyagent",
    "agent:start": "shipmyagent agent on --daemon"
  }
}
```

### Troubleshooting Installation

**"Command not found" after install?**

Add npm global bin to your PATH:

```bash
# Find npm bin path
npm config get prefix

# Add to PATH (macOS/Linux)
export PATH=$(npm config get prefix)/bin:$PATH

# Add to ~/.bashrc or ~/.zshrc for persistence
echo 'export PATH=$(npm config get prefix)/bin:$PATH' >> ~/.bashrc
```

**Permission errors?**

Don't use sudo with npm. Instead:

```bash
# Fix npm permissions
mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
```

## Quick Start (5 Minutes)



### Step 1: Initialize Your Project

Navigate to your project directory and run:

```bash
cd /path/to/your/project
shipmyagent init
```

The interactive setup will ask:

1. **Agent Name**: Give your agent a name (defaults to current directory name, e.g., "my-assistant")
2. **LLM Model**: Pick a model. Supports Claude Sonnet 4 (recommended), Claude Haiku, Claude 3.5 Sonnet, GPT-4, GPT-4o, DeepSeek Chat, etc.
3. **Chat Adapters (Multi-select)**: Optionally enable Telegram / Feishu / QQ. **Only selected adapters are written to `ship.json.services.chat.adapters`**
4. **QQ Sandbox** (only when QQ is selected): Whether to use sandbox environment for testing
5. **Skills Installation** (Optional): Optionally install recommended skill plugins, such as Vercel Best Practices, Web Design Guidelines, Browser Automation, etc.

This creates:

```
your-project/
├── PROFILE.md           # Agent system prompt
├── SOUL.md            # Optional core principles prompt
├── USER.md            # Optional user preference prompt
├── ship.json          # Configuration file
├── .claude/           # Claude compatible directory
│   └── skills/        # Skills storage
└── .ship/             # Agent data directory
    ├── task/          # Scheduled tasks
    ├── logs/          # Operation logs
    ├── .cache/        # Cache data
    ├── profile/       # User profile memory
    │   ├── Primary.md
    │   └── other.md
    ├── data/          # Data storage
    ├── chat/          # Chat records
    ├── public/        # Public resources
    ├── config/        # Configuration files
    ├── skills/        # Project-level skills
    ├── schema/        # JSON Schema (IDE auto-completion)
    │   └── ship.schema.json
    └── .debug/        # Debug information
```

> **Tip**: The Profile memory system saves user preferences, and Schema files provide IDE auto-completion.

### Step 2: Configure Your API Key

Open `ship.json` and add your API key:

```json
{
  "llm": {
    "activeModel": "default",
    "providers": {
      "default": {
        "type": "anthropic",
        "baseUrl": "https://api.anthropic.com/v1",
        "apiKey": "${ANTHROPIC_API_KEY}"
      }
    },
    "models": {
      "default": {
        "provider": "default",
        "name": "claude-sonnet-4-5"
      }
    }
  }
}
```

**Set the environment variable**:

```bash
# Temporary (current session)
export ANTHROPIC_API_KEY="sk-ant-..."

# Permanent (add to ~/.bashrc or ~/.zshrc)
echo 'export ANTHROPIC_API_KEY="sk-ant-..."' >> ~/.bashrc
source ~/.bashrc
```

:::warning
Never commit `ship.json` with hardcoded API keys! Always use environment variables.
:::

### Step 3: Test the Connection

Verify your API key works:

```bash
shipmyagent test-llm
```

You should see:

```
✅ LLM connection successful
Provider: Anthropic
Model: claude-3-5-sonnet-20241022
Response time: 245ms
```

### Step 4: Customize Your Agent

Edit `PROFILE.md` to define your agent's role (and optionally `SOUL.md` + `USER.md` for stable principles and user preferences):

```markdown
# Agent Role

You are a senior full-stack developer specializing in React and Node.js.

## Goals

- Write clean, maintainable code
- Follow best practices and design patterns
- Ensure test coverage for new features
- Review code for security issues

## Context

This project uses:
- React 18 with TypeScript
- Node.js backend with Express
- PostgreSQL database
- Tailwind CSS for styling

## Constraints

- Never modify package.json without approval
- All new code must include tests
- Follow existing code style
- Don't refactor without explicit request
```

### Step 5: Start the Agent

Launch your agent:

```bash
shipmyagent .
# `shipmyagent .`: foreground (current terminal)
# `shipmyagent agent on --daemon [path]`: background (daemon)
```

Output:

```
🚀 Starting ShipMyAgent
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ 100%

✅ Configuration loaded
✅ LLM connection established
✅ Agent initialized: my-assistant
✅ Task scheduler started
✅ Server running on http://localhost:3000

🤖 Your agent is now online!
💬 Chat via Telegram (if configured)
📊 View logs: shipmyagent logs
```

## Your First Interaction

### Via CLI

Ask your agent a question:

```bash
shipmyagent chat "What's the purpose of this project?"
```

### Via Telegram (if configured)

Simply send a message to your bot:

```
Hey, can you review the recent changes?
```

The agent will analyze your repository and respond.

If you send multiple follow-up messages before the agent replies, it will merge them as additional context and produce a single final response (fewer back-and-forth turns).

Ensure everything is working:

- [ ] `shipmyagent --version` shows correct version
- [ ] `shipmyagent init` created configuration files
- [ ] `shipmyagent test-llm` shows successful connection
- [ ] `shipmyagent agent on --daemon` launches without errors
- [ ] Agent responds to questions (CLI or Telegram)
- [ ] Logs are being written: `shipmyagent logs --tail 10`

Run the full diagnostic:

```bash
shipmyagent doctor
```

This checks:
- ✅ Node.js version
- ✅ Configuration validity
- ✅ LLM connectivity
- ✅ File permissions
- ✅ Service status

## Advanced Setup

### Custom Configuration

For non-standard setups:

**Open-compatible gateway (Chat Completions)**:

```json
{
  "llm": {
    "activeModel": "default",
    "providers": {
      "gateway": {
        "type": "open-compatible",
        "baseUrl": "https://your-api.com/v1",
        "apiKey": "${LLM_API_KEY}"
      }
    },
    "models": {
      "default": {
        "provider": "gateway",
        "name": "gpt-4o-mini"
      }
    }
  }
}
```

**Open-responses gateway**:

```json
{
  "llm": {
    "activeModel": "default",
    "providers": {
      "gateway": {
        "type": "open-responses",
        "baseUrl": "https://your-api.com/v1",
        "apiKey": "${LLM_API_KEY}"
      }
    },
    "models": {
      "default": {
        "provider": "gateway",
        "name": "gpt-5.2"
      }
    }
  }
}
```

**Multiple models in one project**:

```json
{
  "llm": {
    "activeModel": "quality",
    "providers": {
      "anthropic_main": {
        "type": "anthropic",
        "apiKey": "${ANTHROPIC_API_KEY}"
      },
      "google": {
        "type": "gemini",
        "apiKey": "${GEMINI_API_KEY}"
      }
    },
    "models": {
      "quality": {
        "provider": "anthropic_main",
        "name": "claude-sonnet-4-5"
      },
      "fast": {
        "provider": "google",
        "name": "gemini-2.5-flash"
      }
    }
  }
}
```

### Production Deployment

Running ShipMyAgent in production:

**Use a process manager**:

```bash
# Install PM2 globally
npm install -g pm2

# Start with PM2
pm2 start shipmyagent --name "agent" -- start

# Configure auto-restart
pm2 startup
pm2 save
```

**Docker setup**:

```dockerfile
FROM node:18-alpine

RUN npm install -g shipmyagent

WORKDIR /app
COPY . .

# Set environment variables
ENV ANTHROPIC_API_KEY=""

CMD ["shipmyagent", "start"]
```

```yaml
# docker-compose.yml
version: '3.8'
services:
  agent:
    build: .
    environment:
      - ANTHROPIC_API_KEY=${ANTHROPIC_API_KEY}
      - TELEGRAM_BOT_TOKEN=${TELEGRAM_BOT_TOKEN}
    restart: unless-stopped
```

## Troubleshooting

**Issue**: Agent won't start

```bash
# Check configuration
shipmyagent validate

# View logs
shipmyagent logs --tail 50

# Run diagnostics
shipmyagent doctor
```

**Issue**: API connection fails

```bash
# Test LLM connection
shipmyagent test-llm --verbose

# Check API key
echo $ANTHROPIC_API_KEY

# Test network
curl https://api.anthropic.com/v1/messages
```

**Issue**: Permissions errors

```bash
# Check file permissions
ls -la .ship/

# Fix permissions
chmod -R 755 .ship/

# Verify config ownership
ls -l ship.json PROFILE.md SOUL.md USER.md
```

For more help, see the [Troubleshooting Guide](/docs/troubleshooting).

## Glossary

- **Agent**: The AI assistant that manages your repository
- **Task**: A scheduled job that runs automatically (via cron)
- **Prompt**: Instructions given to the LLM (in PROFILE.md / SOUL.md / USER.md / tasks)
- **Service**: External service connection (Telegram, GitHub, etc.)
- **Permission**: Rules controlling what the agent can do
- **Approval**: Human confirmation required for sensitive operations
