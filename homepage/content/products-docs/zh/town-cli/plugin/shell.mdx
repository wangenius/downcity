# shell plugin

`shell` 是内建 runtime plugin，用于让 shell session 归属于具体 agent runtime。

当你需要“属于 agent runtime 的 shell 执行”而不是临时本地终端步骤时，使用它。

## 这类 plugin 的含义

- shell session 的归属在托管 shell plugin runtime 上
- start/stop 时的清理归属于 plugin lifecycle
- 命令执行与会话状态都绑定到目标 agent
- shell runtime 参数来自 SDK constructor，可选；CLI 只是调用目标 agent 已注册的 shell plugin

## 推荐阅读路径

- 先看 [Agent Plugins Docs / shell](/zh/plugins-docs/builtins/shell) 理解 lifecycle 与边界
- 当前没有 `town shell` 快捷根命令；脚本化调用时使用 `town plugin command shell <action> --payload <json>`
- 常见 action 包括 `exec`、`start`、`status`、`read`、`write`、`wait`、`close`
