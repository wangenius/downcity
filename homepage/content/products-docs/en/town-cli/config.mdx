# `town config`

Manage project `downcity.json` and alias.

> Naming note: configuration command is `town config`, not `town console config`.

## Usage

```bash
town config <subcommand>
```

## Subcommands

- `get [keyPath]`
- `set <keyPath> <value>`
- `unset <keyPath>`
- `alias`

> Model resources are managed by `city`. Town only reads the project execution binding while hosting Agents.

## Plugin config

`town config` now reads and writes project `downcity.json` only.

For plugin configuration, edit:

- `plugins.*` for agent-local plugin parameters only
- use `town plugin` for plugin lifecycle and runtime state

Examples:

```bash
town config get plugins.asr
town config set plugins.asr.modelId '\"SenseVoiceSmall\"'
```

## Examples

```bash
town config get execution.modelId
town config set execution.modelId fast
town config unset plugins.chat.channels.telegram
town config alias --dry-run
```
