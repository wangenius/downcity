# `town city`

`town city` 管理本机 Agent 调用 City 时使用的用户态连接。

`city` 负责 admin 侧的 City base 与资源管理。`town` 负责 user 登录和本机 Agent runtime 访问。默认 City base 是 `https://base.downcity.ai`。

## 用法

```bash
town city
town city status
town city list
town city connect [url]
town city use [base]
town city login [url]
town city logout
town city disconnect
```

## 状态模型

`town city connect` 和 `town city use` 只选择 City base，不从 `city` 复制 admin 凭证或 user token。

`town city login` 以 user 身份登录，并保存 Town 自己的 user session。Agent runtime 调用 City AIService 时读取这份 Town user session。

`DOWNCITY_CITY_URL`、`DOWNCITY_CITY_TOWN_ID`、`DOWNCITY_CITY_USER_TOKEN` 仍然可以作为 CI 或临时启动时的运行时覆盖项。

## 推荐流程

```bash
city                  # 可选：管理 admin 侧 City base 与资源
town city status      # 默认指向 https://base.downcity.ai
town city login
town agent start
```

使用 `town city list` 可以查看 Town 自己保存的 base，以及从 `city` admin 配置中发现的 base 地址。
