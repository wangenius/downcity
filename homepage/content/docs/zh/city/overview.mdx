# City

**City** 是 Downcity Agent 连接的共享 AI 基础设施层。它负责那些不应该存在于每个独立 Agent 项目中的功能：模型目录、身份认证、用量追踪和计费。

## City 负责什么

- **模型目录** — Federation AIService 列出可用模型和提供商。Agent 通过 ID 绑定模型，无需在每个项目里嵌入提供商密钥。
- **身份与认证** — 全局用户账号、API token 和权限属于 City，不属于每个项目。
- **用量与余额** — 每次模型调用都会被计量和追踪。City 处理计费、配额和支付（通过 Stripe）。
- **Agent 与渠道配置** — Agent 配置和 Telegram、飞书、QQ 等机器人凭证统一存储在全局 `downcity.db`，Agent 通过账号 ID 绑定渠道。

## City 不负责什么

- 项目 `.env` 和 `.agents/skills`
- Agent 会话和对话状态
- 插件行为和工具实现

这些属于 Agent 运行时。

## Agent 如何连接 City

当你运行 `city agent start` 时，CLI 从全局 DB 读取 Agent 配置，从已连接的 City 实例解析 `execution.modelId`，然后将实时模型实例传给 Agent SDK。

当你直接使用 SDK 时，你自己提供模型实例 —— 但 City 仍然拥有背后的目录和计费。
