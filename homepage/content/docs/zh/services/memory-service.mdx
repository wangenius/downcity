# Memory Service

`memory` service 提供运行时记忆操作能力。

## 职责

- 状态查看与索引重建
- 记忆检索与按文件读取
- 显式写入 memory
- 把会话窗口刷写到 daily memory
- 向 agent 注入 memory service 的使用规则

## 接入方式

`memory` 是独立 service，不直接耦合本地 session 执行内核。

- system prompt 会注入一小段来自 `MEMORY.md` 的稳定长期记忆，并附带 memory service 的使用规则
- 注入内容只取长期稳定的 Canon 文本，会去重、限量，并裁剪过长条目
- agent 需要额外记忆时，自行调用 `memory.search`、`memory.get`、`memory.store`
- 不会在每轮 session 执行时自动做 recall / capture

如果你需要检索、读取、写入或刷写记忆，使用 `city memory` 命令。

## 运行态操作

```bash
city service status memory
city memory status
city memory restart
city service command memory search --payload '{"query":"release decision"}'
```
