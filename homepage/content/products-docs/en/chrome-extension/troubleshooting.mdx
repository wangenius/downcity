# Troubleshooting

## 1. Cannot Reach Downcity Town

Check:

- whether `town agent start` is running
- whether `Town URL` is reachable from Chrome
- whether local setup uses `http://127.0.0.1:5314`
- whether you accidentally used `15314`, which is the Agent RPC port

If you just updated the extension code, rebuild first:

```bash
npm run build:extension
```

Then refresh the unpacked extension.

## 2. The Current Town Requires a Token

If you see:

- `Missing bearer token`
- `Invalid bearer token`
- `Permission denied`

the target Town usually requires auth, but the extension does not have a valid token yet.

Fix it by:

1. opening the extension settings page
2. running `town token create my-token` in the local terminal
3. pasting the token into `Town Token`
4. clicking `保存并检查`

## 3. The Remote Town Is Not Exposed Publicly

If you are targeting a remote machine, confirm that the Town runtime HTTP API is externally reachable through its deployment, reverse proxy, tunnel, or hosting layer.

## 4. No Available Agent

This usually means:

- `town agent start` is not running
- the Town URL is wrong
- the Agent is offline

## 5. Side Panel Has No Messages

The extension creates a browser session automatically. If the Side Panel looks empty, send the first message or attach the current page from the composer.

## 6. The Captured Body Looks Wrong

Full-page capture is best-effort, not a perfect reader-mode clone.

The current logic:

- prefers the most article-like `main/article` root
- filters nav, sidebars, and hidden content
- merges multiple strong content roots when needed

Very custom page layouts can still confuse the extractor.
