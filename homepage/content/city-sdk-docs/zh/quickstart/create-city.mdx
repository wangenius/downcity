---
title: 创建 Federation
description: 初始化 Downcity 服务端 Federation，准备默认数据库和 HTTP 服务。
---

服务端运行时的类是 `Federation`（文档里也叫 Federation），产品侧用 `City` 客户端去访问它。两者的区别见 [Federation 与 City](/zh/city-sdk-docs/concepts/federation)。

最快的本地创建和启动方式是：

```bash
fed create ./my-fed
cd my-fed
pnpm install
fed deploy
```

默认模板使用 Node.js + SQLite。`fed deploy` 从 `12314` 开始分配端口，把进程登记到系统级 Fed 管理状态，并自动注入和保存 admin key；之后可以在任意路径运行 `fed` 查看并直接进入管理。下文是需要手动嵌入 Federation runtime 时的 SDK 写法。

安装包：

```bash
pnpm add @downcity/city drizzle-orm better-sqlite3
```

准备好数据库即可。Federation 首次启动时会自动补齐 `DOWNCITY_FEDERATION_ADMIN_SECRET_KEY`、`DOWNCITY_FEDERATION_TOKEN_SIGNING_KEY` 和 `BETTER_AUTH_SECRET`。provider key 则在启动后通过 Admin API 或 `fed` 的 admin workspace 写入：

```bash
OPENAI_API_KEY="..."
OPENAI_BASE_URL="https://api.openai.com/v1"
```

创建 Federation：

```ts
import { Federation } from "@downcity/city";
import Database from "better-sqlite3";
import { drizzle } from "drizzle-orm/better-sqlite3";

const sqlite = new Database("./data.sqlite");
sqlite.pragma("journal_mode = WAL");

const db = Object.assign(drizzle(sqlite), {
  $client: { exec: (sql: string) => sqlite.exec(sql) },
});

const base = new Federation({ db });
```

默认数据库会落到：

```plaintext
./data.sqlite
```

本地 SQLite 推荐使用 `better-sqlite3`。正式部署如果使用 Postgres，也同样是先创建 Drizzle pg db，再传给 `Federation`。

如果要使用 Cloudflare Workers / D1，传入 D1 对应的 Drizzle db：

```ts
import { drizzle } from "drizzle-orm/d1";

const db = drizzle(env.DB);
const base = new Federation({ db });
```

Federation 会在第一次 `health()`、`fetch()`、`table()` 等运行时操作时自动初始化。

如果你的 Federation 还需要自己的业务表，可以通过 Service 或服务声明表。表会挂在对应 Service 的命名空间下，并和 Federation 默认数据库使用同一个数据库连接：

```ts
import { sqliteTable, text } from "drizzle-orm/sqlite-core";
import { Service } from "@downcity/city";

const notes = sqliteTable("notes", {
  id: text("id").primaryKey(),
  title: text("title").notNull(),
  status: text("status").notNull(),
});

const notesService = new Service({
  id: "notes",
  tables: { notes },
});

base.use(notesService);

const table = await base.table("notes.notes");
await table.insert({
  id: "note_1",
  title: "First note",
  status: "draft",
});
```

常规项目不需要手动建表；第一次运行时操作会自动准备 Federation 内置表和 Service 声明的表。

启动服务：

```ts
import { serve } from "@hono/node-server";

await base.health();
serve({ fetch: (request) => base.fetch(request), port: 3001, hostname: "127.0.0.1" });
```

这个服务就是后续多个产品共用的 Federation。管理和业务数据暂时通过你自己的数据库与产品后端处理。
