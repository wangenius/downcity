# `town start`

后台启动 Town runtime。它负责本机宿主、受管 agent registry、运行时 API 和托管 plugin 调用边界。

在交互式终端中，直接运行裸 `town` 也会打开 Town manager，可以在里面选择启动、停止、重启和查看总览，不需要手动输入子命令。

当前 `town start` 不启动 Console UI，也不接受 `--console`、`--public`、`-p` 或 `--host` 这类 Console 参数。浏览器 Console 需要通过产品自身的部署或开发入口单独启动。

## 用法

```bash
town start
```

## 启动时会做什么

- 初始化 Town runtime 目录和受管 agent registry。
- 检查本机 shell sandbox 依赖。Linux 上必须可用 `bubblewrap`（命令名 `bwrap`）和 unprivileged user namespaces。
- 清理旧版本遗留的失联 Town runtime 进程。
- 后台启动 Town runtime。
- 如果当前还没有本机 Bearer Token，会初始化一个本机 token 并只在启动输出里显示一次。
- 尝试发现当前机器的公网 host，并写入 City 侧 Env 的 `DOWNCITY_PUBLIC_HOST`。后续 agent 生成 `contact link` 时会优先使用这个地址，避免 link code 回退到 `127.0.0.1`。

## Linux sandbox 前置条件

Linux shell sandbox 使用 `bubblewrap` 封装 Linux namespaces 等原生内核能力。如果缺少 `bwrap`，或系统禁用了 user namespaces，`town start` 会在启动 runtime 前失败。

请使用系统包管理器安装 `bubblewrap`，例如 `sudo apt install bubblewrap`、`sudo dnf install bubblewrap` 或 `sudo pacman -S bubblewrap`。

## 端口说明

- `5314`：Town runtime API 端口，用于 health、受管 agent、托管 plugin 等本机运行时接口。
