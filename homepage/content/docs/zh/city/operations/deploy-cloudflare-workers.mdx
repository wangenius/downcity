这页是 `cities/edge` 的正式部署操作路径。

如果你只是想理解 City 怎么接入 Workers 和 D1，先读 [Cloudflare Workers](/docs/city/guides/cloudflare-workers)。这一页只讲上线步骤：D1 binding、部署、取回 admin key、写入 env、OAuth callback 和健康检查。

## 前置条件

- 一个 Cloudflare 账号
- 已经在仓库根目录执行过 `pnpm install`
- 已经通过 `pnpm exec wrangler login` 或 `npx wrangler login` 登录 Wrangler

## 1. 确认 edge 街区

参考实现位于 [cities/edge/src/index.ts](/Users/wangenius/Documents/github/downcity/cities/edge/src/index.ts:1)。

它做了四件关键事：

- 把 `env.DB` 接成 Drizzle D1 数据库
- 在 `cities/edge` 内组装 Edge City
- 挂载 accounts、balance、usage、payment、Stripe payment 和 AI services
- 暴露 `/health` 用于部署验证

## 2. 准备 D1

`cities/edge/wrangler.toml` 里声明了 D1 binding：

```toml
[[d1_databases]]
binding = "DB"
database_name = "downcity-db"
database_id = "<your-d1-database-id>"
```

第一次部署时，先创建 D1 数据库，并把返回的 ID 写入 `wrangler.toml`：

```bash
cd cities/edge
npx wrangler d1 create downcity-db
```

如果 `database_id` 为空，`cities/edge/deploy.sh` 也会尝试创建 D1，并把 ID 写回 `wrangler.toml`。

## 3. 部署

```bash
cd cities/edge
pnpm deploy
```

这个脚本会先确认 D1 数据库，再执行 `wrangler deploy`。你也可以直接运行：

```bash
cd cities/edge
npx wrangler deploy
```

部署完成后，Worker URL 通常类似：

```text
https://downcity.<your-subdomain>.workers.dev
```

## 4. 初始化 City

先请求一次 `/health`。这会让 Worker 启动 City，并创建内置的 `env` / `studios` 表。

```bash
curl https://downcity.<your-subdomain>.workers.dev/health
```

返回大致如下：

```json
{
  "ok": true,
  "services": ["env", "studios", "accounts", "balance", "payment", "usage", "payment.stripe", "ai"]
}
```

## 5. 取回 admin key

City 首次启动时会把 `DOWNCITY_CITY_ADMIN_SECRET_KEY` 写入自己的 `env` 表。Workers + D1 部署下，可以直接从远程 D1 查询：

```bash
cd cities/edge
npx wrangler d1 execute downcity-db --remote \
  --command "SELECT value FROM env WHERE key='DOWNCITY_CITY_ADMIN_SECRET_KEY'"
```

请把这个 key 存在可信位置。它是 `Admin Gate`、Bay CLI 或直接调用 admin HTTP 接口时使用的管理凭证。

## 6. 写入 provider 和 service env

Provider API key 和服务密钥应该写入 City 自己的 env 表，不应该进入公开客户端。

示例：

```bash
curl -X POST https://downcity.<your-subdomain>.workers.dev/v1/env/upsert \
  -H "Authorization: Bearer <admin_secret_key>" \
  -H "Content-Type: application/json" \
  -d '{"key":"DEEPSEEK_API_KEY","value":"sk-..."}'
```

常见 key：

| Key | 用途 |
| --- | --- |
| `DEEPSEEK_API_KEY` | edge 参考街区默认使用的 provider key |
| `STRIPE_SECRET_KEY` | 创建 Stripe Checkout 的 API key |
| `STRIPE_WEBHOOK_SECRET` | Stripe webhook 签名密钥 |
| `STRIPE_SUCCESS_URL` | 可选，Checkout 成功后的默认跳转地址 |
| `STRIPE_CANCEL_URL` | 可选，Checkout 取消后的默认跳转地址 |
| `GOOGLE_CLIENT_ID` / `GOOGLE_CLIENT_SECRET` | 可选，Google OAuth 登录 |
| `WECHAT_CLIENT_ID` / `WECHAT_CLIENT_SECRET` | 可选，微信网站应用登录 |

也可以继续读 [Provider 环境变量](/docs/city/guides/provider-env) 和 [环境变量参考](/docs/city/reference/env-vars)。

## 7. 配置 OAuth callback

如果你开启 OAuth 登录，需要把 provider 的 callback URL 配成 Worker 对外地址：

```text
https://downcity.<your-subdomain>.workers.dev/v1/accounts/oauth/callback
```

callback 域名必须和用户实际访问的公开域名一致。如果之后切到自定义域名，也要同步更新 OAuth provider 里的回调地址。

## 8. 验证

部署完成后先检查：

```bash
curl https://downcity.<your-subdomain>.workers.dev/
curl https://downcity.<your-subdomain>.workers.dev/health
```

再用 admin auth 确认模型目录：

```bash
curl https://downcity.<your-subdomain>.workers.dev/v1/ai/models \
  -H "Authorization: Bearer <admin_secret_key>"
```

## 更新部署

修改代码后：

```bash
cd cities/edge
pnpm deploy
```

运行时 env 保存在 D1 里，所以重新部署 Worker 不会丢失 provider key、OAuth secret、studio、余额、usage 记录或 Stripe 支付记录。
