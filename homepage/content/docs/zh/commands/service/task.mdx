# `city task`

任务系统命令组。

## 用法

```bash
city task <subcommand> [options]
```

## 子命令

- `list`
- `create`
- `run <title>`
- `update <title>`
- `delete <title>`
- `status <title> <enabled|paused|disabled>`
- `enable <title>`
- `disable <title>`

## `create` 关键参数

- `--title <title>`（必填）
- `--description <description>`（必填）
- `--session-id <sessionId>`（必填）
- `--when <when>`（默认 `@manual`，支持 cron / `time:ISO8601`）
- `--kind <agent|script>`
- `--status <enabled|paused|disabled>` 或 `--activate`
- `--body <body>`
- `--overwrite`

create 去重：只按 `title` 精确去重。

## `update` 常用参数

- `--title <title>` / `--description <description>`
- `--when <when>` / `--clear-when`
- `--kind <agent|script>`
- `--session-id <sessionId>`
- `--status <status>` / `--activate`
- `--body <body>` / `--clear-body`

## `run` 返回语义

`run` 为异步受理，立即返回：

- `accepted=true`
- `message`
- `executionId`

其中 `message` 会提示调用方：任务已经开始执行，完成后 task 会自动发送给用户，可以直接继续后续流程，无需等待 task 完成。

收到 `accepted=true` 后，调用方不应再手动转发 task 产出，也不应主动读取 run 目录轮询；只有用户明确要求排查或查看产物时，才去检查 run 文件。

## 示例

```bash
city task list
city task create --title "Daily check" --description "Run tests" --session-id telegram-chat-123 --when "0 10 * * 1-5" --activate
city task run daily-check
city task update daily-check --when "@manual"
city task disable daily-check
```
