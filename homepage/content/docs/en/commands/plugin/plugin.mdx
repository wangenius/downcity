# `city plugin`

`city plugin` is the generic entry for the new plugin system.

Use it when you want to:

- inspect which plugins are registered
- inspect a plugin's static enabled state and capability surface
- dispatch a plugin action without using the plugin's short alias

## Common examples

```bash
city plugin list
city plugin status asr
city plugin action asr status
```

## Notes

- `city plugin list` and `city plugin status` are static console-level catalog views.
- They do not require `--path`, `--agent`, or a running agent.
- `city plugin action ...` is the advanced execution form and still targets a specific agent project.
- For day-to-day usage, prefer concrete plugin roots such as `city web ...`, `city skill ...`, `city asr ...`, and `city tts ...`.
- Plugin enablement is now a city-level fact. Static status does not read agent `downcity.json`.
