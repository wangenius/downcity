# 例子：客户端 / 服务端模式

这种模式通常是：

- 服务端进程持有本地 `Agent`
- 服务端通过 `startServer(...)` 暴露 HTTP，或通过 `agent.start({ rpc: ... })` 暴露 RPC
- 客户端进程创建 `RemoteAgent`

这种结构的最大优点是职责清晰：

- 服务端维护执行上下文
- 客户端只负责发请求和消费结果

它适合：

- 前后端分离
- 本地守护进程 + 调用端
- 多个调用方共享同一个 agent 服务
