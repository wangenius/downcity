# CLI

The `downcity` CLI is a unified management tool for the entire Downcity stack. It covers Agent projects, City infrastructure, and Federation configuration from a single terminal interface.

## What it does

### Agent projects

- Creates and initializes Agent projects
- Reads `downcity.json` and passes config to the Agent SDK
- Starts, stops, and restarts Agents
- Manages channel accounts and credentials

### City

- Logs in and out of City identity
- Manages API tokens and permissions
- Inspects usage, balance, and billing status
- Configures channel accounts (Telegram, Feishu, QQ bots)

### Federation

- Lists available model providers and model IDs
- Configures model bindings for projects
- Inspects provider health and status

## Who it is for

The CLI is for users who want to manage the entire Downcity stack from the terminal. If you are building a product that needs programmatic control, use the SDKs directly.

## Key commands

### Agent

| Command | What it does |
|---------|-------------|
| `downcity agent init` | Create a new Agent project |
| `downcity agent start` | Start the Agent in the current project |
| `downcity agent stop` | Stop the running Agent |
| `downcity agent status` | Check if the Agent is running |
| `downcity agent logs` | View runtime logs |

### City

| Command | What it does |
|---------|-------------|
| `downcity login` | Authenticate with City |
| `downcity logout` | Remove local City credentials |
| `downcity chat add` | Add a messaging channel account |
| `downcity chat list` | List registered channel accounts |
| `downcity usage` | View current usage and balance |

### Federation

| Command | What it does |
|---------|-------------|
| `downcity models list` | List available models |
| `downcity providers list` | List registered model providers |
| `downcity providers status` | Check provider health |

## Relationship to SDKs

The CLI is a wrapper around the SDKs. It reads config files, resolves model bindings through City, and calls SDK constructors under the hood. The execution core is the same whether you use the CLI or the SDKs directly.
