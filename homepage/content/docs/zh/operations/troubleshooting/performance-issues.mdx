# 性能问题

## 常见瓶颈

- 模型响应慢
- 任务并发过高
- 单次命令输出过大
- 会话历史过长

## 优化建议

1. 使用更快模型处理高频请求。
2. 降低任务触发频率，避免重叠执行。
3. 控制命令输出规模，优先缩小命令范围、减少无关日志与大段全文输出。

4. 会话 compact 是内置策略。如果仍然偏慢，优先处理：

- 缩小当前工作提示词
- 减少工具输出噪音
- 绑定更快的模型

## 观测方法

```bash
grep -Ei "timeout|slow|context|compact" .downcity/logs/$(date +%F).jsonl
```
