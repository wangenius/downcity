# Runtime Relationship & Process

## 1. Role Relationship

- `console`: manages registry, shared storage, model pool, and UI.
- `agent`: owns one project runtime.
- `service`: executes core workflows.
- `plugin`: adds optional enhancements.
- `capability`: provides stable runtime contracts.
- `asset`: provides install/check/config/use for dependencies.

In one line:

`console` governs, `agent` runs, `service` orchestrates, `plugin` augments, `asset` supplies dependencies.

## 2. Process Startup Model

1. Start console: `city console start`
2. Start agent: `city agent start`
3. Agent loads project config from `ship.json`
4. Runtime assembles services, plugins, capabilities, and assets
5. Inbound messages/tasks enter service workflows
6. Services invoke capabilities when optional enhancements are needed
7. Runtime traces are persisted to `.ship/*`

## 3. Operational Checklist

```bash
city console status
city agent status .
city service list
city plugin list
```
