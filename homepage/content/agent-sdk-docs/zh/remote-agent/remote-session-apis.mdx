# Remote Session API

通过 `RemoteAgent` 拿到 session 后，最常用的方法有：

- `get_info()`
- `prompt()`
- `subscribe()`
- `messages()`
- `system()`
- `approvals()` / `resolve_approval()`
- `approval_mode()` / `set_approval_mode()`
- `fork()`

这些 API 尽量和本地 session 的使用面保持一致，方便你在本地和远程模式之间复用同一套心智模型。

## 获取 session

```ts
const session = await agent.sessions.create();
```

如果你要继续一个已有远程会话：

```ts
const session = await agent.sessions.get("repo-analysis");
```

这样就可以切换到一个已知远程 session，而不是每次都从头开始。

## 发起一轮 prompt

```ts
const turn = await session.prompt({ query: "继续" });
await turn.finished;
```

## 订阅后续事件

```ts
const unsubscribe = session.subscribe((event) => {
  console.log(event);
});

unsubscribe();
```

订阅会收到 `message | part | delta | turn | session` 五类 Mutation。审批请求表现为状态为 `approval-required` 的 Tool Part，可通过回调第二个参数的 `reply.approval(...)` 直接处理。

## 历史消息

```ts
const messages = await session.messages({
  limit: 50,
});
```

## System prompts

```ts
const system = await session.system();
```

## 分叉

```ts
const next = await session.fork();
```
