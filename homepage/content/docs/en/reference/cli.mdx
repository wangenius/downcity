## Runtime model (short)

- `town start`: start Town runtime only
- `town start --console` / `town start -a`: start Town runtime and Console together
- `town console start`: start Console only, after Town runtime is already running
- `town stop`: stop Console, then stop Town runtime and managed agents
- `town restart`: restart Town runtime, restore managed agents, then restart Console
- `town update`: update the currently installed global `downcity` CLI bundle
- `town public`: manage persisted public exposure for Console, with interactive mode plus `on/off/status`
- `town agent`: open the interactive agent manager in a TTY
- `town agent create`: initialize project runtime files
- `town agent list`: inspect registered agents, or use `--running` for active ones only
- `town agent start`: start project agent daemon
- `town agent stop`: stop project agent daemon
- `town agent chat`: chat with an agent in interactive mode or send one-shot messages with `--message`
- `town console`: manage the Console module itself
- `town plugin ...`: inspect plugin catalog, manage plugin lifecycle, or use advanced plugin command forms
- `town chat/task/memory/contact/shell ...`: normal managed-plugin runtime commands
- `town web/skill/asr/tts ...`: normal local plugin commands
- `town model`: open the interactive global model manager in a TTY
- `town model ...`: manage model config for runtime and automation
