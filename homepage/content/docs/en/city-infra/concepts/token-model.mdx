Downcity uses `admin_secret_key` to manage InfraRuntime, and `user_token` to let end-user clients call services.

## user_token

`user_token` is the call credential used by the end user or product client. `AdminClient` requests it from InfraRuntime using `product_id + user_id`.

- `product_id`
- `user_id`
- `metadata`
- `ttl`

The client only carries `user_token` and `product_id`. It never touches provider keys or `admin_secret_key`.

## admin_secret_key

`DOWNCITY_INFRA_ADMIN_SECRET_KEY` is used to manage InfraRuntime: create products, manage env, and issue `user_token` for users under a given product. InfraRuntime generates it automatically and writes it into `.env` on first startup.

```ts
const client = new AdminClient({
  base_url: "https://base.example.com",
  admin_secret_key: process.env.DOWNCITY_INFRA_ADMIN_SECRET_KEY,
});

const product = await client.products.create({
  name: "Chrome Extension",
});
```

## Multi-product reuse

Many clients can belong to the same product. When issuing a token, write `product_id` explicitly:

```ts
const user = await client.products.tokens.apply({
  product_id: product.product_id,
  user_id: "user_123",
  metadata: {
    plan: "pro",
  },
  ttl: "7d",
});

console.log(user.user_token);
```
