`Federation` 是 Downcity 服务端的核心实例（即文档里说的 Federation）。它封装了 HTTP 路由、鉴权、环境变量和数据库基础设施。客户端访问入口是 [`City`](/zh/city-sdk-docs/reference/city-user)，不要和它混淆。

## 最小使用

```ts
import { Federation } from "@downcity/city";

const base = new Federation({ db });
```

## Service 与 Action

每个 Service 是一组 Action。通过 `base.use(service)` 注册：

```ts
import { Service } from "@downcity/city";

const translate = new Service({ id: "translate", name: "翻译" });

// 注册 Action
translate.action("zh2en", async (ctx) => {
  return { result: await api.translate(ctx.input.text, "en") };
});

translate.action("en2zh", async (ctx) => {
  return { result: await api.translate(ctx.input.text, "zh") };
});

base.use(translate);
```

Federation 自动为每个 Action 生成 HTTP 路由：

```
POST /v1/translate/zh2en  →  translate.action("zh2en")
POST /v1/translate/en2zh  →  translate.action("en2zh")
```

## AI Service

AIService 是内置的 Service，用于管理 AI 模型：

```ts
import { createOpenAICompatible } from "@ai-sdk/openai-compatible";
import { AIService, Provider, type OpenAICompatibleClientConfig } from "@downcity/city";

class DeepSeekProvider extends Provider {
  constructor() {
    super({
      id: "deepseek",
      baseURL: "https://api.deepseek.com/v1",
      envKey: "DEEPSEEK_API_KEY",
    });
  }

  protected createClient({ apiKey, baseURL }: OpenAICompatibleClientConfig) {
    return createOpenAICompatible({ apiKey, baseURL, name: "deepseek" });
  }
}

const deepseek = new DeepSeekProvider();

const ai = new AIService();
ai.use(deepseek.model({ id: "deepseek-v4-flash", name: "DeepSeek V4 Flash" }));
base.use(ai);
```

## Action 注册选项

```ts
// 默认 POST，auth=["user"]
svc.action("zh2en", fn);

// GET action
svc.action("list", fn, { method: "GET" });

// Admin action
svc.action("remove", fn, { auth: ["admin"] });

// Guest action（免登录）
svc.action("webhook", fn, { auth: [] });
```

## Hook 三层

```ts
// Action 级 hook（只在这个 action 触发）
const zh2en = svc.action("zh2en", fn);
zh2en.before(checkBalance).after(deductFee);

// Service 级 hook（所有 action 共享）
svc.hook.before(logRequest);

// 执行顺序：
//   action.before → service.before → action.run → service.after → action.after
```

## HTTP Middleware

`base.middle()` 用于注册 Federation 级 HTTP middleware。它们会在内部路由和 Action body 读取之前执行，适合放 CORS、安全响应头、body 大小限制、粗粒度限流和请求超时。

```ts
import {
  bodyLimit,
  clientIp,
  cors,
  memoryRateLimitStore,
  rateLimit,
  requestTimeout,
  securityHeaders,
} from "@downcity/city";

base.middle(securityHeaders());
base.middle(cors({
  origins: ["https://app.example.com"],
  methods: ["GET", "POST", "OPTIONS"],
  headers: ["Content-Type", "Authorization"],
  max_age: 86400,
}));
base.middle(bodyLimit({ max_bytes: 1_000_000 }));
base.middle(rateLimit({
  window_ms: 60_000,
  max: 120,
  key: (ctx) => clientIp(ctx.request),
  match: (ctx) => new URL(ctx.request.url).pathname.startsWith("/v1/"),
  store: memoryRateLimitStore(),
}));
base.middle(requestTimeout({
  ms: 30_000,
  exclude_paths: ["/v1/ai/stream", "/v1/ai/chat/completions"],
}));
```

middleware 按注册顺序进入，按相反顺序退出。middleware 可以直接返回 `Response` 来短路请求，也可以 `await next()` 后修改响应头。

## instruction 文档聚合

`Federation` 可以直接返回当前实例的说明文档字符串：

```ts
const text = await base.instruction();
```

这份文本会聚合：

- Federation 自身怎么使用
- 当前注册的 service
- 每个模块声明的 env 配置
- 服务补充的使用说明

如果你要从远程管理端读取同一份说明，可以调用：

```txt
GET /v1/base/instruction
```

这个接口只允许管理端访问，返回 `text/plain`。

## 鉴权

Federation 自动处理 `user_token` 和 `admin_secret_key` 的校验。Action 默认要求 `user_token`；传 `auth: ["admin"]` 时只允许管理端访问，传 `auth: []` 时表示免登录。

详见 [信任边界](/zh/city-sdk-docs/concepts/trust-boundary)。
