# 安装与连接

## 前置条件

在使用扩展前，你通常需要先启动：

```bash
city start
city start
city agent start
```

如果你是从仓库本地加载扩展，还需要先构建扩展产物：

```bash
npm run build:extension
```

## 在 Chrome 中加载

1. 打开 `chrome://extensions`
2. 打开右上角 `开发者模式`
3. 点击 `加载已解压的扩展程序`
4. 选择 `products/chrome-extension/dist`

## 第一次连接要配置什么

扩展默认连接：

- Host: `127.0.0.1`
- Port: `5315`

如果你的 Console UI 不在默认地址，打开扩展设置页。设置页分成三块：

- `Connect Console`：填写 Console Host、Port，并粘贴 Bearer Token
- `Inline Composer`：选择默认模式、即时执行器、即时模型和即时 ACP Agent
- `Agent / Channel`：选择 Popup 默认投递的 Agent 和默认 Channel Chat

如果你还没有 token，先在本机终端执行：

```bash
city token create my-token
```

如果你更喜欢交互式流程，也可以直接执行：

```bash
city token
```

创建完成后可直接复制 token。

然后把返回的 token 粘贴到扩展设置页的 `Access Token`。

保存 Token 后，扩展会自动在后续请求里附加 `Authorization: Bearer ...`。

## Inline Composer 的两种模式怎么工作

页内输入框支持两种模式：

- `频道模式`：沿用默认 Agent / Channel Chat 投递
- `即时模式`：创建临时 session，当场执行并直接回显结果

`即时模式` 里还可以选择一个 executor：

- `Model Executor`：使用这里选定的默认模型
- `ACP Executor`：使用这里选定的 ACP Agent 项目

如果你切到 `即时模式`，但当前 executor 没有配置可用目标，Inline Composer 会提示你先回设置页补齐配置。

## Agent 和 Chat 怎么选

扩展会先从 Console 拉取 Agent 列表，再读取该 Agent 的可用 chat context。

如果没有可用 Chat，通常说明：

- 对应渠道还没有产生过历史消息
- Agent 还没运行
- Console 地址配置错了

## 更新扩展后该做什么

每次改了 extension 代码后，先重新构建：

```bash
npm run build:extension
```

然后回到 `chrome://extensions`，点击该扩展的刷新按钮。
