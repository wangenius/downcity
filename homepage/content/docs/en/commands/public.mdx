# `bay public`

`bay public` manages the persisted public exposure mode for Console.

This command only controls the Console / control plane bind behavior. It does not change how individual agents listen.

## Usage

```bash
bay public
```

```bash
bay public status
```

```bash
bay public on
```

```bash
bay public on --host 0.0.0.0
```

```bash
bay public off
```

## Actions

- no action: open the interactive public-mode manager in a TTY
- `status`: show persisted public mode and current Console runtime state
- `on`: enable persisted public mode
- `off`: disable persisted public mode

## Common options

- `--host <host>`: bind host to use when public mode is enabled, default `0.0.0.0`

## Important behavior

- `bay public on` saves a persisted Console public-mode setting for future `bay start` and `bay console start`.
- If Console is already running, `bay public on` or `bay public off` automatically restarts Console so the new binding takes effect.
- Explicit runtime flags still win. For example, `bay console start --host 127.0.0.1` overrides the persisted public setting for that launch.
- `bay public` only manages Console exposure. It is not a global switch for every runtime endpoint.

## Typical flow

```bash
# 1) inspect current persisted mode
bay public status

# 2) enable public exposure for future starts
bay public on

# 3) or pin a custom host
bay public on --host 10.0.0.5

# 4) start city as usual
bay start

# 5) disable public exposure again
bay public off
```
