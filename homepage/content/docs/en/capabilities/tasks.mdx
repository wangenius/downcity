# Tasks

Task definitions live in `.ship/task/<taskId>/task.md`.

## Minimal task example

```markdown
---
title: Daily Repo Check
when: "0 9 * * 1-5"
description: Run tests and summarize issues
contextId: telegram-chat-123456
status: enabled
kind: agent
---

Run tests, write channel-delivered body to output.md, and write summary to result.md.
```

## Result Delivery

- Task runs only write artifacts under `.ship/task/<taskId>/<timestamp>/`.
- Results are not auto-sent to chat channels.
- `contextId` is used as the task execution context identifier.

## Directory layout

```text
.ship/task/
  <taskId>/
    task.md
    <timestamp>/
      run.json
      output.md
      result.md
      dialogue.md
```

## Common commands

```bash
sma task list
sma task create --title "Daily check" --description "Run tests" --context-id telegram-chat-123 --when "0 9 * * 1-5" --activate
sma task run daily-check
sma task update daily-check --when "@manual"
sma task disable daily-check
```

## Status

- `enabled`: scheduled
- `paused`: defined but not scheduled
- `disabled`: disabled

## Runtime requirement

Scheduling only works with a running runtime (typically `sma agent start`).
