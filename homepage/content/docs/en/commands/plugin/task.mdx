# `town task`

`town task` is the managed task plugin command group.

Use it when you want a reusable automation definition instead of a one-shot delayed plugin action.

## Usage

```bash
town task <subcommand> [options]
```

## Main subcommands

- `list`
- `create`
- `run <title>`
- `update <title>`
- `delete <title>`
- `status <title> <enabled|paused|disabled>`
- `enable <title>`
- `disable <title>`

## Key boundaries

- Use `town task` for reusable or long-lived automation.
- Use `town chat --delay/--time` or `town memory --delay/--time` for one existing action that should happen later once.
- Task definitions live in project storage and can use `@manual`, cron, or `time:<ISO8601>`.

## Examples

```bash
town task list
town task create --title "Daily check" --description "Run tests" --session-id telegram-chat-123 --when "0 10 * * 1-5" --activate
town task run daily-check
town task update daily-check --when "@manual"
town task disable daily-check
```
