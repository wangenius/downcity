# Products

Products Docs cover the surfaces AI builders operate directly. They sit on top of the shared Downcity runtime: local agent hosting, browser control, web context capture, SDKs, and reusable UI.

## Product Spaces

- [Town CLI](/en/products-docs/town-cli/init): install, initialize, start, stop, configure, and operate local agents.
- [Town Console](/en/products-docs/town-console): use the browser control surface for agents, models, plugins, and runtime status.
- [Chrome Extension](/en/products-docs/chrome-extension/overview): connect web pages to your Town and send page context into agents.

## When To Switch Spaces

- Use [Shared Services SDK](/en/city-sdk-docs) when you are building or deploying the shared service runtime.
- Use [Agent SDK](/en/agent-sdk-docs) when you are embedding programmable agents.
- Use [Services SDK](/en/services-sdk-docs) when you are adding accounts, balance, usage, or payments.
