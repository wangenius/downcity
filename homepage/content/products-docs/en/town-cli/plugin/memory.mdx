# `town memory`

`town memory` is the managed memory plugin command group.

Use it when the agent or operator needs to inspect, write, or organize memory explicitly instead of relying only on prompt-time memory injection.

## Usage

```bash
town memory <subcommand> [options]
```

## Main Subcommands

- `status`
- `search <query> [--include-sources]`
- `read <memoryPath> [--from <line>] [--lines <count>]`
- `remember --content <text> [--topic <topic>] [--wiki-path <path>] [--source <source>]`
- `digest --session-id <sessionId> [--max-messages <count>]`
- `revise <memoryPath> --instruction <text> [--evidence <text>]`

## Notes

- MemoryPlugin uses an LLM Wiki structure: `.downcity/memory/wiki/` is the knowledge layer, and `.downcity/memory/sources/` is the evidence layer.
- The system prompt only injects a small stable wiki summary; deeper retrieval goes through `memory.search` and `memory.read`.
- `memory.search` defaults to wiki and working memory; pass `--include-sources` when evidence is needed.
- All memory actions support one-shot `--delay` / `--time`.

## Examples

```bash
town memory status
town memory search "release decision"
town memory read .downcity/memory/wiki/project-decisions.md --from 10 --lines 20
town memory remember --topic "plugin system" --content "Plugin config should be passed through constructors."
town memory digest --session-id telegram-chat-123 --max-messages 40
town memory revise .downcity/memory/wiki/plugin-system.md --instruction "Merge latest constructor config decision"
```
