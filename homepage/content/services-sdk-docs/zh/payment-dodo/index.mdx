---
title: "Dodo Payment 服务"
description: "把一次性 Dodo Payments Checkout 同步成用户全局余额充值。"
---

`dodoPaymentService()` 会创建 Dodo Payments Checkout，并在 Dodo 通过 webhook 确认支付成功后，完成一笔已经存在的 `balance` topup。

这个服务只负责 Checkout 创建、本地支付记录、webhook 事件记录，以及最后调用 `balance.finishTopup()` 入账。

## 启用方式

```ts
import { balanceService } from "@downcity/services";
import {
  dodoPaymentMethod,
  dodoPaymentService,
  paymentService,
  type DodoPaymentServiceBalanceBridge,
} from "@downcity/services";

const balance = balanceService();
const dodoBalanceBridge: DodoPaymentServiceBalanceBridge = {
  readTopup: async (topup_id) => await balance.readTopup(topup_id),
  finishTopup: async (topup_id, extra) => await balance.finishTopup(topup_id, extra),
};

base.use(balance);
base.use(paymentService({
  methods: [
    dodoPaymentMethod(),
  ],
}));
base.use(dodoPaymentService({
  balance: dodoBalanceBridge,
}));
```

## 必需 env

- `DODO_PAYMENTS_API_KEY`：`dodopayments` SDK 使用的 API key
- `DODO_PRODUCT_ID`：Dodo Checkout 使用的 product
- `DODO_WEBHOOK_KEY`：可选，用于校验 webhook 签名
- `DODO_ENVIRONMENT`：可选，`test_mode` 或 `live_mode`，默认 `test_mode`
- `DOWNCITY_CITY_BASE_URL`：可选，City 对外访问地址，用于生成内置结果页
- `DODO_CURRENCY`：可选，支付方式目录展示用币种
- `DODO_API_BASE_URL`：可选，测试环境 API 地址覆写

Dodo 的 return / cancel URL 会自动基于 `DOWNCITY_CITY_BASE_URL` 生成；未配置时会回退到当前请求 origin，方便本地开发。

## 前端流程

先通过 `balance` 创建可信 topup，再为它创建 Dodo checkout：

```ts
const checkout = await user.service("payment.dodo").action("checkout/create").invoke({
  topup_id: "topup_demo",
});

location.href = checkout.checkout_url;
```

`paymentService()` 也会在支付方式目录里暴露 Dodo：

```ts
const methods = await guest.service("payment").get("methods");
```

只有同时配置 `DODO_PAYMENTS_API_KEY` 和 `DODO_PRODUCT_ID` 时，Dodo method 才会是 enabled。

## Webhook 同步

把 Dodo webhook 事件发到：

```txt
POST /v1/payment.dodo/webhook
```

服务会基于事件 ID 幂等应用 `payment.succeeded`。重复事件只返回已有同步状态，不会重复给余额入账。

## 路由

- `GET /v1/payment/methods`
- `POST /v1/payment.dodo/checkout/create`
- `GET /v1/payment.dodo/payments/me`
- `GET /v1/payment.dodo/payments`
- `GET /v1/payment.dodo/events`
- `POST /v1/payment.dodo/webhook`
- `GET /v1/payment.dodo/redirect/success`
- `GET /v1/payment.dodo/redirect/cancel`
