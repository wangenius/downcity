This section answers one question only: **how do I use it concretely?**

If you are still unsure what Downcity is, why it is split this way, or where the boundaries are, go back to [Concepts](/en/city-sdk-docs/concepts) or [Quickstart](/en/city-sdk-docs/quickstart) first.

## Suggested order

1. Writing product-side code: start with [User City](/en/city-sdk-docs/reference/city-user)
2. Wiring server-side runtime: continue with [Federation](/en/city-sdk-docs/reference/federation)
3. Writing login and token issuance: read [Admin City](/en/city-sdk-docs/reference/city-admin)
4. Hand-writing requests: finish with [HTTP API](/en/city-sdk-docs/reference/http-api)
