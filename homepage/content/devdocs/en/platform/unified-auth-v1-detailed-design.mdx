# Unified Auth V1 Detailed Design

This page continues the V1 implementation doc and goes one layer deeper.

## What this page adds

- table-level schema detail
- API request and response contracts
- middleware ordering
- module responsibilities
- route-permission matrix
- rollout phases
- test matrix

## Schema Focus

Suggested tables:

- `auth_users`
- `auth_roles`
- `auth_permissions`
- `auth_user_roles`
- `auth_role_permissions`
- `auth_tokens`
- `auth_audit_logs`

Key V1 rule:

- store only token hash, never the plaintext token

## API Focus

Suggested V1 auth endpoints:

- `POST /api/auth/bootstrap-admin`
- `POST /api/auth/login`
- `GET /api/auth/me`
- `POST /api/auth/token/create`
- `GET /api/auth/token/list`
- `POST /api/auth/token/revoke`

## Middleware Focus

Recommended order:

1. request id
2. logger
3. origin guard
4. auth middleware
5. permission middleware
6. handler

## Module Focus

Suggested auth root:

- `packages/downcity/src/main/auth/`

Suggested first modules:

- `AuthSchema`
- `AuthStore`
- `AuthService`
- `TokenService`
- `AuthMiddleware`
- `RoutePolicy`
- `PermissionPolicy`
- `AuthRoutes`
- `TokenRoutes`

## First Route Matrix

- `/api/execute` -> `agent.execute`
- `/api/services/list` -> `service.read`
- `/api/services/control` -> `service.write`
- `/api/plugins/list` -> `plugin.read`
- `/api/dashboard/sessions` -> `session.read`
- `/api/dashboard/tasks/run` -> `task.run`
- `/api/ui/env` -> `env.read`
- `/api/ui/command/execute` -> `shell.execute`

## Rollout Phases

1. schema and store
2. login and token
3. auth middleware
4. protect existing routes
5. update Console UI, CLI, and Extension

## Test Focus

At minimum:

- unauthenticated request returns `401`
- revoked token returns `401`
- insufficient permission returns `403`
- bootstrap admin can run only once
- token plaintext is returned only once
