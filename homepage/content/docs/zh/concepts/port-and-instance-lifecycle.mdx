# 端口与实例生命周期

## 统一规则

- agent 端口由 Town runtime 分配。
- agent 生命周期操作不要求用户手工管端口。
- plugin 调用先找目标 agent，再找 endpoint。

## 运行时角色

### Town runtime

- 维护 `~/.downcity/main/agents.json`
- 启停 agent daemon
- 分配端口

### Console

- 提供用于观察和控制 agent 的浏览器 UI
- 不负责拥有本机 runtime 启动

### agent

- 一个项目对应一个 daemon
- runtime API 包括 `/api/plugins/list`、`/api/plugins/catalog` 和 `/api/plugins/*`
- debug 信息写入 `.downcity/.debug`

### plugin 端点

- plugin 调用走 plugin 路由
- 都在目标 runtime 内部分发

## Agent 端口分配

1. 确认 Town runtime 正在运行
2. 校验项目文件
3. 分配可用端口
4. 启动 daemon
5. 写入 debug 信息
6. 更新受管 agent registry

## endpoint 解析

对 plugin API 调用来说：

1. CLI 显式 `--host/--port`
2. `DC_CITY_HOST/DC_CITY_PORT`
3. daemon metadata
4. 默认 `127.0.0.1:5314`

## 建议

- 多 agent 场景下，plugin 操作应该显式解析目标 agent。
- 不要在 shell profile 里残留错误的 `DC_CITY_PORT`。
- runtime 状态异常时，先修复 daemon 元数据并检查 daemon 日志。
