# CLI

`downcity` CLI 是统一的 Downcity 栈管理工具。它通过单一终端界面覆盖 Agent 项目、City 基础设施和 Federation 配置。

## 功能

### Agent 项目

- 创建和初始化 Agent 项目
- 读取 `downcity.json` 并将配置传给 Agent SDK
- 启动、停止和重启 Agent
- 管理渠道账号和凭证

### City

- 登录和登出 City 身份
- 管理 API token 和权限
- 检查用量、余额和计费状态
- 配置渠道账号（Telegram、飞书、QQ 机器人）

### Federation

- 列出可用模型提供商和模型 ID
- 配置项目模型绑定
- 检查提供商健康和状态

## 适用人群

CLI 适合想从终端管理整个 Downcity 栈的用户。如果你正在构建需要编程控制的产品，请直接使用 SDK。

## 关键命令

### Agent

| 命令 | 功能 |
|---------|-------------|
| `downcity agent init` | 创建新 Agent 项目 |
| `downcity agent start` | 在当前项目启动 Agent |
| `downcity agent stop` | 停止运行中的 Agent |
| `downcity agent status` | 检查 Agent 是否正在运行 |
| `downcity agent logs` | 查看运行时日志 |

### City

| 命令 | 功能 |
|---------|-------------|
| `downcity login` | 向 City 认证 |
| `downcity logout` | 移除本地 City 凭证 |
| `downcity chat add` | 添加聊天渠道账号 |
| `downcity chat list` | 列出已注册的渠道账号 |
| `downcity usage` | 查看当前用量和余额 |

### Federation

| 命令 | 功能 |
|---------|-------------|
| `downcity models list` | 列出可用模型 |
| `downcity providers list` | 列出已注册的模型提供商 |
| `downcity providers status` | 检查提供商健康 |

## 与 SDK 的关系

CLI 是 SDK 的包装器。它读取配置文件，通过 City 解析模型绑定，并在底层调用 SDK 构造函数。无论你使用 CLI 还是直接使用 SDK，执行核心都是相同的。
