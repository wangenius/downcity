# `sma service`

Manage service runtime state and forward service commands.

## Command hierarchy

- Top-level command: `sma service`
- Action commands: `sma chat` / `sma task` / `sma memory` / `sma skill`

These action commands belong to the service system and are not top-level commands.

## Usage

```bash
sma service <subcommand>
```

## Subcommands

- `list`
- `status <serviceName>`
- `start <serviceName>`
- `stop <serviceName>`
- `restart <serviceName>`
- `command <serviceName> <command> --payload <json>`

## Notes

`sma service` is path-bound to an agent project.

- You can pass `--path <agentProjectPath>`
- Or pass `--agent <agentName>` to resolve from console registry
- If omitted, it uses current directory `.`
- The resolved path must be registered in console registry
- The resolved path must contain `ship.json` and `PROFILE.md`

Inside agent shell, when `--agent` is omitted and `--path` uses default, it first resolves by injected `SMA_AGENT_NAME` via registry; default `.` also prefers injected `SMA_AGENT_PATH`.

When both are passed, `--agent` takes precedence.

Start agent runtime first:

```bash
sma agent on
```
