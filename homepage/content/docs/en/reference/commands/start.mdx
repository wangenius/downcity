# `sma start`

Start the SMA Manager background process.

## Usage

```bash
sma start
```

## Behavior

- First run creates manager runtime directory and starts a detached process.
- If manager is already running, command returns with an "already running" message.
- Manager is a global control process for agent runtime state, not a business scheduler.

## Runtime files

Written under the user home directory:

- `~/.ship/manager/manager.pid`
- `~/.ship/manager/manager.log`

## Typical flow

```bash
# 1) start manager
sma start

# 2) enter an agent directory and start the agent (background by default)
cd /path/to/agent
sma agent on
```
