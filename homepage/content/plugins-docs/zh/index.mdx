# Agent Plugins Docs

这一组文档专门讲“具体 plugin 本身”。

当你想看 `web`、`skill`、`chat`、`task` 这类具体插件时，应该从这里读。

如果你想先理解 plugin 的设计模式和概念，请先看 [Plugin Overview](/zh/agent-sdk-docs/plugins/overview)。

## 这里主要覆盖什么

- 当前代码里到底有哪些 built-in plugin
- 哪些内建 plugin class 从 `@downcity/plugins` 公开导出
- 哪些 plugin 更偏平台内部或基础设施
- 每个 plugin 的职责边界
- 每个 plugin 的 lifecycle、availability、config 和 action 形状

## 推荐阅读路径

- 先看 [Built-ins Overview](/zh/plugins-docs/builtins/overview)
- 再进入你关心的具体 plugin 页面
- 想看更场景化的写法，再去 [Examples](/zh/plugins-docs/examples)

## 最好先记住的心智模型

这里有两个不同问题：

1. plugin 系统本身是怎么工作的
2. 某一个具体 plugin 到底负责什么

这一组文档主要回答第二个问题。
