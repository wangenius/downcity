# 飞书

飞书渠道的项目层配置方式与 Telegram 类似：

```json
{
  "services": {
    "chat": {
      "channels": {
        "feishu": {
          "enabled": true,
          "channelAccountId": "feishu-main"
        }
      }
    }
  }
}
```

项目配置只负责绑定账号，不负责保存飞书凭据本身。

如果飞书消息链路不通，优先检查：

- 绑定的 `channelAccountId` 是否存在
- agent 是否已经重启
