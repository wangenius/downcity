# 故障排查

## 1. 无法连接到 Downcity Town

先检查：

- `town agent start` 是否正在运行
- `Town URL` 是否能被 Chrome 访问
- 本机开发是否使用 `http://127.0.0.1:5314`
- 是否误用了 `15314`。这是 Agent RPC 端口，不是浏览器 HTTP 地址

如果你刚更新过扩展代码，先执行：

```bash
npm run build:extension
```

然后刷新扩展。

## 2. 当前 Town 要求 Token

如果你看到：

- `Missing bearer token`
- `Invalid bearer token`
- `Permission denied`

通常说明目标 Town 已开启鉴权，但扩展还没有可用 Token。

处理方式：

1. 打开扩展设置页
2. 在本机终端执行 `town token create my-token`
3. 把 token 粘贴到 `Town Token`
4. 点击 `保存并检查`

## 3. 远程 Town 没有对外暴露

如果你访问的是远程机器，需要确认 Town runtime HTTP API 已经通过产品部署、反向代理、隧道或托管层对外可访问。

## 4. 没有可用 Agent

通常表示：

- `town agent start` 还没启动
- Town URL 不对
- 当前 Agent 没有处于运行状态

## 5. Side Panel 没有消息

扩展会自动创建浏览器会话。如果 Side Panel 为空，直接发送第一条消息，或勾选附带当前页面后发送。

## 6. 抓到的正文不对

全页抓取是 best-effort，不是浏览器阅读模式的完全克隆。

当前逻辑会：

- 优先选择最像正文的 `main/article`
- 过滤导航、侧边栏和隐藏内容
- 尝试合并多个强相关主体区块

如果页面结构非常特殊，仍然可能抓到多余内容。
