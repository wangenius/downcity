# Task Service

`task` service manages task definitions and execution.

## Responsibilities

- Create/update/delete task definitions
- Enable/disable/pause task status
- Run tasks immediately
- Manage cron runtime and scheduler reload
- De-duplicate semantically equivalent task creation in the same `contextId` + `kind`

## De-duplication On Create

When calling `create`, task service checks existing tasks under the same `contextId` and `kind`.

- If a task with the same semantic goal is found, service reuses that task instead of creating a duplicate.
- Response returns the existing `taskName` and may include `reusedExisting=true`.

## Naming

- Use one name field only: `taskName`.
- Do not split semantics into separate `taskId` and title fields in user workflows.

## Runtime operations

```bash
sma service status task
sma service command task list --payload '{"status":"enabled"}'
```
