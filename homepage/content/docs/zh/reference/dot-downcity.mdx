# `.downcity/` 目录

`.downcity/` 是 Downcity 的本地运行数据目录，默认建议不提交到 Git。

通过 Downcity 初始化项目时，系统会自动把 `.downcity` 追加到当前项目的 `.gitignore`。

## 常见结构

```text
.downcity/
  logs/
  session/
  memory/
  channel/
  chat/
  task/
  config/
  schema/
  public/
  profile/
  .cache/
```

## 重点目录

### `.downcity/logs/`

每日 JSONL 日志：`<YYYY-MM-DD>.jsonl`。

### `.downcity/agents/<agentId>/sessions/<sessionId>/messages/`

会话消息事实源：`messages.jsonl`，并包含 compact 元数据与归档。

### `.downcity/memory/`

Memory V2 存储层：
- `MEMORY.md`：长期事实/偏好记忆。
- `daily/<YYYY-MM-DD>.md`：按天追加的记忆流水。

### `.downcity/chat/`

chat 元信息与 history 事件流。`access.db` 保存当前 Agent 的 Chat Principal、Request、Allow/Deny Grant 和审计事件；`migration-backup/authorization/` 保存一次性迁移后的旧授权 JSON。

### `.downcity/channel/meta.json`

渠道目标到 `sessionId` 的映射快照（由 chat plugin runtime 维护）。

### `.downcity/task/`

任务定义与每次运行产物（`run.json`、`result.md` 等）。

### Skills 目录位置

`SkillPlugin` 默认扫描 `.agents/skills`（项目级）。如果在构造函数中启用 `use: ["home"]`，也会扫描 `~/.agents/skills`（用户级）。

## 安全建议

- `.downcity/` 可能含敏感对话与执行痕迹，不建议提交到公开仓库。
- 需要排障时优先查看 `logs`、`task`、`session` 三处。
