# `downcity.json` 参考

当前版本采用两层模型配置：

1. `~/.downcity/downcity.db`（console 全局 SQLite）管理完整模型池  
2. 项目 `downcity.json`（agent 本地）只声明 `execution`

## 1) Agent 项目最小结构

```json
{
  "$schema": "./.downcity/schema/downcity.schema.json",
  "name": "my-agent",
  "version": "1.0.0",
  "execution": {
    "type": "acp",
    "agent": {
      "type": "kimi"
    }
  },
  "services": {}
}
```

如果你使用 console 模型池，`execution.type` 应为 `model`，并且 `execution.modelId` 必须能在 `~/.downcity/downcity.db` 的模型池中找到同名 model id。

如果你使用 ACP coding agent，`execution.type` 应为 `acp`，并在 `execution.agent.type` 中声明 `kimi`、`claude` 或 `codex`。

## 2) Console 全局模型池（`~/.downcity/downcity.db`）

模型细节统一在 SQLite：

- providers
- models
- provider.apiKey 为加密存储（`api_key_encrypted`）

通过命令管理：

```bash
city model list
city model get provider openai_main
```

## 3) 运行时解析规则

agent 启动时：

1. 先检查项目是否声明了 `execution`
2. 如果 `execution.type = "model"`，到 `~/.downcity/downcity.db` 的 models 表取模型
3. 再由 `providerId` 关联 providers 表
4. 如果 `execution.type = "acp"`，直接启动对应 ACP agent

`execution` 必须合法，否则启动失败。

## 4) Plugin 配置

插件行为配置写在项目 `downcity.json` 的 `plugins.*`。

插件如果需要额外模型、命令模板或本地路径，也直接写在对应的 `plugins.<name>.*` 下。

例如 asr：

```json
{
  "plugins": {
    "asr": {
      "enabled": true,
      "provider": "local",
      "modelId": "SenseVoiceSmall"
    }
  }
}
```


## 5) Chat 渠道绑定

`downcity.json` 中的 chat 渠道应只绑定全局 bot 账户 ID：

```json
{
  "services": {
    "chat": {
      "channels": {
        "telegram": {
          "enabled": true,
          "channelAccountId": "telegram-main"
        }
      }
    }
  }
}
```

`channelAccountId` 会解析到 `~/.downcity/downcity.db` 的 `channel_accounts.id`。

## 6) ACP 执行模式

当前支持以下 ACP coding agent：

- `kimi`：默认命令 `kimi acp`
- `claude`：默认命令 `npx -y @zed-industries/claude-code-acp`
- `codex`：默认命令 `npx -y @zed-industries/codex-acp`

示例：

```json
{
  "execution": {
    "type": "acp",
    "agent": {
      "type": "kimi"
    }
  }
}
```

也可以覆盖命令：

```json
{
  "execution": {
    "type": "acp",
    "agent": {
      "type": "claude",
      "command": "claude-acp",
      "args": []
    }
  }
}
```

当前限制：

- `execution.type="acp"` 只作用于 session/chat/task 执行链。
- 走模型模式时，必须提供 `execution.modelId`。

## 7) 推荐操作方式

1. 如果你走模型池模式，在 console 层维护 provider/model，再在 agent 层配置 `execution.modelId`  
2. 如果你走 coding agent 模式，在 agent 层配置 `execution.type="acp"`  
3. 修改后重启 agent：

```bash
city agent restart .
```
