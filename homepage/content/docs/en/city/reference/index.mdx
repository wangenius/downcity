This section answers one question only: **how do I use it concretely?**

If you are still unsure what Downcity is, why it is split this way, or where the boundaries are, go back to [Concepts](/en/docs/city/concepts) or [Quickstart](/en/docs/city/quickstart) first.

## Suggested order

1. Writing studio-side code: start with [User Gate](/en/docs/city/reference/gate-user)
2. Wiring server-side runtime: continue with [City](/en/docs/city/reference/city-runtime)
3. Writing login and token issuance: read [Admin Gate](/en/docs/city/reference/gate-admin)
4. Hand-writing requests: finish with [HTTP API](/en/docs/city/reference/http-api)
