## Runtime model (short)

- `bay start`: start Bay runtime only
- `bay start --console` / `bay start -a`: start Bay runtime and Console together
- `bay console start`: start Console only, after Bay runtime is already running
- `bay stop`: stop Console, then stop Bay runtime and managed agents
- `bay restart`: restart Bay runtime, restore managed agents, then restart Console
- `bay update`: update the currently installed global `downcity` CLI bundle
- `bay public`: manage persisted public exposure for Console, with interactive mode plus `on/off/status`
- `bay agent`: open the interactive agent manager in a TTY
- `bay agent create`: initialize project runtime files
- `bay agent list`: inspect registered agents, or use `--running` for active ones only
- `bay agent start`: start project agent daemon
- `bay agent stop`: stop project agent daemon
- `bay agent chat`: chat with an agent in interactive mode or send one-shot messages with `--message`
- `bay console`: manage the Console module itself
- `bay plugin ...`: inspect plugin catalog, manage plugin lifecycle, or use advanced plugin command forms
- `bay chat/task/memory/contact/shell ...`: normal managed-plugin runtime commands
- `bay web/skill/asr/tts ...`: normal local plugin commands
- `bay model`: open the interactive global model manager in a TTY
- `bay model ...`: manage model config for runtime and automation
