These pages are not API reference. They answer the more practical question: how should you use Downcity in real town situations?

If you already understand the basics but are now dealing with real towns, users, and providers, this section is the most useful one.

## Suggested order

1. Building a new town first: read [Pure-client towns](/en/docs/city/guides/pure-client-towns)
2. Already operating multiple towns: continue with [Many towns, one City](/en/docs/city/guides/multi-town-city)
3. Deploying on Cloudflare Workers: continue with [Cloudflare Workers](/en/docs/city/guides/cloudflare-workers)
4. Ready for real users: read [Auth and `user_token`](/en/docs/city/guides/auth-and-user-token)
5. Ready for real providers: continue with [Provider environment](/en/docs/city/guides/provider-env) and [Model routing](/en/docs/city/guides/model-adapters)
