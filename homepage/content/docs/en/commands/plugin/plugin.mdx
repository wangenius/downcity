# `city plugin`

`city plugin` is the generic entry for the new plugin system.

Use it when you want to:

- inspect which plugins are registered
- inspect a plugin's static enabled state and agent-side availability
- dispatch a plugin action without using the plugin's short alias

## Common examples

```bash
city plugin list
city plugin status voice
city plugin action voice status
```

## Notes

- `city plugin list` and `city plugin status` fall back to the console-side static plugin catalog when the agent is unavailable, and they try to reflect plugin enabled state from project `downcity.json`.
- `city plugin action ...` still talks to the target agent API, so start the agent first with `city agent start`.
- Plugin config is stored in project `downcity.json` under `plugins.*`.
- `plugins.<name>.enabled` controls whether a plugin enters the real execution path. When disabled, its hooks and `plugin.system` injection are skipped.
- Disabled plugins also have their explicit `plugin action` calls rejected by the registry instead of executing the action body.
- If `plugins.<name>` exists but does not explicitly set `enabled`, the agent falls back to the plugin's own default instead of forcing it to `true`.
