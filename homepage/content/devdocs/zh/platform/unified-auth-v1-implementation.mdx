# 统一账户系统 V1 实施稿

这一页把统一账户系统架构继续收敛成第一阶段可直接执行的实施计划。

## V1 目标

V1 只做一件事：

```text
建立统一账户系统，并把所有受保护接口改成统一 Bearer Token 鉴权。
```

## V1 包含

- 管理员 bootstrap
- 登录
- Bearer Token 签发与校验
- 鉴权中间件
- 路由保护策略
- 基础审计日志

## 建议的数据表

- `auth_users`
- `auth_roles`
- `auth_permissions`
- `auth_user_roles`
- `auth_role_permissions`
- `auth_tokens`
- `auth_audit_logs`

## 建议的新模块区域

- `packages/downcity/src/main/auth/`

建议第一批模块：

- `AuthSchema`
- `AuthStore`
- `AuthService`
- `TokenService`
- `AuthMiddleware`
- `RoutePolicy`
- `PermissionPolicy`
- `AuthRoutes`
- `TokenRoutes`

## 第一批受保护接口

这些接口全部应要求 Bearer Token：

- `/api/execute`
- `/api/services/*`
- `/api/plugins/*`
- `/api/dashboard/*`
- `/api/ui/*`

## 第一批落地顺序

1. 建 schema 和 auth store
2. 加管理员 bootstrap 和 login
3. 加 token 创建、校验、吊销
4. 接统一鉴权中间件
5. 收口 runtime 和 console 路由
6. 让 Console UI、CLI、Chrome Extension 都开始带 token

## 完成判定

V1 完成应满足：

- 系统可以初始化首个管理员
- 登录可以返回 Bearer Token
- 受保护接口会拒绝未鉴权请求
- 高危操作可以按权限拦截
- Console UI 和扩展都能正确使用 token
