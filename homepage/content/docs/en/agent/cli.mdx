# Agent CLI

The `downcity` CLI is a convenience wrapper around the Agent SDK. It reads `downcity.json` from your project directory and passes those values to the Agent SDK constructor, then handles lifecycle commands like start, stop, and restart.

## Project initialization

```bash
downcity agent init my-project
```

This generates:

- `downcity.json`
- `PROFILE.md`
- `SOUL.md`
- `.downcity/`

## Project directory structure

```text
my-project/
├── downcity.json
├── PROFILE.md
├── SOUL.md
├── .downcity/
└── your application code
```

## Key commands

- `downcity agent start` — start the agent in the current project
- `downcity agent stop` — stop the running agent
- `downcity agent restart` — restart the agent
- `downcity agent status` — check if the agent is running
- `downcity agent logs` — view runtime logs

## What the CLI does

1. Reads `downcity.json` from the project directory
2. Resolves `execution.modelId` through the connected Federation AIService
3. Passes the resolved config to `new Agent(...)`
4. Calls `agent.start()`
5. Maintains daemon state so you can inspect, stop, or restart later

## CLI vs SDK

The CLI is for day-to-day project management. The SDK is for embedding agent behavior into your own applications. Both use the same execution core.
