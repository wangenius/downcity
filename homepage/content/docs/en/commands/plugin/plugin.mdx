# `studio plugin`

`studio plugin` is the generic entry for the current plugin system.

> Recommended reading: conceptual plugin docs live in [Agent SDK / Plugins](/en/agent-sdk-docs/plugins/overview), and concrete manuals live in [Plugins Docs](/en/plugins-docs).

Use it when you want to:

- inspect which plugins are registered
- inspect a plugin's static enabled state and capability surface
- start, stop, or restart one managed plugin against a target agent
- dispatch a managed plugin command without using a short alias
- execute a local plugin action directly inside the current project
- inspect or cancel one-shot scheduled plugin jobs

## Common examples

```bash
studio plugin
studio plugin list
studio plugin status asr
studio plugin start chat --agent my-agent
studio plugin command task run --payload '{"title":"daily-check"}' --agent my-agent
studio plugin action asr status
studio plugin schedule list --path /path/to/agent
```

## Notes

- Running bare `studio plugin` in an interactive terminal opens a plugin manager.
- `studio plugin list` and `studio plugin status` are static console-level catalog views.
- Managed plugins such as `chat`, `task`, `memory`, `contact`, and `shell` support `start/stop/restart/command`.
- `studio plugin action ...` is the local execution form and runs directly inside the current project by default.
- For day-to-day usage, prefer concrete command roots such as `studio chat ...`, `studio task ...`, `studio memory ...`, `studio shell ...`, `studio web ...`, `studio skill ...`, `studio asr ...`, and `studio tts ...`.
