# `city console config`

Manage project `downcity.json` and alias.

> Naming note: configuration command is `city console config`, not `city config`.

## Usage

```bash
city console config <subcommand>
```

## Subcommands

- `get [keyPath]`
- `set <keyPath> <value>`
- `unset <keyPath>`
- `alias`

> Model pool management has moved to [`city console model`](/en/docs/commands/console/model).

## Plugin config

`city console config` now reads and writes project `downcity.json` only.

For plugin configuration, edit:

- `plugins.*` for plugin behavior and plugin-owned dependency config

Examples:

```bash
city console config get plugins.asr
city console config set plugins.asr.enabled true
city console config set plugins.asr.modelId '\"SenseVoiceSmall\"'
```

## Examples

```bash
city console config get model.primary
city console config set model.primary fast
city console config unset services.chat.channels.telegram
city console config alias --dry-run
```
