# City 用户身份

Federation 统一签发用户身份，City 客户端使用 `user_token` 访问 Federation 或产品后端。一个 Federation 可以服务多个产品，每个产品由 `city_id` 隔离。

## 身份边界

- **Federation** — 保存 Ed25519 私钥并签发 `user_token`
- **City** — 保存 `federation_url + user_token` 并发起用户请求
- **产品后端** — 使用 `FedBureau` 和 Federation 公开公钥识别用户
- **FederationAdmin** — 独立的 Federation 管理控制面

## 认证流程

1. 用户登录时选择产品 `city_id`
2. Federation 使用私钥签发绑定该 `city_id` 的 `user_token`
3. 用户通过 `new City({ federation_url, user_token })` 调用 Federation
4. 用户携带同一个 Bearer Token 调用产品后端
5. 产品后端的 `FedBureau` 获取并缓存 Federation JWKS，然后识别用户身份

```ts
const city = new City({
  federation_url: "https://fed.example.com",
  user_token,
});
```

产品后端：

```ts
const bureau = new FedBureau({
  federation_url: "https://fed.example.com",
  city_id: "city_product_a",
});

const identity = await bureau.identify(request);
```

公钥可以公开，它只能验证 token，不能签发或伪造 token。Federation 自动发布：

```text
/.well-known/downcity.json
/.well-known/jwks.json
```

产品后端必须预先配置可信的 `federation_url` 和自己的 `city_id`，不能根据未验证 token 动态选择 JWKS。

## 渠道账号

聊天平台的机器人凭证存储在 `~/.downcity/downcity.db` 中，并通过 ID 从项目引用：

```json
{
  "plugins": {
    "chat": {
      "channels": {
        "telegram": {
          "channelAccountId": "telegram-main"
        }
      }
    }
  }
}
```
