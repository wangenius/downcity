# 命令环境要求

这页只回答一个问题：不同 `city` 命令组运行时，各自需要什么环境。

## 0）一级命令边界

一级命令组只有：

- `city console`
- `city agent`
- `city plugin`

像 `city chat`、`city task`、`city memory`、`city shell`、`city skill`、`city asr`、`city tts` 这些，都是 plugin 命令，不是一级根命令。

## 1）按命令组看环境要求

| 命令组 | 必需环境 | 可选环境 | 说明 |
| --- | --- | --- | --- |
| `city init` | 可写 `~/.downcity/` | 无 | 写入 `~/.downcity/downcity.db` |
| `city start/stop/restart/status` | 可写 `~/.downcity/console/` | 无 | 管理全局 console 进程与 registry |
| `city console` | Console 已运行；浏览器可访问 UI 端口 | `--host`、`--port` | 一个 UI 可切换多个运行中的 agent |
| `city agent create` | 目标目录可写 | 无 | 生成 `downcity.json`、`PROFILE.md`、`.downcity/` |
| `city agent start` | Console 已运行；项目包含 `downcity.json` + `PROFILE.md` | `--host` | agent 端口由 console 分配并登记进 registry |
| `city agent restart/status/doctor` | 目标 agent 已登记在 console registry | 无 | 通过 registry 解析 |
| `city config ...` | 目标路径下存在合法 `downcity.json` | `--path` | 只读写项目配置，不调用 agent API |
| `city plugin list/status` | 无 | `--json` | 静态 plugin catalog，不解析 agent |
| `city plugin start/stop/restart/command` | 目标 agent 正在运行 | `--agent`、`--path`、`--host`、`--port` | 面向托管 plugin |
| `city plugin action ...` | 项目存在 `downcity.json` | `--path` | 高级本地执行形式 |
| `city <managed-plugin> <action>` | 目标 agent 正在运行 | `--path`、`--host`、`--port` | 如 `city chat`、`city task`、`city memory`、`city shell` |
| `city <local-plugin> <action>` | 项目存在 `downcity.json` | `--path` | 如 `city skill`、`city web`、`city asr`、`city tts` |

## 2）运行时传输

默认规则：

1. 不传 `--host` / `--port`：托管 runtime 命令默认走本地 IPC
2. 传了 `--host` 或 `--port`：改走远程 HTTP

只有进入远程 HTTP 模式之后，Bearer Token 解析才有意义。

## 3）`DC_AUTH_TOKEN`

对大多数本地使用场景，你不需要关心 auth 环境变量。

本地调用默认走 IPC。`DC_AUTH_TOKEN` 只在你显式走远程 HTTP 时才生效。

优先级顺序：

1. 显式 `--token`
2. `DC_AUTH_TOKEN`

## 4）常见命令的上下文变量

### `city chat ...`

- 省略 `--chat-key` 时，会尝试 `DC_CTX_CHAT_KEY`
- 省略 `--session-id` 时，会先尝试 `DC_SESSION_ID`

### `city task ...`

- task 创建与运行需要 `sessionId`
- 可以显式传 `--session-id`，或依赖 `DC_SESSION_ID`

## 5）推荐用法

日常操作优先直接使用 plugin 根命令：

```bash
city chat send --chat-key ctx_xxx --text "hi"
city task list
city memory search "release decision"
city plugin list
```
