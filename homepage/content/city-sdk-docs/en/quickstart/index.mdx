This section only does one thing: help you run a complete Downcity call chain as quickly as possible.

What you end up with is not just a demo page. It is the smallest city-shaped system:

- one reusable City
- one model record
- one `city_id`
- one `user_token` that an end user can actually use
- one `User City` that successfully calls City

You do not need a separate AI backend for every new city. A new project can start as a client first and connect to an already deployed City.

## Recommended path

1. [Use it now](/en/city-sdk-docs/quickstart/use-now): run the shortest end-to-end path with `cities/node` and `cli/city`
2. [Create City](/en/city-sdk-docs/quickstart/create-city): create City and let the first startup initialize the default database automatically
3. [Add model](/en/city-sdk-docs/quickstart/add-model): register models with AIChannel and AIService
4. [Issue `user_token`](/en/city-sdk-docs/quickstart/issue-token): give a city user a valid call credential
5. [Connect City](/en/city-sdk-docs/quickstart/connect-client): call Federation from the product with `User City`

## Assumptions in this quickstart

- the example City uses `sqlite:./data.sqlite`
- models come from runtime registration code, not handwritten model tables
- all cities currently share the same AIService model directory
- models are defined through `AIChannel.model()` and registered with `AIService.use()`
