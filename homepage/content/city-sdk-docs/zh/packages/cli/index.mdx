`downcity` 包会安装两组命令：

- `fed` / `downfed`：管理 Federation 基础设施，例如创建项目、部署 Worker、保存 admin 连接、直接请求 Federation HTTP API。
- `city` / `downcity`：管理本机 Agent、本地插件、chat/gateway，以及本机加入和登录某个 Federation 的用户态连接。

如果你要部署一套共享后端，用 `fed`。如果你要让本机 Agent 连接、登录、运行和对话，用 `city`。

## fed / downfed

`fed` 面向 Federation 管理者。常用命令：

```bash
fed
fed create .
fed deploy
fed deploy --dry-run
fed deploy --verify
fed server add
fed server manage
fed query GET /health
```

它负责的是共享后端本身：Federation 项目、Cloudflare Workers 部署、D1 / Queue / R2 资源、admin 工作区和 HTTP 调试。

## city / downcity

`city` 面向本机工作区和 Agent。常用命令：

```bash
city
city federation status
city federation join <url>
city federation login
city agent create .
city agent list
city agent start
city agent chat
```

它负责的是本机状态：选择哪个 Federation、保存用户登录态、运行本机 Agent、管理插件和 chat/gateway。

## 边界

- 不要用 `city deploy` 部署 Federation；部署入口是 `fed deploy`。
- 不要用 `city create` 创建 Federation 项目；创建入口是 `fed create`。
- 不要把 `fed` 当成本机 Agent 管理器；Agent 入口是 `city agent ...`。
- 程序化管理仍然优先使用 `Admin City`，CLI 更适合人坐在终端前操作。
