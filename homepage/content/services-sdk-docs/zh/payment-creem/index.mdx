---
title: "Creem Payment 服务"
description: "把一次性 Creem Checkout 支付同步成用户全局余额充值。"
---

`creemPaymentService()` 会创建 Creem 托管 Checkout，并在 Creem 通过 webhook 确认支付成功后，完成一笔已经存在的 `balance` topup。

它不是 subscription 或 entitlement 服务。这个服务只负责 Checkout 创建、本地支付记录、webhook 事件记录，以及最后调用 `balance.finishTopup()` 入账。

## 启用方式

```ts
import { balanceService } from "@downcity/services";
import {
  creemPaymentMethod,
  creemPaymentService,
  paymentService,
  type CreemPaymentServiceBalanceBridge,
} from "@downcity/services";

const balance = balanceService();
const creemBalanceBridge: CreemPaymentServiceBalanceBridge = {
  readTopup: async (topup_id) => await balance.readTopup(topup_id),
  finishTopup: async (topup_id, extra) => await balance.finishTopup(topup_id, extra),
};

base.use(balance);
base.use(paymentService({
  methods: [
    creemPaymentMethod(),
  ],
}));
base.use(creemPaymentService({
  balance: creemBalanceBridge,
}));
```

## 必需 env

- `CREEM_API_KEY`：用于创建 Checkout 的 API key
- `CREEM_PRODUCT_ID`：Creem 托管 Checkout 使用的 product
- `CREEM_WEBHOOK_SECRET`：可选，用于校验 `creem-signature`
- `CREEM_SUCCESS_URL`：可选，默认成功跳转地址
- `CREEM_CURRENCY`：可选，支付方式目录展示用币种
- `CREEM_API_BASE_URL`：可选，测试环境 API 地址覆写

如果你没有直接配置成功跳转地址，服务会优先基于 `DOWNCITY_CITY_BASE_URL`，再回退到当前请求 origin 自动生成默认结果页地址。

## 前端流程

先通过 `balance` 创建可信 topup，再为它创建 Creem checkout：

```ts
const checkout = await user.service("payment.creem").action("checkout/create").invoke({
  topup_id: "topup_demo",
});

location.href = checkout.checkout_url;
```

`paymentService()` 也会在支付方式目录里暴露 Creem：

```ts
const methods = await guest.service("payment").get("methods");
```

只有同时配置 `CREEM_API_KEY` 和 `CREEM_PRODUCT_ID` 时，Creem method 才会是 enabled。

## Webhook 同步

把 Creem webhook 事件发到：

```txt
POST /v1/payment.creem/webhook
```

服务会基于事件 ID 幂等应用 `checkout.completed`。重复事件只返回已有同步状态，不会重复给余额入账。

## 路由

- `GET /v1/payment/methods`
- `POST /v1/payment.creem/checkout/create`
- `GET /v1/payment.creem/payments/me`
- `GET /v1/payment.creem/payments`
- `GET /v1/payment.creem/events`
- `POST /v1/payment.creem/webhook`
- `GET /v1/payment.creem/redirect/success`
- `GET /v1/payment.creem/redirect/cancel`
