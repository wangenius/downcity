# Shell 与沙箱

Agent 使用 `@downcity/shell` 执行本地命令、管理沙箱运行环境、处理交互式审批。

## 安装

```bash
pnpm add @downcity/shell
```

## Shell

`Shell` 类管理命令执行和交互式审批反馈。

### 最小示例

```ts
import { Shell } from "@downcity/shell";
import { Agent } from "@downcity/agent";

const shell = new Shell({ autoApprove: false });

const agent = new Agent({
  id: "demo",
  path: process.cwd(),
  shell,
});
```

### 构造参数

| 选项 | 类型 | 说明 |
|------|------|------|
| `autoApprove` | `boolean` | 是否自动审批命令执行 |
| `allowedCommands` | `string[]` | 限制允许执行的命令列表 |
| `workdir` | `string` | 命令执行的工作目录 |

### 方法

- `shell.approve(input)` — 批准待审批的 shell action
- `shell.deny(input)` — 拒绝待审批的 shell action
- `shell.dispose()` — 释放 shell 资源

## 沙箱

沙箱通过宿主传入的 Agent 配置或 Shell 构造参数配置。沙箱后端会根据宿主平台自动选择：

- `macos-seatbelt` — macOS 沙箱
- `linux-bubblewrap` — Linux 沙箱
- `unrestricted-host` — 不启用沙箱

SDK 内部的沙箱运行器负责 shell 子进程创建、平台后端调用和一次性命令执行。通常不需要直接使用。

### ConfigResolver

`SandboxConfigResolver` 从项目环境加载沙箱配置。

`SandboxPreflight` 执行沙箱前置检查，验证沙箱环境是否就绪。
