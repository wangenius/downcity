# Agent 类

本地 `Agent` 的核心公开使用面包括：

- `new Agent(options)`
- `agent.start(options?)`
- `agent.stop()`
- `agent.session(sessionId?)`
- `agent.sessions()`
- `agent.plugins`

## 构造参数

- `id`
- `path`
- `tools`
- `instruction`
- `services`
- `plugins`
- `platform`
- `configureSession`

## 核心语义

- `Agent` 本身是本地执行壳
- 真正的执行对象是它创建出来的 session

从 API 设计上看，它更像一个“session 工厂 + 长期运行能力入口”。

## `start()` 和 `stop()`

`agent.start()` 负责启动当前 agent 的长期运行能力。

典型例子包括：

- 启动 HTTP 服务
- 启动本地 RPC 服务
- 启动 service lifecycle

推荐写法：

```ts
await agent.start({
  http: {
    host: "127.0.0.1",
    port: 15314,
  },
  rpc: true,
});
```

如果你不调用 `agent.start()`：

- 仍然可以直接 `await agent.session()` 然后 `session.run()`
- 但不会启动 HTTP / RPC
- 也不会启动后台 service lifecycle

`agent.stop()` 则负责关闭当前 agent 已启动的长期运行能力。

## 生命周期入口

不过在大多数用户场景里，更推荐统一使用：

```ts
await agent.start({
  http: {
    host: "127.0.0.1",
    port: 15314,
  },
  rpc: true,
});
```

本地 `Agent` 不再暴露 `agent.http` 与 `agent.rpc` 这类细粒度 transport 入口。

如果你需要：

- 只启动 HTTP：`agent.start({ http: { host, port } })`
- 只启动 RPC：`agent.start({ rpc: true })`
- 同时启动两者：`agent.start({ http: { host, port }, rpc: true })`

`agent.plugins` 是当前本地 SDK 新增的重要公开面。

你可以直接用它做：

- `agent.plugins.list()`
- `agent.plugins.availability(pluginName)`
- `agent.plugins.runAction({ plugin, action, payload })`
- `agent.plugins.pipeline(pointName, value)`
- `agent.plugins.guard(pointName, value)`
- `agent.plugins.effect(pointName, value)`
- `agent.plugins.resolve(pointName, value)`
