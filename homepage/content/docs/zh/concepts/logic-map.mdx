# 架构逻辑图

这页回答一个问题：

`console`、`agent`、`service`、`plugin`、`capability`、`asset` 各自负责什么，以及一次请求如何穿过它们。

## 1. 职责边界

- `console`：全局控制面。管理 daemon、registry、模型池、env、共享存储。
- `agent`：项目执行面。加载项目配置与上下文，持有单个 runtime。
- `service`：核心业务流程。
- `plugin`：可选增强模块。
- `capability`：稳定调用契约。
- `asset`：插件依赖资源。

## 2. 系统关系

```mermaid
flowchart LR
  subgraph CP["控制面（Console）"]
    CLI["city CLI"] --> CONSOLE["city console start"]
    CLI --> CFG["console config / model / env"]
    CLI --> AGENTOPS["city agent ..."]
    CLI --> PLUGINOPS["city plugin ..."]
    CONSOLE --> REG["~/.ship/console/agents.json"]
  end

  subgraph DP["执行面（Per Project）"]
    AGENT["agent daemon"] --> SERVICES["services"]
    AGENT --> PLUGINS["plugins"]
    PLUGINS --> CAPS["capabilities"]
    PLUGINS --> ASSETS["assets"]
    AGENT --> SHIP[".ship/*"]
  end

  CFG --> AGENT
  PLUGINOPS --> AGENT
```

## 3. 请求流

```mermaid
sequenceDiagram
  participant User as User
  participant Channel as Telegram/Feishu/QQ
  participant Agent as agent daemon
  participant Service as service(chat/skill/task/memory)
  participant Capability as capability(audio.transcribe)
  participant Asset as asset(voice.transcriber)

  User->>Channel: 发送消息
  Channel->>Agent: 进入 runtime
  Agent->>Service: 执行业务流程
  Service->>Capability: 调稳定契约
  Capability->>Asset: 使用依赖资源
  Asset-->>Capability: 返回结果
  Capability-->>Service: 返回结果
  Service-->>Agent: 返回业务结果
```
