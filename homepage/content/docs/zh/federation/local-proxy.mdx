# 本地代理

当你需要一个本地 Federation 服务，只暴露 Codex 或 Claude Code 的模型能力，而不注册到已有 Fed server 时，可以使用 Downcity local proxy skill。

生成的项目会启动：

- `127.0.0.1:8317` 上的 CLIProxyAPI
- `127.0.0.1:3000` 上的 Downcity 本地 Federation
- `/v1/ai/models` 和 `/v1/ai/chat/completions` 模型接口

## 安装 skill

使用 Vercel Labs `skills` CLI 安装：

```bash
npx skills add https://github.com/wangenius/downcity-local-proxy-skill \
  --skill downcity-local-proxy \
  --agent codex \
  --global \
  --yes
```

## 让 Codex 创建本地服务

安装后，对 Codex 发送这个 prompt：

```text
Use downcity-local-proxy to create ./downcity/local-proxy with Codex and Claude Code enabled, then install dependencies and start the local Federation service.
```

## 手动启动

skill 会创建一个独立项目。进入生成目录后运行：

```bash
cd ./downcity/local-proxy
pnpm install
pnpm start --codex
pnpm start --claude-code
pnpm start --codex --claude-code
```

当 CLIProxyAPI 需要 OAuth 登录或刷新时，添加 `--login`：

```bash
pnpm start --codex --login
pnpm start --claude-code --login
```

本地代理只会在生成项目内的 `.downcity/` 写入运行时文件，不会修改全局 Downcity CLI 状态，也不会把自己注册到已有 Fed server。
