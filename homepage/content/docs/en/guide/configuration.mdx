# Configuration

## ship.json

`ship.json` is the main configuration file for ShipMyAgent, controlling the Agent's capabilities and behavior boundaries.

### Complete Example

```json
{
  "$schema": "./.ship/schema/ship.schema.json",
  "name": "my-agent",
  "version": "1.0.0",
  "start": {
    "port": 3000,
    "host": "0.0.0.0",
    "webui": false,
    "webport": 3001
  },
  "llm": {
    "activeModel": "quality",
    "providers": {
      "google": {
        "type": "gemini",
        "apiKey": "${GEMINI_API_KEY}"
      },
      "compat_gateway": {
        "type": "open-compatible",
        "baseUrl": "https://your-api.com/v1",
        "apiKey": "${LLM_API_KEY}"
      }
    },
    "models": {
      "quality": {
        "provider": "google",
        "name": "gemini-2.5-pro",
        "temperature": 0.7
      },
      "fast": {
        "provider": "google",
        "name": "gemini-2.5-flash"
      },
      "fallback": {
        "provider": "compat_gateway",
        "name": "gpt-4o-mini"
      }
    },
    "logMessages": true
  },
  "services": {
    "skills": {
      "paths": [".claude/skills"]
    },
    "chat": {
      "egress": {
        "chatSendMaxCallsPerRun": 30,
        "chatSendIdempotency": true
      },
      "adapters": {
        "telegram": {
          "enabled": true,
          "botToken": "${TELEGRAM_BOT_TOKEN}",
          "followupWindowMs": 600000,
          "groupAccess": "anyone"
        },
        "feishu": {
          "enabled": true,
          "appId": "${FEISHU_APP_ID}",
          "appSecret": "${FEISHU_APP_SECRET}",
          "domain": "https://open.feishu.cn",
          "adminUserIds": ["ou_xxx", "ou_yyy"]
        },
        "qq": {
          "enabled": false,
          "appId": "${QQ_APP_ID}",
          "appSecret": "${QQ_APP_SECRET}",
          "sandbox": false
        }
      }
    }
  }
}
```

### Field Explanation

#### `$schema` (Schema Reference)

- **$schema**: Points to the JSON Schema file path for IDE auto-completion and configuration validation.

#### `name` and `version`

- **name**: Agent project name to identify your Agent application.
- **version**: Agent version number, semantic versioning is recommended.

#### `start` (Startup Configuration)

Controls the Agent service startup parameters:

- **port**: Port number for the Agent service to listen on (default `3000`).
- **host**: Host address to bind the service, `0.0.0.0` means allow external access.
- **webui**: Whether to enable interactive web interface (default `false`).
- **webport**: Port for the interactive web interface (default `3001`).

#### `llm` (Language Model)

- **activeModel**: Active model key, must exist in `llm.models`.
- **providers**: Provider connection map, key is user-defined (for example: `google`, `openai_main`).
  - **type** options: `anthropic`, `openai`, `deepseek`, `gemini`, `open-compatible`, `open-responses`, `moonshot`, `xai`, `huggingface`, `openrouter`.
  - **baseUrl**: Optional. If omitted, the runtime uses the default URL by provider type.
  - **apiKey**: API key, **strongly recommended** to use `${ENV_VAR}` to reference environment variables.
- **models**: Model map, key is user-defined (for example: `quality`, `fast`).
  - **provider**: Provider key (must exist in `llm.providers`).
  - **name**: Upstream model name.
  - Optional sampling fields: `temperature`, `maxTokens`, `topP`, `frequencyPenalty`, `presencePenalty`.
- **logMessages**: Whether to log outbound LLM request payloads (default `true`).

> Notes:
> - `open-compatible` uses Chat Completions-compatible flow (`/chat/completions`).
> - `moonshot` uses Moonshot (Kimi) official Chat Completions-compatible endpoint.
> - `open-responses`, `openai`, and `deepseek` use Responses-compatible flow (`/responses`).
> - `gemini` uses native Google Generative AI endpoint (`/v1beta/models/*:generateContent`).

#### `services.chat.method` (Reply Delivery Mode)

- **method**: Optional `cmd` or `direct` (default `direct`).
  - `direct`: assistant text generated by the model is auto-delivered to the current chat channel.
    Control parameters are read from frontmatter metadata: `chatKey`, `reply(message_id)` / `message_id` / `messageId`, `react/reactions`.
    `reply` must be a target `message_id` value (not `true/false`).
    When `react/reactions` omits `messageId`, runtime reuses `reply/message_id` in the same `chatKey`.
    Attachments still use `<file type="...">path</file>` in body text.
    Delivery is step-wise (each model step can be sent), not only one final merged send.
    For delayed/scheduled delivery, use shell execution with `sma chat send --delay <ms>` or `sma chat send --time <time>`.
  - `cmd`: model sends user-visible replies by executing `sma chat send` (Bash-first).

#### `services.chat.channels` (Chat Adapters)

Configure chat adapters for various platforms to deploy your Agent on different chat platforms:

**telegram** (Telegram Bot)
- **enabled**: Whether to enable the adapter
- **botToken**: Bot Token, supports environment variables
- **auth_id**: (Optional) Owner auth ID
- **followupWindowMs**: (Optional) Followup window duration, default 10 minutes
- **groupAccess**: (Optional) Group chat access control, `"anyone"` (default) or `"initiator_or_admin"`
- **Typing indicator**: While the Agent is executing, the bot will show Telegram “typing…” status (auto-refreshed until completion).

**feishu** (Feishu/Lark Bot)
- **enabled**: Whether to enable the adapter
- **appId**: Feishu application ID
- **appSecret**: Feishu application secret
- **domain**: (Optional) API domain, default `https://open.feishu.cn`, use `https://open.larksuite.com` for international version
- **adminUserIds**: (Optional) Admin user ID whitelist

**qq** (QQ Official Bot)
- **enabled**: Whether to enable the adapter
- **appId**: QQ application ID
- **appSecret**: QQ application secret
- **sandbox**: (Optional) Whether to use sandbox environment, default `false`
- **groupAccess**: (Optional) Group chat access control, `"anyone"` (default) or `"initiator_or_admin"`

> For detailed configuration and service guides, please refer to [Service Guide](/en/docs/guide/services).

#### `services.skills` (Skills)

Configure available skill plugins for the Agent:

- **paths**: Array of search paths for skill files, supports relative and absolute paths.
  - Recommended to use `.claude/skills` or `.ship/skills` directory to store skill files.
  - Supports mixing built-in skills and custom skills.

> For detailed skill development guide, please refer to [Skill System](/en/docs/guide/skills).

#### `services.chat.egress` (Chat Egress)

> Mainly applies to `cmd` mode.

- **chatSendMaxCallsPerRun**: Maximum `chat_send` calls in a single run.
- **chatSendIdempotency**: Enable best-effort idempotency dedupe for `chat_send`.

#### `context` (Context)

Controls how much "conversation history" is injected at each execution and Chat scheduling strategy.

- **context.messages**: Each `chatKey` maintains a continuously growing `UIMessage[]` as the unique history (stored in `.ship/chat/<encodedChatKey>/messages/messages.jsonl`), and automatically compacts when exceeding context window.
  - **keepLastMessages**: Number of recent messages to keep after compact (default `30`)
  - **maxInputTokensApprox**: Input budget (approximate token count, used to trigger compact early, default `16000`)
  - **archiveOnCompact**: Whether to archive collapsed original segments during compact (default `true`)
- **services.chat.queue**: Scheduler divided by `chatKey` lanes (serial for same chatKey, concurrent for different chatKeys; merges new messages at step boundaries during a run).

## PROFILE.md

`PROFILE.md` is the carrier of the System Prompt. It determines the Agent's "Personality" and "Professional Ethics".

### Writing Tips

#### 1. Define Role

```markdown
# Agent Role

You are a senior frontend architect, focusing on React performance optimization.
```

#### 2. Set Goals

```markdown
## Goals

- Review all new component code submissions.
- Ensure no unnecessary re-renders are introduced.
- Keep component code concise and reusable.
```

#### 3. Set Constraints

```markdown
## Constraints

- Never modify dependency versions in `package.json`.
- All CSS styles must use Tailwind class names, inline styles are prohibited.
- You must explain the reason before modifying code.
```

#### 4. Provide Context

You can write important project specifications and code style guides directly in `PROFILE.md`, and the Agent will bear these rules in mind during every conversation.
