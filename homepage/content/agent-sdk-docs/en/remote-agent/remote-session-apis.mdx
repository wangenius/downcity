# Remote Session API

Remote Sessions provide `get_info()`, `set({ modelId })`, `prompt()`, `stop()`, `subscribe()`, `messages()`, `system()`, approval methods, and `fork()`.

```ts
const session = await agent.sessions.create();
await session.set({ modelId: "quality" });

const unsubscribe = session.subscribe(apply_mutation);
const turn = await session.prompt({ query: "Continue" });
await turn.finished;

const page = await session.messages();
unsubscribe();
```

The subscription emits `message | part | delta | turn | session`. An approval request is an approval-required Tool Part and can be resolved with the callback's `reply.approval(...)` or with `session.resolve_approval(...)`.

Remote clients cannot send a local model instance. The server resolves `modelId` and owns the resulting model instance.
