# Service Runtime

Services are the runtime module type for core business workflows.

Typical built-in services include:

- `chat`
- `skill`
- `task`
- `memory`

## What Service Owns

- user-facing workflows
- runtime orchestration
- context-aware execution
- stable service commands and actions

## What Service Does Not Own

- install/check/config of external dependencies
- plugin-private implementation details
- reusable optional enhancements that should be decoupled

## Recommended Interaction Model

- service -> capability
- capability -> plugin implementation
- plugin -> asset

For example, `chat` does not need to know voice model details. It invokes `audio.transcribe`, and the voice plugin plus its asset handle the rest.

## Typical Commands

```bash
city service list
city service status chat
city service command chat status
```
