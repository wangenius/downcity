# Profile

Agent 的行为由项目目录中的两个自然语言提示词文件塑造：`PROFILE.md` 和 `SOUL.md`。Agent 在启动时读取两者，并将它们包含在每个会话的系统上下文中。

## PROFILE.md

定义 Agent 的可见角色：

- 它是谁
- 它负责什么
- 它应该如何回答
- 它应该遵守哪些边界

把它理解为角色定义，而不是运行时配置。如果你想调整输出风格、优先级或协作语气，这里是首选。

### 示例

```markdown
# Profile

你是一个名叫 DevBot 的编程助手。

你擅长：
- 审查 Pull Request
- 解释代码变更
- 建议重构

你应该：
- 简洁但全面
- 在有帮助时使用示例
- 当上下文不清楚时提出澄清问题
```

## SOUL.md

定义更深层的、更稳定的运行原则。它更接近基础工作风格，而不是一次性角色提示。只有当你真正想改变 Agent 的长期行为时才修改它。

### 示例

```markdown
# Soul

始终偏好简单方案而非巧妙方案。
不确定时，先询问而非假设。
尊重用户隐私 —— 绝不记录敏感数据。
```

## 最佳实践

- `PROFILE.md` 聚焦当前角色和职责。
- `SOUL.md` 聚焦 timeless 原则和价值观。
- 在一个健康的项目中，`SOUL.md` 修改得比 `PROFILE.md` 更少。
- 两个文件都是可选的，但大多数项目都受益于同时拥有两者。
