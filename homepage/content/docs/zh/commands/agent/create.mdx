# `city agent create`

在目标路径创建/初始化一个 Downcity Agent 项目。

## 用法

```bash
city agent create [path]
```

## 参数

- `path`（可选）：项目路径，默认当前目录。
- 交互默认的 `Agent name` 会基于目录名自动推导，并将 `_`/`-` 转为空格（仅作为默认值，可手动修改）。
- 交互过程中现在可以直接选择执行模式：`API Model Pool`、`Local llama (~/.models)`、`Kimi ACP`、`Claude ACP`、`Codex ACP`。
- 创建出的项目会直接写入统一的 `downcity.json.execution`，不再拆成 `model.primary + session.agent`。
- 当你选择 Local 模式时，命令会额外把模型文件写入 `plugins.lmp.model`，后续通过 `city lmp ...` 管理本地模型。

## 选项

- `-f, --force [enabled]`：允许覆盖已有 `downcity.json`（危险操作）

## 示例

```bash
# 在当前目录初始化
city agent create .

# 在指定目录初始化
city agent create ./my-agent-project
```

## 示例输出（基于当前实现）

以下示例直接对齐 `packages/downcity/src` 当前命令实现（含默认输出模式与字段结构），示例值使用占位符表示。

```text
$ city agent create .
city version: 1.0.0
🚀 Initializing Downcity project: <projectRoot>
✅ Created PROFILE.md
✅ Created SOUL.md
✅ Created downcity.json
✅ Created .downcity/ directory structure
✅ Created downcity.schema.json
```

如果你选择 ACP 或 Local 模式，即使 console 模型池为空也可以初始化。

Local 模式创建后，常用后续命令：

```bash
city lmp status
city lmp models
city lmp use <gguf-file>
```
