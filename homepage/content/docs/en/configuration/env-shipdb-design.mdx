# Env and ship.db Design

Use one rule:

- shared secrets and shared infrastructure live in `~/.ship/ship.db`
- project behavior lives in `ship.json`

## `ship.db` owns

- `env_entries`
- `channel_accounts`
- `model_providers`
- `models`
- other console-owned encrypted settings

## Project `ship.json` owns

- `model.primary`
- `services.*`
- `plugins.*`
- `assets.*`

## Why

- secrets should not be duplicated into project config
- plugin enablement and asset selection are project behavior
- console storage should not hide project-specific plugin state

## Runtime load order

1. read global env from `ship.db`
2. read agent-private env from `ship.db`
3. read project `.env`
4. read project `ship.json`
5. resolve `plugins.*` and `assets.*`
6. resolve channel credentials and model pool from `ship.db`
