# Service System 文本

service 可以实现：

```ts
system(context) {
  return "some service-level system text";
}
```

它的语义是：这个 service 希望补充一段 service 维度的 system 文本。

## 它适合承载什么

- service 自己的使用规则
- service 的行为边界
- 需要长期稳定注入的一段领域说明

## 在本地 SDK 中如何生效

当前 SDK local session 使用 `SdkSessionSystemComposer`。

当你把 service 显式传入 `new Agent({ services: [...] })` 后，SDK 会在每次 session run 解析 system prompt 时收集这些 service 的 `system(context)` 文本。

注入顺序是：

- `PROFILE.md`
- `SOUL.md`
- core prompt
- `service.system(context)`
- 已注册 plugin 的 `plugin.system(context)`
- runtime clock prompt

如果某个 service 的 `system(context)` 抛错，SDK 会跳过这一个 service system 文本，不会阻断本轮 session run。

## 和 lifecycle 的关系

`service.system(context)` 不要求 `service.lifecycle.start()` 已经执行。

这意味着：

- 只提供提示词规则的 service 可以直接生效
- 依赖后台连接、queue worker 或外部进程状态的 system 文本，需要自己在实现中处理未启动状态

## 适合放在哪里

如果一段规则属于整个 agent 的长期人格或项目约束，优先放在 `PROFILE.md` / `SOUL.md`。

如果一段规则属于某个 service 自己，比如 chat service 的发送规则、某个业务 service 的领域边界，就适合放在 `service.system(context)`。
