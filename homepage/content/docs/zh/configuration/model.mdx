# 模型配置

模型配置拆分为两部分：

1. 共享 AI service 维护模型目录和 provider 侧细节。
2. Agent 在 `downcity.json` 只选择 `execution.modelId`。

## 工作流

1. 先确认 `town model list` 能看到可用模型。
2. 在 agent 项目设置 `execution.modelId`。
3. 重启 agent 并进行一次实际调用验证。

## 命令示例

```bash
town model list
town model use default --path .
town config set execution.modelId default
town agent restart .
```

## 失败场景

- `execution.modelId` 不存在：agent 启动失败。
- provider/key 无效：调用模型时报鉴权或网络错误。
