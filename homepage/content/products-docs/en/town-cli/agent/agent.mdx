# `town agent`

Manage agent lifecycle and registry-aware operations.

This is the Town CLI command group for project Agents. For the conceptual model, read [What Is an Agent](/en/docs/agent/overview/what-is-agent).

Run `town agent` without a subcommand in an interactive terminal to open the agent manager. Run bare `town` to enter the full-screen Town TUI dashboard first. Use subcommands when you want a scriptable command.

## Command hierarchy

Core command groups are: `town start/stop/restart/status`, `town city`, `town agent`, `town plugin`, plus plugin shortcut roots.

## Usage

```bash
town agent
town agent <subcommand>
```

## Version inspection

- `town -v` shows the city CLI version only.
- `town agent -v` shows both the current town CLI version and the `@downcity/agent` runtime version actually bundled with that town installation.

## Interactive manager

The interactive manager is list-first:

- Opening `town agent` shows the Agent list immediately.
- Focusing an Agent shows its status, execution binding, chat account connections, and project path in the main panel.
- Press Enter on an Agent to open its management panel, then start, stop, restart, chat, configure the Agent ID, or connect Town-level chat accounts.
- Use the bottom list actions to create a new Agent or exit.

It does not replace the subcommands below. It is a guided entry point for common local operations.

Chat credentials stay in Town-level chat accounts managed from `town chat`. The agent manager only connects an agent to those existing accounts through `plugins.chat.channels.<platform>.channelAccountId`. If an agent points at a missing account, the manager removes that dangling binding automatically.

## Subcommands

- `create`: create/initialize an agent project
- `list`: list registered agent projects
- `start`: start the agent
- `stop`: stop the agent daemon
- `chat`: chat with an agent in interactive or one-shot mode
- `history clean`: hard-clean a targeted session history
- `status`: show agent status
- `doctor`: diagnose state files and optionally fix stale pid/meta
- `restart`: restart the agent

## Environment variable injection

On every `town` execution, CLI resolves current agent context and injects variables into the `town` process and its child processes:

- `DC_AGENT_ID`: current agent runtime id
- `DC_AGENT_PATH`: absolute path of current agent

> Note: this is process-scoped injection and does not mutate your parent shell session.
