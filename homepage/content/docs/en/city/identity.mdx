# City User Identity

Federation issues user identities, while the City client uses a `user_token` to access Federation or a product backend. Trusted product backends use a `bureau_token` to identify users online. One Federation can serve multiple products, isolated by `city_id`.

## Identity boundaries

- **Federation** — stores the Ed25519 private key and issues `user_token` values
- **City** — stores `federation_url + user_token` and makes user requests
- **Bureau** — stores `federation_url + bureau_token` and identifies users or performs authorized management operations

## Authentication flow

1. The user selects a product `city_id` during login
2. Federation signs a `user_token` bound to that `city_id`
3. The user calls Federation through `new City({ federation_url, user_token })`
4. The user sends the same Bearer Token to the product backend
5. The backend's `Bureau` sends `bureau_token + user_token` to Federation
6. Federation reads `city_id` from the Bureau Token record, validates the user token, and queries the current Accounts record

```ts
const city = new City({
  federation_url: "https://fed.example.com",
  user_token,
});
```

Product backend:

```ts
const bureau = new Bureau({
  federation_url: "https://fed.example.com",
  bureau_token: process.env.DOWNCITY_BUREAU_TOKEN!,
});

const identity = await bureau.identify(request);
```

`Bureau` requires a `bureau_token` and does not accept `city_id`. Federation binds the Token to a City when it is issued and stores only its hash. `identify()` always queries Federation online, so Token revocation, City suspension, and Accounts state changes take effect immediately.

Federation still publishes Ed25519 public keys for its own token validation and other protocols:

```text
/.well-known/downcity.json
/.well-known/jwks.json
```

Management backends use the same `Bureau` class. The only difference is whether the `bureau_token` has the `federation:admin` capability.

## Channel accounts

Bot credentials for messaging platforms are stored in `~/.downcity/downcity.db` and referenced by ID from the project:

```json
{
  "plugins": {
    "chat": {
      "channels": {
        "telegram": {
          "channelAccountId": "telegram-main"
        }
      }
    }
  }
}
```
