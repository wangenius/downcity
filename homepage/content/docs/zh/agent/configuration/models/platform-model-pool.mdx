# Federation 模型目录

Agent 项目本身不保存完整模型定义。

## 模型目录在哪里

模型目录属于 Federation AIService（部署在共享后端 Federation 上的内建模型服务）。Federation 的整体定位见 [Downcity 核心概念](/zh/docs/concepts/downcity-core-concepts)。

从用户视角理解：

- Federation 保存 provider、模型名、API key 等完整信息
- 项目层只保存 `execution.modelId`

这也是为什么项目里通常看不到完整 provider 配置。

## 为什么这样设计

- 多个项目可以共用同一模型定义
- API key 不需要在每个项目里重复保存
- 项目切模型时只改绑定字段

如果你同时维护多个项目，这个收益会非常明显。

## 你通常怎么操作

1. 先在 Federation AIService 准备好模型
2. 再把项目的 `execution.modelId` 绑定到目标模型

一旦绑定完成，后续切换模型本质上就是切换 `modelId`。
