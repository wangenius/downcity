# `studio console`

`studio console` 只负责管理 Console 模块本身。

Console 是 Downcity 的 Web 控制台：

- 单实例 Console
- 多 agent 切换查看
- 全局 model / env / auth / plugin 管理
- 选中 agent 后代理到对应运行时

## 用法

```bash
studio console [action] [options]
```

## action

- `start`：后台启动 Console
- `stop`：停止 Console
- `restart`：重启 Console
- `status`：查看 Console 状态

如果省略 `action`，默认执行 `start`。

## 常用参数

- `-p, --public [enabled]`：以公网模式启动 Console（绑定 `0.0.0.0`）
- `-h, --host <host>`：Console 主机，默认 `127.0.0.1`

如果传 `-p` 而不传 `-h`，CLI 会自动把监听主机切到 `0.0.0.0`，方便服务器直接对外暴露。

## 关键行为

- Console 依赖 Studio runtime，先执行 [`studio start`](/zh/docs/commands/start)。
- Console 不是 Studio runtime 本身，而是运行在其上的独立模块。
- 一个 Console 实例可以切换查看多个运行中的 agent。
- `start` 不会改写已经运行中的监听地址；如果当前是本机模式，要切到公网模式请执行 `studio console restart -p`。
- 如果你希望公网模式在后续启动里持续生效，请使用 [`studio public`](/zh/docs/commands/public)。
- `studio status` 会同时显示 Studio runtime 与 Console 状态。

## 典型流程

```bash
# 1) 一起启动 runtime + Console
studio start --console

# 1.1) 一步启动并对外暴露 Console
studio start -p

# 1.2) 只启动 Console，并对外暴露
studio console start -p

# 1.3) 为后续启动持久化公网模式
studio public on

# 2) 启动多个 agent
studio agent start /path/to/agent-a
studio agent start /path/to/agent-b

# 3) 查看 Console 状态
studio console status

# 4) 只重启 Console
studio console restart
```

## 对外访问说明

- 默认 `studio console start` 绑定 `127.0.0.1:5315`，只能本机访问。
- `studio console start -p` 会绑定 `0.0.0.0:5315`，可以从外部访问。
- 如果 Console 已经在运行，`studio console start -p` 不会把已有进程从 `127.0.0.1` 改成 `0.0.0.0`；请使用 `studio console restart -p`。
- `studio public on` 会把相同的公网行为持久化到后续 `studio start` 和 `studio console start`，并在 Console 已运行时自动重启应用新绑定。
- 如果使用了 `0.0.0.0` 绑定，浏览器访问时请改用服务器 IP 或域名。
- 公网模式下，启动结果会额外打印 `Public URL`。
- 如果运行环境处于 NAT / 反向代理后面，建议显式设置 `DOWNCITY_PUBLIC_HOST` 或 `DOWNCITY_PUBLIC_URL`。

## 端口说明

- `5314`：Studio runtime API 端口，供托管 plugin / plugin 等接口使用。
- `5315`：Console Web UI 端口，供浏览器访问 Console。
