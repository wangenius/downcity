# legacy downcity.json

新项目不再需要 `downcity.json`。`city agent create` 会把 Agent ID、启动参数、模型绑定和 plugin 参数写入 CLI 全局 DB，项目目录只保留 `PROFILE.md`、`SOUL.md`、`.env` 和 `.downcity/` 运行时数据。

如果旧项目中仍存在 `downcity.json`，CLI 会把它作为 legacy 迁移来源读取一次；后续配置写入都进入全局 DB。

## 旧结构

```json
{
  "$schema": "./.downcity/schema/downcity.schema.json",
  "id": "my-agent",
  "version": "1.0.0",
  "start": {
    "port": 5314,
    "host": "0.0.0.0"
  },
  "execution": {
    "type": "api",
    "modelId": "default"
  },
  "plugins": {}
}
```

## 关键字段

- `id` — 稳定的 Agent 标识符。用于会话存储和运行时目录归属。
- `start` — 运行时启动配置（端口、host）。
- `execution` — 将项目绑定到已连接 Federation AIService 中的模型 ID。
- `plugins` — plugin 运行参数。

在新模型中，这些字段保存在 CLI 全局 DB 中，而不是项目文件中。

## 不应该放在这里的内容

- 完整的模型供应商标识（那些属于 City / Federation AIService）。
- 明文 API 密钥或机器人凭证（那些属于 CLI 全局 DB 或通过环境变量注入）。
- 控制台级别的控制平面设置。

## 与 SDK 的关系

当你直接使用 SDK 时，你以构造参数形式传入相同的值，例如 `new Agent({ id, path, model, plugins, config })`。SDK 不要求项目目录存在 `downcity.json`。
