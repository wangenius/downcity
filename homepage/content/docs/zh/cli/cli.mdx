# Agent CLI

`city` CLI 是 Agent SDK 的便利包装器。它从 CLI 全局 DB 读取 Agent 配置，将模型、plugin 和启动参数传给 Agent SDK 构造函数，然后处理启动、停止和重启等生命周期命令。

## 项目初始化

```bash
city agent create my-project
```

这会生成：

- `.agents/skills/`
- `.downcity/`
- `.env`
- CLI 全局 DB 中的 Agent 配置

## 项目目录结构

```text
my-project/
├── .env
├── .agents/skills/
├── .downcity/          # 仅运行数据
└── 你的业务代码
```

## 关键命令

- `city agent start` — 在当前项目启动 Agent
- `city agent stop` — 停止运行中的 Agent
- `city agent restart` — 重启 Agent
- `city agent status` — 检查 Agent 是否正在运行
- `city agent logs` — 查看运行时日志

## CLI 做了什么

1. 从 CLI 全局 DB 读取 Agent 配置
2. 通过已连接的 Federation AIService 解析 `execution.modelId`
3. 将解析后的配置传给 `new Agent(...)`
4. 启动 Agent 进程
5. 维护守护进程状态，以便后续检查、停止或重启

## CLI vs SDK

CLI 用于日常项目管理。SDK 用于将 Agent 能力嵌入自己的应用。两者使用相同的执行核心。
