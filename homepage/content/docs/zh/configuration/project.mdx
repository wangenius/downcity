# 项目配置（downcity.json）

`downcity.json` 是项目级运行入口，常用字段如下：

```json
{
  "name": "my-agent",
  "execution": {
    "type": "api",
    "modelId": "quality"
  },
  "start": {
    "host": "127.0.0.1",
    "port": 5314
  }
}
```

## 注意事项

1. 项目现在必须声明 `execution`，执行模式固定为 `api`。
2. `execution.modelId` 必须存在于已连接的 City AIService 中。
3. `start.host/start.port` 会影响 CLI 的默认路由目标。
4. 项目层统一在 `downcity.json` 中维护 `plugins` 与执行相关配置。
5. 本地 shell / CLI 命令默认强制运行在 sandbox 中。Safe Sandbox 边界由 runtime 固定生成，不通过 `downcity.json` 配置。

## 执行模式

如果你希望项目使用 City AIService 模型：

```json
{
  "execution": {
    "type": "api",
    "modelId": "default"
  }
}
```

## Shell Sandbox

当前版本里，`shell` / CLI 命令默认运行在 **Safe Sandbox** 中。Safe Sandbox 边界由 runtime 固定生成，不通过 `downcity.json` 调整。

当前版本的行为：

- 作用于 `@downcity/shell` 命令执行与 `kind=script` 的 task 执行链
- 当前支持 macOS 与 Linux：macOS 使用系统自带 `sandbox-exec`，Linux 使用 `bubblewrap`（命令名 `bwrap`）
- 不支持关闭 Safe Sandbox，也不会静默回退到宿主机普通子进程执行
- 未挂入 sandbox 的宿主路径默认不可见
- 项目根目录和 `.downcity/sandbox` 默认可写
- sandbox 子进程共享 `.downcity/sandbox` 作为持久 `HOME`，并使用 `.downcity/sandbox/tmp` 作为常见临时目录变量与 zsh `TMPPREFIX` 目标、`.downcity/sandbox/.cache` 作为 `XDG_CACHE_HOME`
- 支持项目内安装与缓存，例如 `.venv`、`node_modules`、下载到项目内的模型和缓存文件
- `brew install`、`npm install -g`、`pip install --user`、`gh auth login` 或访问宿主服务初始化能力时，agent 可以显式请求 `sandbox="unrestricted"` 并提供 `reason`
- Unrestricted Sandbox 由 Downcity Runtime 管理；每次执行前都会向用户确认，用户拒绝或超时则不会执行命令
- unrestricted `shell_start` 的审批只授权启动该进程；后续每一次向同一个 unrestricted session 调用 `shell_write`，也都必须提供独立 `reason` 并再次等待用户确认
- `sudo`、Xcode 工具安装、修改 SSH/keychain/shell profile 或明显破坏性命令会被默认拒绝
- Safe Sandbox 默认允许 shell 打开网络访问

Linux 使用前需要先安装 `bubblewrap`，并确保当前系统允许 unprivileged user namespaces。缺少 `bwrap` 或系统禁用 user namespace 时，shell 命令会失败，不会回退到普通宿主子进程执行。

## 去哪里修改

Shell Safe Sandbox 边界当前不可通过项目配置修改。需要宿主级能力时，agent 必须显式请求 `sandbox="unrestricted"` 并等待用户审批。
