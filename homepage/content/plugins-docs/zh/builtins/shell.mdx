# Shell Built-In

Shell 不再是 plugin。它是 `@downcity/shell` 提供的内建能力，通过 `new Shell()` 挂到 Agent 上。

```ts
import { Agent } from "@downcity/agent";
import { Shell } from "@downcity/shell";

const agent = new Agent({
  id: "repo-helper",
  path: "/path/to/project",
  shell: new Shell(),
});
```

## Tools

传入 `shell: new Shell()` 后，模型会获得这些工具：

- `shell_exec`
- `shell_start`
- `shell_status`
- `shell_read`
- `shell_write`
- `shell_wait`
- `shell_close`

## Sandbox 模式

Shell tools 默认进入 Safe Sandbox。

只有需要宿主级能力时才使用 `sandbox: "unrestricted"`，例如 `brew install`、`npm install -g`、`pip install --user`、`gh auth login`，或初始化项目 sandbox 外的宿主服务。

```ts
shell_exec({
  cmd: "brew install ffmpeg",
  sandbox: "unrestricted",
  reason: "Homebrew 需要写入宿主级 package 目录。",
});
```

Unrestricted 执行每次都需要用户审批。缺少 `reason` 会在执行前被拒绝。

## 审批 API

前端或 SDK 客户端通过 Agent API 审批：

```ts
const pending = await agent.approvals();

await agent.approve({ approval_id: pending[0].approval_id });
await agent.deny({ approval_id: pending[0].approval_id });
```

`RemoteAgent` 通过 Town Agent HTTP gateway 暴露同样的 API。

## Session Approval Mode

Shell approval mode 只作用于当前 session：

- `ask`：默认模式，每次 unrestricted 请求都进入审批队列。
- `always-allow`：当前 session 内自动通过 shell approval，不再弹出新的 pending approval。

```ts
const modes = agent.approval_modes();

agent.set_approval_mode({
  session_id: "consoleui",
  mode: "always-allow",
});

const current = agent.approval_mode({ session_id: "consoleui" });
```

`always-allow` 不改变 sandbox 模式，只是把原本需要手动批准的 unrestricted 请求自动记录为 approved。已有 pending approval 不会被自动批准。

## 关键语义

- Shell 归属于 `@downcity/shell`，不属于 `@downcity/plugins`。
- Agent 只挂载 Shell tools，并转发 session event。
- Safe Sandbox 命令可读写项目，并使用配置的 sandbox 目录。
- Unrestricted 命令只有在逐次审批通过后才会离开 Safe Sandbox 执行。
- `always-allow` 只影响当前 session 的新 approval 请求，不影响其它 session。
- 对 unrestricted `shell_start` 会话执行的每一次 `shell_write` 也需要审批。
- `agent.stop()` 会释放 Shell sessions。
