# `town model`

`town model` is no longer part of the Town command surface.

Town is the local Agent host and Agent Plugin manager. City models are managed by the `city` CLI and the shared City service runtime.

Use:

```bash
city
town city use
town city status
```

If an Agent project needs a City model ID in `downcity.json.execution.modelId`, choose that model from City-side administration and keep Town focused on hosting Agents and managing Plugins.
