# `town plugin`

`town plugin` 是 Town 查看 Agent Plugin 能力目录的入口。

> 推荐先看：plugin 概念层文档在 [Agent SDK / Plugins](/zh/agent-sdk-docs/plugins/overview)，具体手册在 [Agent Plugins Docs](/zh/plugins-docs)。

适合这些场景：

- 查看当前注册了哪些 plugin
- 检查某个 plugin 的能力面
- 不走短别名，直接把托管 plugin command 转发给指定 agent
- 在当前本地项目内直接执行本地 plugin action
- 查看或取消一次性 plugin 调度任务

## 常用示例

```bash
town plugin
town plugin list
town plugin command chat list --agent my-agent
town plugin command task run --payload '{"title":"daily-check"}' --agent my-agent
town plugin schedule list --path /path/to/agent
```

## 说明

- 在交互式终端里直接运行 `town plugin`，会进入 plugin 目录浏览器。
- `town plugin list` 和 `town plugin info` 是静态 catalog 视图。Town 不拥有 plugin 运行态。
- `town plugin command <pluginName> <command>` 会把 command 转发给目标 agent 的运行中 plugin。
- `town plugin action ...` 是本地执行形式，默认直接在当前项目里运行。
- `town chat ...`、`town task ...`、`town memory ...`、`town contact ...`、`town skill ...` 是当前可用的快捷入口。
- Shell 执行现在是 Agent 内建能力；unrestricted sandbox 决策通过 Agent SDK 审批 API 完成。
