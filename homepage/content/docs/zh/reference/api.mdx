# HTTP API 参考

> 面向用户的运行时 API。更多细粒度命令建议优先使用 CLI（`sma ...`）。

## 基础接口

### `GET /health`

```json
{ "status": "ok", "timestamp": "2026-03-08T12:00:00.000Z" }
```

### `GET /api/status`

```json
{ "name": "shipmyagent", "status": "running", "timestamp": "..." }
```

### `POST /api/execute`

请求体：

```json
{
  "instructions": "帮我总结今天的改动",
  "chatId": "default"
}
```

## Service Runtime API

### `GET /api/services/list`

列出 service 运行状态。

### `POST /api/services/control`

```json
{ "serviceName": "chat", "action": "restart" }
```

`action` 支持：`start | stop | restart | status`。

### `POST /api/services/command`

```json
{
  "serviceName": "task",
  "command": "list",
  "payload": { "status": "enabled" }
}
```

## Extension Runtime API

### `GET /api/extensions/list`

列出 extension 运行状态。

### `POST /api/extensions/control`

```json
{ "extensionName": "voice", "action": "status" }
```

### `POST /api/extensions/command`

```json
{
  "extensionName": "voice",
  "command": "status",
  "payload": {}
}
```

## Console UI API（摘要）

### `GET /api/ui/extensions`

返回 extension 运行状态 + 配置摘要（适用于 Console UI 面板）。

返回字段示例：

```json
{
  "success": true,
  "extensions": [
    {
      "name": "voice",
      "state": "idle",
      "supportsLifecycle": true,
      "supportsCommand": true,
      "config": {
        "lifecycle": { "start": true, "stop": true, "command": true },
        "actions": [
          {
            "name": "status",
            "supportsCommand": true,
            "supportsApi": true,
            "apiMethod": "GET",
            "apiPath": "/extension/voice/status"
          }
        ]
      }
    }
  ]
}
```

### `POST /api/tui/contexts/:contextId/execute`

在指定上下文执行一轮（例如你在 UI 里选中的 chatKey）。

带附件请求示例：

```json
{
  "instructions": "请阅读附件 markdown，并给出可执行摘要。",
  "attachments": [
    {
      "type": "document",
      "fileName": "page.md",
      "contentType": "text/markdown; charset=utf-8",
      "content": "# Captured Web Page\n\n..."
    }
  ]
}
```

说明：

- `contextId` 应传目标上下文本身（聊天回传场景下，直接使用已选中的 chatKey）。
- 附件会先被 runtime 落盘，再以 `@attach` 指令行自动注入到用户消息前部。

## Action 直连路由

每个 service/extension action 还会暴露默认路由：

- `/service/<serviceName>/<actionName>`
- `/extension/<extensionName>/<actionName>`

例如：

- `POST /service/chat/send`
- `GET /extension/voice/status`

## 调试建议

先确认 runtime 已启动：

```bash
sma agent start
curl http://localhost:5314/health
```
