# `studio plugin`

`studio plugin` 是当前 plugin 系统的通用入口。

> 推荐先看：plugin 概念层文档在 [Agent SDK / Plugins](/zh/agent-sdk-docs/plugins/overview)，具体手册在 [Plugins Docs](/zh/plugins-docs)。

适合这些场景：

- 查看当前注册了哪些 plugin
- 检查某个 plugin 的静态启用态与能力面
- 面向指定 agent 启动、停止、重启托管 plugin
- 不走短别名，直接转发托管 plugin command
- 在当前本地项目内直接执行本地 plugin action
- 查看或取消一次性 plugin 调度任务

## 常用示例

```bash
studio plugin
studio plugin list
studio plugin status asr
studio plugin start chat --agent my-agent
studio plugin command task run --payload '{"title":"daily-check"}' --agent my-agent
studio plugin action asr status
studio plugin schedule list --path /path/to/agent
```

## 说明

- 在交互式终端里直接运行 `studio plugin`，会进入 plugin 管理器。
- `studio plugin list` 和 `studio plugin status` 是静态 console-level catalog 视图。
- `chat`、`task`、`memory`、`contact`、`shell` 这类托管 plugin 支持 `start/stop/restart/command`。
- `studio plugin action ...` 是本地执行形式，默认直接在当前项目里运行。
- 日常使用更推荐具体命令入口，比如 `studio chat ...`、`studio task ...`、`studio memory ...`、`studio shell ...`、`studio web ...`、`studio skill ...`、`studio asr ...`、`studio tts ...`。
