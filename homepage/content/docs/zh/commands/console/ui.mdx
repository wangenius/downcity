# `city console ui`

管理由 `console` 托管的 UI 网关。

该 UI 不绑定某一个 `agent start` 进程，而是可以在同一页面切换查看多个已运行 agent。
它是状态优先的 Dashboard，对话只在内置 `consoleui` channel 中开放（默认 context：`consoleui-chat-main`）。

## 用法

```bash
city console ui [action] [options]
```

## 常用参数

- `-p, --port <port>`：UI 端口（默认 `5315`）
- `-h, --host <host>`：UI 主机（默认 `127.0.0.1`）

## action

- `start`：后台启动 UI（默认）
- `stop`：停止后台 UI
- `restart`：重启后台 UI
- `status`：查看 UI 状态

## 行为说明

- 需要先启动 console：`city console start`。
- `start` / `restart` 前会先尝试清理没有 pid 文件但仍占着 UI 端口的旧 `console ui run` 进程。
- UI 启动后会读取 console registry 中的 agent 列表（运行中 + 历史）。
- `Global Agents` 面板会展示每个 agent 的运行状态、runtime 地址、当前绑定模型与 chat 身份快照（例如 Telegram bot 用户名）；对历史记录可直接点击“启动”重新拉起。
- `Global Agents` 右上角主动作现在是“打开文件夹”：先通过系统目录选择器选中项目目录。
- 如果该目录已经存在 `downcity.json` 且完成最小初始化，UI 会直接启动该 agent。
- 只有当目录尚未完成初始化时，才会进入“初始化并启动”流程：补齐 `PROFILE.md`、`SOUL.md`、`downcity.json` 与 `.downcity/` 结构后再拉起 runtime。
- 侧边栏按三层分区：`Global` / `Agent` / `Context`。
- `Global`：全局概览、console runtime、model 管理、agents 管理、plugins。
- `Global / Model` 现已支持 provider/model 全量操作（创建、更新、测试、发现、暂停、删除）。
- `Global / Overview` 仅展示全局级健康信号（console、agent fleet、channels、plugins、config risk）。
- `Global / Overview` 顶部摘要区域会显示当前 DC 版本号。
- `Global / Env` 现已支持手动维护 Console 共享 env 与 Agent 私有 env，统一写入 `~/.downcity/downcity.db` 的 `env_entries`。
- `Global / Env` 点击右上角 `+` 时，会自动检测当前剪贴板是否为 `.env` 文本；若匹配则直接批量导入（支持一次多个键），否则回退到手动创建弹窗。
- `Global / Console Runtime` 展示 console 运行态与 console 级配置文件状态。
- `Agent`：`Overview`、`Authorization`、`Services`、`Command`、`Tasks`、`Logs`。
- `Agent / Overview` 聚焦当前 agent（bootstrap 配置、chat 身份、模型绑定），不再混入 global 信息。
- `Agent / Overview` 现会额外显示当前 agent 是否有 context 正在执行，以及执行中的 context 数量。
- `Agent / Authorization` 已重构为角色权限模型：维护 `role -> permission`，再把用户绑定到角色。
- `Agent / Authorization` 的配置统一写入 console `~/.downcity/downcity.db`，不会明文落到项目 `downcity.json`。
- `Agent / Authorization` 中新用户会落到默认角色；群聊/频道是否可用，取决于当前发消息用户所属权限组。
- `Agent / Authorization` 现已改为“概览 + 目录”工作台：主页面只显示角色摘要、渠道默认组摘要、权限说明与用户目录。
- `Agent / Authorization` 中角色编辑与渠道默认组配置都收进 dialog；角色支持维护 `name` / `description`，并可删除非保留角色。
- 如果你想完整了解这套模型，可继续阅读：[权限说明](/zh/docs/security/permissions)。
- `Agent / Overview` 的 `chat overview` 现支持 channel 级快捷动作：`open` / `close` / `test` / `reconnect`，可直接做连通性与开关操作。
- `Agent / Command` 提供终端风格命令面板：在当前 agent 项目根目录执行 shell 命令，并展示 stdout/stderr 与退出码。
- `Agent / Command` 现支持输入原地预测与快捷输入：`Tab` 接受整条预测、`Ctrl/⌘ + →` 接受下一个词、`↑/↓` 浏览会话命令历史。
- `Agent / Tasks` 会展示任务定义详情与每次执行记录（run history + run detail）。
- `Agent / Tasks` 支持执行中实时进度：手动 `run` 后会显示 `running`、当前阶段、轮次与进度事件，完成后自动切换到最终状态。
- `Agent / Tasks` 支持任务管理：可直接切换任务状态（`enabled` / `paused` / `disabled`）、删除任务定义、删除单条 run 历史记录（执行中 run 不允许删除）。
- `Agent / Tasks` 支持一键清空当前任务全部 run log（会自动跳过执行中的 run 目录）。
- `Context`：`Context Overview` + 每个 context 条目（包含内置 `consoleui` channel）。
- `Context Overview` 内会展示 channel 链路状态与 context 对应关系，并提供 `open` / `close` / `test` / `reconnect`。
- `Context Overview` 支持直接修改 channel 参数（例如 qq 的 `appId/appSecret/sandbox`），保存后自动重载该渠道。
- 只有 `consoleui` channel 支持发送消息；其他 context 在 UI 中为只读。
- `Context Workspace` 会展示当前 context 的消息历史、channel 历史、system prompt 解析结果。
- chat context 的 `system prompt` 现在只注入“当前 context 所属 channel”的规则，不再把其他已启用 channel 的 prompt 一起拼进去。
- `Context Overview` 与 `Context Workspace` 现都会显示当前 context 是否正在执行（`executing` / `idle`）。
- `Context Workspace` 现支持消息管理：所有 chat context 都可分别清理 `context messages` 与 `chat history`。
- `Context Workspace` 现支持查看 compact archive：可在右侧 `archive` 面板按归档批次查看被 compact 折叠前的历史消息。
- `city console status` 会同时显示 console 与 UI 状态。

## 典型流程

```bash
# 1) 启动 console
city console start

# 2) 启动多个 agent
city agent start /path/to/agent-a
city agent start /path/to/agent-b

# 3) 后台启动 UI
city console ui start --port 5315

# 4) 查看 UI 状态
city console ui status

# 5) 重启 UI（加载最新网关代码/路由）
city console ui restart

# 6) 停止 UI
city console ui stop
```
