# UI SDK

`@downcity/ui` is the React + Tailwind component package extracted from Downcity's shared interface layer.

## What It Includes

- Console-style primitives such as `Button`, `Badge`, `Card`, `Checkbox`, `Input`, `Label`, `Textarea`, `Separator`, `Dialog`, `Sheet`, `DropdownMenu`, `Popover`, and `Toaster`
- A shared `cn` utility for Tailwind class composition
- Theme tokens in `styles.css` for Downcity's default surface, border, and interactive colors

## Install

```bash
npm install @downcity/ui
```

## Use In A Tailwind App

Import Tailwind first, then import the SDK stylesheet in your app CSS:

```css
@import "tailwindcss";
@import "@downcity/ui/styles.css";
```

Then import components in React:

```tsx
import { Button, Card, CardContent, Input, Label } from "@downcity/ui";

export function ExampleCard() {
  return (
    <Card>
      <CardContent className="space-y-3">
        <div className="space-y-1">
          <Label htmlFor="agent-name">Agent Name</Label>
          <Input id="agent-name" placeholder="For example: research-writer" />
        </div>
        <div className="flex items-center justify-between gap-4">
          <span>Run an agent task</span>
          <Button>Open Console</Button>
        </div>
      </CardContent>
    </Card>
  );
}
```

## Current Scope

- The package is focused on reusable primitives instead of product-specific screens
- Theme defaults follow Downcity's neutral console palette
- Host apps can still extend styling with `className`
