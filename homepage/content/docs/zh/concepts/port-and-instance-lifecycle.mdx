# 端口与实例生命周期

## 统一规则

- agent 端口由 `console` 分配。
- `city agent start/restart` 不要求用户手工管端口。
- `service` / `plugin` 命令先找目标 agent，再找 endpoint。

## 运行时角色

### console

- 维护 `~/.ship/console/agents.json`
- 启停 agent daemon
- 分配端口

### agent

- 一个项目对应一个 daemon
- runtime API 包括 `/api/services/*`、`/api/plugins/*`
- debug 信息写入 `.ship/.debug`

### service / plugin

- `city service ...` 走 service 路由
- `city plugin ...` 走 plugin 路由
- 都在目标 runtime 内部分发
