# 模型配置（Console + Agent）

模型配置拆分为两部分：

1. Console 维护模型池（provider/model 元信息、密钥、endpoint）。
2. Agent 在 `downcity.json` 只选择 `model.primary`。

## 工作流

1. 在 console 中配置 provider/model。
2. 在 agent 项目设置 `model.primary`。
3. 重启 agent 并进行一次实际调用验证。

## 命令示例

```bash
city console model list
city console model use default --path .
city console config set model.primary default
city agent restart .
```

## 失败场景

- `model.primary` 不存在：agent 启动失败。
- provider/key 无效：调用模型时报鉴权或网络错误。
