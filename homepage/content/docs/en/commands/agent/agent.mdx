# `city agent`

Manage Agent Runtime start/restart.

## Command hierarchy

Top-level commands are fixed to 4: `city console`, `city agent`, `city service`, `city extension`.

## Usage

```bash
city agent <subcommand>
```

## Subcommands

- `create`: create/initialize an agent project
- `start`: start runtime
- `status`: show runtime status
- `doctor`: diagnose state files and optionally fix stale pid/meta
- `restart`: restart runtime

## Environment variable injection

On every `city` execution, CLI resolves current agent context and injects variables into the `city` process and its child processes:

- `DC_AGENT_PATH`: absolute path of current agent
- `DC_AGENT_NAME`: agent name (prefer `ship.json.name`, fallback to directory name)

> Note: this is process-scoped injection and does not mutate your parent shell session.
