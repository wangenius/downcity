# Console 配置存储（downcity.db）

从当前版本开始，Console 不再使用：

1. `~/.downcity/downcity.json`
2. `~/.downcity/.env`

Console 全局配置统一存储在：

- `~/.downcity/downcity.db`

## 存储内容

`downcity.db` 中主要包含以下数据域：

1. 模型池（providers/models）
2. Console 共享 env（`global_env`）
3. Agent 私有 env（`agent_env`）
4. Bot 账号凭据池（`channel_accounts`）

其中敏感字段（如 API Key）采用加密存储。

## 与 Agent 项目配置的边界

以下文件仍然是 Agent 项目级配置（保留不变）：

1. `<agent>/downcity.json`
2. `<agent>/.env`

也就是：

- Console：`~/.downcity/downcity.db`
- Agent：项目目录下的 `downcity.json/.env`

## 环境变量作用域（重要）

当前版本按明确分层工作：

1. Console 共享 env：`global_env`
2. Agent 私有 DB env：`agent_env`
3. Agent 本地运行时覆盖：`<agent>/.env`（用户维护）
4. Chat 渠道凭据：`channel_accounts`（由 `downcity.json` 的 `channelAccountId` 绑定）

关键行为：

1. agent env 最终是 `agent_env + <agent>/.env`（`.env` 优先）。
2. Agent 的 `.env` 只在当前 agent 内生效，不污染全局 `process.env`。
3. `downcity.json` 只保存渠道绑定（`channelAccountId`），不保存明文密钥。
4. 渠道真实凭据来自 `channel_accounts`。
5. plugin 配置来自项目 `downcity.json`。

## 常用操作

初始化 Console（写入 `downcity.db`）：

```bash
city console init
```

查看/修改项目 plugin 配置：

```bash
city console config get plugins.asr
city console config set plugins.asr.enabled true
```

管理模型池：

```bash
city console model list
city console model create
```
