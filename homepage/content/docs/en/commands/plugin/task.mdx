# `bay task`

`bay task` is the managed task plugin command group.

Use it when you want a reusable automation definition instead of a one-shot delayed plugin action.

## Usage

```bash
bay task <subcommand> [options]
```

## Main subcommands

- `list`
- `create`
- `run <title>`
- `update <title>`
- `delete <title>`
- `status <title> <enabled|paused|disabled>`
- `enable <title>`
- `disable <title>`

## Key boundaries

- Use `bay task` for reusable or long-lived automation.
- Use `bay chat --delay/--time` or `bay memory --delay/--time` for one existing action that should happen later once.
- Task definitions live in project storage and can use `@manual`, cron, or `time:<ISO8601>`.

## Examples

```bash
bay task list
bay task create --title "Daily check" --description "Run tests" --session-id telegram-chat-123 --when "0 10 * * 1-5" --activate
bay task run daily-check
bay task update daily-check --when "@manual"
bay task disable daily-check
```
