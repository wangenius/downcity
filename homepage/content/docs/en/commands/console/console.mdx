# `sma console`

`console` is the global control plane.

Use it to manage console runtime and inspect managed agent runtimes.

## Command hierarchy

Top-level commands are fixed to 4:

- `sma console`
- `sma agent`
- `sma service`
- `sma extension`

Under `console`, use `init/config/start/stop/status/agents/ui`.

## Usage

```bash
sma console init
sma console config <subcommand>
sma console start
sma console stop
sma console restart
sma console status
sma console agents [--json]
sma console ui [start|stop|status] [--port 3001]
```

## Behavior

- `init`: initialize global console config (`~/.ship/*`).
- `config`: manage project `ship.json` and llm/provider/model/alias.
- `start`: start console in background.
- `stop`: stop console and managed child agent runtimes.
- `restart`: restart console.
- `status`: print console process state and active managed agents.
- `agents`: print active managed agents list.
- `ui`: manage single UI instance (start/stop/status) and switch across agents in one page.

## Typical Flow

```bash
# 1) start global console
sma console start

# 2) start project runtime
cd /path/to/agent-project
sma agent create .
sma agent on

# 3) check status
sma console status
sma agent status .
```
