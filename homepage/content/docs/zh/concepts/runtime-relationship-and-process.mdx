# 关系与进程模型

本文聚焦两件事：

- 四类角色如何协作
- 进程如何启动与连接

## 1. 角色关系

- `console` 是控制中心：维护 agent registry，承接 `config/service/extension` 运维命令。
- `agent` 是执行中心：在项目内处理消息、任务、工具调用与落盘。
- `service` 是业务域：承载核心流程能力。
- `extension` 是扩展域：承载跨 agent 共享的增强能力。

可以用一句话概括：

`console` 负责“管”，`agent` 负责“跑”，`service/extension` 负责“提供能力”。

## 2. 进程启动模型

1. 启动 console：`sma console start`
2. 在项目内启动 agent：`sma agent start`
3. agent runtime 加载配置（项目 `ship.json.services.*` + console 全局 `~/.ship/ship.db` 的 `extensions_config.*` + 全局模型池）
4. 入站消息/任务进入 agent 调度
5. agent 调用 service/extension 完成执行
6. 执行轨迹写入 `.ship/*`

## 3. 运维检查清单

```bash
sma console status
sma agent status .
sma service list
sma extension list
```
