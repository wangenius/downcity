# RemoteAgent Quickstart

## Start HTTP on the server side first

Server-side code:

```ts
const started = await agent.start({
  http: {
    host: "127.0.0.1",
    port: 15314,
  },
});
```

## Connect from the client

```ts
import { RemoteAgent } from "@downcity/agent";

const agent = new RemoteAgent({
  baseUrl: "http://127.0.0.1:15314",
});

const session = await agent.createSession();
const turn = await session.prompt({
  query: "Summarize the current repository structure",
});

const result = await turn.finished;
```

On the client side, you do not need to care about project prompts, shared model catalog details, or runtime assembly. You only consume the remote session capability that has already been exposed.

## Good fit for

- another process already owns the local Agent
- you only want to consume agent capability through HTTP
- you want to separate the caller from the executor

If you have a frontend and backend, a local daemon plus callers, or several consumers sharing one agent host, this mode feels very natural.
