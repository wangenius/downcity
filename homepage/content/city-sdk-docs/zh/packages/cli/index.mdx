`downcity` 是 Downcity 的全局安装聚合包。它会安装两个命令：`city` 负责 City 管理，`town` 负责本机 Agent 宿主。

如果说 `Admin City` 负责“程序里的可信调用”，那 `city` 负责“人坐在终端前的直接操作”。

## 这组文档怎么读

建议按下面顺序读：

1. 先读 [快速开始](/zh/city-sdk-docs/packages/cli/quickstart)，先知道怎么连到 City 数据库。
2. 再读 [模型与数据表管理](/zh/city-sdk-docs/packages/cli/models-and-tables)，理解最常见的表查看和模型维护动作。
3. 最后读 [运维与排查](/zh/city-sdk-docs/packages/cli/operations-and-debug)，把日常检查、故障定位和人工修复串起来。

## 适合什么时候用

- 你想直接查看 City 数据库里有哪些表
- 你要增删改查 `downcity_models`
- 你要在本地排查 runtime 数据是否写对了

## 安装

```bash
npm i -g downcity
```

## 默认用法

```bash
city
```

在交互式终端里，默认 `city` 会进入全屏 City TUI dashboard；在非交互环境里，它会输出帮助信息。

CLI 也支持语言切换：

```bash
city --lang en
city --lang zh --help
DC_CLI_LANG=zh city
```

在交互模式下，`city` 也可以直接从菜单修改默认语言，并把选择持久化到本地。语言解析优先级是：先读 `--lang`，再读交互菜单里保存的默认语言，最后再读 `DC_CLI_LANG`、`LC_ALL`、`LC_MESSAGES`、`LANG`。

如果还没有连接任何 City，dashboard 会先引导你添加一个 City；之后你可以：

- 通过 `Manage Servers` 管理多个 City server
- 进入 Admin 流程管理 env / models / towns / payment
- 进入 User 流程登录、充值和调用服务

## 可脚本化的 city 命令

```bash
city manage
city create .
city deploy
city deploy --dry-run
city deploy --verify
city env refresh
```

这些命令负责 City 项目、部署，以及 City runtime env cache。

## 和 `town city` 的边界

`town city` 是 Town 的命令组。当本机 Town runtime 或 agent 需要选择 City base、保存 Town user 登录态时，使用它：

```bash
town city use
town city login
town city status
```

一句话区分：

- `city`：人在终端里直接操作 City
- `town city`：让 Town 和本地 agent 连接到某个 City base

## 它和其他 package 的边界

- 不替代 [@downcity/city](/zh/city-sdk-docs/packages/city/city-runtime) 里的 `Admin City`
- 不替代 [@downcity/city](/zh/city-sdk-docs/packages/city) 的运行时能力
- 更适合“人工排查”和“初始化运维”，而不是嵌进产品代码里

更准确地说：

- `Admin City` 负责程序里的可信调用
- `city` 负责终端里的人工运维和排查
