# 常见误用对比

## 误用 1

需求：

- 给消息补默认字段

错误做法：

- 用 `guard`

更好做法：

- 用 `pipeline`

原因：

- 这是改值，不是拦截。

## 误用 2

需求：

- 没权限就阻断

错误做法：

- 用 `effect`

更好做法：

- 用 `guard`

原因：

- `effect` 不是拦截语义。

## 误用 3

需求：

- 每次执行完都写审计日志

错误做法：

- 用 `pipeline`

更好做法：

- 用 `effect`

原因：

- 这里没有主值加工需求。

## 误用 4

需求：

- 最终用户角色是什么

错误做法：

- 用 `pipeline`

更好做法：

- 用 `resolve`

原因：

- 这要求唯一答案，不是逐步增强。

## 误用 5

需求：

- 维护一个长期缓存和后台同步 worker

错误做法：

- 全塞进 plugin hook

更好做法：

- 用 `service` 持有长期状态
- hook 只保留链路扩展部分

原因：

- 长期运行实例不应该硬塞进 hook 语义里。

## 读完这篇最该留下的感觉

不要按“我在哪写起来方便”来分层。

要按“当前这段能力在运行时到底扮演什么角色”来分层。
