# 环境变量加载

项目 agent 启动时，会读取项目环境变量，并把 `downcity.json` 里的 `${VAR}` 做替换。

## 常见场景

- 你在配置中引用密钥
- 你不想把明文直接写进 `downcity.json`

## 推荐做法

- 把密钥放到项目 `.env` 或可控运行环境里
- 配置里只写 `${VAR_NAME}`

这样做的直接好处是：

- 配置更容易共享
- 密钥不容易误提交
- 环境切换更容易

## 修改后要做什么

通常需要重启 agent：

```bash
city agent restart
```

因为 agent 启动时会读取并解析这批配置，单纯改环境变量不会自动刷新当前进程里的运行时快照。
