# NotesService 场景

## 场景

你在做一个本地研发助手，希望它能在一次运行期间持续积累“临时笔记”：

- 记录用户刚刚确认过的约束
- 记录某个仓库里已经验证过的事实
- 让多个 session 共用这些临时笔记

这时它已经不像一个一次性 tool，更像一个长期存在的状态对象。

## 为什么这里更适合 service

因为你要的是：

- 长期 in-memory 状态
- 多个 session 共用
- 一组稳定 action

如果你把它做成单纯 tool，后面通常会开始往 tool 外面补：

- 全局 Map
- 状态同步逻辑
- 生命周期控制

那就已经在往 service 靠了。

## 一个更像真实应用的写法

```ts
import {
  Agent,
  BaseService,
  type ServiceActions,
} from "@downcity/agent";

class NotesService extends BaseService {
  readonly name = "notes";

  private readonly notes: Array<{
    id: string;
    text: string;
    createdAt: number;
  }> = [];

  addNote(text: string) {
    const clean = String(text || "").trim();
    if (!clean) {
      throw new Error("note text is required");
    }
    const item = {
      id: `note-${this.notes.length + 1}`,
      text: clean,
      createdAt: Date.now(),
    };
    this.notes.push(item);
    return item;
  }

  listNotes() {
    return [...this.notes];
  }

  clearNotes() {
    this.notes.length = 0;
  }

  readonly actions: ServiceActions = {
    add: {
      execute: async ({ payload }) => {
        const body = payload as { text?: unknown };
        const item = this.addNote(String(body.text || ""));
        return {
          success: true,
          data: item as never,
        };
      },
    },
    list: {
      execute: async () => {
        return {
          success: true,
          data: {
            items: this.listNotes(),
          },
        };
      },
    },
    clear: {
      execute: async () => {
        this.clearNotes();
        return {
          success: true,
          data: {
            cleared: true,
          },
        };
      },
    },
  };
}

const notesService = new NotesService();

const agent = new Agent({
  id: "repo-helper",
  path: "/path/to/project",
  tools: {},
  services: [notesService],
});
```

## 这个例子里的关键点

### 1. app 代码可以直接持有 service 引用

比如：

```ts
notesService.addNote("Repository uses pnpm workspaces");
console.log(notesService.listNotes());
```

这在当前 SDK 里很实用，因为“泛化的 service action 调用端口”并不是 local SDK 的主 public 入口。

### 2. `actions` 仍然值得保留

即使你在 app 代码里更可能直接调实例方法，保留 `actions` 仍然有价值：

- 结构和完整 runtime 一致
- 后续扩展更自然
- 这组能力的边界更清晰

### 3. 这是典型的“多 session 共用状态”

如果同一个 agent 创建多个 session，它们都能共享同一个 `NotesService` 实例里的笔记。

## 什么时候这个例子还不够

如果你的笔记需要：

- 重启后仍保留
- 跨进程共享

那就不应该只停留在内存里，而应该把状态持久化到文件或数据库。

但“要不要持久化”不影响它首先更像 service 这件事。
