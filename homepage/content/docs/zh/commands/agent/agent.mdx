# `sma agent`

管理 Agent Runtime 的启动、停止和重启。

## 命令层级

一级命令固定为 4 个：`sma console`、`sma agent`、`sma service`、`sma extension`。

## 用法

```bash
sma agent <subcommand>
```

## 子命令

- `create`：创建/初始化 agent 项目
- `on`：启动 runtime
- `off`：停止 runtime
- `status`：查看 runtime 状态
- `doctor`：诊断状态文件，并可修复僵尸 pid/meta
- `restart`：重启 runtime

## 环境变量注入

每次执行 `sma` 时，CLI 会自动推导当前 agent 上下文，并在 `sma` 进程及其子进程中注入：

- `SMA_AGENT_PATH`：当前 agent 绝对路径
- `SMA_AGENT_NAME`：当前 agent 名称（优先 `ship.json.name`，否则目录名）

> 注意：这是进程级注入，不会直接回写你当前 shell 会话的环境变量。
