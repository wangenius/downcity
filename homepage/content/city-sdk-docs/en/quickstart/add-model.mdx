## Scenario 1: OpenAI-Compatible Provider (Auto Passthrough)

For DeepSeek and other OpenAI-compatible providers:

```ts
import { Provider, AIService } from "@downcity/city";

const deepseek = new Provider("deepseek", {
  env: { DEEPSEEK_API_KEY: "DeepSeek API Key" },
  baseURL: "https://api.deepseek.com",
  envKey: "DEEPSEEK_API_KEY",       // auto-passthrough uses this
  text: deepseekTextHandler,        // SDK pathway
  stream: deepseekStreamHandler,    // SDK pathway
  // no openai → auto-passthrough
});

const ai = new AIService();
ai.use(deepseek.model({
  id: "deepseek-v4-flash",
  name: "DeepSeek V4 Flash",
  description: "DeepSeek text model",
  default: true,
}));
base.use(ai);
```

## Scenario 2: Custom OpenAI-Compatible Handler

Use an `openai` handler when an OpenAI-compatible upstream needs request or response customization:

```ts
const openaiCustom = new Provider("openai-custom", {
  env: { OPENAI_API_KEY: "OpenAI API Key" },
  text: openaiTextHandler,
  stream: openaiStreamHandler,
  openai: async (ctx) => {
    // Downstream: customize OpenAI body
    // Upstream: normalize response
    // (streaming: SSE event-by-event conversion)
  },
});
```

## Handler Signatures

| Pathway | ctx.input | Return |
|---|---|---|
| `text` | `{ prompt }` | `UIMessage` |
| `stream` | `{ prompt }` | `Response` (SSE) or `UIMessageStream` |
| `openai` | `{ model, messages, stream, ... }` | `Response` |
