# Getting Started

Downcity has several user surfaces. This page is a map, not a command tutorial.

Use it to decide where to start:

- Use [Downcity CLI](/en/docs/reference/cli) when you want to run local agents, manage project agents, or use built-in plugin commands.
- Use [City CLI](/en/city-sdk-docs/packages/cli) when you want to create, manage, or deploy a City project.
- Use [Agent SDK](/en/agent-sdk-docs) when you want to embed agent behavior in your own Node.js application.
- Use [City SDK](/en/city-sdk-docs) when you want to build services, model providers, billing, usage, or admin surfaces.
- Use [Agent Plugins Docs](/en/plugins-docs) when you want to understand built-in plugin behavior or author plugin integrations.
- Use [Products Docs](/en/docs) when you want product-facing usage for Downcity CLI, Downcity Console, or Chrome Extension.

## Recommended Order

First understand the model:

1. [How Downcity Fits Together](/en/docs/concepts/project-logic-overview)
2. [Agent Overview](/en/docs/agent/overview/what-is-agent)
3. [Configuration Model](/en/docs/configuration/overview)

Then choose the practical surface:

- Local CLI usage: [Downcity CLI](/en/docs/reference/cli)
- Project agent workflows: [Agent CLI product docs](/en/docs/reference/cli)
- Plugin workflows: [Plugin CLI product docs](/en/docs/reference/cli)
- City administration CLI: [City CLI](/en/city-sdk-docs/packages/cli)
- SDK integration: [Agent SDK](/en/agent-sdk-docs)

## What This Space Explains

Downcity docs explain why the system is split into Downcity, City, Agent, plugins, services, permissions, and configuration layers.

Concrete command flags, examples, and step-by-step operation guides live in product or SDK documentation.
