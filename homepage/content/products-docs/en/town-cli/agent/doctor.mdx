# `town agent doctor`

Diagnose agent state files for a target agent and optionally clean stale pid/meta files.

Use this command when daemon state looks inconsistent.

## Usage

```bash
town agent doctor [path] [--fix]
```

## Key option

- `--fix [enabled]`: clean stale state files automatically

## Behavior

- `path` defaults to current directory `.`
- Target agent must be registered in the managed agent registry
- Without `--fix`, it only reports issues and does not modify files

## What stale means

`stale` usually means the pid/meta state files still exist, but the daemon process does not.

Common causes include abnormal process exit, machine restart, or incomplete cleanup from a previous launch. In most cases, `doctor --fix` is the right first move before starting again.
