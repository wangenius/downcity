`@downcity/city` is a terminal tool for InfraRuntime operators. It is not part of product runtime traffic. It is the operational entry you use during setup, inspection, and debugging.

If `AdminClient` owns "trusted programmatic calls", `@downcity/city` owns "a human operating InfraRuntime directly from the terminal".

## How to read this section

The recommended order is:

1. Start with [Quickstart](/en/docs/city-infra/packages/city-terminal/quickstart) to learn how the CLI connects to your InfraRuntime database.
2. Continue with [Models and tables](/en/docs/city-infra/packages/city-terminal/models-and-tables) for the most common table inspection and model-management workflows.
3. Finish with [Operations and debugging](/en/docs/city-infra/packages/city-terminal/operations-and-debug) to connect routine checks, incident diagnosis, and manual fixes.

## Use it when

- you want to inspect which tables exist in the InfraRuntime database
- you need to add, update, or remove entries in `downcity_models`
- you want to debug runtime data directly from the terminal

## Install

```bash
npm i -g @downcity/city
```

## Default usage

```bash
downcity
```

The default entry opens an interactive panel. If no server exists yet, the CLI first requires you to add one. After that, you can:

- manage multiple InfraRuntime servers from `Manage Servers`
- enter the Admin flow for env / models / products / payment
- enter the User flow for login, recharge, and service calls

## Common commands

```bash
downcity models
downcity models add
downcity models update
downcity --database-url "$DOWNCITY_INFRA_DATABASE_URL"
```

## Boundaries with other packages

- It does not replace `AdminClient` from [@downcity/conduit](/en/docs/city-infra/packages/conduit).
- It does not replace the runtime capabilities in [@downcity/infra](/en/docs/city-infra/packages/infra).
- It is better for human-operated inspection and maintenance than for code you embed into a product.

- `AdminClient` is for trusted programmatic calls
- `@downcity/city` is for human-operated terminal workflows
