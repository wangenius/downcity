---
title: Global wallet model
description: "Why @downcity/services stores balance by user instead of creating one wallet per bay."
---

`@downcity/services` keeps balance by user, not by bay.

That means:

- user A spends inside bay X
- user A spends inside bay Y
- both consume the same shared balance

## Why this fits Downcity better

The typical Downcity shape is:

- one City serving multiple client bays
- the same user moving across those bays
- one shared AI account layer behind them

If balance is split per bay, both bay UX and operations become fragmented.

## Does bay still matter

Yes, but it belongs in ledger metadata rather than in the wallet primary key.

```ts
await balance.sub(ctx.user!.user_id, 10, {
  note: "agent chat",
  meta: {
    bay_id: ctx.bay?.bay_id,
    model_id: ctx.variant?.id,
  },
});
```

That keeps the wallet global while still preserving where the spend came from.
