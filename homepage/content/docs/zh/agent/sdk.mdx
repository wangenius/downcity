# Agent SDK

Agent SDK (`@downcity/agent`) 是执行 Agent 工作的核心运行时。其他所有东西 —— CLI、控制台、插件 —— 都建立在它之上或连接到它。

## 创建 Agent

```ts
import { Agent } from "@downcity/agent";

const agent = new Agent({
  id: "repo-helper",
  path: "/path/to/project",
  model: myModel,
  tools: [myTool],
  plugins: [myPlugin],
  env: { ... },
});
```

## 构造参数

- `id` — 稳定的 Agent 标识符。用于会话存储和运行时目录归属。
- `path` — 项目目录。Agent 从这里读取 `.env` 并保存 `.downcity` 运行数据。
- `model` — 实时模型实例。直接传入，不从目录查询。
- `tools` — 可供 Agent 使用的工具实例数组。
- `plugins` — 启动时加载的插件实例数组。
- `env` — 宿主提供的环境变量，与项目 `.env` 叠加。
- `config` — 宿主显式传入的 Agent 配置；SDK 不读取项目配置文件。
- `plugin_config` — 宿主 Plugin 配置持久化接口；未提供时持久化类 action 会明确失败。
- `start` — 可选的 HTTP 服务器配置（端口、host）。
- `execution` — 可选的执行绑定配置。

## 模型

模型是最重要的依赖。它可以通过以下方式提供：

- 直接传入构造函数
- 通过 `session.set({ model })` 按会话设置
- 当 CLI 管理项目时从 CLI 全局 DB 中的 Agent 配置解析

## 工具

工具是 Agent 在执行过程中可调用的函数。显式挂载：

```ts
const agent = new Agent({
  tools: [searchTool, calculatorTool],
});
```

## 插件

插件扩展 Agent 的能力，如聊天、任务、记忆和 Shell。显式挂载：

```ts
const agent = new Agent({
  plugins: [chatPlugin, memoryPlugin],
});
```

## 环境变量

Agent 在启动时合并两层环境变量：

1. 宿主提供的环境变量（来自 `new Agent({ env })` 或 CLI 运行时）。
2. 项目根目录 `.env`（叠加在宿主层之上）。

运行时元数据变量如 `DC_AGENT_ID`、`DC_AGENT_PATH`、`DC_SESSION_ID` 是为当前命令上下文注入的。

## 启动 Agent

```ts
await agent.start();
```

这会加载插件、初始化会话，并在配置了 HTTP 服务器时启动它。

## 停止 Agent

```ts
await agent.stop();
```

这会优雅地关闭插件、持久化状态并清理资源。

## Session 消息与流式订阅

Session 使用一条线性 Message 序列。顶层类型只有 `user`、`assistant`、`action` 和 `error`；文本、推理、工具调用与文件都保存在 `assistant.parts` 中。

```ts
const session = await agent.sessions.get("main");

const unsubscribe = session.subscribe((mutation) => {
  if (
    mutation.type === "assistant-part-delta" &&
    mutation.part_type === "text"
  ) {
    process.stdout.write(mutation.delta);
  }
});

const turn = await session.prompt({ query: "检查这个项目" });
const result = await turn.finished;
unsubscribe();
```

`subscribe()` 中的每条 Mutation 都已经成功写入当前 Session 的 `messages.jsonl`。`message_id`、`sequence` 和 `revision` 可用于把实时变化稳定地应用到同一条 Message；`commit_sequence` 用于检测遗漏和断线续传。

读取当前 Message 快照：

```ts
const page = await session.messages({ limit: 100 });

for (const message of page.items) {
  console.log(message.sequence, message.type);
}
```

断线后从最后一次提交继续读取：

```ts
const changes = await session.message_changes({
  after_commit_sequence: last_commit_sequence,
  limit: 200,
});

last_commit_sequence = changes.next_commit_sequence;
```

同一个 reducer 可以处理实时 `subscribe()` Mutation 与 `message_changes()` 返回值。调用方应按 `commit_sequence` 应用 Mutation，按 `message_id` 定位 Message，并按 Message `sequence` 展示；不要根据 role 数量或数组位置匹配流式消息。

HTTP 端点使用相同协议：

- `GET /api/sdk/sessions/:sessionId/messages`
- `GET /api/sdk/sessions/:sessionId/message-changes?after_commit_sequence=0`
