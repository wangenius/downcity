# `studio agent`

管理 Agent 生命周期和 registry 相关操作。

> 推荐阅读：新的 Agent 完整文档已经迁移到独立文档域，优先从 [Agent 总览](/zh/docs/agent/overview/what-is-agent) 和 [Agent CLI 总览](/zh/docs/agent/cli/overview) 开始。

在交互式终端中直接运行 `studio agent` 会进入 agent manager。需要脚本化调用时，继续使用明确子命令。

## 命令层级

一级命令组为：`city console`、`studio agent`、`studio plugin`。

## 用法

```bash
studio agent
studio agent <subcommand>
```

## 版本查看

- `studio -v` 只显示 city CLI 自身版本。
- `studio agent -v` 会同时显示当前 city 版本，以及这份 city 实际绑定的 `@downcity/agent` runtime 版本。

## 交互式管理器

交互式管理器可以完成：

- 查看已登记 agents
- 创建 agent 项目
- 启动 agent
- 选择 agent 时直接看到当前状态、名称、模型和聊天渠道关联
- 配置 agent id、模型绑定，或连接 city 级聊天渠道账号

它不会替代下面的子命令，只是把常用本地操作收进一个引导入口。

聊天渠道凭据属于 `studio chat` 中的 city 级 channel account。agent manager 只负责把当前 agent 连接到已有账号，即写入 `plugins.chat.channels.<channel>.channelAccountId`。如果发现 agent 绑定了不存在的 account，会自动清理这个悬空关联。

## 子命令

- `create`：创建/初始化 agent 项目
- `list`：列出已登记的 agent 项目
- `start`：启动 agent
- `stop`：停止 agent daemon
- `chat`：在终端中与 agent 对话，支持交互式和一次性模式
- `status`：查看 agent 状态
- `doctor`：诊断状态文件，并可修复僵尸 pid/meta
- `restart`：重启 agent

## 环境变量注入

每次执行 `studio` 时，CLI 会自动推导当前 agent 上下文，并在 `studio` 进程及其子进程中注入：

- `DC_AGENT_ID`：当前 agent runtime id
- `DC_AGENT_PATH`：当前 agent 绝对路径

> 注意：这是进程级注入，不会直接回写你当前 shell 会话的环境变量。
