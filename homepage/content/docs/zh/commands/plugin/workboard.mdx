# workboard 插件

`workboard` 是一个内建 plugin，用来把当前 agent 的对外状态整理成结构化快照，供 Console 和 UI 组件直接消费。

## 它会展示什么

- 当前是处于展开中、停留中还是待命中
- 最近是否发生过新的更新
- 对外公开的模糊线索
- 不泄漏内部上下文的状态摘要

## 适合什么场景

- 你想快速知道当前 agent 现在呈现出什么状态
- 你需要在 Console 里观察近期变化
- 你想把这种公开状态接到自定义 host 或 UI SDK 组件上

## 数据接口

当 agent 运行时，`workboard` 会提供：

- `GET /api/workboard/snapshot`

该接口返回结构化 JSON，但只包含对外安全的概览状态，不会直接暴露内部 session、service、task 或路径细节。

## Console 中的表现

Console 会通过 workboard 快照展示一个独立的 Agent 主视图：

- 在侧边栏的 Agent 导航里单独进入 `Workboard`
- 主区域展示当前公开状态和近期片段
- 右侧 inspector 展示选中状态与公开线索

## 说明

- `workboard` 默认随内建 plugin 一起提供
- 如果你关闭该 plugin，对应快照接口会不可用
- 当前版本优先使用确定性采集，不依赖额外模型推理
- 当前版本默认面向对外展示，因此只输出概览式公开态
