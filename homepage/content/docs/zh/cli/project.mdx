# 项目结构

Agent 项目是 CLI 能够识别和运行的目录。项目目录主要保存提示词、环境变量和运行时数据；Agent ID、模型绑定、启动参数和 plugin 参数由 CLI 全局 DB 管理。

## 目录布局

```text
my-project/
├── PROFILE.md         # Agent 角色定义
├── SOUL.md            # 长期原则
├── .env               # 环境变量
├── .downcity/         # 运行时数据
│   ├── sessions/      # 持久化的对话历史
│   ├── logs/          # 运行时日志
│   └── cache/         # 插件缓存和临时文件
└── src/               # 你的代码（可选）
```

## 关键文件

### PROFILE.md

定义 Agent 的可见角色。这包含在每个会话的系统上下文中。见 [Profile](/zh/docs/cli/profile)。

### SOUL.md

定义更深层的、更稳定的运行原则。也包含在系统上下文中。见 [Profile](/zh/docs/cli/profile)。

### .env

项目专属环境变量。这些叠加在宿主环境之上。Agent SDK 会自动读取此文件。

### .downcity/

运行时数据目录。这里的一切由 Agent SDK 和插件管理。你不应该直接编辑里面的文件，但可以检查日志和会话以进行调试。

## 多个项目

你可以在同一个机器上有多个项目。每个项目都是独立的：自己的全局 DB 配置、自己的会话、自己的运行时。CLI 操作当前工作目录中的项目。

## 配置存放在哪里

`city agent create` 会把 Agent 配置写入 CLI 全局 DB，而不是项目目录中的 `downcity.json`。旧项目里的 `downcity.json` 仍可被读取一次用于迁移；迁移后新的写入都进入全局 DB。
