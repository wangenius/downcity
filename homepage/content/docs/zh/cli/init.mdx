# 项目初始化

`downcity agent init` 命令创建一个新 Agent 项目，包含运行所需的最小文件。

## 基本用法

```bash
downcity agent init my-project
```

这会创建：

```text
my-project/
├── downcity.json
├── PROFILE.md
├── SOUL.md
├── .downcity/
└── .env
```

## 生成的文件

- `downcity.json` — 项目配置。见 [downcity.json](/zh/docs/cli/downcity-json)。
- `PROFILE.md` — Agent 的可见角色和输出风格。见 [Profile](/zh/docs/cli/profile)。
- `SOUL.md` — 更深层的长期运行原则。见 [Profile](/zh/docs/cli/profile)。
- `.downcity/` — 运行时数据目录（会话、日志、缓存）。
- `.env` — 项目环境变量。

## 选项

```bash
# 使用特定模板初始化
downcity agent init my-project --template chatbot

# 跳过创建默认文件
downcity agent init my-project --minimal

# 指定自定义目录
downcity agent init my-project --dir ./projects
```

## 初始化后

1. 编辑 `downcity.json` 设置你的 Agent ID 和模型绑定。
2. 编辑 `PROFILE.md` 和 `SOUL.md` 塑造行为。
3. 运行 `downcity agent start` 启动 Agent。
