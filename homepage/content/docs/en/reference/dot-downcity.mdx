# `.downcity/` Directory

`.downcity/` is Downcity's local runtime data directory. In most projects, it should not be committed to Git.

## Typical layout

```text
.downcity/
  logs/
  context/
  memory/
  channel/
  chat/
  task/
  config/
  schema/
  public/
  profile/
  .cache/
```

## Key folders

### `.downcity/logs/`

Daily JSONL logs: `<YYYY-MM-DD>.jsonl`.

### `.downcity/context/<contextId>/messages/`

Conversation source-of-truth (`messages.jsonl`) with compact metadata/archive.

### `.downcity/memory/`

Memory V2 storage:
- `MEMORY.md` for long-term facts/preferences.
- `daily/<YYYY-MM-DD>.md` for append-only daily memory.
- `index.sqlite` for local FTS index (rebuildable).

### `.downcity/chat/`

Chat metadata and history event stream.

### `.downcity/channel/meta.json`

Channel-target to `contextId` mapping snapshot (managed by chat service).

### `.downcity/task/`

Task definitions and per-run artifacts (`run.json`, `result.md`, etc.).

### Skills location

Skills are discovered from `.agents/skills` (project) and `~/.agents/skills` (user).

## Security guidance

- `.downcity/` may contain sensitive conversations and execution traces.
- For debugging, prioritize `logs`, `task`, and `context`.
