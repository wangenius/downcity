# HTTP API 参考

> 面向用户的运行时 API。更多细粒度命令建议优先使用 CLI（`sma ...`）。

## 基础接口

### `GET /health`

```json
{ "status": "ok", "timestamp": "2026-03-08T12:00:00.000Z" }
```

### `GET /api/status`

```json
{ "name": "shipmyagent", "status": "running", "timestamp": "..." }
```

### `POST /api/execute`

请求体：

```json
{
  "instructions": "帮我总结今天的改动",
  "chatId": "default"
}
```

## Service Runtime API

### `GET /api/services/list`

列出 service 运行状态。

### `POST /api/services/control`

```json
{ "serviceName": "chat", "action": "restart" }
```

`action` 支持：`start | stop | restart | status`。

### `POST /api/services/command`

```json
{
  "serviceName": "task",
  "command": "list",
  "payload": { "status": "enabled" }
}
```

## Extension Runtime API

### `GET /api/extensions/list`

列出 extension 运行状态。

### `POST /api/extensions/control`

```json
{ "extensionName": "voice", "action": "status" }
```

### `POST /api/extensions/command`

```json
{
  "extensionName": "voice",
  "command": "status",
  "payload": {}
}
```

## Action 直连路由

每个 service/extension action 还会暴露默认路由：

- `/service/<serviceName>/<actionName>`
- `/extension/<extensionName>/<actionName>`

例如：

- `POST /service/chat/send`
- `GET /extension/voice/status`

## 调试建议

先确认 runtime 已启动：

```bash
sma agent start
curl http://localhost:5314/health
```
