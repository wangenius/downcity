# Agent 生命周期

`new Agent(options)` 完成装配后会立即启动 plugin lifecycle 与 ActionSchedule。不存在 `agent.start()` 或 `agent.stop()`。

```ts
const agent = new Agent({
  id: "my-agent",
  path: process.cwd(),
  prepare_session: async (session) => await session.set({ model }),
});

await agent.ready();

const session = await agent.sessions.create();
const turn = await session.prompt({ query: "开始" });
await turn.finished;

await agent.dispose();
```

通常 Session 会在首次执行前等待 Agent 就绪，因此 `ready()` 是可选的显式检查点；在读取 plugin 状态、暴露网络服务前使用它最合适。`dispose()` 会关闭 plugin lifecycle、ActionSchedule 和 Shell 等后台资源，但不会关闭由 `AgentHTTP` 或 `AgentRPC` 创建的 transport，调用方需要分别关闭它们。
