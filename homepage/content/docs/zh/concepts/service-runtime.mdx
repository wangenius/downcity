# Service Runtime

Service 是 Agent Runtime 里的核心业务模块。

常见内置 service：

- `chat`
- `skill`
- `task`
- `memory`

## 1. Service 的核心职责

- 对外提供 command/action（CLI 与 HTTP 都可调用）。
- 通过注册机制告诉 agent：有哪些能力、每个 action 的入参与语义是什么。
- 承载业务流程与状态落盘（`.ship/*`）。

## 2. 调用方向（关键）

- `agent -> service`：agent 在执行阶段调度 service action。
- `service -> service`：service 之间可以互调。
- `service -> extension`：service 可以直接调用 extension 能力。
- `extension -/-> service`：extension 不负责反向编排业务流程。

例如：`chat` service 在语音附件场景会直接调用 `voice` extension 的转写 action。

## 3. 作用域与隔离

- Service 配置是 **每个 agent 项目独立** 的：`ship.json.services.*`。
- 不同 agent 可启用不同 service 组合、不同参数与不同运行环境。

## 4. 常用运维命令

```bash
city service list
city service status chat
city service restart task
city service command memory search --payload '{"query":"release"}'
```
