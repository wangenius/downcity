---
title: Issue `user_token`
description: Give the bay client a user-scoped credential that can call City.
---

`user_token` must be issued by a trusted environment. Your backend uses `Admin Visa` and asks City for a token based on `bay_id + user_id`.

```ts
import { Visa } from "@downcity/city";

const client = new Visa({
  role: "admin",
  city_url: "https://base.example.com",
  admin_secret_key: process.env.DOWNCITY_CITY_ADMIN_SECRET_KEY,
});

const bay = await client.bays.create({
  name: "Web App",
});

const user = await client.bays.tokens.apply({
  bay_id: bay.bay_id,
  user_id: "user_123",
  metadata: {
    plan: "pro",
  },
  ttl: "7d",
});
```

Return `user.user_token` and `bay.bay_id` to the bay client.

## What `metadata` is for

`metadata` carries business context such as plan or organization into hooks:

```ts
const action = service.action("generate", async (ctx) => {
  return generateSomething(ctx.input);
});

action.before(async (ctx) => {
  await quotaService.check({
    bay_id: ctx.bay?.bay_id,
    user_id: ctx.user?.user_id,
    plan: ctx.user?.metadata?.plan,
  });
});
```

This is how multiple bay clients share one auth and quota layer cleanly.
