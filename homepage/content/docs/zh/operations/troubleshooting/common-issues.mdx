# 常见问题

## 命令找不到（`city: command not found` / `downcity: command not found`）

```bash
npm config get prefix
echo $PATH
export PATH=$(npm config get prefix)/bin:$PATH
rehash
city --version
```

或直接用：

```bash
npx downcity --version
```

说明：

- `city` 与 `downcity` 是同一个可执行命令。
- 新版本安装后不应该再需要手动 `chmod +x`。
- 如果 `sudo city ...` 失败，但普通 `city ...` 正常，通常是 `sudo` 的 PATH 问题，不是安装失败。

## 端口冲突（5314 被占用）

```bash
lsof -nP -iTCP:5314 -sTCP:LISTEN
city agent doctor . --fix
city agent restart .
```

## 启动后无法调用 `city task/chat/...`

先确认 daemon 运行中：

```bash
city agent start
city service list
```

## LLM 配置错误

检查：

```bash
city console model list
city console config get model.primary
```

并确认：

1. `model.primary` 指向 `~/.downcity/downcity.db` 中存在的 model id
2. console 全局 SQLite 的 providers/models 记录完整可用
3. 如果是 Kimi（Moonshot），`provider.type` 必须是 `moonshot`
4. 如果是 Kimi（Moonshot），`baseUrl` 留空时默认走 `https://api.moonshot.ai/v1`
5. 如果是 Kimi（Moonshot），建议优先使用当前官方模型名，例如 `kimi-k2.5`
