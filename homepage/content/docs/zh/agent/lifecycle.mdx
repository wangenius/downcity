# Agent 生命周期

Agent 实例经历一个简单的生命周期：创建、启动、执行、停止。

## 1. 创建

用构造参数实例化 Agent：

```ts
const agent = new Agent({
  id: "my-agent",
  model: myModel,
  tools: [myTool],
  plugins: [myPlugin],
});
```

在这个阶段，Agent 已配置但未运行。没有加载插件，没有会话，也没有 HTTP 服务器在监听。

## 2. 启动

```ts
await agent.start();
```

启动期间，Agent 会：

- 验证项目目录（如果提供了 `path`）
- 加载 `PROFILE.md` 和 `SOUL.md`（如果在项目目录中找到）
- 初始化所有插件（调用它们的 `init` 钩子）
- 启动 HTTP 服务器（如果在 `start` 中配置了）

`start()` 之后，Agent 已准备好接收消息和任务。

## 3. 执行

启动后，Agent 可以通过多种表层进入：

- HTTP 请求（如果启用了 HTTP 服务器）
- 直接 SDK 调用（`agent.session(...)`）
- 插件驱动的事件（聊天消息、定时任务）

所有路径都收敛到同一个会话和执行核心。

## 4. 停止

```ts
await agent.stop();
```

关闭期间，Agent 会：

- 停止接收新请求
- 优雅地关闭所有活动会话
- 调用所有插件的 `destroy` 钩子
- 停止 HTTP 服务器（如果在运行）
- 持久化任何待处理状态

## 恢复

如果进程意外退出，下次 `start()` 会：

- 从 `.downcity/` 恢复已持久化的会话
- 重新初始化插件
- 恢复任何定时任务
