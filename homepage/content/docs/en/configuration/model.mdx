# Model Configuration

Model setup is split into two responsibilities:

1. The shared AI service manages the model catalog and provider-side details.
2. Agent project chooses `execution.modelId` in `downcity.json`.

## Workflow

1. Make sure a model is available through `town model list`.
2. Set `execution.modelId` in project.
3. Restart agent and validate one real invocation.

## Commands

```bash
town model list
town model use default --path .
town config set execution.modelId default
town agent restart .
```

## Failure cases

- missing `execution.modelId` target: agent startup fails.
- invalid provider/key: model invocation fails with auth/network errors.
