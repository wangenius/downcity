# 本地 Agent 的 HTTP 服务

本地 Agent 不只可以在当前进程里直接调用，也可以对外暴露服务端点。

## 推荐启动方式

现在更推荐直接使用统一生命周期入口：

```ts
const started = await agent.start({
  http: {
    host: "127.0.0.1",
    port: 15314,
  },
});

console.log(started.http?.baseUrl);
```

这样会在一个调用里统一启动：

- HTTP 服务
- plugin lifecycle（默认开启）

## 单独启动 HTTP

```ts
const started = await agent.start({
  http: {
    host: "127.0.0.1",
    port: 15314,
  },
});

console.log(started.http?.baseUrl);
```

当前默认端口是 `15314`。

如果你没有手动指定端口，这就是本地 SDK HTTP 的默认监听端口。

## `start()` 和不 `start()` 的区别

不调用 `agent.start()` 时：

- 你仍然可以直接 `await agent.createSession()`，再用 `session.prompt()` / `session.subscribe()`
- 适合把 Agent 当作当前进程里的本地库
- 不会启动 HTTP
- 不会启动 plugin 的后台生命周期

调用 `agent.start()` 之后：

- Agent 会开始对外暴露 HTTP 端点
- 默认会启动 plugins
- chat、task、ActionSchedule 和其他后台 loop 才会真正持续运行

可以把它理解成：

- 不 `start`：library mode
- `start`：long-lived runtime mode

## 什么时候用 HTTP

- 你想让另一个进程通过 `RemoteAgent` 调它
- 你想把本地 agent 暴露成一个轻量服务

它特别适合“执行方和调用方分进程”的结构。

## 启动前会发生什么

调用 `agent.start({ http: { ... } })` 时，SDK 默认会先确保当前 agent 的 plugins 已启动。

这样可以减少“端点已经起来了，但关键 plugin 还没 ready”的不一致状态。
