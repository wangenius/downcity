# Service Runtime

Service 是核心业务工作流模块。

典型内置 service：

- `chat`
- `skill`
- `task`
- `memory`

## Service 应该负责什么

- 用户主流程
- 运行时编排
- 上下文感知执行
- 稳定的 service actions

## Service 不应该负责什么

- 外部依赖的 install/check/config
- plugin 私有实现细节
- 可复用但应解耦的增强逻辑

## 推荐调用模型

- service -> capability
- capability -> plugin 实现
- plugin -> asset

例如 `chat` 不需要知道 voice 模型细节，只调用 `audio.transcribe` 即可。
