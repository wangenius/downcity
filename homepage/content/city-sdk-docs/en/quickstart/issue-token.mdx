---
title: Issue `user_token`
description: Give the town client a user-scoped credential that can call City.
---

`user_token` must be issued by a trusted environment. Your backend uses `Admin City` and asks City for a token based on `town_id + user_id`.

```ts
import { City } from "@downcity/city";

const client = new City({
  role: "admin",
  city_url: "https://base.example.com",
  admin_secret_key: process.env.DOWNCITY_CITY_ADMIN_SECRET_KEY,
});

const town = await client.towns.create({
  name: "Web App",
});

const user = await client.towns.tokens.apply({
  town_id: town.town_id,
  user_id: "user_123",
  metadata: {
    plan: "pro",
  },
  ttl: "7d",
});
```

Return `user.user_token` and `town.town_id` to the town client.

## What `metadata` is for

`metadata` carries business context such as plan or organization into hooks:

```ts
const action = service.action("generate", async (ctx) => {
  return generateSomething(ctx.input);
});

action.before(async (ctx) => {
  await quotaService.check({
    town_id: ctx.town?.town_id,
    user_id: ctx.user?.user_id,
    plan: ctx.user?.metadata?.plan,
  });
});
```

This is how multiple town clients share one auth and quota layer cleanly.
