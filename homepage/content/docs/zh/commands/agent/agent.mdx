# `city agent`

管理 Agent 生命周期和 registry 相关操作。

## 命令层级

一级命令组为：`city console`、`city agent`、`city service`、`city plugin`。

## 用法

```bash
city agent <subcommand>
```

## 子命令

- `create`：创建/初始化 agent 项目
- `list`：列出已登记的 agent 项目
- `start`：启动 agent
- `chat`：在终端中与 agent 持续对话
- `quest`：向 agent 的 Console 主会话发送一轮消息
- `status`：查看 agent 状态
- `doctor`：诊断状态文件，并可修复僵尸 pid/meta
- `restart`：重启 agent

## 环境变量注入

每次执行 `city` 时，CLI 会自动推导当前 agent 上下文，并在 `city` 进程及其子进程中注入：

- `DC_AGENT_PATH`：当前 agent 绝对路径
- `DC_AGENT_NAME`：当前 agent 名称（优先 `downcity.json.name`，否则目录名）

> 注意：这是进程级注入，不会直接回写你当前 shell 会话的环境变量。
