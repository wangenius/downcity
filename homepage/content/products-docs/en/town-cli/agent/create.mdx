# `town agent create`

Create/initialize a Downcity agent project in the target path.

> This page remains as a command entry point. For the fuller project initialization guide, see [Create Your First Agent](/en/docs/agent/getting-started/create-your-first-agent) and [CLI / create](/en/docs/agent/cli/lifecycle/create).

## Usage

```bash
town agent create [path]
```

## Arguments

- `path` (optional): project path. Defaults to current directory.
- The interactive default `Agent id` is derived from the directory name and normalized to snake_case (default only; you can edit it).
- The interactive flow reads available execution targets from the connected City context and writes the selected ID into `downcity.json.execution.modelId`.
- Created projects keep execution binding in `downcity.json.execution` instead of splitting it across legacy model and session fields.

## Options

- `-f, --force [enabled]`: allow overwriting existing `downcity.json` (dangerous)

## Example

```bash
# Initialize in current directory
town agent create .

# Initialize in another directory
town agent create ./my-agent-project
```
