# Skills

Downcity 兼容 SKILL.md 形态的 skills。

## 默认扫描目录

- 项目级：`.agents/skills`
- 用户级：`~/.agents/skills`
- 额外目录：`downcity.json.services.skills.paths`

## 推荐工作流

1. `city skill list`（先看本地已安装/已学会 skills）
2. `city skill lookup <name>`（命中后直接读取并执行）
3. 若 `list` 中没有目标 skill，再执行 `find -> install -> lookup`

示例：

```bash
city skill list
city skill lookup playwright

# 仅当缺失时再安装
city skill find playwright
city skill install owner/repo@playwright
city skill lookup playwright
```

## 关键原则

- 使用 skill 前先 `lookup`，不要只凭记忆执行。
- `list` 中列出的 skills 都是已安装可用，不需要重复 `install`。
- 任务匹配时，优先使用已安装 skills。
- Agent 运行时中，`lookup` 回包仅返回状态；`SKILL.md` 正文会以 `<skill>...</skill>` user message 注入。
- `SKILL.md` 中的 `allowed-tools` 约束应严格遵守。
- 运行时会自动发现新增/变更的 skill 文件。

## 相关配置

```json
{
  "services": {
    "skills": {
      "paths": [".agents/skills"],
      "allowExternalPaths": false
    }
  }
}
```
