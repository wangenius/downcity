If you already know which package you want to inspect, this section is the fastest route.

Downcity keeps the package graph intentionally small: start with `@downcity/infra` for the InfraRuntime itself, add `@downcity/conduit` for product calls, create a Drizzle db in your host runtime, and add official services only when the product needs them.

That order maps to four clear roles:

- `infra` is the Kernel for the InfraRuntime itself
- `services` are Capabilities that add official behavior
- `conduit` and `city terminal` are Interfaces for product and operator entry
- `blocks/*` is the Composition layer that assembles runnable examples

If you only care about npm packages, the three most important pieces to remember first are `infra`, `services`, and `conduit`.

## Recommended order

1. Start with [@downcity/infra](/en/docs/city-infra/packages/infra) to understand the InfraRuntime itself
2. Then read [@downcity/conduit](/en/docs/city-infra/packages/conduit) to see how products call it
3. If you want terminal tooling, read [@downcity/city](/en/docs/city-infra/packages/city-terminal)
4. Add `@downcity/services` when your product needs auth, wallet, usage, or Stripe payment flows

## If you only want the integration picture first

Read the package layer as this chain:

```txt
@downcity/infra
  -> provides the InfraRuntime Kernel
@downcity/services
  -> adds official capabilities into InfraRuntime
@downcity/conduit
  -> uses UserClient and AdminClient to call those capabilities
```

So `@downcity/conduit` is not only for AI calls.

It also unifies:

- AI service calls
- custom service calls
- official service calls
- trusted-side product, token, and env management calls

## One responsibility map

```txt
Kernel        -> @downcity/infra
Capabilities  -> @downcity/services
Interfaces    -> @downcity/conduit + @downcity/city
Composition   -> blocks/node + blocks/edge
```

## Typical combinations

### Minimal AI product

- `@downcity/infra`
- `@downcity/conduit`

### Minimal registration and login

- `@downcity/infra`
- `@downcity/conduit`
- `@downcity/services`

### Usage metering

- `@downcity/infra`
- `@downcity/conduit`
- `@downcity/services`

### Global wallet and topup orders

- `@downcity/infra`
- `@downcity/conduit`
- `@downcity/services`

### One-time Stripe topups

- `@downcity/infra`
- `@downcity/conduit`
- `@downcity/services`
