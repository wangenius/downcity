# 上下文工程

ShipMyAgent 把“上下文”拆成可组合、可落盘的几层数据：

1. `PROFILE.md` / `SOUL.md` / `USER.md`
2. `ship.json` 配置
3. `.ship/context/<contextId>/messages/messages.jsonl` 对话历史
4. `.ship/context/<contextId>/memory/Primary.md` 长期记忆

## 为什么这样设计

- 历史记录可审计，不依赖隐式内存。
- 超长对话可 compact，保留最近消息与摘要。
- 各 `contextId` 相互隔离，减少串话。

## 实践建议

- `PROFILE.md` 写清楚边界、目标、风格。
- 用 `contextId` 区分不同渠道/用户/业务流。
- 定期查看 `.ship/logs/<date>.jsonl` 与任务 run 目录。
