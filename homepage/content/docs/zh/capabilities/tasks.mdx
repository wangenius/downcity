# Tasks

Task 是可复用、可调度的 Agent 工作定义。

它不只是“延迟发送一条聊天消息”。一个 task 有自己的定义、调度、运行产物和结果投递语义。

## Task 负责什么

一个 task 定义描述：

- 要执行什么
- 什么时候执行
- 结果应该回到哪个 session
- 当前是 enabled、paused，还是 disabled

task runtime 会把运行产物写进项目 `.downcity/` 状态树。这样定时工作始终归属于拥有它的项目 Agent。

## 结果投递

`sessionId` 决定任务结果返回哪里。

它不是认证身份，而是结果回到既有 session 或 chat context 的投递路线。

## 时间语义

task schedule 由 runtime 进程解释。cron 表达式因此使用 runtime 所在环境的时区。

一次性 time trigger 应该显式带时区，避免存储后的调度语义变得模糊。
