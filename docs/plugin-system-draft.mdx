# Plugin System Draft（已过时）

这个文件保留为历史记录，但其中基于 `capability` 的插件系统设计已经不再代表当前实现。

当前已经落地的新模型是：

- `service` 拥有生命周期与业务流程
- `plugin` 是被动扩展层，不拥有独立生命周期
- 扩展通过 service 定义的 plugin 点完成
- plugin 点统一分为 `pipeline`、`guard`、`effect`、`resolve`

请以以下内容为准：

- `packages/downcity/src/types/Plugin.ts`
- `packages/downcity/src/console/plugin/HookRegistry.ts`
- `packages/downcity/src/console/plugin/PluginRegistry.ts`
- `packages/downcity/src/services/chat/runtime/PluginPoints.ts`
- `homepage/content/devdocs/zh/architecture/system-logic.mdx`
- `homepage/content/devdocs/zh/package/runtime-and-services.mdx`

如果你是在继续设计或迁移插件系统，请不要再参考旧的 capability 草稿内容。
