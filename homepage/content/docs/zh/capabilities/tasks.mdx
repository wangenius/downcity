# Tasks

任务定义存放在 `.downcity/task/<taskId>/task.md`。

> 想看完整执行链路（状态机、校验规则、run 目录语义、排障）：  
> 参考 [Task Service 详细说明](/zh/docs/services/task/task-service)。

## 最小任务示例

```markdown
---
title: 每日仓库巡检
when: "0 9 * * 1-5"
description: 运行测试并总结异常
contextId: telegram-chat-123456
status: enabled
kind: agent
---

请执行任务并把最终输出正文写入 `output.md`，并在 `result.md` 写摘要。
```

## 执行结果通知

- 任务完成后只会写入 `.downcity/task/<taskId>/<timestamp>/` 产物。
- 不会自动把结果发送到 chat channel。
- `contextId` 仅作为任务执行上下文标识使用。

## 目录结构

```text
.downcity/task/
  <taskId>/
    task.md
    <timestamp>/
      run.json
      output.md
      result.md
      dialogue.md
```

## 常用命令

```bash
city task list
city task create --title "Daily check" --description "Run tests" --context-id telegram-chat-123 --when "0 9 * * 1-5" --activate
city task run daily-check
city task update daily-check --when "@manual"
city task disable daily-check
```

## 状态说明

- `enabled`：参与调度
- `paused`：保留定义，不调度
- `disabled`：禁用

## 调度生效条件

任务调度依赖运行中的 runtime（建议 `city agent start`）。
