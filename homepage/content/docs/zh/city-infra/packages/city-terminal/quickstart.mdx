`@downcity/city` 是给 InfraRuntime 管理者使用的终端工具，不是产品运行时 SDK。

## 什么时候应该先打开 city terminal

- 你想快速确认数据库里到底写了什么
- 你在本地初始化 InfraRuntime
- 你怀疑某个 service 没有正常建表或写数据
- 你不想先写临时脚本，只想人工查看模型、表、schema

## 安装

```bash
npm i -g @downcity/city
```

如果你正在当前仓库里迭代 `city terminal`，可以直接运行：

```bash
npm run build:city terminal
```

这个命令会先构建本地 `packages/city-terminal`，然后自动刷新你机器上的全局 `downcity` 命令。构建通过后，下一次执行 `downcity` 时就是当前仓库这份代码。

如果你已经装过 `downcity`，之后也可以直接运行：

```bash
downcity update
```

在仓库里运行时，它会优先用当前 workspace 重新 build 并刷新全局安装；不在仓库里时，则会更新到 npm 上的最新版本。

## 第一次运行

```bash
downcity
```

默认会先显示当前 CLI 版本，再进入交互式面板。这通常是第一次排查时最直接的入口。

如果你还没有配置任何 server，CLI 不会给你塞一个默认地址，而是会先要求你添加 server。

## 第一次先添加 server

现在 `city terminal` 把 server 当成一等资源管理。第一次启动时，需要先填写：

- `base_url`
- `admin_secret_key`

保存后，这个 server 会自动成为当前 active server。之后你再从顶层菜单进入：

- `Admin`
- `User`
- `Manage Servers`

## 之后怎么管理 server

以后不再用“切换当前 server”这种临时入口，而是统一通过 `Manage Servers`：

- `List Servers`
- `Add Server`
- `Select Active Server`
- `Edit Server`
- `Remove Server`

## 一个最常见的上手流程

```bash
npm i -g @downcity/city
downcity
```

这三步通常就足够你做第一轮确认：

1. 先添加一个可用的 InfraRuntime server
2. 当前 active server 配置已经保存
3. 你可以继续进入 Admin / User 流程

## CLI 最适合解决什么问题

### 场景一：人工看数据

当你的问题是：

- “数据到底有没有写进去？”
- “usage 事件到底有没有记录？”
- “模型配置是不是数据库里就错了？”

CLI 往往比写脚本更快。

### 场景二：初始化或补录模型

```bash
downcity
```

适合：

- 刚建好 InfraRuntime
- 调整模型目录
- 本地验证新模型配置

这些动作现在都从交互式菜单进入，而不是依赖默认 server。

### 场景三：手动发起 Stripe 充值

如果你的 InfraRuntime 启用了 `payment.stripe`，CLI 现在也可以直接：

- 读取 Stripe 服务暴露的 env requirements
- 以 Admin 身份补齐 `STRIPE_SECRET_KEY`、`STRIPE_WEBHOOK_SECRET`
- 在 `Admin -> Payment` 里直接看到当前 server 对应的 Stripe webhook endpoint
- 以 User 身份创建 topup
- 继续创建 Stripe Checkout，并自动打开浏览器支付

如果你没有单独配置 `STRIPE_SUCCESS_URL` / `STRIPE_CANCEL_URL`，Stripe 服务会优先基于
`DOWNCITY_INFRA_BASE_URL`，再回退到当前请求 origin 自动生成默认 success / cancel 页面。

Stripe webhook endpoint 本身则固定是：

- `{base_url}/v1/payment.stripe/webhook`

## 它不适合什么

- 不适合嵌进产品代码
- 不适合替代管理后台逻辑
- 不适合做长期自动化流程

这类需求更适合交给 `AdminClient` 或你自己的后端脚本。

## 常用入口

这一页最常见的 CLI 入口就是：

- `downcity`
- `downcity update`
- `Manage Servers`
- `Admin`
- `User`

## 常见误解

### CLI 不是给终端用户用的

它是给 InfraRuntime 管理者、开发者、运维者用的，不是产品用户入口。

### CLI 不是 AdminClient 的替代品

CLI 适合“人坐在终端前的操作”；`AdminClient` 适合“程序里长期存在的可信调用”。

### CLI 不只是在看表

它也是非常好的初始化、排查和模型维护入口。
