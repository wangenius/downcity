# `city service`

`city service` is the console-level catalog entry for services.

Use it when you want to:

- inspect which services are registered
- inspect a service's static capability surface
- use advanced agent-targeted lifecycle or command forwarding

## Command hierarchy

- Top-level command: `city service`
- Concrete service commands: `city chat` / `city task` / `city memory`

These concrete commands are the normal agent-facing runtime entry points.

## Usage

```bash
city service <subcommand>
```

## Subcommands

- `list`
- `status <serviceName>`
- `start <serviceName>`
- `stop <serviceName>`
- `restart <serviceName>`
- `command <serviceName> <command> --payload <json>`
- `schedule list`
- `schedule info <jobId>`
- `schedule cancel <jobId>`

## Common Scheduled Execution

All service action commands now support common scheduling options:

- `--delay <ms>`: delay execution
- `--time <time>`: schedule execution (Unix sec/ms timestamp or timezone-aware ISO datetime)

Notes:

- scheduled jobs are persisted in the current agent project at `.downcity/schedule.sqlite`
- unfinished jobs are restored after agent restart
- `--delay` and `--time` cannot be used together
- this is for one-shot delayed/scheduled execution; use `task` service for cron and longer workflows

For detailed behavior, recovery rules, and HTTP examples, see:

- [`Service Action Scheduling`](/en/docs/commands/service/scheduling)

## Notes

- `city service list` and `city service status` are static console-level catalog views.
- They do not require `--path`, `--agent`, or a running agent.
- `city service start/stop/restart/command` are advanced commands for a specific agent target.
- For day-to-day runtime operations, prefer the concrete command roots:
  - `city chat ...`
  - `city task ...`
  - `city memory ...`
- If you need to target a specific agent from `city service start/stop/restart/command`, use `--agent <name>` or `--path <path>`.

Typical flow:

```bash
city service list
city service status memory
city memory status
city memory restart
```
