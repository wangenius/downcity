# `town stop`

Stop Town runtime and managed agents.

The current implementation also tries to clean up detached orphan processes:

- Town runtime (`run`) processes that are still alive without a pid file
- legacy Console UI runtime processes left by older versions
- agent daemons (`agent start --foreground true`) that are still alive without a pid file

On macOS and Linux, `town stop` stops the matching process group first. ACP executors, shell sessions, and watch child processes started by an agent receive the same stop signal, which prevents orphaned children from continuing to consume memory after the parent process exits.

## Usage

```bash
town stop
```

## Related commands

- [`town start`](/en/products-docs/town-cli/start)
- [`town status`](/en/products-docs/town-cli/status)

## Output Behavior

- It cleans up legacy runtime processes, stops managed agents, and then stops Town runtime.
- If runtime is already stopped, it still sweeps detached orphan processes and prints the current state.
