# 安全与权限总览

Downcity 的安全模型分为 3 层：

1. 运行隔离：一个 agent runtime 绑定一个 `projectRoot`，默认不会跨项目串上下文。
2. 权限控制：通过 `ship.json.permissions` 控制仓库写入与命令执行。
3. 数据保护：模型 provider 的 `apiKey` 以加密形式存储在 `~/.ship/ship.db`。

## 你需要重点关注

1. 不要在终端残留错误的 `DC_SERVER_HOST/DC_SERVER_PORT`，避免把命令路由到错误 runtime。
2. `permissions.shell.deny` 默认应保留高风险命令（例如 `rm`）。
3. 生产环境优先使用 `DC_MODEL_DB_KEY` 管理密钥，不要把明文 key 写入仓库。
