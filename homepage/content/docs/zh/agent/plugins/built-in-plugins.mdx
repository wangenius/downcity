# 内建 Plugin 导览

当前 Downcity 内建 plugin 共有六个：

- `auth`
- `skill`
- `web`
- `asr`
- `tts`
- `workboard`

这六个 plugin 是当前代码里的内建清单，也是你理解 plugin 体系最重要的一组参考实现。

## 这组文档怎么读

如果你是第一次接触 Downcity plugin，最推荐这样读：

1. 先看 [内建 Plugin 总览](/zh/docs/agent/plugins/builtins/overview)
2. 再按自己关心的能力看单独页面
3. 读到某个运行机制不理解时，再跳去专题页

推荐顺序：

1. [skill Plugin](/zh/docs/agent/plugins/builtins/skill)
2. [web Plugin](/zh/docs/agent/plugins/builtins/web)
3. [asr Plugin](/zh/docs/agent/plugins/builtins/asr)
4. [tts Plugin](/zh/docs/agent/plugins/builtins/tts)
5. [workboard Plugin](/zh/docs/agent/plugins/builtins/workboard)
6. [auth Plugin](/zh/docs/agent/plugins/builtins/auth)

## 每个内建 Plugin 分别解决什么问题

### `auth`

负责 chat 入口授权、用户与群聊观测、以及用户角色解析。

它不是一个“可有可无的能力插件”，而是当前平台规则的一部分。

### `skill`

负责 skills 的发现、安装、列出与读取。

如果你想让 agent 学会新的 skill，通常会先经过它。

### `web`

负责选择联网 provider，并把联网能力接到 agent 上。

它本身不是搜索引擎，也不是浏览器执行器，而是 `web-access` 与 `agent-browser` 的适配层。

### `asr`

负责语音转写。

它既可以显式转写一个本地音频文件，也可以在 chat 入站链路里自动把语音附件增强成文本。

### `tts`

负责文本转语音。

它会把文本生成本地音频文件，并返回可复用的音频文件标签，方便后续发送到渠道。

### `workboard`

负责暴露运行时工作快照。

它更像观测面和控制面能力，而不是一个主要靠 action 调用的业务插件。
