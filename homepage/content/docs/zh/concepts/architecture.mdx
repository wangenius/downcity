# 架构总览

ShipMyAgent 现在可以直接按两大主干理解：

- `console`：全局控制与运维中台
- `agent`：项目执行运行时

在这两大主干内部，`service` 与 `extension` 作为能力层被统一调度。

## 分层结构

1. **Console 层（全局）**
- 命令：`sma console ...`、`sma console config ...`、`sma service ...`、`sma extension ...`
- 职责：全局进程管理、运维入口、模型与运行配置治理

2. **Agent 层（项目）**
- 命令：`sma agent create/on/off/status/restart ...`
- 职责：加载项目上下文，执行模型与工具调用，落盘运行状态

3. **Capability 层（运行时能力）**
- `service`：核心业务域（`chat` / `skill` / `task` / `memory`）
- `extension`：共享扩展域（如 `voice`）

4. **State 层（可审计落盘）**
- 路径：`.ship/context`、`.ship/logs`、`.ship/.cache`、`.ship/tasks`、`.ship/debug`
- 目标：可回放、可定位、可审计

## 架构原则

- 控制与执行分离：`console` 负责管控，`agent` 负责执行。
- 核心与扩展分离：`service` 是核心，`extension` 是增强。
- 命令入口统一：同一套能力可通过 CLI/HTTP 访问。
- 配置变更前置：模型等关键配置先通过 console 入口管理，再进入 agent 执行。
