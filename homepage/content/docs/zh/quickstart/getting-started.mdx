# 快速开始

Downcity 有多个用户表层。这个页面是入口地图，不是命令教程。

你可以按目标选择入口：

- 想运行本地 Agent、管理项目 Agent、使用内建 plugin 命令，看 [Town CLI](/zh/products-docs/town-cli)。
- 想创建、管理或部署 City 项目，看 [City CLI](/zh/city-sdk-docs/packages/cli)。
- 想把 Agent 能力嵌入自己的 Node.js 应用，看 [Agent SDK](/zh/agent-sdk-docs)。
- 想构建 service、模型供应、计费、usage 或 admin 表层，看 [City SDK](/zh/city-sdk-docs)。
- 想理解内建 plugin 行为或编写 plugin 集成，看 [Agent Plugins Docs](/zh/plugins-docs)。
- 想查 Town CLI、Town Console、Chrome Extension 的产品用法，看 [Products Docs](/zh/products-docs)。

## 推荐阅读顺序

先理解模型：

1. [Downcity 的整体关系](/zh/docs/concepts/project-logic-overview)
2. [Agent 总览](/zh/docs/agent/overview/what-is-agent)
3. [配置模型](/zh/docs/configuration/overview)

再选择具体使用表层：

- 本地 CLI 使用：[Town CLI](/zh/products-docs/town-cli)
- 项目 Agent 工作流：[Agent CLI 产品文档](/zh/products-docs/town-cli/agent/agent)
- Plugin 工作流：[Plugin CLI 产品文档](/zh/products-docs/town-cli/plugin/plugin)
- City 管理命令：[City CLI](/zh/city-sdk-docs/packages/cli)
- SDK 集成：[Agent SDK](/zh/agent-sdk-docs)

## 这个 Space 解释什么

Downcity docs 解释为什么系统要拆成 Town、City、Agent、plugins、services、permissions 和 configuration 这些层。

具体命令参数、示例和一步步操作指南，放在产品文档或 SDK 文档里。
