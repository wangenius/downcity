Downcity uses `admin_secret_key` to manage City, and `user_token` to let end-user clients call services.

## user_token

`user_token` is the call credential used by the end user or product client. `Admin City` requests it from City using `town_id + user_id`.

- `town_id`
- `user_id`
- `metadata`
- `ttl`

The client only carries `user_token` and `town_id`. It never touches provider keys or `admin_secret_key`.

## admin_secret_key

`DOWNCITY_CITY_ADMIN_SECRET_KEY` is used to manage City: create towns, manage env, and issue `user_token` for users under a given town. City generates it automatically and writes it into `.env` on first startup.

```ts
const client = new City({
  role: "admin",
  city_url: "https://base.example.com",
  admin_secret_key: process.env.DOWNCITY_CITY_ADMIN_SECRET_KEY,
});

const town = await client.towns.create({
  name: "Chrome Extension",
});
```

## Multi-town reuse

Many clients can belong to the same town. When issuing a token, write `town_id` explicitly:

```ts
const user = await client.towns.tokens.apply({
  town_id: town.town_id,
  user_id: "user_123",
  metadata: {
    plan: "pro",
  },
  ttl: "7d",
});

console.log(user.user_token);
```
