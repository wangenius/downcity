# City 用户身份

Federation 统一签发用户身份，City 客户端使用 `user_token` 访问 Federation 或产品后端。可信产品后端使用 `bureau_token` 在线识别用户。一个 Federation 可以服务多个产品，每个产品由 `city_id` 隔离。

## 身份边界

- **Federation** — 保存 Ed25519 私钥并签发 `user_token`
- **City** — 保存 `federation_url + user_token` 并发起用户请求
- **Bureau** — 保存 `federation_url + bureau_token`，供产品后端识别用户或执行被授权的管理操作

## 认证流程

1. 用户登录时选择产品 `city_id`
2. Federation 使用私钥签发绑定该 `city_id` 的 `user_token`
3. 用户通过 `new City({ federation_url, user_token })` 调用 Federation
4. 用户携带同一个 Bearer Token 调用产品后端
5. 产品后端的 `Bureau` 将 `bureau_token + user_token` 发送给 Federation
6. Federation 从 Bureau Token 记录读取 `city_id`，验证用户 Token 并查询 Accounts 当前用户记录

```ts
const city = new City({
  federation_url: "https://fed.example.com",
  user_token,
});
```

产品后端：

```ts
const bureau = new Bureau({
  federation_url: "https://fed.example.com",
  bureau_token: process.env.DOWNCITY_BUREAU_TOKEN!,
});

const identity = await bureau.identify(request);
```

`Bureau` 必须配置 `bureau_token`，但不配置 `city_id`。Federation 在签发 Bureau Token 时把它绑定到 City，并且数据库只保存 Token hash。`identify()` 每次在线查询 Federation，因此 Token 撤销、City 暂停和 Accounts 用户状态会立即生效。

Federation 仍会发布 Ed25519 公钥，供 Federation 自身和其他协议验证用户 Token：

```text
/.well-known/downcity.json
/.well-known/jwks.json
```

管理后端使用同一个 `Bureau` 类，区别仅在 `bureau_token` 是否拥有 `federation:admin` capability。

## 渠道账号

聊天平台的机器人凭证存储在 `~/.downcity/downcity.db` 中，并通过 ID 从项目引用：

```json
{
  "plugins": {
    "chat": {
      "channels": {
        "telegram": {
          "channelAccountId": "telegram-main"
        }
      }
    }
  }
}
```
