# `shell` Plugin

`shell` 是 `@downcity/shell` 包对应的内建 plugin adapter。

它提供：

- one-shot shell 执行
- 持久 shell session
- status、read、write、wait、close 这些 action

## 主要形状

- `lifecycle`
- `actions`

## 它主要解决什么问题

如果你需要这些能力，就应该关注 `shell`：

- 由 `@downcity/shell` 实现的 shell session
- per-agent 的 shell 状态边界
- 一套围绕 shell 执行的 runtime action 层

## 用户怎么使用

当前没有根命令 `town shell`。使用 plugin command 形式：

```bash
town plugin command shell exec --payload '{"cmd":"pwd"}'
town plugin command shell start --payload '{"cmd":"pnpm test","inlineWaitMs":1000}'
town plugin command shell status --payload '{"shellId":"<shellId>"}'
town plugin command shell read --payload '{"shellId":"<shellId>","fromCursor":0}'
town plugin command shell write --payload '{"shellId":"<shellId>","chars":"q\n"}'
town plugin command shell wait --payload '{"shellId":"<shellId>","timeoutMs":5000}'
town plugin command shell close --payload '{"shellId":"<shellId>","force":true}'
```

`town plugin command ...` 面向托管 agent runtime。要在当前项目内直接本地执行，可以使用高级形式：`town plugin action shell <action> --payload '<json>'`。

## SDK action 用法

可以直接实例化，并按需覆盖 runtime 参数；不传时使用默认值：

```ts
import { ShellPlugin } from "@downcity/plugins";

const shell = new ShellPlugin({
  maxActiveShells: 64,
  defaultExecTimeoutMs: 60_000,
  defaultWaitTimeoutMs: 10_000,
  cleanupDelayMs: 10 * 60 * 1000,
});
```

```ts
await agent.plugins.runAction({
  plugin: "shell",
  action: "exec",
  payload: {
    cmd: "pwd",
  },
});
```

action 名称是 `exec`、`start`、`status`、`read`、`write`、`wait`、`close`。

## 关键语义

- shell sessions 由 `@downcity/shell` 实现；`ShellPlugin` 负责把它绑定到某个 agent 实例
- constructor 参数都是可选 runtime 调参；默认值保持内建行为不变
- sandbox backend 也在 `@downcity/shell` 中。每条命令都会进入当前 agent sandbox 执行，持久目录是 `.downcity/sandbox`
- `shellId` 只标识 `.downcity/shell/<shellId>` 下的会话日志与进程生命周期，不决定 sandbox HOME、cache 或权限
- sandbox 命令默认可读写项目目录、可联网，并共享 `.downcity/sandbox` 作为 `HOME`
- sandbox 命令不能写真实用户 HOME，也不能写 `/usr/local`、`/opt/homebrew`、`/System` 等宿主系统目录
- `start` lifecycle 负责绑定 runtime
- `stop` lifecycle 会关闭全部 shell sessions，并清空内存状态

## 公开性说明

`ShellPlugin` 仍然可以从 `@downcity/plugins` 包根直接导入。命令执行、sandbox backend、session lifecycle 与 approval primitives 位于 `@downcity/shell`。
