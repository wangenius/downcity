# Codex 安装与入门教程

> 目标：从 0 到 1 学会把 Codex 用在日常工作里

# Codex 是什么

Codex 是 OpenAI 的 **coding agent**，可以读代码、改代码、运行代码，也可以在 IDE 里和你并排协作，或者把较长任务委托到云端去做。官方文档明确这样描述它。

你可以把它理解成：

> 不是一个只会聊天的 AI，
> 而是一个可以直接在工作区里动手做事的执行型助手。

---

# 安装前准备

这一部分很重要，尤其是你说的**网络环境**。

## 准备网络环境

需要科学上网。这是实操里最容易踩坑的点。很多人以为：

> 浏览器能打开网页 = Codex 能用

其实不一定。

:::note
需要注意： VPN 要特别检查“终端是否可用”
:::

执行下列命令，如果 显示 HTTP/2 200 那说明是可以的。

```bash
curl -I https://www.google.com
```

## Intel芯片

### 下载 VS Code

[https://code.visualstudio.com/](https://code.visualstudio.com/)

### 安装 Codex 插件

在 VS Code 中：

1. 打开 Extensions
2. 搜索 **Codex**
3. 找到 OpenAI 的 Codex 扩展
4. 点击安装
5. 安装完成后重启 VS Code（如果没显示出来）

---

### 登录 Codex

- 用 **ChatGPT 账号登录**

---

#### 登录失败时先别怀疑插件，先查网络

这里是实操中最重要的一句。

如果出现这些情况：

- 登录页打不开
- 登录完没有反应
- 插件一直转圈
- 发消息后无响应
- 扩展市场异常
- terminal 请求经常超时

优先检查：

1. VPN 是否真的开启
2. terminal 是否也走了代理
3. VS Code 是否被代理覆盖到
4. 切换一下代理模式或节点
5. 重启 VS Code 再试

# 使用

这一部分讲最常见的 3 个使用方式：

1. 在当前项目里让 Codex 帮你看代码、改代码
2. 使用 Skills 让它按固定套路执行任务
3. 把重复任务整理成自动化流程

如果你是第一次用，建议你先按这个顺序来：

**先学会在一个工作空间里提清楚需求，再学 Skills，最后再做自动化。**

---

## 工作空间

工作空间可以理解成：

> Codex 当前能看到、能操作的项目目录。

你打开哪个项目，Codex 就主要围绕这个项目来工作。

你可以放任何内容在这个文件目录下

## Skills

Skills 可以理解成：

> 一套固定任务模板，告诉 Codex “遇到这类事情，你按这个套路做”。

[https://skills.sh/](https://skills.sh/)

---

## 自动化任务

自动化任务适合处理这类事情：

> 你不想每次手动说一遍，但又希望它按固定流程重复执行。
