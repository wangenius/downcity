# HTTP API 参考

> 面向用户的运行时 API。更多细粒度命令建议优先使用 CLI（`city ...`）。

## 基础接口

### `GET /health`

```json
{ "status": "ok", "timestamp": "2026-03-08T12:00:00.000Z" }
```

### `GET /api/status`

```json
{ "name": "downcity", "status": "running", "timestamp": "..." }
```

### `POST /api/execute`

请求体：

```json
{
  "instructions": "帮我总结今天的改动",
  "chatId": "default"
}
```

## Service Runtime API

### `GET /api/services/list`

列出 service 运行状态。

### `POST /api/services/control`

```json
{ "serviceName": "chat", "action": "restart" }
```

`action` 支持：`start | stop | restart | status`。

### `POST /api/services/command`

```json
{
  "serviceName": "task",
  "command": "list",
  "payload": { "status": "enabled" }
}
```

## Plugin Runtime API

### `GET /api/plugins/list`

列出已注册的 plugin。

### `POST /api/plugins/availability`

```json
{ "pluginName": "voice" }
```

### `POST /api/plugins/action`

```json
{
  "pluginName": "voice",
  "actionName": "status",
  "payload": {}
}
```

## Console UI API（摘要）

### `GET /api/ui/plugins`

返回 plugin 摘要与可用性信息（适用于 Console UI 面板）。

返回字段示例：

```json
{
  "success": true,
  "plugins": [
    {
      "name": "voice",
      "actions": ["status", "on", "off"],
      "pipelines": ["chat.augmentInbound"],
      "guards": [],
      "effects": [],
      "resolves": [],
      "requiredAssets": ["voice.transcriber"],
      "hasSystem": true,
      "hasAvailability": true,
      "availability": {
        "enabled": true,
        "available": true,
        "reasons": []
      },
      "config": {
        "actions": [
          {
            "name": "status",
            "supportsCommand": true,
            "supportsApi": true,
            "apiMethod": "",
            "apiPath": ""
          }
        ]
      }
    }
  ]
}
```

### `POST /api/dashboard/sessions/:sessionId/execute`

在指定会话执行一轮（例如你在 UI 里选中的 chatKey）。

带附件请求示例：

```json
{
  "instructions": "请阅读附件 markdown，并给出可执行摘要。",
  "attachments": [
    {
      "type": "document",
      "fileName": "page.md",
      "contentType": "text/markdown; charset=utf-8",
      "content": "# Captured Web Page\n\n..."
    }
  ]
}
```

说明：

- `sessionId` 应传 runtime 返回的目标会话 ID。聊天场景下该 ID 由 chat service 维护映射（不再要求可由平台字段直接拼接）。
- 附件会先被 runtime 落盘，再以 `<file ...>` 标签自动注入到用户消息前部。
- 当 `sessionId` 映射到 chat 会话时，会复用 chat 平台同一条执行队列（与平台入站消息一致），并在执行前写入 chat history。
- chat 上下文调用会立即返回排队结果（`queued=true`、`queueItemId`、`queuePosition`）。

## Action 直连路由

每个 service/plugin action 还会暴露默认路由：

- `/service/<serviceName>/<actionName>`
- `/api/plugins/action`

例如：

- `POST /service/chat/send`
- `POST /api/plugins/action`

## 调试建议

先确认 runtime 已启动：

```bash
city agent start
curl http://localhost:5314/health
```
