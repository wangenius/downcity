`AIService` 是 Federation 的 AI 服务。它注册 `AIModelDefinition`，负责鉴权、模型可用性、fallback、reasoning、计量和计费。

| 概念 | 作用 |
|---|---|
| `AIChannel` | Federation 到一个上游 AI 渠道的执行边界 |
| `AIChannelStreamInput` | 显式语言调用、最终模型、env 与 reasoning |
| `AIChannelActionInput` | 图片及其它非语言 action 的受限输入 |
| `AIModelSpec` | `channel.model()` 的模型声明参数 |
| `AIModelDefinition` | AIService 内部完整运行时模型 |
| `CityModelDescriptor` | 客户端公开模型目录结构 |
| `CityModel` | 实现 AI SDK `LanguageModelV3` 的客户端模型 |

所有语言模型入口都统一到 `AIChannel.stream(input)`：

```text
city.ai.text()          -> LanguageModelV3 -> AIChannel.stream
city.ai.stream()        -> CityModel       -> AIChannel.stream
CityModel.doGenerate()  -> CityModel       -> AIChannel.stream
chat/completions        -> OpenAI adapter   -> AIChannel.stream
```

OpenAI 兼容端点只负责协议转换，支持 system/user/assistant/tool、图片 URL、function tools、tool choice、reasoning、JSON/SSE。它不会把请求原样转发给上游，也不需要 `AIChannel.openai()`。

详见 [AIChannel 与模型注册](./provider)。
