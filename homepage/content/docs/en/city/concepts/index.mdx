The concepts section is not extra theory. It exists to keep the Downcity boundary clear before you wire it into real studios.

If you start by treating Downcity as “just another backend framework” or “a generic provider gateway,” the service boundary, studio split, and trust model will all drift in the wrong direction.

## Start with this picture

## When this section is the right starting point

- you want multiple AI studios to share the same AI infrastructure
- you need to understand the role of `studio_id`, `user_token`, and the model directory
- you are still deciding where City, your business backend, and an AI Gateway should stop

If you only want one working call chain first, go straight to [Quickstart](/en/docs/city/quickstart).
