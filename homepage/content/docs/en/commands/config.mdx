# `bay config`

Manage project `downcity.json` and alias.

> Naming note: configuration command is `bay config`, not `bay console config`.

## Usage

```bash
bay config <subcommand>
```

## Subcommands

- `get [keyPath]`
- `set <keyPath> <value>`
- `unset <keyPath>`
- `alias`

> Model pool management has moved to [`bay model`](/en/docs/commands/model).

## Plugin config

`bay config` now reads and writes project `downcity.json` only.

For plugin configuration, edit:

- `plugins.*` for agent-local plugin parameters only
- use `bay <plugin> on/off` for plugin lifecycle

Examples:

```bash
bay config get plugins.asr
bay config set plugins.asr.modelId '\"SenseVoiceSmall\"'
```

## Examples

```bash
bay config get model.primary
bay config set model.primary fast
bay config unset plugins.chat.channels.telegram
bay config alias --dry-run
```
