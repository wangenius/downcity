# Chrome Extension 架构

`chrome-extension/` 是 Downcity 在浏览器侧的上下文入口，负责把“正在看的页面”和“想让 agent 做的事”连接起来。

## 主要职责

- 获取当前页面标题与 URL
- 提取可读正文并转换成 Markdown
- 选择目标 agent 与 chat context
- 把执行请求投递到 Downcity Console gateway

## 主要模块

- `src/extension-popup/`：扩展弹窗 UI
- `src/options/`：设置页
- `src/services/downcityApi.ts`：访问 agent、context、任务投递的 API 边界
- `src/services/pageMarkdown.ts`：页面结构化抓取与 Markdown 生成
- `src/services/storage.ts`：扩展本地偏好存储
- `src/inline-composer/`：Inline Composer 交互与页内发送逻辑

## 执行链路

```text
用户操作
  -> Extension Popup 或 Inline Composer
    -> 采集页面 / 选区上下文
      -> 生成 Markdown 附件
        -> POST 到 console gateway
          -> agent context 执行
```

## 设计说明

- Extension Popup 与 options 是 React 页面
- 页面内选区面板仍然由 content script 驱动，但样式产物已经接入统一构建链
- 请求投递优先用 `sendBeacon`，回退到 `keepalive fetch`，确保扩展弹窗关闭后请求仍能继续发出
