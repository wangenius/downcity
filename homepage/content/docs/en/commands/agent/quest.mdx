# `city agent quest`

Send one message to a target agent through the Console main session.

This command reuses the same default session as Console UI: `consoleui-chat-main`.

## Usage

```bash
city agent quest --to <agentName> <message...>
```

## Key options

- `-t, --to <name>`: target agent name from console registry
- `--host <host>` / `--port <port>`: force remote HTTP transport
- `--token <token>`: explicit Bearer token for remote HTTP mode
- `--json [enabled]`: output structured JSON instead of plain reply text

## Behavior

- The target agent must already be running.
- The command resolves the target by agent name, not by current directory.
- The message is executed in the agent's Console main session, so follow-up `quest` calls continue in the same session context.
- In normal text mode, the CLI prints the agent reply directly.
- In `--json` mode, the CLI returns session metadata and raw execute result.
