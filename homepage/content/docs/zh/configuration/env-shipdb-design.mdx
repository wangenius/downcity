# Env 与 ship.db 设计

只记一个原则：

- 共享密钥和共享基础设施放 `~/.ship/ship.db`
- 项目行为放 `ship.json`

## `ship.db` 负责

- `env_entries`
- `channel_accounts`
- `model_providers`
- `models`
- 其他 console 持有的加密设置

## 项目 `ship.json` 负责

- `model.primary`
- `services.*`
- `plugins.*`
- `assets.*`

## 为什么这么分

- 密钥不应该散落到项目配置
- plugin 开关和 asset 选择属于项目行为
- console 存储不应该偷偷承载项目级 plugin 状态

## 运行时加载顺序

1. 从 `ship.db` 读全局 env
2. 从 `ship.db` 读 agent 私有 env
3. 读项目 `.env`
4. 读项目 `ship.json`
5. 解析 `plugins.*` 与 `assets.*`
6. 从 `ship.db` 解析渠道凭据和模型池
