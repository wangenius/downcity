# 配置详解

## ship.json

`ship.json` 是 ShipMyAgent 的主配置文件，控制着 Agent 的能力和行为边界。

### 完整示例

```json
{
  "$schema": "./.ship/schema/ship.schema.json",
  "name": "my-agent",
  "version": "1.0.0",
  "start": {
    "port": 3000,
    "host": "0.0.0.0",
    "webui": false,
    "webport": 3001
  },
  "llm": {
    "activeModel": "quality",
    "providers": {
      "google": {
        "type": "gemini",
        "apiKey": "${GEMINI_API_KEY}"
      },
      "compat_gateway": {
        "type": "open-compatible",
        "baseUrl": "https://your-api.com/v1",
        "apiKey": "${LLM_API_KEY}"
      }
    },
    "models": {
      "quality": {
        "provider": "google",
        "name": "gemini-2.5-pro",
        "temperature": 0.7
      },
      "fast": {
        "provider": "google",
        "name": "gemini-2.5-flash"
      },
      "fallback": {
        "provider": "compat_gateway",
        "name": "gpt-4o-mini"
      }
    },
    "logMessages": true
  },
  "services": {
    "skills": {
      "paths": [".claude/skills"]
    },
    "chat": {
      "egress": {
        "chatSendMaxCallsPerRun": 30,
        "chatSendIdempotency": true
      },
      "adapters": {
        "telegram": {
          "enabled": true,
          "botToken": "${TELEGRAM_BOT_TOKEN}",
          "followupWindowMs": 600000,
          "groupAccess": "anyone"
        },
        "feishu": {
          "enabled": true,
          "appId": "${FEISHU_APP_ID}",
          "appSecret": "${FEISHU_APP_SECRET}",
          "domain": "https://open.feishu.cn",
          "adminUserIds": ["ou_xxx", "ou_yyy"]
        },
        "qq": {
          "enabled": false,
          "appId": "${QQ_APP_ID}",
          "appSecret": "${QQ_APP_SECRET}",
          "sandbox": false
        }
      }
    }
  }
}
```

### 字段说明

#### `$schema` (Schema 引用)

- **$schema**: 指向 JSON Schema 文件路径，用于提供 IDE 智能提示和配置验证。

#### `name` 和 `version`

- **name**: Agent 项目名称，用于标识你的 Agent 应用。
- **version**: Agent 版本号，推荐使用语义化版本。

#### `start` (启动配置)

控制 Agent 服务的启动参数：

- **port**: Agent 服务监听的端口号（默认 `3000`）。
- **host**: 服务绑定的主机地址，`0.0.0.0` 表示允许外部访问。
- **webui**: 是否启用交互式 Web 界面（默认 `false`）。
- **webport**: 交互式 Web 界面的端口（默认 `3001`）。

#### `llm` (语言模型)

- **activeModel**: 当前激活模型 key，必须存在于 `llm.models`。
- **providers**: Provider 连接配置表，key 由你自定义（例如：`google`、`openai_main`）。
  - **type** 可选值：`anthropic`、`openai`、`deepseek`、`gemini`、`open-compatible`、`open-responses`、`moonshot`、`xai`、`huggingface`、`openrouter`。
  - **baseUrl**: 可选。不填时按 provider 类型使用内置默认地址。
  - **apiKey**: API 密钥，**强烈建议**使用 `${ENV_VAR}` 环境变量引用。
- **models**: 模型配置表，key 由你自定义（例如：`quality`、`fast`）。
  - **provider**: Provider key（必须存在于 `llm.providers`）。
  - **name**: 上游模型名称。
  - 可选采样字段：`temperature`、`maxTokens`、`topP`、`frequencyPenalty`、`presencePenalty`。
- **logMessages**: 是否记录发给 LLM 的请求 payload（默认 `true`）。

> 说明：
> - `open-compatible` 走 Chat Completions 兼容链路（`/chat/completions`）。
> - `moonshot` 走 Moonshot(Kimi) 官方 Chat Completions 兼容接口。
> - `open-responses`、`openai`、`deepseek` 走 Responses 兼容链路（`/responses`）。
> - `gemini` 走 Google 原生接口（`/v1beta/models/*:generateContent`）。

#### `services.chat.method` (回复投递方式)

- **method**: 可选 `cmd` 或 `direct`（默认 `direct`）。
  - `direct`：模型输出的 assistant 文本会自动发送到当前会话 channel。
    控制参数从 frontmatter metadata 读取：`chatKey`、`reply(message_id)` / `message_id` / `messageId`、`delay`、`sendAtMs/sendAt/time`、`react/reactions`。
    `reply` 的值必须是目标 `message_id`（不接受 `true/false`）。
    `react/reactions` 未显式传 `messageId` 时，会默认复用 `reply/message_id`（同一 chatKey）。
    附件仍通过正文中的 `<file type="...">path</file>` 发送。
    投递为 step 级分条发送，而不是仅最终合并一次发送。
  - `cmd`：模型通过执行 `sma chat send` 发消息（Bash-first）。

#### `services.chat.channels` (聊天适配器)

配置各种平台的集成适配器，支持将 Agent 部署到不同的聊天平台：

**telegram** (Telegram 机器人)
- **enabled**: 是否启用适配器
- **botToken**: Bot Token，支持环境变量
- **auth_id**: (可选) 主人鉴权 ID（单值）
- **followupWindowMs**: (可选) 跟进窗口时长，默认 10 分钟
- **groupAccess**: (可选) 群聊访问控制，`"anyone"`（默认）或 `"initiator_or_admin"`

**feishu** (飞书/Lark 机器人)
- **enabled**: 是否启用适配器
- **appId**: 飞书应用 ID
- **appSecret**: 飞书应用密钥
- **domain**: (可选) API 域名，默认 `https://open.feishu.cn`，国际版使用 `https://open.larksuite.com`
- **adminUserIds**: (可选) 管理员用户 ID 白名单

**qq** (QQ 官方机器人)
- **enabled**: 是否启用适配器
- **appId**: QQ 应用 ID
- **appSecret**: QQ 应用密钥
- **sandbox**: (可选) 是否使用沙箱环境，默认 `false`
- **groupAccess**: (可选) 群聊访问控制，`"anyone"`（默认）或 `"initiator_or_admin"`

> 详细配置说明和接入指南请参考 [集成指南](/zh/docs/guide/services)。

#### `services.skills` (技能)

配置 Agent 可用的技能插件：

- **paths**: 技能文件的搜索路径数组，支持相对路径和绝对路径。
  - 推荐使用 `.claude/skills` 或 `.ship/skills` 目录存放技能文件。
  - 支持内置技能和自定义技能的混合使用。

> 详细技能开发指南请参考 [技能系统](/zh/docs/guide/skills)。

#### `services.chat.egress` (消息出站控制)

> 主要作用于 `cmd` 模式。

- **chatSendMaxCallsPerRun**：单次 run 内 `chat_send` 最大调用次数。
- **chatSendIdempotency**：是否启用 `chat_send` 的 best-effort 幂等去重。

#### `context` (上下文)

控制每次执行时注入多少"对话历史"和 Chat 的调度策略。

- **context.messages**：每个 `chatKey` 维护一份持续增长的 `UIMessage[]` 作为唯一 history（落盘在 `.ship/chat/<encodedChatKey>/messages/messages.jsonl`），并在超出上下文窗口时自动 compact。
  - **keepLastMessages**: compact 后保留的最近消息条数（默认 `30`）
  - **maxInputTokensApprox**: 输入预算（近似 token 数，用于提前触发 compact，默认 `16000`）
  - **archiveOnCompact**: compact 时是否归档被折叠的原始段（默认 `true`）
- **services.chat.queue**：按 `chatKey` 分 lane 的调度器（同 chatKey 串行、不同 chatKey 可并发；运行中在 step 边界合并新消息）。

## PROFILE.md

`PROFILE.md` 是系统提示词（System Prompt）的载体。它决定了 Agent 的"性格"和"职业操守"。

### 编写技巧

#### 1. 定义角色

```markdown
# Agent Role

你是一个资深的前端架构师，专注于 React 性能优化。
```

#### 2. 设定目标 (Goals)

```markdown
## Goals

- 审查所有新提交的组件代码。
- 确保没有引入多余的重渲染。
- 保持组件代码简洁、可复用。
```

#### 3. 设定约束 (Constraints)

```markdown
## Constraints

- 永远不要修改 `package.json` 中的依赖版本。
- 所有的 CSS 样式必须使用 Tailwind 类名，禁止写内联样式。
- 修改代码前必须先解释原因。
```

#### 4. 提供上下文

你可以将重要的项目规范、代码风格指南直接写在 `PROFILE.md` 中，Agent 会在每次对话时都谨记这些规则。
