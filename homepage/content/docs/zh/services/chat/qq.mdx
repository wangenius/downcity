# QQ 机器人（QQBot）集成

使用 QQ 官方机器人（QQBot）把 Downcity 接入到 QQ：支持 **群聊消息全量接入** 与 **C2C 私聊** 两种场景。

> 当前 QQ channel 为 dev 版本，建议优先用于测试、联调与验证，再逐步投入正式使用。

## 配置步骤

### 1）创建 QQBot 应用并获取凭证

1. 前往 QQBot 平台创建机器人应用（QQ 开放平台 / QQBot 控制台）
2. 在应用信息中获取 **App ID**、**App Secret**
3. 确认机器人接入方式为 **WebSocket**，并开启群聊 / C2C 相关事件（用于接收消息）

### 2）创建 Channel Account 并绑定到项目

在 Console UI 的 `Global / Channel Accounts` 创建 QQ 账户（例如 `qq-main`）：

1. `channel = qq`
2. 填入 `appId`、`appSecret`
3. 按需设置 `sandbox`

然后在 `downcity.json` 里绑定该账户：

```json
{
  "services": {
    "chat": {
      "channels": {
        "qq": {
          "enabled": true,
          "channelAccountId": "qq-main"
        }
      }
    }
  }
}
```

### 3）启动 Downcity

```bash
city agent start
```

启动成功时，你会在日志中看到类似输出：

- `QQ service enabled`
- `🤖 正在启动 QQ 机器人...`
- `WebSocket 连接已建立`
- `QQ Bot 已就绪`

## 使用方式

### C2C 私聊

直接给机器人发送消息即可开始对话。

### 群聊

把机器人添加到群里后，群成员发送的普通消息会直接进入 Agent（不需要 `@` 机器人）。

### 语音消息（voice/audio）自动转写

当 `asr` plugin 已启用，且 `plugins.asr` 中的转写配置可用时，QQ 入站的 `voice/audio` 附件会触发本地语音转写（best-effort），转写文本会追加到用户指令中再执行。

建议先完成 asr 扩展初始化：

```bash
city asr on SenseVoiceSmall
city asr status
```

说明：
- 转写链路失败不会中断主流程（仍可处理同一条消息中的文本内容）。
- QQ 侧语音事件依赖网关 `AUDIO_ACTION` 能力订阅；若平台权限未开通，可能收不到语音附件字段。

## 常见问题排查

- **看起来在线但实际上已断（假连接）**：新版本会基于心跳 ACK 做活性检查；当出现 `heartbeat_timeout` 时会自动触发 QQ 重连。你也可以手动执行 `city chat reconnect --channel qq`。
- **能收到消息但发不回去**：回发链路现在会在网络波动/鉴权异常时自动重试一次，并触发链路自愈重连；如果仍失败，优先检查日志中的 `QQ send failed` 明细（HTTP 状态码与平台返回体）。
- **启动后完全没反应**：先看启动日志；如果你用的是前台模式可 `Ctrl+C` 停止；daemon 模式用 `city agent restart` 停止。运行日志在 `.downcity/logs/`。
- **群里收不到消息**：检查 QQBot 控制台是否已开启群聊相关事件/权限，特别是普通群消息事件（`GROUP_MESSAGE_CREATE`）。
- **语音消息没有被转写**：先执行 `city asr status` 检查 plugin 和转写配置是否可用，同时确认 QQBot 侧已开通音频事件权限（`AUDIO_ACTION`）。
- **凭据错误/权限不足**：日志里通常会出现获取 `access_token` 或 WebSocket 鉴权失败；检查绑定的 channel account（`qq-main`）里的 `appId/appSecret/sandbox` 配置是否正确。
- **报错 `QQ requires chatType + messageId to send a reply`**：QQ 当前走的是“被动回复”接口，需要带上触发这次对话的 `messageId` 才能回消息。只要目标 chatKey 下已经有对方发过的消息（会落盘到 `.downcity/chat/<encodedChatKey>/messages/messages.jsonl`），工具就能从 history 里拿到最新 `messageId`；否则需要对方先发一条消息建立上下文。
- **想确认 QQ 实际发了哪些字段（尤其是 username/nickname）**：开启原始事件抓取后查看落盘的 JSON 快照：
  - 启用抓取：`SHIP_QQ_CAPTURE_EVENTS=dispatch`（只抓取业务事件）或 `SHIP_QQ_CAPTURE_EVENTS=all`（包含心跳/握手等）
  - 可选自定义目录：`SHIP_QQ_CAPTURE_DIR=...`
  - 默认目录：`<projectRoot>/.downcity/.debug/qq-events/`
