# 模型配置

模型配置拆分为两部分：

1. City 维护模型目录和 provider 侧细节。
2. Agent 在 `downcity.json` 只选择 `execution.modelId`。

## 职责流程

1. 用 `city` 准备或检查模型目录。
2. 让 Downcity 以本机用户身份连接 City。
3. 在 agent 项目设置 `execution.modelId`。
4. 重启 agent 并进行一次实际调用验证。

## 命令在哪里

具体工作流见 [Downcity City 文档](/zh/docs/reference/cli)、[Downcity config 文档](/zh/docs/reference/cli) 和 [Agent restart 文档](/zh/docs/reference/cli)。

## 失败场景

- `execution.modelId` 不存在：agent 启动失败。
- provider/key 无效：调用模型时报鉴权或网络错误。
