# `city agent`

Manage agent lifecycle and registry-aware operations.

> Recommended reading: the new complete Agent documentation now lives in a dedicated docs domain. Start with [Agent Overview](/en/docs/agent/overview/what-is-agent) and [Agent CLI Overview](/en/docs/agent/cli/overview).

Run `city agent` without a subcommand in an interactive terminal to open the agent manager. Use subcommands when you want a scriptable command.

## Command hierarchy

Top-level command groups are: `city console`, `city agent`, `city service`, `city plugin`.

## Usage

```bash
city agent
city agent <subcommand>
```

## Interactive manager

The interactive manager lets you:

- list registered agents
- create an agent project
- start an agent
- pick an agent and see current status, name, model, and chat channel connections before choosing an action
- configure the agent name, model binding, or city-level chat channel account connections

It does not replace the subcommands below. It is a guided entry point for common local operations.

Chat channel credentials stay in city-level channel accounts managed from `city chat`. The agent manager only connects an agent to those existing accounts through `services.chat.channels.<channel>.channelAccountId`. If an agent points at a missing account, the manager removes that dangling binding automatically.

## Subcommands

- `create`: create/initialize an agent project
- `list`: list registered agent projects
- `start`: start the agent
- `stop`: stop the agent daemon
- `chat`: chat with an agent in interactive or one-shot mode
- `status`: show agent status
- `doctor`: diagnose state files and optionally fix stale pid/meta
- `restart`: restart the agent

## Environment variable injection

On every `city` execution, CLI resolves current agent context and injects variables into the `city` process and its child processes:

- `DC_AGENT_PATH`: absolute path of current agent
- `DC_AGENT_NAME`: agent name (prefer `downcity.json.name`, fallback to directory name)

> Note: this is process-scoped injection and does not mutate your parent shell session.
