# `schedule` Plugin

`schedule` 是基础设施型 plugin。

它负责轮询调度任务，并把到点的 plugin action 重新派发执行。

## 主要形状

- `lifecycle`
- 没有业务 action

## 它主要解决什么问题

如果你想理解这些问题，应该看 `schedule`：

- 为什么调度能力归属于 plugin lifecycle，而不是 agent 顶层壳
- 延迟/定时 plugin action 是怎么被重新调用的
- 为什么一个 plugin 可以没有用户可见业务 action，却依然是核心基础设施

## 关键语义

- 它主要轮询 `PluginScheduleStore`
- 启动恢复时会把中断的 running jobs 重置回 pending
- 它是核心基础设施模式，不是普通用户能力型 plugin

## 公开性说明

`SchedulePlugin` 在代码里是 built-in，但主要是平台基础设施 plugin，不属于典型公开 SDK 能力插件。
