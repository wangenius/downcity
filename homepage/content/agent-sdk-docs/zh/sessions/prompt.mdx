# `session.prompt()`

`session.prompt()` 现在是本地 `Agent` 和 `RemoteAgent` 统一的 Session 输入 API。

调用方不需要判断这条输入到底是：

- 第一轮输入
- 运行中纠偏
- 还是下一轮排队输入

统一都这样调用：

```ts
const turn = await session.prompt({
  query: "分析这个仓库",
});

await turn.finished;
```

## 传入 parts 数组

`query` 也支持直接传入 `SessionUserMessagePart` 数组，可以携带图片、文件等：

```ts
import type { SessionUserMessagePart } from "@downcity/agent";

const parts: SessionUserMessagePart[] = [
  { type: "text", text: "分析这张图片" },
  { type: "file", mediaType: "image/png", url: "https://example.com/image.png" },
];

const turn = await session.prompt({
  query: parts,
});

await turn.finished;
```

图片 `file` part 也可以传本机绝对路径或 `file://` URL；本地 `Agent` 会在请求模型前读取文件并转换为 `data:<mediaType>;base64,...`：

```ts
const turn = await session.prompt({
  query: [
    { type: "text", text: "这张图片写了什么？" },
    {
      type: "file",
      mediaType: "image/png",
      filename: "image.png",
      url: "/Users/me/project/.downcity/input/image.png",
    },
  ],
});
```

## 例子

```ts
const session = await agent.sessions.get("repo-analysis");

const turn = await session.prompt({
  query: "分析这个仓库",
});

await session.prompt({
  query: "只看 Downcity 这一条链路",
});

await turn.finished;
```

## 它会返回什么

`prompt()` 只有在这条新输入已经被绑定到某个确定 turn 之后才会返回。

返回的 handle 里最重要的是：

- `turn.id`
- `turn.finished`
- `turn.result`

## Session 内部会做什么

- 如果当前 session 空闲，就启动新 turn
- 如果当前 turn 还来得及吸收新的用户输入，就并入当前 turn；消息成功写入后，`subscribe()` 会收到对应的 `variant: "message"` action Mutation
- 否则把这条输入排到下一 turn

调用方不需要自己维护第二套队列或第二套运行时。
