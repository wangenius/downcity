# `town city`

`town city` manages the City user connection used by local Agents.

`city` is for admin-side City base management. `town` is for user login and local Agent runtime access. By default, Town uses `https://base.downcity.ai`.

## Usage

```bash
town city
town city status
town city list
town city connect [url]
town city use [base]
town city login [url]
town city logout
town city disconnect
```

Running bare `town` opens the interactive console. Choose `Connect City` to view the signed-in user's balance or start a recharge from the user menu.

## State Model

`town city connect` and `town city use` select a City base. They do not copy admin credentials or user tokens from `city`.

`town city login` signs in as a user and stores Town's own user session. Agent runtime reads that Town user session when it calls City AIService.

`View User Balance` reads the current signed-in user's `balance` account. `User Recharge` creates a topup for that user and starts a checkout with an available payment method; by default, Town opens the checkout page in your browser.

Environment variables such as `DOWNCITY_CITY_URL`, `DOWNCITY_CITY_TOWN_ID`, and `DOWNCITY_CITY_USER_TOKEN` are still supported as runtime overrides for CI or temporary launches.

## Recommended Flow

```bash
city                  # optional: manage admin-side City bases and resources
town city status      # defaults to https://base.downcity.ai
town city login
town                  # choose Connect City -> View User Balance or User Recharge
town agent start
```

Use `town city list` to see Town bases plus base URLs discovered from `city` admin configuration.
