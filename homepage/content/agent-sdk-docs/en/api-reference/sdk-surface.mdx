# SDK Surface

The SDK is session-first. `Agent` owns local runtime assembly and `RemoteAgent` connects to an exposed runtime.

## `Agent`

- `new Agent(options)`, `ready()`, `dispose()`
- `sessions.create()`, `get()`, `list()`, `archive()`, `archived()`, `clean_archive()`
- `setInstruction()`, `getEnv()`, `setEnv()`, `patchEnv()`
- `getConfig()`, `getLogger()`, `getContext()`, `getShell()`, and `plugins`

Local plugin actions use `agent.plugins.runAction(...)`. HTTP and RPC transports are supplied by `AgentHTTP` and `AgentRPC` in `@downcity/server`.

## `RemoteAgent`

- `new RemoteAgent({ url, token? })`
- `sessions` collection methods matching local Agent
- `runPluginAction()` and `close()`

HTTP URLs use the server root such as `http://127.0.0.1:5314`; RPC uses `rpc://127.0.0.1:15314`.

## Sessions

Both local and remote Sessions provide `get_info()`, `prompt()`, `stop()`, `subscribe()`, `messages()`, `system()`, `approvals()`, `approval_mode()`, `set_approval_mode()`, `resolve_approval()`, and `fork()`.

Local Sessions accept `set({ model })` or `set({ modelId })`. Remote Sessions reject a local model instance and accept only `set({ modelId })`.

`subscribe()` emits the unified live Mutation stream after the subscription is established. Load initial Message and Session snapshots, and reload them after reconnecting.
