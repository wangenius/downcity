# `town city`

`town city` 管理本机 Agent 调用 City 时使用的用户态连接。

`city` 负责 admin 侧的 City 与资源管理。`town` 负责账号登录和本机 Agent runtime 访问。默认 City 是 `https://base.downcity.ai`。

## 用法

```bash
town city
town city status
town city list
town city whoami
town city connect [url]
town city use [city]
town city login [url]
town city logout
town city disconnect
```

运行裸 `town` 会进入交互式操作台。选择 `连接 City` 后，菜单会按状态、City、账号、设置分区展示；登录后余额会直接显示在账号分区里，并提供 `充值`。

## 状态模型

`town city connect` 和 `town city use` 只选择 City，不从 `city` 复制 admin 凭证或 user token。

`town city login` 以 user 身份登录，并保存 Town 自己的 user session。Agent runtime 调用 City AIService 时读取这份 Town user session。

`town city whoami` 会显示 Town 当前实际使用的 City 账号，包括 City URL、town id、user id，以及是否被环境变量覆盖。

登录后交互菜单会直接读取当前账号的 `balance` 账户并展示余额。`充值` 会为当前账号创建充值单，并通过可用支付方式创建 checkout 页面；默认会打开浏览器完成支付。

余额、充值、Agent runtime 默认都使用 `town city login` 保存的当前 City 账号。`DOWNCITY_CITY_URL` 可以临时选择另一个 City，并读取该 City 对应的 Town 登录态；只有显式设置 `DOWNCITY_CITY_USER_TOKEN` 时，才会覆盖当前账号身份。`DOWNCITY_CITY_TOWN_ID` 可以覆盖 town id。

## 推荐流程

```bash
city                  # 可选：管理 admin 侧 City 与资源
town city status      # 默认指向 https://base.downcity.ai
town city login
town city whoami
town                  # 选择「连接 City」→ 查看余额或选择「充值」
town agent start
```

使用 `town city list` 可以查看 Town 自己保存的 City，以及从 `city` admin 配置中发现的 City 地址。
