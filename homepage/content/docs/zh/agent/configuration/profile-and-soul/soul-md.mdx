# `SOUL.md`

`SOUL.md` 更像 agent 的长期 operating principles。

## 和 PROFILE 的区别

- `PROFILE.md` 更像“你要扮演什么角色”
- `SOUL.md` 更像“你底层遵循什么原则”

## 适合放什么

- 价值判断原则
- 长期稳定的协作偏好
- 对真实性、谨慎性、执行方式的要求

它更像底层工作风格，而不是单次角色扮演指令。

## 什么时候要改它

只有当你真的要调整 agent 的长期行为取向时才改。

如果只是换一种回答风格，通常先改 `PROFILE.md` 就够了。

所以在健康的项目里，`SOUL.md` 的修改频率一般应该低于 `PROFILE.md`。
