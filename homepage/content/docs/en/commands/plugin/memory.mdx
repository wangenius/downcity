# `city memory`

`city memory` is the managed memory plugin command group.

Use it when the agent or operator needs to inspect or write memory explicitly instead of relying only on prompt-time memory injection.

## Usage

```bash
city memory <subcommand> [options]
```

## Main subcommands

- `status`
- `search <query>`
- `get <memoryPath> [--from <line>] [--lines <count>]`
- `store --content <text> [--target <longterm|daily|working>] [--session-id <sessionId>]`
- `flush --session-id <sessionId> [--max-messages <count>]`

## Notes

- Memory is enabled by default and belongs to the managed plugin runtime.
- The system prompt only injects a bounded longterm pack; deeper retrieval still goes through `memory.search` and `memory.get`.
- All memory actions support one-shot `--delay` / `--time`.

## Examples

```bash
city memory status
city memory search "release decision"
city memory get .downcity/memory/MEMORY.md --from 10 --lines 20
city memory store --target longterm --content "User prefers concise release notes."
city memory flush --session-id telegram-chat-123 --max-messages 40
```
