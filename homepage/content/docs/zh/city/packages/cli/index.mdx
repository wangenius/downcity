`@downcity/cli` 是给 City 管理者用的终端工具。它不是产品运行时的一部分，而是你在开发、初始化和排查阶段的运维入口。

如果说 `AdminClient` 负责“程序里的可信调用”，那 `@downcity/cli` 负责“人坐在终端前的直接操作”。

## 这组文档怎么读

建议按下面顺序读：

1. 先读 [快速开始](/docs/city/packages/cli/quickstart)，先知道怎么连到 City 数据库。
2. 再读 [模型与数据表管理](/docs/city/packages/cli/models-and-tables)，理解最常见的表查看和模型维护动作。
3. 最后读 [运维与排查](/docs/city/packages/cli/operations-and-debug)，把日常检查、故障定位和人工修复串起来。

## 适合什么时候用

- 你想直接查看 City 数据库里有哪些表
- 你要增删改查 `downcity_models`
- 你要在本地排查 runtime 数据是否写对了

## 安装

```bash
npm i -g @downcity/cli
```

## 默认用法

```bash
downcity
```

默认会进入交互式面板。没有 server 时，CLI 会先要求你添加一个 server；之后你可以：

- 通过 `Manage Servers` 管理多个 City server
- 进入 Admin 流程管理 env / models / studios / payment
- 进入 User 流程登录、充值和调用服务

## 常用命令

```bash
downcity models
downcity models add
downcity models update
studio --database-url "$DOWNCITY_CITY_DATABASE_URL"
```

## 它和其他 package 的边界

- 不替代 [@downcity/gate](/docs/city/packages/gate) 里的 `AdminClient`
- 不替代 [@downcity/city](/docs/city/packages/city) 的运行时能力
- 更适合“人工排查”和“初始化运维”，而不是嵌进产品代码里

更准确地说：

- `AdminClient` 负责程序里的可信调用
- `@downcity/cli` 负责终端里的人工运维和排查
