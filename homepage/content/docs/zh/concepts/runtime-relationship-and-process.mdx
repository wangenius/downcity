# Runtime Relationship And Process

当前运行关系可以概括成：

- `console` 负责管
- `agent` 负责承载
- `execution context` 负责暴露统一能力
- `session` 负责执行
- `plugin runtime` 负责增加或拥有能力模块

## 启动顺序

1. console 启动
2. agent 在 console registry 下启动
3. agent 初始化模型访问、session store 与 plugin registry
4. 托管 plugin 进入可用状态

## 请求路径

1. 调用先进入某个托管 plugin 或本地 SDK surface
2. plugin 在需要时解析目标 session context
3. plugin 触发 actions 或 hooks
4. 执行进入 `context.session`
5. session 产出结果

## 最重要的结论

当前系统已经不需要单独的公开附加层概念来解释请求归属。

能力模块统一是 plugin；真正执行仍然归属于 session。
