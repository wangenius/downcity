# `downcity.json` 参考

项目 `downcity.json` 是本地 Agent 的运行契约。它声明 Agent 身份、执行绑定和插件配置。City 模型、Service 细节以及 Shell Safe Sandbox 边界不直接写进项目文件。

## 1) Agent 项目最小结构

```json
{
  "$schema": "./.downcity/schema/downcity.schema.json",
  "name": "my-agent",
  "version": "1.0.0",
  "execution": {
    "type": "api",
    "modelId": "quality"
  },
  "plugins": {}
}
```

`execution.type` 现在固定为 `api`，并且 `execution.modelId` 必须能对应已连接 Federation AIService 暴露的某个模型。

## 2) City 模型绑定

项目里只保存选中的模型 ID。

在 City 管理侧选择模型 ID，然后在项目里保留绑定。

具体工作流见 [Downcity City 文档](/zh/docs/reference/cli)、[Downcity config 文档](/zh/docs/reference/cli) 和 [Agent restart 文档](/zh/docs/reference/cli)。

## 3) 运行时解析规则

agent 启动时：

1. 先检查项目是否声明了 `execution`
2. 如果 `execution.type = "api"`，通过已连接的 Federation AIService 解析模型 ID
3. 把解析后的模型注入 Agent session runtime

`execution` 必须合法，否则启动失败。

## 4) Plugin 配置

插件行为配置可以写在项目 `downcity.json` 的 `plugins.*`。并不是所有 plugin 都从这里读取运行参数；例如 `asr` 这类能力型 plugin 的真实转写函数从 SDK constructor 传入。

例如 chat 渠道绑定：

```json
{
  "plugins": {
    "chat": {
      "channels": {
        "telegram": {
          "enabled": true,
          "channelAccountId": "telegram-main"
        }
      }
    }
  }
}
```

## 5) Chat 渠道绑定

`downcity.json` 中的 chat 渠道应只绑定全局 bot 账户 ID：

```json
{
  "plugins": {
    "chat": {
      "channels": {
        "telegram": {
          "enabled": true,
          "channelAccountId": "telegram-main"
        }
      }
    }
  }
}
```

`channelAccountId` 会解析到 `~/.downcity/downcity.db` 的 `channel_accounts.id`。

## 6) 推荐配置方式

1. 在 City 层维护 provider/model
2. 在 agent 层配置 `execution.type="api"` 与 `execution.modelId`
3. 项目级插件统一放在 `plugins.*`
4. 本地 shell / CLI 命令默认强制运行在固定 Safe Sandbox 中
5. 修改后需要让 agent runtime 重新载入项目配置

## 7) Sandbox

`downcity.json` 当前不提供 sandbox 配置块。Shell Safe Sandbox 边界由 runtime 固定生成，不通过项目配置调整。

当前实现边界：

- 当前支持 macOS 与 Linux：macOS 使用 `sandbox-exec`，Linux 使用 `bubblewrap`（命令名 `bwrap`）
- shell 命令强制经过 sandbox，不支持关闭，也不会回退到宿主机普通子进程执行
- `kind=script` 的 task 正文也通过同一套 sandbox backend 执行
- 未挂入 sandbox 的宿主路径默认不可见
- agent sandbox 持久目录是 `.downcity/sandbox/`
- sandbox 子进程使用 `.downcity/sandbox/` 作为 `HOME`，`.downcity/sandbox/tmp` 作为常见临时目录变量与 zsh `TMPPREFIX` 目标，`.downcity/sandbox/.cache` 作为 `XDG_CACHE_HOME`
- 项目文件、项目内 `.venv`、`node_modules`、下载到项目内的模型和缓存默认可写
- 真实用户 HOME 以及 `/usr/local`、`/opt/homebrew`、`/System` 等系统目录默认不可写
- Linux 需要预先安装 `bubblewrap`，并确保系统允许 unprivileged user namespaces
