# `town stop`

停止 Town runtime 与受管 agent。

当前实现还会额外尝试清理“失联的旧进程”：

- 没有 pid 文件但仍在后台运行的 Town runtime（`run`）
- 旧版本可能遗留的 Console UI runtime 进程
- 没有 pid 文件但仍在后台运行的 agent daemon（`agent start --foreground true`）

在 macOS / Linux 上，`town stop` 会优先停止对应的进程组。这样 agent 内部拉起的 ACP 执行器、shell 会话、watch 子进程也会一起收到停止信号，避免父进程退出后留下孤儿进程继续占用内存。

## 用法

```bash
town stop
```

## 相关命令

- [`town start`](/zh/products-docs/town-cli/start)
- [`town status`](/zh/products-docs/town-cli/status)

## 输出内容

- 会清理旧 runtime 进程、停止受管 agent，并停止 Town runtime。
- 如果 runtime 已经停止，会继续清扫失联进程并输出当前状态。
