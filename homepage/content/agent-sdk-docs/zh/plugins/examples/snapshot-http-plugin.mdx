# SnapshotHttpPlugin 场景

## 场景

你想让某个 plugin 对外提供一个只读快照接口：

- `GET /api/notes/snapshot`

这类事情就很适合 `plugin.http.runtime`。

## 示例

```ts
import type { Plugin } from "@downcity/agent";

export const snapshotHttpPlugin: Plugin = {
  name: "notes_http",
  title: "Notes Snapshot",
  description: "Expose one read-only notes snapshot endpoint.",
  http: {
    runtime: {
      authPolicies: [
        {
          path: "/api/notes/*",
          method: "GET",
          requireAuth: true,
          anyPermissions: ["agent.read"],
        },
      ],
      register({ app, pluginName }) {
        app.get("/api/notes/snapshot", async (c) => {
          return c.json({
            success: true,
            plugin: pluginName,
            snapshot: {
              count: 3,
            },
          });
        });
      },
    },
  },
};
```

## 这个例子最想说明什么

- 路由由 plugin 声明
- server 负责统一装配
- 鉴权策略也由 plugin 声明

## 当前边界

这个能力针对的是完整 runtime HTTP 装配链。

也就是说：

- `plugin.http.runtime` 不是 `agent.plugins.runAction(...)` 这一类显式调用
- 当前最小 SDK `agent.start({ http: { ... } })` 只暴露 session API，不会自动把 plugin HTTP 路由挂进去
- 如果你需要这一层，就应该把它理解成更完整的 host/runtime 集成能力
