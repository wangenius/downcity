# `plugins` 配置

项目中的 `plugins` 用来声明 agent 级 plugin 配置。

## 最常见的是 chat

例如：

```json
{
  "plugins": {
    "chat": {
      "channels": {
        "telegram": {
          "enabled": true,
          "channelAccountId": "telegram-main"
        }
      }
    }
  }
}
```

对大多数用户来说，这也是第一次接触 `plugins` 时最先会写的一段配置。

## 这段配置表达的不是

它不是在保存 Telegram bot token。

它只是在说：

- 当前 agent 启用了 Telegram 渠道
- 它要绑定 city 全局层的 `telegram-main` 这个 channel account

所以如果聊天入口有问题，优先检查的通常应该是：

- 对应全局账号是否存在
- `channelAccountId` 是否写对
- 修改后是否已经重启 agent

## 推荐理解方式

- 凭据在全局层
- 绑定在项目层
