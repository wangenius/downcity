# `city chat`

用于向指定会话发送消息、贴表情、查看上下文和读取历史。

> 层级说明：`city chat` 是 service 动作命令，不是一级命令。一级入口是 `city service`。

## 用法

```bash
city chat <subcommand> [options]
```

## 子命令

- `status`：查看渠道连接状态
- `test`：测试渠道连通性
- `reconnect`：重连渠道（默认全部，可按渠道）
- `open`：打开渠道（写入 `ship.json` 为 `enabled=true`，已配置会尝试启动）
- `close`：关闭渠道（写入 `ship.json` 为 `enabled=false`，并停止运行）
- `configuration`：查看渠道配置元信息（字段名/类型/说明/来源）
- `configure`：更新渠道参数（写入 `ship.json`，默认重载该渠道）
- `list`：列出当前 agent 已记录的会话（chatTitle/chatKey）
- `info`：查看指定会话信息（路由快照/本地路径/上下文快照）
- `send`：发送消息
- `react`：给消息贴表情（当前主要 Telegram）
- `context`：查看当前会话上下文快照
- `delete`：彻底删除会话（路由映射 + chat 历史 + context 目录）
- `history`：读取历史消息

## `status/test/reconnect/open/close` 常用选项

- `--channel <telegram|feishu|qq>`：只针对单个渠道执行（不传则全部）

## `configure` 常用选项

- `--channel <telegram|feishu|qq>`：目标渠道
- `--config-json <json>`：配置 patch（JSON 对象）
- `--restart`：配置后立即重载渠道

## `configuration` 常用选项

- `--channel <telegram|feishu|qq>`：仅查看指定渠道（不传则返回全部）

## `list` 常用选项

- `--channel <telegram|feishu|qq>`：渠道过滤
- `--limit <n>`：返回最近 N 条（默认 50）
- `--q <text>`：关键词过滤（title/chatId/contextId/actor）

## `info` 常用选项

- `--chat-key <chatKey>`：目标会话（不传则尝试读取 `DC_CTX_CHAT_KEY`）
- `--context-id <contextId>`：显式指定 contextId（优先级更高）

## `send` 常用选项

- `--text <text>`：消息正文
- `--stdin`：从标准输入读消息正文
- `--text-file <file>`：从文件读消息正文
- `--chat-key <chatKey>`：目标会话
- `--delay <ms>`：延迟发送
- `--time <time>`：定时发送（时间戳秒/毫秒或 ISO）
- `--reply`：reply_to_message 发送

## `react` 常用选项

- `--emoji <emoji>`：表情，默认 👍
- `--message-id <id>`：目标消息 ID
- `--chat-key <chatKey>`：目标会话
- `--big`：大表情（Telegram `is_big`）

## `delete` 常用选项

- `--chat-key <chatKey>`
- `--context-id <contextId>`

## `history` 常用选项

- `--chat-key <chatKey>`
- `--context-id <contextId>`
- `--limit <n>`
- `--direction <all|inbound|outbound>`
- `--before-ts <ts>` / `--after-ts <ts>`（毫秒）

## 示例

```bash
city chat list --limit 20
city chat info --chat-key ctx_xxxxxxxxxxxxxxxx
city chat send --chat-key ctx_xxxxxxxxxxxxxxxx --text "done"
city chat react --chat-key ctx_xxxxxxxxxxxxxxxx --emoji "✅"
city chat context --chat-key ctx_xxxxxxxxxxxxxxxx
city chat delete --chat-key ctx_xxxxxxxxxxxxxxxx
city chat history --chat-key ctx_xxxxxxxxxxxxxxxx --limit 50
city chat status
city chat test --channel telegram
city chat reconnect --channel qq
city chat open --channel qq
city chat close --channel telegram
city chat configuration --channel qq
city chat configure --channel qq --config-json '{"channelAccountId":"qq-main","enabled":true}'
```

## 说明

该命令组通常需要已启动的 agent runtime：

```bash
city agent start
```
