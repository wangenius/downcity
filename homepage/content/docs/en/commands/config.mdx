# `studio config`

Manage project `downcity.json` and alias.

> Naming note: configuration command is `studio config`, not `studio console config`.

## Usage

```bash
studio config <subcommand>
```

## Subcommands

- `get [keyPath]`
- `set <keyPath> <value>`
- `unset <keyPath>`
- `alias`

> Model pool management has moved to [`studio model`](/en/docs/commands/model).

## Plugin config

`studio config` now reads and writes project `downcity.json` only.

For plugin configuration, edit:

- `plugins.*` for agent-local plugin parameters only
- use `studio <plugin> on/off` for plugin lifecycle

Examples:

```bash
studio config get plugins.asr
studio config set plugins.asr.modelId '\"SenseVoiceSmall\"'
```

## Examples

```bash
studio config get model.primary
studio config set model.primary fast
studio config unset plugins.chat.channels.telegram
studio config alias --dry-run
```
