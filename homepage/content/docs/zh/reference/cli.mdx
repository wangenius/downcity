# CLI 命令

Downcity docs 解释设计模型、运行边界和配置语义。

`downcity` / `city` 负责本机 Agent 宿主、本机项目管理，以及连接到某个 Federation。Federation 创建、部署和基础设施运维由 `fed` / `downfed` 负责；精确参数以本机安装的 CLI help 为准，SDK 侧 CLI 边界详见 City SDK 文档。

- [CLI：fed 与 city](/zh/city-sdk-docs/packages/cli)

## 本地帮助

精确参数以本机安装的 CLI help 为准。产品文档负责说明命令组和常见工作流。

## 交互式 TUI

不带子命令运行 `city` 或 `downcity` 时，现在默认进入交互式 TUI。`city` 侧聚焦本机 Agent 与 Federation 连接状态；`fed` 侧聚焦 Federation 管理与部署。两个 CLI 都保持同一套稳定布局：左侧 sidebar 承载菜单层级，顶部显示 breadcrumb，loading、输入表单、表格、JSON、消息与选择详情都在右侧 section 内完成，直到你主动退出 TUI。

## CLI 语言切换

`city` 和 `downcity` 现在都支持中文与英文。

- 通过 `--lang zh` 或 `--lang en` 显式切换当前命令语言。
- 如果没有传 `--lang`，CLI 会优先读取最近一次在交互菜单中保存的默认语言，然后再依次读取 `DC_CLI_LANG`、`LC_ALL`、`LC_MESSAGES`、`LANG`。
- 在交互模式下，`city` 和 `downcity` 都可以直接从菜单切换语言，并把该选择持久化到本地。

示例：

```bash
downcity --lang en --help
city --lang zh
DC_CLI_LANG=zh downcity agent --help
```
