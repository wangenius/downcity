The `@downcity/agent` package now exposes two primary SDK entries:

- `Agent`: local embeddable agent runtime
- `RemoteAgent`: remote agent client

## Install

```bash
pnpm add @downcity/agent
```

If you want to use built-in plugin classes such as `ChatPlugin`, install the built-in plugin package too:

```bash
pnpm add @downcity/plugins
```

## The most important distinction here

- pure SDK embedding mode: the host owns the model directly
- Downcity project mode: the model usually comes from `execution.modelId` + `City AIService`

So not every scenario should default to `session.set({ model })`.

## Local Agent

Use `new Agent(...)` when your app runs in the same process and can provide tools, plugins, and models directly.

```ts
import { Agent } from "@downcity/agent";
import { createOpenAI } from "@ai-sdk/openai";

const openai = createOpenAI({
  apiKey: process.env.OPENAI_API_KEY!,
});

const agent = new Agent({
  id: "vibecape-official",
  path: "/path/to/agent-project",
  tools: {},
  model: openai.responses("gpt-5"),
});

const session = await agent.createSession();

const turn = await session.prompt({
  query: "Rename the local file to match the new convention",
});
const result = await turn.finished;

console.log(result.text);
```

If you did not provide a default model on `Agent`, you can still call `session.set({ model })` for one specific session.

## Start RPC

```ts
const started = await agent.start({
  rpc: {
    host: "127.0.0.1",
    port: 15314,
  },
});

console.log(started.rpc?.url);
```

## Expose HTTP

```ts
const server = await startServer({
  host: "127.0.0.1",
  port: 15314,
  getAgentRuntime: () => agent.getRuntime(),
  getAgentContext: () => agent.getContext(),
  sessionCollection: agent.getSessionCollection(),
});
```

## Remote Agent

Use `RemoteAgent` when another process has already exposed the Agent over HTTP or RPC.

```ts
import { RemoteAgent } from "@downcity/agent";

const agent = new RemoteAgent({
  url: "http://127.0.0.1:15314",
});
```
