# Chat Access

Chat Access is an admission capability of the Chat Plugin. It answers one question: may a direct or group message from an external Chat user enter a specific Agent?

It is not an Agent Core permission system or a general Role/Permission model. After admission, the Chat Plugin passes the message to the Agent without propagating roles or permissions into Sessions or other Plugins.

## Core rules

- Default deny: without an explicit `allow` grant, the message does not enter the Agent.
- Per-Agent isolation: approval for Agent A does not grant access to Agent B.
- Per-account isolation: identity uses `channel + issuer + subject_id`, so the same user contacting two bots has two principals.
- Explicit scopes: persisted grants are only `direct` and `group`; `all` in management commands expands to two grants.
- `deny` differs from `revoke`: denial prevents new requests, while revocation lets the user request access again.

## Approval flow

When an unapproved user sends a direct message, the Chat Plugin creates or reuses a pending request and returns a command like:

```bash
downcity chat access approve req_xxx --agent assistant
```

Administrators can inspect requests first:

```bash
downcity chat access requests --agent assistant --status pending
```

Unapproved group messages are silent by default, but their pending requests appear in the same management list.

## Storage

Each Agent stores its own data at:

```text
<agent_project>/.downcity/chat/access.db
```

The database contains principals, requests, allow/deny grants, and audit events. It does not contain bot tokens, app secrets, or message bodies. Chat accounts and credentials remain in the City-global `~/.downcity/downcity.db`; Agent configuration stores only the bound `channelAccountId`.

## Difference from City Auth

City Auth protects HTTP and Console control-plane login and route permissions. Chat Access decides whether an external Telegram, Feishu, or QQ user may send messages to a specific Agent. Their data, identities, and decision paths are separate.
