`downcity` is the global installation package for Downcity's CLI products. It installs two commands: `city` for City administration and `town` for local Agent hosting.

If `Admin City` owns "trusted programmatic calls", `city` owns "a human operating City directly from the terminal".

## How to read this section

The recommended order is:

1. Start with [Quickstart](/en/city-sdk-docs/packages/cli/quickstart) to learn how the CLI connects to your City database.
2. Continue with [Models and tables](/en/city-sdk-docs/packages/cli/models-and-tables) for the most common table inspection and model-management workflows.
3. Finish with [Operations and debugging](/en/city-sdk-docs/packages/cli/operations-and-debug) to connect routine checks, incident diagnosis, and manual fixes.

## Use it when

- you want to inspect which tables exist in the City database
- you need to add, update, or remove entries in `downcity_models`
- you want to debug runtime data directly from the terminal

## Install

```bash
npm i -g downcity
```

## Default usage

```bash
city
```

In an interactive terminal, the default `city` entry opens the City panel. In non-interactive environments it prints help instead.

The CLI also supports language switching:

```bash
city --lang en
city --lang zh --help
DC_CLI_LANG=zh city
```

If no City is connected yet, the interactive flow first asks you to add one. After that, you can:

- manage multiple City servers from `Manage Servers`
- enter the Admin flow for env / models / towns / payment
- enter the User flow for login, recharge, and service calls

## Scriptable city commands

```bash
city manage
city create .
city deploy
city deploy --dry-run
city deploy --verify
city env refresh
```

These commands manage City projects, deployment, and City runtime env cache.

## Boundary with `town city`

`town city` is a Town command group. Use it when a local Town runtime or agent needs to select a City base and store a Town user login:

```bash
town city use
town city login
town city status
```

So the short version is:

- `city`: operate City from the terminal
- `town city`: connect Town and local agents to a City base

## Boundaries with other packages

- It does not replace `Admin City` from [@downcity/city](/en/city-sdk-docs/packages/city/city-runtime).
- It does not replace the runtime capabilities in [@downcity/city](/en/city-sdk-docs/packages/city).
- It is better for human-operated inspection and maintenance than for code you embed into a town.

- `Admin City` is for trusted programmatic calls
- `city` is for human-operated terminal workflows
