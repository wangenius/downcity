# Session 类型与结果

最常见的输入输出对象有：

- `AgentSession`
- `RemoteAgentSession`
- `AgentSessionSetInput`
- `AgentSessionPromptInput`
- `AgentSessionStopResult`
- `AgentSessionTurnHandle`
- `AgentSessionTurnResult`
- `SessionMutation`
- `AgentSessionInfo`

## `AgentSessionPromptInput`

核心字段：

- `query`

## `AgentSessionTurnHandle`

核心字段：

- `id`
- `result`
- `finished`

## `AgentSessionTurnResult`

核心字段：

- `turnId`
- `success`
- `error`
- `text`
- `assistantMessage`（本地 session 通常可用）

对大多数用户来说，最常直接消费的是 `text`。如果你需要最终的结构化 assistant 结果，可以读取 `assistantMessage`；如果你需要在 `finished` 兑现前实时更新 UI，配合 `session.subscribe()` 返回的 `SessionMutation` 即可。

## `AgentSessionStopResult`

核心字段：

- `stopped`
- `turnId`
- `cancelledQueuedPrompts`
- `reason`

`session.stop()` 会停止当前 turn，并取消尚未被当前 turn 吸收的排队 prompt。空闲时调用不会抛错，会返回 `stopped: false` 和 `reason: "idle"`。
