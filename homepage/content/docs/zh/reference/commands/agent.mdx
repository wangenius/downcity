# `sma agent`

管理 Agent Runtime 的启动、停止和重启。

## 用法

```bash
sma agent [path]
sma agent on [path] [options]
sma agent off [path]
sma agent status [path]
sma agent restart [path] [options]
```

## 子命令

- `on`：启动 runtime
- `off`：停止 daemon runtime
- `status`：查看 daemon runtime 状态
- `restart`：重启 daemon runtime

## 参数

### `on` 选项

- `-p, --port <port>`：服务端口（也可在 `ship.json.start.port` 配置）
- `-h, --host <host>`：服务主机（也可在 `ship.json.start.host` 配置）
- `--webui [enabled]`：启用交互式 Web UI（也可在 `ship.json.start.webui` 配置）
- `--webport <port>`：Web UI 端口（也可在 `ship.json.start.webport` 配置）
- `--foreground [enabled]`：前台启动（仅当前终端）
- `--daemon [enabled]`：后台启动兼容参数（保留旧写法）

### `restart` 选项

- `-p, --port <port>`：服务端口（也可在 `ship.json.start.port` 配置）
- `-h, --host <host>`：服务主机（也可在 `ship.json.start.host` 配置）
- `--webui [enabled]`：启用交互式 Web UI（也可在 `ship.json.start.webui` 配置）
- `--webport <port>`：Web UI 端口（也可在 `ship.json.start.webport` 配置）

## 新运行逻辑

- `sma agent .`：等价 `sma agent on .`
- `sma agent on .`：默认后台 daemon 启动
- `sma agent on . --foreground`：前台启动
- `sma agent off .`：关闭当前目录对应的 agent daemon
- `path` 不传时默认当前目录 `.`

## 环境变量注入

每次执行 `sma` 时，CLI 会自动推导当前 agent 上下文，并在 `sma` 进程及其子进程中注入：

- `SMA_AGENT_PATH`：当前 agent 绝对路径
- `SMA_AGENT_NAME`：当前 agent 名称（优先 `ship.json.name`，否则目录名）

> 注意：这是进程级注入，不会直接回写你当前 shell 会话的环境变量。

## 示例

```bash
# 当前目录后台启动（默认）
sma agent .

# 显式后台启动
sma agent on .

# 前台启动
sma agent on . --foreground

# 停止当前目录 agent
sma agent off .

# 查看当前目录 agent 状态
sma agent status .

# 重启当前目录 agent
sma agent restart .
```
