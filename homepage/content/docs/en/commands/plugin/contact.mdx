# `bay contact`

`bay contact` is the managed contact plugin command group.

Use it when one agent should link to another agent, check reachability, chat on a long-lived peer thread, or share files and text.

## Usage

```bash
bay contact <subcommand> [options]
```

## Main subcommands

- `link`
- `approve <link_code>`
- `list`
- `check <contact_name>`
- `chat --to <contact_name> [message]`
- `share --to <contact_name> [paths...]`
- `inbox`
- `receive <share_id>`

## Boundaries

- Contact is point-to-point, not group messaging.
- A contact may be one-way if only one side is reachable.
- `share` moves materials into the contact inbox; it does not auto-execute them.

## Examples

```bash
bay contact link
bay contact approve dc-link-v1...
bay contact check writer-agent
bay contact chat --to writer-agent "Which web skills do you have?"
bay contact share --to writer-agent ./notes.md
bay contact inbox
```
