# 命令环境要求

这页只回答一个问题：不同 `studio` 命令组运行时，各自需要什么环境。

## 0）一级命令边界

一级命令组只有：

- `studio console`
- `studio agent`
- `studio plugin`

像 `studio chat`、`studio task`、`studio memory`、`studio shell`、`studio skill`、`studio asr`、`studio tts` 这些，都是 plugin 命令，不是一级根命令。

## 1）按命令组看环境要求

| 命令组 | 必需环境 | 可选环境 | 说明 |
| --- | --- | --- | --- |
| `studio init` | 可写 `~/.downcity/` | 无 | 写入 `~/.downcity/downcity.db` |
| `studio start/stop/restart/status` | 可写 `~/.downcity/console/` | 无 | 管理全局 console 进程与 registry |
| `studio console` | Console 已运行；浏览器可访问 UI 端口 | `--host`、`--port` | 一个 UI 可切换多个运行中的 agent |
| `studio agent create` | 目标目录可写 | 无 | 生成 `downcity.json`、`PROFILE.md`、`.downcity/` |
| `studio agent start` | Console 已运行；项目包含 `downcity.json` + `PROFILE.md` | `--host` | agent 端口由 console 分配并登记进 registry |
| `studio agent restart/status/doctor` | 目标 agent 已登记在 console registry | 无 | 通过 registry 解析 |
| `studio config ...` | 目标路径下存在合法 `downcity.json` | `--path` | 只读写项目配置，不调用 agent API |
| `studio plugin list/status` | 无 | `--json` | 静态 plugin catalog，不解析 agent |
| `studio plugin start/stop/restart/command` | 目标 agent 正在运行 | `--agent`、`--path`、`--host`、`--port` | 面向托管 plugin |
| `studio plugin action ...` | 项目存在 `downcity.json` | `--path` | 高级本地执行形式 |
| `studio <managed-plugin> <action>` | 目标 agent 正在运行 | `--path`、`--host`、`--port` | 如 `studio chat`、`studio task`、`studio memory`、`studio shell` |
| `studio <local-plugin> <action>` | 项目存在 `downcity.json` | `--path` | 如 `studio skill`、`studio web`、`studio asr`、`studio tts` |

## 2）运行时传输

默认规则：

1. 不传 `--host` / `--port`：托管 runtime 命令默认连接目标项目解析出的本地 agent daemon HTTP 端点
2. 传了 `--host` 或 `--port`：覆盖端点，按给定 host/port 走 HTTP

只有目标 HTTP 端点当前启用了鉴权时，Bearer Token 解析才有意义。

## 3）`DC_AUTH_TOKEN`

对大多数本地使用场景，你不需要关心 auth 环境变量。

托管 runtime 调用默认走本地 daemon HTTP。`DC_AUTH_TOKEN` 只在目标 HTTP 端点要求鉴权时才生效。

优先级顺序：

1. 显式 `--token`
2. `DC_AUTH_TOKEN`

## 4）常见命令的上下文变量

### `studio chat ...`

- 省略 `--chat-key` 时，会尝试 `DC_CTX_CHAT_KEY`
- 省略 `--session-id` 时，会先尝试 `DC_SESSION_ID`

### `studio task ...`

- task 创建与运行需要 `sessionId`
- 可以显式传 `--session-id`，或依赖 `DC_SESSION_ID`

## 5）推荐用法

日常操作优先直接使用 plugin 根命令：

```bash
studio chat send --chat-key ctx_xxx --text "hi"
studio task list
studio memory search "release decision"
studio plugin list
```
