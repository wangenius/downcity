# Agent 类

本地 `Agent` 的核心公开使用面包括：

- `new Agent(options)`
- `agent.start(options?)`
- `agent.stop()`
- `agent.createSession(input?)`
- `agent.getSession(sessionId)`
- `agent.listSessions(input?)`
- `agent.getConfig()`
- `agent.getLogger()`
- `agent.plugins`

`agent.createSession()` 和 `agent.getSession()` 返回 `AgentSession`。

`agent.getConfig()` 和 `agent.getLogger()` 是更窄的高级集成辅助入口。本地 SDK 不再把 `agent.getRuntime()` 与 `agent.getContext()` 作为公开使用面。

## 构造参数

- `id`
- `path`
- `model`
- `env`
- `mode`
- `tools`
- `instruction`
- `plugins`

## 核心语义

- `Agent` 本身是本地执行壳
- 真正的执行对象是它创建出来的 session

从 API 设计上看，它更像一个“session 工厂 + 长期运行能力入口”。

## `start()` 和 `stop()`

`agent.start()` 负责启动当前 agent 的长期运行能力。

典型例子包括：

- 启动 HTTP 服务
- 启动 plugin lifecycle

推荐写法：

```ts
await agent.start({
  http: {
    host: "127.0.0.1",
    port: 15314,
  },
});
```

如果你不调用 `agent.start()`：

- 仍然可以直接 `await agent.createSession()` 然后 `session.prompt()`
- 但不会启动 HTTP
- 也不会启动后台 plugin lifecycle

`agent.stop()` 则负责关闭当前 agent 已启动的长期运行能力。

## 生命周期入口

不过在大多数用户场景里，更推荐统一使用：

```ts
await agent.start({
  http: {
    host: "127.0.0.1",
    port: 15314,
  },
});
```

本地 `Agent` 不再暴露细粒度的运行时内部入口。

如果你需要服务端点，就使用：

- HTTP：`agent.start({ http: { host, port } })`

`agent.plugins` 是当前本地 SDK 新增的重要公开面。

你可以直接用它做：

- `agent.plugins.list()`
- `agent.plugins.availability(pluginName)`
- `agent.plugins.runAction({ plugin, action, payload })`
- `agent.plugins.pipeline(pointName, value)`
- `agent.plugins.guard(pointName, value)`
- `agent.plugins.effect(pointName, value)`
- `agent.plugins.resolve(pointName, value)`
