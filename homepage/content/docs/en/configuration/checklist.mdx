# Configuration Checklist

Use this checklist before release:

1. Model binding
- Verify `downcity.json.execution.modelId` exists in the connected City AIService catalog.

2. Routing and network
- Verify `start.host/start.port` match runtime daemon.
- Ensure no stale `DC_SERVER_*` env values.

3. Channels and plugins
- Enable only required channels/plugins.
- Run one send/receive test per enabled channel.

4. Security and secrets
- Ensure `DC_MODEL_DB_KEY` is injected securely.
- Verify chat authorization groups and channel-account bindings are correct.

5. Observability
- Confirm log paths are writable and traceable.
- Check `town status` and `town agent status` are healthy.
