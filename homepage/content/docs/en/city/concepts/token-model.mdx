Downcity uses `admin_secret_key` to manage City, and `user_token` to let end-user clients call services.

## user_token

`user_token` is the call credential used by the end user or bay client. `Admin Visa` requests it from City using `bay_id + user_id`.

- `bay_id`
- `user_id`
- `metadata`
- `ttl`

The client only carries `user_token` and `bay_id`. It never touches provider keys or `admin_secret_key`.

## admin_secret_key

`DOWNCITY_CITY_ADMIN_SECRET_KEY` is used to manage City: create bays, manage env, and issue `user_token` for users under a given bay. City generates it automatically and writes it into `.env` on first startup.

```ts
const client = new Visa({
  role: "admin",
  city_url: "https://base.example.com",
  admin_secret_key: process.env.DOWNCITY_CITY_ADMIN_SECRET_KEY,
});

const bay = await client.bays.create({
  name: "Chrome Extension",
});
```

## Multi-bay reuse

Many clients can belong to the same bay. When issuing a token, write `bay_id` explicitly:

```ts
const user = await client.bays.tokens.apply({
  bay_id: bay.bay_id,
  user_id: "user_123",
  metadata: {
    plan: "pro",
  },
  ttl: "7d",
});

console.log(user.user_token);
```
