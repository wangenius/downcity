# Session 事件

`session.subscribe()` 当前会发出的核心事件类型：

- `turn-start`
- `text-delta`
- `reasoning-delta`
- `tool-call`
- `tool-result`
- `assistant-step`
- `action`
- `turn-finish`
- `error`

## 最常见的是 `text-delta`

如果你只是做实时文本输出，通常只需要先处理：

```ts
if (event.type === "text-delta") {
  process.stdout.write(event.text);
}
```

如果你要做更完整的体验，通常还应同时处理：

- `turn-start`
- `turn-finish`
- `error`
- `assistant-step`
- `action`
- `tool-call`
- `tool-result`

`assistant-step` 对 chat bridge 这类 transport 很有用，因为它可以拿到 step 级可见回复，同时又不需要自己接管 Session history 落盘。

## `action`

`action` 表示 Session 内部辅助工作，例如上下文压缩、模型切换和历史分叉。它不是 assistant 正文。

模型切换 action 的 `title` 会直接包含原模型与新模型名称，例如 `Session model switched from GPT-5 to Claude Sonnet`，UI 可以直接展示标题，不需要自行拼接模型名称。

Action event 同时会作为 `type: "action"` record 持久化到 Session JSONL。同一个 action 复用同一个 `id`，并随着生命周期把 `state` 更新为 `running`、`completed` 或 `failed`。Action record 会出现在 UI history/timeline 中，但 Agent 在构建 LLM 输入前会过滤掉它。

Agent 当前内置的辅助 action 包括：

- `compacting`
- `model-switching`
- `history-forking`

```ts
session.subscribe((event) => {
  if (event.type === "action") {
    console.log(event.state, event.title, event.description);
  }
});
```
