# Agent SDK

`@downcity/agent` provides `Agent` for in-process execution and `RemoteAgent` for a runtime exposed over HTTP, HTTPS, or RPC. The working object in both modes is a Session.

## Local embedding

```ts
const agent = new Agent({
  id: "repo-helper",
  path: process.cwd(),
  prepare_session: async (session) => await session.set({ model }),
  tools: { search: search_tool },
});

const session = await agent.sessions.create();
const turn = await session.prompt({ query: "Summarize this repository" });
await turn.finished;
await agent.dispose();
```

The host owns model selection and injects a runtime model through `prepare_session` or a direct local `session.set({ model })` call.

## Remote access

```ts
const agent = new RemoteAgent({ url: "http://127.0.0.1:5314" });
const session = await agent.sessions.create();
```

Remote Sessions do not expose model configuration. The server host owns model IDs, defaults, recovery, and runtime injection.

## Session API

- `agent.sessions.create()` / `get()` / `list()`
- `session.prompt()` and `turn.finished`
- `session.subscribe()` for the unified live Mutation stream
- `session.messages()` for snapshots
- `session.stop()`, `system()`, and `fork()`

Use `AgentHTTP` or `AgentRPC` from `@downcity/server` when the runtime must be exposed over the network.
