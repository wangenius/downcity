## 运行模型（简版）

- `bay start`：仅启动 Bay runtime
- `bay start --console` / `bay start -a`：同时启动 Bay runtime 和 Console
- `bay console start`：只启动 Console，前提是 Bay runtime 已经运行
- `bay stop`：先停止 Console，再停止 Bay runtime 与托管 agent
- `bay restart`：重启 Bay runtime，恢复之前运行中的 agent，再重启 Console
- `bay update`：更新当前全局安装来源对应的 `downcity` CLI 聚合包
- `bay public`：管理 Console 的持久化公网暴露，支持交互式 manager 和 `on/off/status`
- `bay agent`：在交互式终端中打开 agent manager
- `bay agent create`：初始化项目运行文件
- `bay agent list`：查看已登记 agent，也可加 `--running` 只看正在运行的项目
- `bay agent start`：启动项目 agent daemon
- `bay agent stop`：停止项目 agent daemon
- `bay agent chat`：在终端中与 agent 对话，也可通过 `--message` 发送一次性消息
- `bay console`：管理 Console 模块本身
- `bay plugin ...`：查看 plugin catalog、管理 plugin lifecycle，或使用高级 plugin command 形式
- `bay chat/task/memory/contact/shell ...`：日常托管 plugin runtime 命令
- `bay web/skill/asr/tts ...`：日常本地 plugin 命令
- `bay model`：在 TTY 中打开交互式全局模型管理器
- `bay model ...`：管理运行时模型配置与自动化脚本入口
