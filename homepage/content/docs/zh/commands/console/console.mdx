# `city console`

`city console` 只负责管理 Console 模块本身。

Console 是 Downcity 的 Web 控制台：

- 单实例 Console
- 多 agent 切换查看
- 全局 model / env / auth / plugin 管理
- 选中 agent 后代理到对应运行时

## 用法

```bash
city console [action] [options]
```

## action

- `start`：后台启动 Console
- `stop`：停止 Console
- `restart`：重启 Console
- `status`：查看 Console 状态

如果省略 `action`，默认执行 `start`。

## 常用参数

- `-p, --port <port>`：Console 端口，默认 `5315`
- `-h, --host <host>`：Console 主机，默认 `127.0.0.1`

## 关键行为

- Console 依赖 city runtime，先执行 [`city start`](/zh/docs/commands/start)。
- Console 不是 city runtime 本身，而是运行在其上的独立模块。
- 一个 Console 实例可以切换查看多个运行中的 agent。
- `city status` 会同时显示 city runtime 与 Console 状态。

## 典型流程

```bash
# 1) 一起启动 runtime + Console
city start --console

# 2) 启动多个 agent
city agent start /path/to/agent-a
city agent start /path/to/agent-b

# 3) 查看 Console 状态
city console status

# 4) 只重启 Console
city console restart
```
