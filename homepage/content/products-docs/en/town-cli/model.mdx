# `town model`

`town model` is no longer part of the Town command surface.

Town is the local Agent host and Agent Plugin manager. Models come from the selected City base; Town does not manage or store model resources.

Use:

```bash
city
town city login
town city status
```

If an Agent project needs a City model ID in `downcity.json.execution.modelId`, choose that model from the City base and keep Town focused on hosting Agents and managing Plugins.
