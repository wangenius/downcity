# `ship.json` 参考

当前版本采用两层模型配置：

1. `~/.ship/ship.db`（console 全局 SQLite）管理完整模型池  
2. 项目 `ship.json`（agent 本地）只声明 `model.primary`

## 1) Agent 项目最小结构

```json
{
  "$schema": "./.ship/schema/ship.schema.json",
  "name": "my-agent",
  "version": "1.0.0",
  "model": {
    "primary": "default"
  },
  "services": {}
}
```

`model.primary` 的值必须能在 `~/.ship/ship.db` 的模型池中找到同名 model id。

## 2) Console 全局模型池（`~/.ship/ship.db`）

模型细节统一在 SQLite：

- providers
- models
- provider.apiKey 为加密存储（`api_key_encrypted`）

通过命令管理：

```bash
city console model list
city console model get provider openai_main
```

## 3) 运行时解析规则

agent 启动时：

1. 读取项目 `model.primary`
2. 到 `~/.ship/ship.db` 的 models 表取模型
3. 再由 `providerId` 关联 providers 表

任何一步缺失都会启动失败。

## 4) Plugin / Asset 配置

插件行为配置写在项目 `ship.json` 的 `plugins.*`。

插件依赖的底层资源配置写在项目 `ship.json` 的 `assets.*`。

例如 voice：

```json
{
  "plugins": {
    "voice": {
      "enabled": true
    }
  },
  "assets": {
    "voice.transcriber": {
      "provider": "local",
      "modelId": "SenseVoiceSmall"
    }
  }
}
```

## 5) Chat 渠道绑定

`ship.json` 中的 chat 渠道应只绑定全局 bot 账户 ID：

```json
{
  "services": {
    "chat": {
      "channels": {
        "telegram": {
          "enabled": true,
          "channelAccountId": "telegram-main"
        }
      }
    }
  }
}
```

`channelAccountId` 会解析到 `~/.ship/ship.db` 的 `channel_accounts.id`。

## 6) 推荐操作方式

1. 在 console 层维护模型池（provider/model）  
2. 在 agent 层只改 `model.primary` 绑定目标模型 ID  
3. 修改后重启 agent：

```bash
city agent restart .
```
